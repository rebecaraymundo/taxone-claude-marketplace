# [Interfaces] [ORACLE Interdados - V11R01 EBS] [Geral] Limpando a Tabela MSAFI_MSFTS_PROC_LOG para Melhorar a Performance das Cargas no ODI

**Problema:** As cargas no ODI estão demorando muito tempo durante a etapa de exclusão (delete) na tabela MSAFI_MSFTS_PROC_LOG, gerando travamentos.

**Causa Raiz:** Como a tabela "MSAFI_MSFTS_PROC_LOG" é responsável por armazenar todos os logs de transação, é possível que ela fique um tamanho relativamente grande, o que pode gerar alguns problemas de desempenho

**Resolução:** No momento o Interdados V11 não prevê uma execução de limpeza dessa tabela e **NÂO** recomendamos a limpeza da mesma devido o armazenamento de logs que usamos para consultas e avaliação de integrações, dessa forma, será necessário que o cliente acione a consultoria ou a equipe de desenvolvimento responsável para ajuste.

Mas caso seja necessária a limpeza, comando SQL **TRUNCATE**. Isso removerá todos os dados da tabela sem afetar as cargas de dados futuras, uma vez que a tabela irá apagar os dados antigos, armazenando os novos

Passo a passo:

  1. Acesse o banco de dados onde está alocada a tabela MSAFI_MSFTS_PROC_LOG.
  2. Execute o comando SQL: `TRUNCATE TABLE MSAFI_MSFTS_PROC_LOG;`
  3. Verifique se a tabela está vazia rodando: `SELECT * FROM MSAFI_MSFTS_PROC_LOG;`
