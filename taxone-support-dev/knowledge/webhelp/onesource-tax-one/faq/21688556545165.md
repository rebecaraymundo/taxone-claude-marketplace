Para resolver o erro 90566: 'Código do Tributo não cadastrado' mesmo tendo o código cadastrado é necessário verificar os campos abaixo.

**Passo 1: Identificação do Problema** Verifique o arquivo TXT gerado pelo sistema SAFX53. Abra o arquivo e localize os campos COD_TRIBUTO e ESP_TRIBUTO para verificar se estão formatados corretamente.

**Passo 2: Verificação dos Campos COD_TRIBUTO e ESP_TRIBUTO** Certifique-se de que os campos COD_TRIBUTO e ESP_TRIBUTO contenham exatamente dois dígitos. Por exemplo, se o código for '2', deve ser apresentado como '02'; se for '0', deve ser exibido como '00'.

**Passo 3: Edição do Arquivo TXT** Caso os campos estejam com um único dígito (exemplo: '2' em vez de '02'), edite manualmente o arquivo TXT para corrigir essa formatação. Use um editor de texto como o Bloco de Notas (Windows) ou TextEdit (Mac) para fazer as alterações.

**Passo 4: Ajuste e Salve as Mudanças** Faça as correções necessárias nos campos COD_TRIBUTO e ESP_TRIBUTO para garantir que todos os códigos estejam formatados com dois dígitos. Salve o arquivo após as alterações.

**Passo 5: Teste e Validação** Após editar o arquivo, faça um teste importando-o novamente para o sistema SAFX53. Certifique-se de que o arquivo atualizado não exibe mais o erro 90566 relacionado ao código do tributo.

**Passo 6: Reenvio do Arquivo Corrigido** Caso o teste tenha sido bem-sucedido e o erro não seja mais exibido, o arquivo corrigido pode ser reenviado para o sistema ou processo que estava gerando a falha.

**Passo 7: Verificação Final** Após o reenvio do arquivo corrigido, confirme se o sistema aceitou o arquivo sem gerar mais o erro 90566. Certifique-se de que os campos COD_TRIBUTO e ESP_TRIBUTO estão corretamente formatados com dois dígitos.

Seguindo esses passos, você deve conseguir resolver o problema relacionado ao erro 90566 devido à formatação inadequada dos campos COD_TRIBUTO e ESP_TRIBUTO no arquivo TXT gerado pelo sistema SAFX53. Certifique-se sempre de manter uma cópia do arquivo original antes de fazer edições para evitar perda de dados.
