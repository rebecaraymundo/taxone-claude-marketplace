**Assunto:**
Esta FAQ tem por objetivo orientar sobre fluxo primário para atendimento de chamado de Autenticação SSO

Obs:. Sempre verificar pois este Fluxo pode sofrer alterações.

**O Que é o SSO:**

Fluxo para atendimento de chamados referente ao serviço de SSO - Atendimento é feito via Tickets no Zendesk.

A Thomson Reuters oferece o Federated Single Sign-On para o ONESOURCE TAX ONE.
Esse recurso permite que o cliente se autentique em sua rede corporativa primeiro e, em seguida, se autentique automaticamente em aplicativos da Thomson Reuters com a tecnologia Single Sign-On (SSO).
O Single Sing On (SSO) no Tax One é um serviço contratado adicionalmente.

**Solução:**

Fluxo Primário:

-**> Solicitação** = Orientar para fazer contato com o Account Manager responsável pela conta do Cliente para solicitação. Após a contratação, um profissional do time responsável acompanhará a implantação do SSO, enviando todas instruções necessárias e ficará disponível para solucionar dúvidas sobre a configuração.

**- > Chamados de Configuração** = Direcionamos para Contratos/Licenciamentos

**- > Chamados de Erro** = Direcionamos para TechOps

**- > Dúvidas sobre o serviço** = Devem ser avaliadas e trocar informações entre os Times de Suporte para ver se temos algo mapeado como documentação ou FAQ para responder o cliente. As dúvidas mais técnicas que não tivermos o devido conhecimento iremos abri pra TechOps e se necessário eles podem solicitar apoio para o Global. O suporte jamais deve responder algo sem saber, sem ter o tema documentado, sem ter o devido respaldo do time global ou de TechOps.

Como DFe e Tax One são produtos diferentes o Arquivo Metadata também deverá ser diferente. Solicitação SSO de Tax One o cliente deve abrir chamado para o Time de Tax One, solicitações SSO de DFe a solicitação deve ir para o Time de DFe.

Devemos verificar com o cliente se o arquivo Metadata enviado tem configurações só de um ou de ambos os produtos.

Sendo arquivos diferentes, um não deve impactar no outro. Caso o Cliente utilize o mesmo arquivo com informações de DFe e de Tax One unificadas, o cliente vai precisar subir o arquivo nos dois ambientes (DFe e Tax One) de forma apartada para surtir efeito em cada produto.
