Segue demonstração da causa e a solução, para os casos em que os arquivos externos foram importados com sucesso, porém, o 'Log' da 'Geração Prévia' dos 'Movimentos' informa que '**Não existem dados para geração do evento R40X0 (JCP).** ':

Exemplo:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21196190453133)

Isso pode ocorrer quando o arquivo foi importado para o ambiente '**1 - Produção** ' e a geração prévia dos movimentos está sendo executada no ambiente de '**2 -Produção Restrita** '.

Exemplo incorreto:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21196196959373)

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21196174405133)

A 'Geração Prévia' dos 'Movimentos' deve ser efetuada para o respectivo ambiente ao qual pertencem os arquivos externos (JCP) dos bancos, importados na solução, conforme exemplo destacado abaixo.

**1** \- Importação dos Arquivos Externos

Os arquivos devem ser previamente importados para o Tax One.

Para importação dos arquivos e posterior geração prévia dos movimentos, pode ser verificado os seguintes artigos:

[TAX ONE - Importação de Arquivos Externos (JCP) XML dos Bancos](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/21162751356685>),

[TAX ONE - Arquivos externos (JCP) XML dos bancos não estão disponíveis para seleção e importação](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/21163304240397>), e

[TAX ONE - Geração e envio de eventos relativos a arquivos externos (JCP) XML dos bancos](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/21165659218445>).

**2** \- Em Parâmetros > Dados Iniciais:

  * Selecione o Tipo de Ambiente.
