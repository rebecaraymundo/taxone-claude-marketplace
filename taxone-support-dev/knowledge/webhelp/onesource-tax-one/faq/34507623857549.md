## Problema:

Usuário precisa corrigir o código de receita para o evento R-4020 no REINF, após ter utilizado um código de serviço incorreto inicialmente.

## Causa Raiz:

O problema ocorreu devido à seleção inicial de um código de serviço inadequado para o evento R-4020, o que gerou a necessidade de retificação da informação.

## Resolução:

Siga os passos abaixo para corrigir o código de receita para o evento R-4020:

  * Acesse o módulo 'Ferramentas' 'Tabelas Internas' 'Manter' 'Tabelas' 'Código de Receita'.
  * Insira manualmente os códigos de receita corretos que serão utilizados, conforme orientação fiscal.
  * Vincule os códigos de receita no Parâmetro de Identificação de Natureza de Rendimento.
  * Verifique se o código de DARF utilizado está correto para o código de Natureza de Rendimento e código de Receita informado no documento. Essas informações a área fiscal do cliente precisa validar.
  * Reprocesse o evento R-4020 através da tela de Painel de Controle de Eventos no Reinf, selecionando o evento original e clicando em 'Reprocessar Evento'.


## Próximos Passos:

Após reprocessar o evento, verifique se o código de DARF utilizado está correto para o código de Natureza de Rendimento e código de Receita informados. Valide essas informações com a área fiscal.

## Dicas e Recomendações:

Assegure-se de sempre verificar a tabela MOR da Receita para confirmar os códigos de receita adequados para cada natureza de rendimento antes de realizar quaisquer alterações ou submissões no sistema.
