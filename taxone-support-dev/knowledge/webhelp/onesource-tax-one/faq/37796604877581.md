# [Estadual][Ativo Permanente] TAX ONE - Valor Duplicado no Relatório CIAP (Bloco G) Gerando Erro no Campo 05 do G125 – Como Corrigir?

Para o Relatório de Conferência do CIAP (EFD – Bloco G) o cálculo é realizado da seguinte maneira: subtrai-se o Total de Baixas Parciais (B) do Total de ICMS da Aquisição (A). Onde:


  * **A = VLR_CRED_ICMS + VLR_CRED_DIF_ALIQ + VLR_ICMS_FRETE + VLR_DIFAL_FRETE + VLR_ICMSS**
  * **B = Somatório dos valores das baixas parciais, que inclui: VLR_ICMS_BX + VLR_DIFAL_BX + VLR_ICMS_FRETE_BX + VLR_DIFAL_FRETE_BX + VLR_ICMSS_BX**


Para recuperar as baixas parciais, é necessário acessar a tabela de alienações (APT_ALIENACAO) utilizando os seguintes critérios:


  * APT_ALIENACAO.COD_EMPRESA = APT_AQUISICAO.COD_EMPRESA
  * APT_ALIENACAO.COD_ESTAB = APT_AQUISICAO.COD_ESTAB
  * APT_ALIENACAO.NUM_CIAP = APT_AQUISICAO.NUM_CIAP
  * APT_ALIENACAO.DAT_OPER <= Data da Apuração informada em tela
  * APT_ALIENACAO.TIPO_BAIXA = ‘2’ ou ‘3’ (referente a baixas parciais)


Pode haver várias ou nenhuma baixa para uma aquisição específica. Quando existem múltiplas baixas, os valores são acumulados conforme a regra mencionada acima.

Este campo é preenchido com o valor do ICMS próprio que foi utilizado no cálculo do CIAP. Trata-se do valor original da entrada do Bem, menos as baixas parciais que possam ter sido realizadas até o período.


  * Subtrai o valor total das baixas parciais do valor do ICMS registrado na aquisição. Ou seja, subtrai-se o campo "Vlr.ICMS" das alienações, que possuem o Tipo de Baixa igual a Parcial e data de operação menor ou igual à Data Final do período, do campo “Valor ICMS” do cadastro.


Em termos técnicos, isso seria representado por:


  * **(APT_AQUISICAO.VLR_CRED_ICMS) – (somatório de APT_ALIENACAO.VLR_ICMS_BX)**


Onde:


  * **APT_AQUISICAO.VLR_CRED_ICMS** representa o valor de crédito de ICMS da aquisição.
  * **APT_ALIENACAO.VLR_ICMS_BX** é o somatório dos valores de ICMS das baixas, considerando todas as baixas parciais (TIPO_BAIXA = 2 ou 3) que ocorreram até o final do período especificado (DAT_OPER <= DATA_FIM do G110).


**Observação:** O Guia Prático descreve este campo como sendo o valor da entrada do bem. No entanto, é crucial considerar as baixas parciais já realizadas para utilizar o valor exato do saldo restante. Isso é importante porque o validador irá verificar se o campo 10-VL_PARC_PASS é igual à soma dos campos (05+06+07+08) dividida pelo campo 09-NUM_PARC.
