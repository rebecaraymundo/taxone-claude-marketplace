#### Problema:

Ao gerar a SAFX53 via Interdados, o campo "MES_COMPETENCIA" não está sendo preenchido corretamente em IR

#### Causa Raiz:

O erro em questão deve ser tratado de acordo com o imposto sendo gerado, pois para cara tipo de imposto, o campo validado é diferente

#### Resolução:

O primeiro passo é verificar qual o imposto sendo gerado e qual o extrator sendo utilizado.

Caso seja o extrator "**LACLS_ExtractApSpedData_Incremental.xdo** ", são impostos IR. Para IR, precisamos verificar o campo _INVOICE_RECEIVED_DATE_ dentro da _G_INVOICES:_

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40859463736461)

Caso seja o Extrator "**LACLS_ExtractTaxWithholdingData.xdo** ", são impostos PCC. Para PCC, é utilizado o campo _TAX_POINT_DATE_ dentro da  _G_TAXES_WITHHELD_V:_

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40859435995789)

#### Próximos Passos:

Após realizar as alterações recomendadas, é importante fazer uma auditoria nos documentos afetados para confirmar que o mês de competência está sendo definido corretamente. Caso ocorra algum problema, entre em contato com o suporte Thomson Reuters para apoio.
