# Como Resolver o Problema do Campo COD_SERVICO Vazio ao Processar Cargas

## Problema

O campo COD_SERVICO está vindo vazio ao processar cargas, mesmo após alterar a parametrização para Service_list_code e a tag estar sendo preenchida adequadamente no Oracle.

## Causa Raiz

O problema ocorre devido a uma regra configurada no campo COD_SERVICO durante o processo de edição do layout dentro do procedimento SAFX09E. Esta regra (105) está impedindo que o valor seja carregado corretamente.

## Resolução

Siga os passos abaixo para corrigir o problema:

  1. No Interdados, acesse a tela de procedimento.
  2. Localize o procedimento SAFX09E e clique no ícone de lápis para editar.
  3. Na aba **Layout** , procure pelo campo **COD_SERVICO** (campo 13).
  4. Dê um duplo clique no campo COD_SERVICO para acessar suas configurações.
  5. Remova a regra **105** do campo.
  6. Após a remoção, clique em **Salvar**.
  7. Realize uma nova carga para confirmar que o problema foi resolvido.


![Procedimento no Interdados](https://atendimentotr.zendesk.com/attachments/token/qm1PIJiMNFHb1Ra6Jb7zHwqZi/?name=image.png)

## Próximos Passos

Após realizar a nova carga, monitore o sistema para garantir que o campo COD_SERVICO esteja sendo preenchido corretamente. Caso o problema persista, entre em contato com o suporte para uma análise aprofundada.

## Dicas e Recomendações

  * Verifique regularmente a parametrização dos campos no layout, garantindo que regras desnecessárias sejam removidas.
  * Mantenha seus processos documentados para facilitar o diagnóstico e resolução de problemas semelhantes no futuro.
  * Considere criar uma base de testes antes de realizar alterações em ambientes produtivos.
  * Acompanhe atualizações e boas práticas recomendadas pelo suporte para otimizar o uso dos sistemas.
