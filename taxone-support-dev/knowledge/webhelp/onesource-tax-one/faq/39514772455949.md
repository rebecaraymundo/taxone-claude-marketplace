## Problema:

A senha de integração entre os sistemas Interdados e TaxOne expirou e após atualização via interface, porém somente o campo SENHA_DW_WS foi atualizado, não alterando o campo necessário SENHA_TAX_WS no banco de dados.

## Causa Raiz:

O problema ocorre porque a atualização da senha através do menu Interdados V11R01 Patch045 altera somente o campo SENHA_DW_WS na tabela MSAFI_MSFTS_CONFIG. A senha utilizada pelo sistema TaxOne é armazenada no campo SENHA_TAX_WS, que não é atualizado através da interface padrão.

## Resolução:

Para resolver este problema, é necessário realizar a atualização diretamente no banco de dados da seguinte maneira:

  1. Acesse o banco de dados onde a tabela MSAFI_MSFTS_CONFIG está armazenada.
  2. Realize um backup da tabela antes de realizar quaisquer alterações.
  3. Utilize a consulta SQL para realizar a atualização:
`UPDATE MSAFI_MSFTS_CONFIG SET SENHA_TAX_WS = 'nova_senha' WHERE nome_do_campo_identificador = valor_identificador;`
  4. Faça um novo teste de integração para validar a atualização


## Próximos Passos:

Após realizar a atualização, é importante monitorar os logs do sistema para garantir que não haja erros na integração e que a nova senha esteja funcionando como esperado. Caso ocorra algum erro, entre em contato com o suporte Thomson Reuters para apoio
