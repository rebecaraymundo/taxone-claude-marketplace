# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Como corrigir a importação dos números de notas fiscais no campo NUM_DOCFIS da SAFX53?

O objetivo deste artigo é orientar consultores e clientes sobre como resolver o problema de importação incorreta dos números das notas fiscais através da interface SAFX53, na qual os números não correspondem aos registrados no ERP.

O problema reside na limitação do campo NUM_DOCFIS na interface SAFX53, que possui capacidade para apenas 12 caracteres, enquanto alguns números de notas fiscais no sistema ERP Oracle Cloud do cliente podem ter até 15 caracteres.

Como o campo é obrigatório, ele não pode ser deixado em branco. A Thomson Reuters segue a regra determinada pelo SPED Fiscal, que limita o tamanho do campo a 12 caracteres.

Para solucionar o problema, siga o processo abaixo no Layout da SAFX53:

  1. Acesse o **Interdados** e clique em "**Procedimentos** ".
  2. Localize a **SAFX53** e clique no botão “**MAIS** ”. Em seguida, clique em "**Layout** ".
  3. Clique duas vezes no campo **NUM_DOCFIS**.
  4. Na tela de Layout, marque a opção “**Zero à Esquerda** ” e clique em "**SALVAR** ".
  5. Se necessário, marque também "**Remover** " para eliminar os zeros à esquerda e ajustar o número para 12 dígitos.
  6. Realize um teste para verificar se a importação dos números está correta.
  7. Com essa alteração, o sistema ajustará os números para 12 posições ao remover os zeros à esquerda.
  8. Após realizar as alterações e testes, monitore os próximos ciclos de importação para garantir a consistência dos dados. Caso o problema persista, entre em contato com o suporte técnico.


**Dicas e Recomendações:**

  * Revise regularmente as configurações das interfaces de importação para garantir que estão compatíveis com as atualizações dos sistemas ERP.
  * Para mais informações ou suporte, visite nossa página de suporte técnico ou entre em contato diretamente.
