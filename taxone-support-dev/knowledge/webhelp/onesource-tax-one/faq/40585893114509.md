O objetivo deste artigo é orientar o usuário na resolução de possíveis inconsistências envolvendo os campos COD_TRIBUTO e ESP_TRIBUTO na SAFX53.

## Problema

Os campos **COD_TRIBUTO** e **ESP_TRIBUTO** na extração SAFX53 estão sendo gerados com informações incorretas, especificamente com o valor "LA", o que não condiz com os dados esperados.

## Causa Raiz

O problema ocorre devido à ausência de parametrizações adequadas nas regras De/Para no ambiente, que definem os valores esperados para os campos COD_TRIBUTO e ESP_TRIBUTO. Os valores estão sendo preenchidos diretamente com base no campo TAX_TYPE_CODE do XML, sem a aplicação das regras necessárias. E como o tamanho do campo é de duas posições, a interface assumia os dois primeiros caracteres enviados à tag TAX_TYPE_CODE.

## Resolução

  1. Acesse o ambiente e localize as regras De/Para correspondentes aos campos COD_TRIBUTO e ESP_TRIBUTO.
  2. Para o campo COD_TRIBUTO, aplique a **Regra 19** , preenchendo os valores de acordo com as especificações do layout.
  3. Para o campo ESP_TRIBUTO, aplique a **Regra 20** , também preenchendo os valores conforme o layout esperado.
  4. Certifique-se de que todos os valores enviados no campo TAX_TYPE_CODE, que contemplam o IR e o PCC, estejam devidamente parametrizados nas regras De/Para (19 e 20).
  5. Após realizar os ajustes, reprocesse a carga e verifique os resultados nos campos COD_TRIBUTO e ESP_TRIBUTO.


## Próximos Passos

  1. Depois de implementar as correções, realize uma nova extração para confirmar se os campos estão sendo gerados corretamente.
  2. Verifique se todos os campos impactados estão com os valores apropriados.
  3. Entre em contato com o suporte, caso continue enfrentando problemas ou tenha dúvidas adicionais.


## Dicas e Recomendações

  * Documente as parametrizações realizadas para facilitar futuras manutenções e auditorias.
  * Antes de realizar qualquer ajuste, faça um registro das configurações.
  * Certifique-se de revisar regularmente as parametrizações De/Para para evitar problemas similares.
