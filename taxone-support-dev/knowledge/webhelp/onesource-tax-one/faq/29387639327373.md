# [SPED][EFD - Contribuições] Tax One - Resolução do campo Tipo de Pagamento (13 - IND_PGTO registro C100) não está preenchido no EFD Contribuições, como resolver?

Nos casos onde o campo Tipo de Pagamento do Registro C100 no EFD Contribuições não está sendo preenchido automaticamente como "Outros" (Tipo 2) mesmo quando essa informação está correta, isso pode acontecer na importação das notas fiscais para o sistema Tax one, onde o campo 99 - IND_FATURA, que corresponde ao Tipo de Faturamento, não está sendo preenchido. Isso faz com que o sistema não reconheça automaticamente o tipo de pagamento como "Outros" no EFD Contribuições.

Para resolver, siga as orientações:

**Nota:** Quando é consultada uma nota em tela, por padrão o campo 'outros' aparece marcado mas isso não quer dizer que esta gravado em banco, pois é possível incluir notas pela tela. Deve ser verificado se na SAFX07 foi preenchido o campo 99 - IND_FATURA (que é o campo tipo de faturamento) ou fazer a manutenção da nota pela tela a salvando novamente para gravar o campo.

  1. Verifique o arquivo de integração SAFX07 para confirmar se o campo 99 - IND_FATURA está sendo preenchido corretamente.
  2. Se o campo não estiver preenchido, faça a manutenção da nota pela tela e salve novamente para garantir que o campo seja gravado corretamente.
  3. Após a correção, equalize o datamart e gere o EFD Contribuições novamente.
  4. Após implementar a solução, monitore os próximos lançamentos para verificar se o campo Tipo de Pagamento está sendo preenchido corretamente. Se o problema persistir, entre em contato com o suporte técnico.


**Importante:** Verifique regularmente os arquivos de integração e as configurações no sistema Tax One para prevenir problemas de preenchimento nos registros do EFD Contribuições. Considere implementar uma rotina de verificação após cada importação para assegurar a precisão dos dados.
