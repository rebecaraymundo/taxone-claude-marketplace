Caso seja apresentado o erro "TR00455 - Não foi encontrada conta contábil com nível superior" ao realizar a carga da CAD2002 no Onesource ECF, mesmo quando a conta contábil de nível superior está cadastrada e presente no arquivo de integração. Esse problema pode ocorrer devido a divergências nas informações de data e CNPJ entre os registros cadastrados e os dados fornecidos no arquivo de integração. As datas e o CNPJ devem corresponder exatamente aos dados cadastrados no Onesource BR para que a integração seja bem-sucedida.

Para resolver, siga as orientações:

  1. Verifique se as contas sintéticas associadas às contas analíticas estão inclusas no arquivo de integração.
  2. Confirme se a data inicial das contas é a mesma que a do Cadastro do Estabelecimento e da Central de Cadastros.
  3. Ajuste o CNPJ e a data no arquivo CAD0000 para corresponder exatamente ao cadastrado no Onesource.
  4. Desmarque a opção "Considerar apenas Contas Contábeis e centro de custo movimentadas na exportação dos Cadastros" se estiver marcada.
  5. Realize a geração do arquivo novamente e faça os testes.
  6. Após realizar os ajustes e a integração, verifique no módulo Onesource ECF se há erros ou avisos no Painel de Controle. Se os problemas persistirem, envie os arquivos corrigidos e prints das telas relevantes para análise adicional.


Nota: Confira sempre se os dados no arquivo de integração correspondem exatamente aos dados cadastrais no sistema. Mantenha os registros atualizados e sincronizados para evitar erros de integração. Consulte o [Manual de Layout TXT](<https://suporte.thomsonreuters.com.br/hc/en-us>) para mais detalhes sobre o formato correto dos arquivos.
