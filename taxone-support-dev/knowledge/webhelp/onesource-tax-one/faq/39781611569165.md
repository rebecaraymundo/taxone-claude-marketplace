### O objetivo deste artigo orientar o consultor\cliente como proceder quando é realizada uma customização.

**Problema:**

O cliente enfrenta uma discrepância na execução de uma carga de dados que não retorna os resultados esperados, mais especificamente no que diz respeito ao campo COD_ESTOQUE, onde valores deveriam ser carregados como texto (varchar) e não como número (number).

**Resolução:**

O problema ocorre porque a carga customizada CTM01 alterou o formato do campo COD_ESTOQUE de número para texto, gerando uma inconsistência com a carga BASE (CORE) que processa esse campo como número. Essa alteração na customização interfere na execução correta da carga.

  1. Verificar o mapeamento da carga CORE, confirmar o tipo de dado esperado para o campo COD_ESTOQUE.
  2. Ajustar a query da carga customizada CTM01 para alinhar o tipo de dado do campo COD_ESTOQUE com o esperado pela carga CORE.
  3. Executar novamente a carga customizada CTM01 para validar se o erro foi resolvido.
  4. Caso persista o erro, comparar os logs de erro da carga CORE e CTM01 para identificar quaisquer outras discrepâncias.


**Próximos Passos:**

Após a resolução e teste da carga corrigida, monitorar as próximas execuções para garantir que o erro não se repita. Caso o problema persista, consultar a equipe técnica para uma análise mais aprofundada.

**Dicas e Recomendações:**

Sempre alinhar as customizações com as configurações da carga BASE para evitar discrepâncias ou erros de execução. Recomenda-se realizar testes de validação a cada customização implementada para garantir que as funcionalidades estejam conforme o esperado antes da execução em ambiente de produção.
