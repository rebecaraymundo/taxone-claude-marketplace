O objetivo deste artigo é orientar o consultor\cliente na solução do problema em que, no momento da execução de Importação de tabelas de cadastro do TaxOne, as novas empresas não são carregadas.

**Problema:**
Usuários não conseguem importar as tabelas de cadastro de empresas do TaxOne para o Interdados, mesmo após a execução das cargas e importações, gerando um cenário onde as empresas parametrizadas no TaxOne não são exibidas no Interdados.
A causa do problema reside na configuração e na comunicação entre as APIs de importação/exportação e as bases de dados entre TaxOne e Interdados. Apesar de a importação informar sucesso, há uma falha no processo que não traz as novas empresas parametrizadas para o sistema de


**Resolução:**
Para solucionar este problema, siga os passos abaixo:

  * Verifique as configurações de URL das APIs de Importação e Envio de Dados.
  * Assegure que as portas 80 e 443 estão abertas e funcionando corretamente.
  * Confira os arquivos de configuração como Exporta.ini, ajustando quaisquer espaços incorretos nas parametrizações.
  * Realize testes de conexão para garantir que não há bloqueios de comunicação.
  * Reinicie o Interdados após realizar os ajustes.
  * Se persistir, solicite uma revisão do arquivo Wrapper.conf e wrapper.log para análise mais aprofundada.


**Próximos Passos:**

Após implementar a solução, realize novamente a carga 1000 e a importação das tabelas. Verifique se as novas empresas estão sendo exibidas corretamente. Se o problema persistir, entre em contato com o suporte técnico para assistência adicional.



**Dicas e Recomendações:**

Mantenha sempre as versões de software atualizadas e siga as melhores práticas recomendadas pelo fabricante para configurações de sistema. Regularmente, revise as configurações de rede e de API para evitar problemas similares.
