O objetivo deste artigo é orientar o usuário à tomar as seguintes ações para resolução de Erro ao Salvar XML no Interdados para as SAFX07E, 08E e 09E.

## Problema:

Usuários não conseguem executar nenhuma carga via extrator para o procedimento da SAFX07E, com erro ao salvar o arquivo XML, apesar do arquivo ser criado no Oracle Cloud.

## Causa Raiz:

O problema foi identificado como uma configuração incorreta nos parâmetros do extrator no sistema e na tela de edição do procedimento.

A tela de parâmetrização "Parâmetros Extrator" é responsável por enviar os parâmetros de extração ao Oracle Cloud, é importante manter os parâmetros coerentes para evitar insconsistências na extração dos dados.

É importante também, para esse procedimento, manter o flag no campo "Serviço Java" ativo.

## Resolução:

  1. Acesse a tela de Parâmetros Extrator no sistema.
  2. Altere os parâmetros do extrator, o padrão mais comum para esse procedimento é:

P_LEGAL_ENTITY
P_LRU
P_LAST_UPDATE_DATE
P_LAST_UPDATE_TO


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39300776493965)

  3. Inclua os flags para os campos "Programação por Inventário" e "Serviço Java" na tela de edição do procedimento.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39300807655437)
  4. Realize novos testes para verificar se o problema foi resolvido.


## Próximos Passos:

Após implementar as modificações sugeridas, monitore o sistema para garantir que o erro ao salvar o XML não ocorra mais. Caso o problema persista, entre em contato com o suporte para uma análise mais detalhada.

## Dicas e Recomendações:

Regularmente verifique e atualize os parâmetros de configuração no sistema para evitar problemas similares. Mantenha os registros de alterações feitas para facilitar o diagnóstico de problemas futuros.
