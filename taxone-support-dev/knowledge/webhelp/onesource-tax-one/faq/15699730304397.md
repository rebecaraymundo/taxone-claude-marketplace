Para clientes que encapsulam as request transportando os objetos de um ambiente para o outro, o encapsulamento e transporte devem ser feitos **ANTES **de realizar o processo “Atualiza Tabelas”.

Este processo ocorre na transação /N/MSAFX/SAFTR057ATU quando executado e/ou na transação /N/MSAFX/SAFTR001A quando clicado no ícone disquete para salvar a versão.

Na execução as tabelas intermediárias são zeradas e consequentemente são levadas zeradas para o outro ambiente, causando atualização incompleta nos outros ambientes.

*** Caso o transporte já tenha sido realizado e ocorreu o problema de não atualização de tabelas no outro ambiente, existem duas opções para solucionar:

  1. Aplicar diretamente a request de atualização no ambiente.

  2. Reaplicar as requests no ambiente de DEV e realizar o transporte para QA antes de aplicar a Atualização das Tabelas.
