O objetivo deste artigo é orientar a consultoria\cliente, sobre o processo de programação automática onde a data é alterada com processamento com um dia a mais que a programação.





**Problema:**

O problema enfrentado é a alteração automática da data de início inserida durante a programação de extração na interface de uma SAFX, onde o sistema processa o dia seguinte ao da data inicial especificada pelo usuário.
Sua causa raiz foi identificada como um erro na programação da interface de uma SAFX, que faz com que o sistema interprete incorretamente a data de início inserida, avançando-a para o dia seguinte.

**Resolução:**

Para resolver o problema, siga os passos abaixo:

  * Exclua qualquer programação anterior que possa estar causando o erro.
  * Criar uma programação na interface correspondente a SAFX e teste inserindo a data desejada.
  * Realize um novo teste para garantir que a data inserida esteja sendo corretamente processada e que não sofra alterações.


**Próximos Passos:**

Após seguir os passos mencionados, o usuário deve observar se o problema foi completamente resolvido ao realizar novas inserções de data. Em caso de recorrência do problema ou surgimento de novas questões, recomenda-se contactar o suporte técnico para uma investigação mais aprofundada.



**Dicas e Recomendações:**

É recomendável sempre revisar as configurações de programação inseridas para garantir que todos os campos estejam corretamente preenchidos conforme desejado. Estar atento a quaisquer inconsistências nas datas pode ajudar a evitar falhas futuras. Qualquer dúvida ou dificuldade técnica, não hesite em entrar em contato com a nossa equipe de suporte.
