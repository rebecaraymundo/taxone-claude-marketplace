O objetivo deste artigo é auxiliar na correção do problema de conexão do Interdados com o banco de dados após a migração para o Windows Server 2016.

O problema ocorre no momento em que o Interdados é acionado, causando o erro abaixo mesmo com o SQLPlus funcionando normalmente.
![](https://suporte.thomsonreuters.com.br/hc/article_attachments/29707778715789)

Após clicar em OK, o sistema trás a tela abaixo mostrando Erro no teste de conexão com Base.


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/29707811162125)



A causa do problema pode estar relacionada à configuração do Oracle Client e à versão do Java utilizada. O Interdados não está homologado para o Java Corretto 11 e embora o Oracle Database 19c esteja sendo utilizado, o cliente Oracle deve ser da versão 12c de 32 bits.

Para resolver o problema podemos realizar os passos a seguir:

1- Verifique a versão do JAVA instalada. O Interdados V11 é compatível somente com o JAVA da Sun versão 8 de 32 bits. Caso necessário, proceda com a instalação desta versão específica.

2- Confirme se o Oracle Client 12c de 32 bits está instalado e corretamente configurado. Caso duas versões do Oracle Client estejam instaladas (32 bits e 64 bits), ajuste a variável de ambiente PATH para que o caminho do Oracle de 32 bits tenha prioridade.

  * Para ajustar o PATH:
    1. Na barra de pesquisa do Windows, digite "Env" e abra "Editar as variáveis de ambiente do sistema".
    2. Clique em "Variáveis de ambiente" e na lista "Variáveis do sistema", selecione "PATH" e clique em "Editar".
    3. Reorganize os valores para que o caminho do Oracle Client de 32 bits venha antes do de 64 bits.
  * Salve as alterações e reinicie o sistema para que as alterações tenham efeito.
  * Realize novos testes de conexão com o Interdados para verificar se o problema foi resolvido.


Após realizar as etapas acima, teste a conexão utilizando tanto o Interdados quanto o SQLPlus com auxilio de um DBA, assim poderá confirmar que a configuração está correta.

Se com as orientações anteriores não obtiver êxito, realizar os passos a seguir:

  1. Remover todos os Clients Oracle existentes no Servidor, realizar a limpeza do registro do Windows, remover as pastas de instalação e reinicialização do servidor.
  2. Após a instalação do Client Oracle 12c 32 bits, para que os arquivos sqlnet.ora e tnsnames, assumam os valores corretos, assim como a reconfiguração das entradas no tnsnames.ora.
  3. Caso não funcione, verificar a possibilidade de mudar a versão do S.O. para uma versão anterior que tem histórico de funcionamento correto no cliente, lembrando que a versão atual do S.O está homologada para o Interdados, indicamos que um DBA Oracle seja consultado.


**Dicas e Recomendações:**

É essencial manter todas as aplicações e drivers atualizados e homologados conforme as especificações do fabricante para evitar problemas de compatibilidade.

Sempre confira a documentação técnica e as notas de versão antes de realizar upgrades ou instalações em ambientes de produção. Além disso, manter um ambiente de teste que espelhe o ambiente de produção pode ajudar a identificar problemas antes que eles afetem as operações diárias.
