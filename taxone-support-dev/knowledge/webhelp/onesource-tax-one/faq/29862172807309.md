**Objetivo** : Resolver o problema das cargas de dados que não chegam ao Tax One devido ao não reconhecimento de caracteres especiais, o que causa falhas na carga, mesmo quando a conclusão no Interdados On-Premise é bem-sucedida.

Passo a passo para solucionar problemas com caracteres especiais em cargas de dados para o Tax One:

  1. Abra o arquivo "interfaces-erp-taxone.txt" e identifique o PST_ID da carga em questão. Verifique o status final da carga. Se o status for "Status ODI alterado para F", significa que a carga foi finalizada e os dados deveriam ter sido enviados ao Tax One.
  2. Caso os dados não tenham sido enviados ao Tax One, verifique se a carga contém algum caractere especial que possa estar causando problemas no recebimento pelo Tax One.
  3. Utilize o Datadog para identificar se existem caracteres especiais. Pesquise pelo Lote gerado pela carga e, em seguida, pelo Protocolo. ![](https://suporte.thomsonreuters.com.br/hc/article_attachments/30136856018189)


  1. Se houver caracteres especiais, será exibida uma mensagem "Ilegal escape" mostrando a linha em que o caractere está presente.
  2. Abra o arquivo JSON da carga em questão no Notepad++ e localize a linha indicada pela mensagem "Ilegal escape". Identifique o caractere especial presente.![](https://suporte.thomsonreuters.com.br/hc/article_attachments/30136812289933)


  1. Solicite ao cliente que verifique no ERP onde está a informação contendo o caractere especial e faça a remoção do caractere, como a barra invertida "\" no exemplo fornecido.
  2. Após a remoção do caractere especial realize uma nova carga de dados.


**Observação** : Caso o cliente não consiga encontrar o arquivo JSON, pode ser que ele não esteja sendo gerado. Nesse caso, o cliente pode seguir o processo descrito no KBA "[Interfaces] [ORACLE Interdados - V11R01 EBS] [Web Service] Geração do arquivo Json" para gerar o arquivo JSON.
