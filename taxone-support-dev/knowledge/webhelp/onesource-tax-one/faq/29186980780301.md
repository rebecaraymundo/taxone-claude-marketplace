Quando o relatório "**Conferência Contas x Referencia** l" não abre na aba de processos gerados após sua geração no sistema, a causa pode ser a ausência de dados para os parâmetros selecionados ao gerar o relatório. Nesse cenário, sem dados para processar, o sistema não gera o arquivo do relatório. Através do log do processo de geração é possível verificar que há uma mensagem, descrevendo que não foram encontrados dados para os parâmetros selecionados.

Passo a passo para resolver o problema do relatório "Conferência Contas x Referencial" não estar sendo gerado:

  1. Verifique se os parâmetros selecionados para a geração do relatório estão corretos e se há dados disponíveis para esses parâmetros.
  2. Vá até a aba de geração de relatórios e selecione novamente o relatório "Conferência Contas x Referencial".
  3. Selecione os parâmetros adequados que você sabe que contêm dados.
  4. Gere o relatório novamente.
  5. Se o relatório for gerado, ele deverá aparecer na aba de processos gerados para visualização e download.
  6. Após seguir os passos acima, se o problema persistir, entre em contato com o suporte técnico fornecendo detalhes adicionais do problema para uma investigação mais aprofundada.


**Dicas e Recomendações:**

  * Este relatório é para apoio ao usuário, com objetivo de efetuar algumas validações de inconsistência de dados ou parâmetros. Sendo assim, só deverá ser gerado se houver dados 'inconsistentes" enquadrados nessas validações.
  * Assegure-se de revisar o [Manual Funcional](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/Help/WebHelp/Manuais_Funcionais/SPED/ECD_Escrituracao_Contabil_Digital/SPED_ECD_Relatorios/SPED_ECD_relatorios_confcontasrefe.htm>) do módulo para entender melhor os requisitos de dados para cada tipo de relatório. Isso pode evitar problemas de geração de relatórios no futuro devido à seleção incorreta de parâmetros.
