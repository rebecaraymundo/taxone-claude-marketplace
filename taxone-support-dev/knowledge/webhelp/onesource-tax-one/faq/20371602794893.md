Quando o cliente\consultoria está com versões muito inferiores do último patch (Patch 046), por exemplo, patch 033 e não deseja mais utilizar o ODI logo será necessário realizar uma nova instalação, onde neste caso, seria para a última versão, Patch 046 sem ODI.

Passo a passo para atualização do Patch 046 sem ODI quando o cliente está com mais de três patches desatualizados:

  1. Realize todo o processo inicialmente em um ambiente UAT (User Acceptance Testing) para testes. Somente após validação, faça a atualização no ambiente produtivo.

  2. Verifique se os seguintes requisitos estão atendidos:

     * Versão do EBS R12.2 ou superior
     * Versão do Oracle 19c com Oracle Client 32 bits
     * Versão do Java 8 JDK 32 bits (download: [https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html - jdk-8u202-windows-i586.exe](<https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html%20-%20jdk-8u202-windows-i586.exe> "Hyperlink \[https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html - jdk-8u202-windows-i586.exe\]"))
     * Observação: Java 11 Correto não é homologado.
  3. Faça um mapeamento de todas as customizações existentes no cliente e realize backup dessas customizações.

  4. Realize um merge das customizações com as queries atuais de cada SAFX que houver customização.

  5. Aplique as customizações em um objeto customizado, apontando para o sinônimo correspondente.

  6. Realize testes de validação.

  7. Faça o download do Patch 046 sem ODI e as documentações no link: <https://suporte.thomsonreuters.com.br/hc/pt-br/articles/360054863151--Interface-Oracle-Interdados-V11R1-Extrator-Oracle-EBS>


Lembre-se de seguir rigorosamente os procedimentos e observações mencionados, uma vez que a atualização envolve a realização de uma nova instalação devido à grande defasagem de versão do cliente em relação ao Patch 046.
