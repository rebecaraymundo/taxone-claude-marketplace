## Problema:

Usuários notaram que após a importação de dados para tabelas relacionadas à Produção Conjunta - Terceiros, o campo DATA_INI_OP é automaticamente preenchido com a data padrão 01/01/1900, mesmo quando não é fornecido qualquer valor para esse campo.

## Causa Raiz:

A causa dessa situação é que o campo DATA_INI_OP é considerado uma chave na tabela e, portanto, não pode permanecer vazio. O sistema, por padrão, preenche automaticamente esse campo com a data 01/01/1900 quando o tipo de produção conjunta é definido como '2'.

## Resolução:

  * Verifique o tipo de produção conjunta que está sendo usado (deve ser '2' para esse comportamento ocorrer).
  * Prossiga com a importação dos dados como normalmente faria.
  * Considere a data 01/01/1900 como um valor de preenchimento padrão e ignore-a em análises subsequentes, pois ela não será gerada em obrigações fiscais.
