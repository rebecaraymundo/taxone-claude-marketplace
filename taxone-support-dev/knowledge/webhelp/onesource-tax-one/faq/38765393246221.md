# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Divergência na geração do campo UF_ORIG_DEST da SAFX07E.

O objetivo deste artigo é orientar o usuário referente à possíveis causas na divergência na geração do campo UF_ORIG_DEST da SAFX07E.

**Problema:**

Usuário relata que o campo está sendo preenchido com SA, ao invés de SP, sigla para o estado de São Paulo.

**Causa Raiz:**

O campo UF_ORIG_DEST busca valor presente no arquivo XML na tag G_SUPPLIERS campo STATE. O valor presente no campo STATE do XML está SAO PAULO e o tamanho do campo é 2. Com isso, após o processamento o campo UF_ORIG_DEST assumia o valor SA.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/38765399346445)

**Resolução:**

  * Alterar na origem o valor enviado ao XML para a tag**** G_SUPPLIERS campo STATE.


**Próximos Passos:**

Após realizar a alteração, recomendamos:

  * Realize novos processamentos e verifique se o campo está sendo gerado no Interdados de acordo com o que está sendo enviado no arquivo XML, tag G_SUPPLIERS campo STATE.


**Dicas e Recomendações:**

Para evitar problemas similares no futuro, siga essas melhores práticas:

  * Mantenha um registro atualizado da configuração do ambiente, incluindo alterações feitas nos parâmetros.
  * Estabeleça um processo de revisão para garantir que as configurações estejam corretas antes de realizar homologações ou testes.
  * Considere testar as alterações em um ambiente isolado para limitar potenciais impactos.
