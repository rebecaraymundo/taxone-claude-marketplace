# Identificação do Campo de Dados para Carregar Informações do Tipo de Documento no TAX ONE

## Problema:

O usuário está tem dúvidas em como inserir um novo código no campo "tipo de documento" abaixo que fica na capa da nota fiscal

 _ref.: Layout SAFX07-Campo 05_

 _Tela Basicos > Data Warehouse > Manutenção > Documento Fiscal > Doc Fiscal de Mercadoria e Serviços)_

## Causa Raiz:

O problema ocorre devido a uma falta de clareza sobre qual campo ou tabela do sistema o TAX ONE consulta para obter o valor do campo "tipo de documento".

## Resolução:

Localize a tabela **SAFX2005 (BÁSICOS > DATA WAREHOUSE > Manutenção > Códigos > Tipo de Documento)**. Este é o local onde o TAX ONE consulta as informações para carregar os dados no campo "tipo de documento"

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/41203764821005)

Verifique o conteúdo do **campo 05** na tabela **SAFX07** , pois é neste campo que o **"tipo de documento"** é preenchido com os valores consultados na tabela SAFX2005.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/41203764823437)

Certifique-se de que as informações cadastradas na tabela SAFX2005 estejam corretas e completas. Qualquer inconsistência pode causar erros no SPED.

## Próximos Passos:

Após realizar as verificações e ajustes necessários, monitore o sistema para garantir que os dados estão sendo carregados corretamente. Caso o problema persista, entre em contato com o suporte para assistência adicional.

## Dicas e Recomendações:

Recomenda-se a revisão periódica das tabelas e campos no seu sistema para assegurar que todas as informações estejam atualizadas e precisas. Isso pode prevenir problemas futuros com o TAX ONE. Utilize o manual de layout SAFXs como um guia para entender melhor o mapeamento de dados.
