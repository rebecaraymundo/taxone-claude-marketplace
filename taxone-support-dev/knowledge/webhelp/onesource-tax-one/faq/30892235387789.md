# [Interfaces] [ORACLE Interdados - V11R01 EBS] [Geral] Como corrigir o problema de não carregamento das tabelas de importação do cadastro do Tax One no Interdados V11R01?

Quando o consultor ou cliente clica em _Menu > Cadastro Tax One > Configuração e Importação Cadastro Tax One_, espera-se que sejam exibidos os Códigos de Tabela e as Tabelas conforme a imagem na página 2 do Manual do Usuário, na seção de interface e cadastro do Tax One:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/30892225962509)

No entanto, as informações não estão sendo carregadas conforme a imagem abaixo:


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/30892235375885)


A seguir, algumas verificações que o consultor ou cliente deve realizar:

  1. **Verificação do Java:**
     * De acordo com o Manual de Pré-requisitos do projeto (link disponível no final deste artigo), em uma configuração de máquina dedicada, o cliente deve ter instalado o Java JRE8 32 bits ou 64 bits. O Java 11 não foi homologado para o Interdados V11R01.
  2. **Configuração de Máquina Dedicada:**
     * A Thomson Reuters recomenda que cada ambiente seja configurado em uma máquina dedicada, conforme os pré-requisitos informados na seção "Configuração de Máquina Dedicada". A Thomson Reuters não se responsabiliza nem oferece suporte a máquinas dedicadas que contenham dois ambientes, como um ambiente de Homologação e outro de Produção configurados na mesma máquina, seja ela Desktop/Servidor, ou dois produtos na mesma máquina, devido ao uso de versões diferentes do Java.
  3. **Verificação das Portas:**
     * Verifique se as portas 80 e 443 estão abertas, pois são necessárias para que as URLs das APIs de Importação e Envio de Dados funcionem corretamente no Interdados X TaxOne.
  4. **URLs das APIs:**
     * Certifique-se de que as URLs das APIs de Importação e Envio de Dados estão configuradas corretamente:
       * **API Importação de Dados:** https://uat01.taxone.thomsonreuters.com/taxone-integrator
       * **API Envio de Dados: **https://uat01.taxone.thomsonreuters.com/safx
  5. **Configuração do Arquivo Exporta. ini:**
     * Após verificar todos os pontos acima, no arquivo Exporta.ini, verifique se existem espaços nas parametrizações e se a pasta \bin no JAVAHOME=C:\Program Files\Java\jre1.8.0_202 existe. O exemplo abaixo é como não deve estar configurado.
     * Se você se deparar com o exemplo acima, altere-o conforme indicado a seguir: O \bin no final do parâmetro JAVAHOME deve sempre existir para que ele possa acessar os executáveis do Java. Após seguir os passos acima, feche o Interdados, abra-o novamente e execute o processo de importação das tabelas do Tax One.


Se o problema persistir, entre em contato com o Service Desk da Thomson Reuters.
