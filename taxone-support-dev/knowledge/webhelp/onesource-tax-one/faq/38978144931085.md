### **Erro em carga/importação de Lotes: Requisição Indevida de Processamentos OBI**

**Problema:** Durante a integração de lotes de documentos, o sistema retorna um erro, como no caso das SAFX112 e SAFX113, mesmo que a importação dos documentos principais (CT-es) ocorra com sucesso.

**Causa Raiz:** O erro acontece porque a **requisição na API Gateway do OBI** aciona o processamento de uma SAFX (ex: SAFX112) mesmo quando o arquivo de origem (XML) não contém as informações necessárias para gerar esse registro. A lógica do sistema é que, ao ser acionado, o processamento será executado.

Por exemplo, se a sua requisição inclui o `cod_processamento` da SAFX112 (que trata de observações) em um XML que não tem observações, o sistema tentará processá-la e gerará um erro na carga/importação.

**Resolução:** Para solucionar o problema e evitar que esses erros sejam gerados, a **requisição na API Gateway do OBI** deve ser ajustada. O ideal é que a **requisição na API Gateway** tenha uma validação prévia para verificar o conteúdo do arquivo XML e, somente se as informações necessárias para aquela SAFX estiverem presentes, acionar o processamento correspondente.

**Próximos Passos:** Entre em contato com a equipe de TI ou o responsável pela **requisição na API Gateway do OBI** para que essa condição de validação seja implementada no código.

**Dicas e Recomendações:** É fundamental que a **requisição na API Gateway do OBI** possua a inteligência necessária para avaliar o conteúdo dos arquivos de origem antes de acionar os processamentos, evitando assim erros indevidos no Tax One.
