### **Problemas na leitura de tags XMLs devido ambiguidade no nome das tags**

**Problema:** O OBI não consegue ler o valor de um campo a partir de uma estrutura XML, mesmo que o caminho das tags pareça estar correto. Por exemplo, isso pode acontecer com o campo `VLR_TOT_NOTA`.

**Causa Raiz:** O problema ocorre devido a uma ambiguidade na estrutura do XML. A mesma tag (`<servico>`) é utilizada em diferentes níveis da hierarquia (`<servico><valor><servico>`). O OBI, como a maioria dos sistemas de leitura de XML, precisa de um caminho claro e único para identificar e extrair os dados. A duplicação do nome da tag causa um conflito que impede o reconhecimento.

**Resolução:** Para solucionar o problema, é necessário corrigir a estrutura do XML para eliminar a ambiguidade.

  1. **Ajuste o XML:** Renomeie a tag aninhada para um nome único. O seu XML deve ser ajustado, por exemplo, de:

XML

         <servico>
           <valor>
             <servico>10518.39</servico>
           </valor>
         </servico>

para:

XML

         <servico>
           <valor>
             <valorServico>10518.39</valorServico>
           </valor>
         </servico>

  2. **Atualize a Configuração no OBI:** Após a alteração no XML, a parametrização para o campo afetado no OBI (como o `VLR_TOT_NOTA`) deve ser ajustada para o novo caminho, por exemplo, `valor;valorServico`.


**Próximos Passos e Recomendações:**

  * Para evitar erros futuros, evite usar nomes de tags duplicados em diferentes níveis de hierarquia no mesmo XML.
