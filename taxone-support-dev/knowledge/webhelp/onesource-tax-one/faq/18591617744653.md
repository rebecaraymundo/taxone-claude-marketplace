Objetivo: auxiliar na solução de falta de informações referente ao campo IND_CRED_ICMSS quando realizada a extração da SAFX08.

  1. Quando efetuada a extração de notas que contém valores de substituição tributária, e o campo 78 da SAFX08 (IND_CRED_ICMSS) não está sendo populado com a informação "S" para os itens que contém valores, verifique se esta nota esta respeitando a seguinte regra:


  * Se campo MOVTO_E_S diferente de Saída e valor de BASE_SUB_TRIB_ICMS for maior que zero, então campo receberá 'S';
  * Se campo MOVTO_E_S igual Saída e valor de BASE_SUB_TRIB_ICMS for maior que zero, então campo receberá '@';
  * Se campo MOVTO_E_S igual Saída e valor de BASE_SUB_TRIB_ICMS for igual a zero, então campo receberá 'N';


  1. Caso esteja realizando a importação para o DW, também existe uma regra de preenchimento com algumas condições quando este campo recebe arroba:


IF P_IND_CRED_ICMSS IS NULL THEN

/* PARA REGISTROS DE SAIDA OU ENTRADA COM NORM_DEV = '2' GRAVAR 'S' CASO ESTEJA NULO */

IF (P_MOVTO_E_S = '9') OR

(P_MOVTO_E_S IN ('1', '2', '3', '4', '5') AND P_NORM_DEV = '2') THEN

IND_CRED_ICMSS_W := 'S';

ELSE

IND_CRED_ICMSS_W := 'N';

END IF;

ELSIF P_IND_CRED_ICMSS NOT IN ('S', 'N') THEN

P_VALOR_ERRO := P_IND_CRED_ICMSS;

COD_ERRO_W := 90788;

RAISE ERRO_W;

ELSE

IND_CRED_ICMSS_W := P_IND_CRED_ICMSS;

END IF;
