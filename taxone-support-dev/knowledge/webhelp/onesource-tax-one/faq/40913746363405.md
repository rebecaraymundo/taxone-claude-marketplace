## Problema:

Erro de comunicação ao extrair safx via OBI na Namespace.

## Causa Raiz:

O problema é causado por uma mudança no certificado do lado do OBI, que necessita da atualização para um novo certificado.

## Resolução:

  1. Acessar a transação STRUST.
  2. Selecionar o diretório SSL cliente SSL Client(Standard).
  3. Clicar no botão “Importar Certificado”.
  4. No popup, selecionar o arquivo "Sectigo Public Server Authentication Root R46 1.crt".
  5. Confirmar com o flag verde (ENTER).
  6. Confirmar a permissão solicitada pelo SAP.
  7. Disponibilizar o conteúdo do arquivo nos campos respectivos e pressionar o botão: “Incluir na lista de certificados”.
  8. Após realizar essa etapa, o certificado estará disponível na aba Lista de certificado.
  9. Concluir o procedimento clicando no botão salvar no menu.


##

### Próximos Passos:

Após a atualização do certificado, monitorar o sistema para verificar se a comunicação foi estabelecida corretamente. Caso o erro persista, entre em contato com o suporte técnico para uma análise mais detalhada.

##

### Dicas e Recomendações:

É recomendável manter todos os certificados atualizados e verificar regularmente as configurações de segurança para evitar erros de comunicação. Mantenha-se informado sobre as atualizações do sistema OBI para garantir compatibilidade.
