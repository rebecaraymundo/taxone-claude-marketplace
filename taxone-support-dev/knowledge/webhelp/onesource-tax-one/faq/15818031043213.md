# [SPED][EFD - ECD] TAXONE - EFD  ECD - Como fazer a geração das demonstrações contábeis e meio magnético com erro ORA-06512 (linhas 9483 e 10860)

Passo a passo para corrigir o erro ORA-06512 na geração de demonstrações contábeis e meio magnético, especificamente nas linhas 9483 para demonstrações contábeis, e 10860 para o meio magnético.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/15817939208333)

Ao gerar as demonstrações contábeis no menu Geração > Demonstrações Contábeis, o logo apresenta o erro ORA-106512.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/15817848765965)

Meio magnético

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/15817906501773)

Na geração do meio magnético, os registros não são gerados, e o log apresenta o mesmo erro.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/15817841903885)

1\. Para corrigir o erro, acesse o módulo Básicos > Ferramentas > Parâmetros do sistema > Preferências.


2\. Verifique o valor configurado para o **Número da Operação de COMMIT**.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/15817992172941)

3\. De acordo com o manual operacional, o valor recomendado é **500** operações/transações.


4\. Caso o valor esteja muito superior a 500, ajuste-o para um valor adequado.




**Nota** : O COMMIT é um comando que confirma as operações a serem registradas no banco de dados. Nesse campo, deve ser informado o número de operações/transações a serem atualizadas nas tabelas do banco de dados a partir das informações existentes na área temporária (Rollback). A cada 500 operações/transações executadas, as tabelas do banco de dados são atualizadas com as informações da área temporária (Rollback).




5\. Após ajustar o valor, efetue novamente a geração das demonstrações contábeis e do meio magnético.
