## **Problema:**

Após o vencimento do Certificado de Autenticação SSO, os acessos são desabilitados e o usuário deve realizar atualização do arquivo XML para Single Sign On (SSO) com SAML e solicitar que seja aplicado no sistema Tax One.

## **Causa Raiz:**

A atualização é necessária devido à expiração do certificado anterior, que impacta a autenticação SSO do usuário no sistema Tax One.

## **Resolução:**

Para solicitar a atualização do Certificado o usuário deve seguir os passos a seguir:

  1. Abrir chamado através do portal com título: ATUALIZAÇÃO DO CERTIFICADO DE AUTENTICAÇÃO SSO TAX ONE, ou algum texto similar que dê a entender que se trata de uma atualização e não de uma implantação.
  2. No corpo do ticket, é necessário que o usuário confirme o ambiente correto que deve ser atualizado UAT ou PROD e deve ser enviado o DB ID do ambiente no Tax One.
  3. É importante que o cliente que gere o novo arquivo XML com o certificado atualizado e este seja anexado ao chamado.
  4. O Time de Suporte fará o envio do novo arquivo XML para a equipe de SSO global para atualização no sistema.
  5. Aguarde a confirmação de que o certificado foi atualizado na plataforma.
  6. Após receber a confirmação o usuário deve validar o funcionamento do SSO no sistema Tax One.


## Próximos Passos:

Após a atualização e confirmação do funcionamento do SSO, nos manteremos disponível para qualquer suporte adicional que o usuário possa necessitar.

## Dicas e Recomendações:

Recomenda-se monitorar a validade dos certificados SSO para evitar interrupções no acesso ao sistema. Planeje as renovações de certificados com antecedência para garantir uma transição suave.
