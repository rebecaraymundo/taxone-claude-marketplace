# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Resolução de Dados Vazios na Exportação e Importação com TaxOne Caracteres Especiais

O objetivo deste artigo é orientar o usuário à possíveis causas de dados vazios na exportação e importação com TaxOne

**Problema:**

Clientes enfrentam problemas de campos vazios na exportação da SAFX08 no Interdados e divergência na importação para o TaxOne, resultando em valores zerados dos itens processados.

Causa Raiz:

A causa do problema está nos caracteres especiais presentes nos campos exportados que, ao serem importados para o TaxOne, causam a quebra do arquivo JSON, impedindo que a carga seja processada corretamente, por exemplo os caracteres " : " , " ; ".

**Resolução:**

Para resolver o problema, siga os passos abaixo:

  * Acesse o layout da SAFX08E através da opção "procedimentos";
  * Localize o campo _DESCRICAO_COMPL_ e clique duas vezes sobre ele;
  * Insira os caracteres especiais a serem removidos no campo indicado, utilizando o delimitador "|". Por exemplo, insira ".|;|&|!|:".
  * Salve as modificações e execute novas cargas para verificação.


**Próximos Passos:**

Após realizar os ajustes, monitore as próximas importações para garantir que os campos são preenchidos corretamente e que não há divergências ou campos vazios afetando o processamento dos dados.

**Dicas e Recomendações:**

É recomendável revisar periodicamente os layouts de exportação e importação para se assegurar de que não existem inconformidades nos dados, especialmente após atualizações do sistema ou mudanças nos procedimentos.
