Este artigo tem como objetivo direcionar uma possível correção para o erro "O campo Codigo de Tipo serviço não está cadastrado" ao processar uma carga da SAFX09E originada do Interdados

Problema:

Ao fazer a importação da SAFX09E no TaxOne e o código de tipo de serviço cadastrado não foi mapeado corretamente no sistema TAXONE, retorna a mensagem de erro em questão.

Causa Raiz:

Ausência do cadastro do código de serviço enviado no TaxOne

Resolução:

Para resolver o problema, siga os passos abaixo:

1 -Valide a regra De/Para código 105 Cod Serviço para confirmar que a regra consta cadastrada corretamente;

2-Confirme no campo 12 do XML para validar que o a informação está sendo enviada conforme a regra De/Para 105 Cod Serviço;

Caso os campos esteja corretos na regra e no JSON de envio, será um indicador que o o código de serviço ainda não foi cadastrado no TaxOne, dessa forma, será necessário cadastrar manualmente o código dentro do TaxOne ou executar a SAFX2018 - CODIGO SERVIÇO.

Após a finalização da execução, gere novamente a SAFX09E e confirme se o mesmo erro será retornado no TaxOne.

Dicas e Recomendações:

Mantenha os parâmetros da rotina SAFX atualizados e faça revisão periódica para garantir que todos os mapeamentos estejam corretos. Isso evitará problemas de divergência no futuro e garantirá a integridade dos dados dentro do sistema TAXONE.
