# Resolução de Problema no Campo "COD_SERVICO_NAT_REND" Não Sendo Preenchido

## Problema:

O campo "COD_SERVICO_NAT_REND" não estava sendo preenchido após a execução da carga no Interdados Cloud PRD, via procedimento 53.

## Causa Raiz:

A causa do problema foi a parametrização incorreta do tipo de imposto no XML. O tipo "LACLS_IRPF_WT" estava cadastrado como IRRF, quando deveria estar como IR, o que causava a entrada errada no processo de PCC em vez do processo de IR.

## Resolução:

  1. Acesse o modulo de parametrização do Interdados Cloud.

  2. Altere o tipo de imposto "LACLS_IRPF_WT" de IRRF para IR.

  3. Verifique que todos os tipos de impostos relacionados a IR estejam configurados para serem mapeados como IR.

  4. Execute novamente o procedimento 53 para processar a carga.


## Próximos Passos:

Após realizar as alterações e executar o procedimento, verifique se o campo "COD_SERVICO_NAT_REND" está sendo corretamente preenchido. Monitore os resultados das próximas cargas para garantir a consistência dos dados.

## Dicas e Recomendações:

Verifique regularmente as parametrizações no sistema para garantir que todas estejam corretas e atualizadas, evitando problemas nos processos de carga e relatórios. Em caso de dúvidas ou problemas recorrentes, não hesite em entrar em contato com o suporte.
