## Problema:

Ao executar a importação de dados na interface Interdados, o campo **IND_PRODUTO** não é preenchido corretamente, resultando em erro na importação e bloqueio do projeto.

## Causa Raiz:

O problema ocorre devido à ausência de valores na tag **ITEM_TYPE** no arquivo XML de origem. Sem essa informação, a interface não consegue aplicar o DE-PARA configurado na regra 04, resultando em valores inválidos (@) no campo **IND_PRODUTO**.

## Resolução:

Siga os passos abaixo para corrigir o problema:

  1. Verifique no arquivo XML de origem se o campo **ITEM_TYPE** está preenchido corretamente para os produtos que apresentam erros.
  2. Certifique-se de que o DE-PARA correspondente à regra 04 no Interdados está configurado corretamente.
  3. Realize a execução da carga novamente, utilizando a configuração correta do DE-PARA na regra.
  4. Teste a importação para confirmar se o campo **IND_PRODUTO** foi preenchido corretamente.


## Próximos Passos:

Após a implementação das correções, realize os seguintes passos:

  * Confirme o preenchimento correto do campo **IND_PRODUTO** nos produtos impactados.
  * Realize testes adicionais para garantir que todos os valores esperados estão sendo populados corretamente.
  * Monitore futuras execuções de importação para validar a estabilidade da solução aplicada.


## Dicas e Recomendações:

Para evitar problemas semelhantes no futuro, recomendamos:

  * Manter o XML de origem sempre atualizado e verificar a integridade dos dados antes da importação.
  * Realizar revisões periódicas das regras de DE-PARA configuradas na interface Interdados.
  * Documentar os procedimentos de configuração e importação para facilitar consultas futuras.
  * Em caso de dúvidas, entre em contato com o suporte técnico especializado para assistência.


Para mais informações sobre a configuração de DE-PARA e gestão de importação de dados, consulte nossa [base de conhecimento](<https://suporte.tr.com/interdados>).
