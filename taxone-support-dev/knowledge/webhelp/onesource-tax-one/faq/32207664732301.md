Problemas ao reabrir um evento de abril de 2022 no sistema Onesource, pois o evento permanecia com o status de "enviando para mensageria" e não é enviado ao governo, retornando ao status de "Pendente de envio". A causa desse problema é a utilização de uma versão antiga do layout do Reinf (1.5.1), que não permite a visualização e processamento do evento na versão atual (2.1.2). Além disso, é necessário que exista um evento R-1000 autorizado pelo fisco com a data de validade aberta para que o evento seja processado corretamente.

Passo a passo para resolver problemas com reabertura de eventos na versão antiga do Reinf:

  1. Verifique se há um evento R-1000 recebido com sucesso pelo fisco e com data de validade aberta.
  2. Caso necessário, encerre a validade do evento R-1000 atual no e-CAC e envie um novo R-1000 gerado na versão 2.1.2.
  3. No campo de informações do contribuinte, informe a data inicial como o primeiro dia do mês subsequente ao fechamento no e-CAC, deixando a data final em branco.
  4. Na tela de **geração prévia dos cadastros** , informe o mês atual como data inicial e final. O sistema irá gerar o XML corretamente com a data final em branco, igual ao cadastro.
  5. Prossiga com o evento de reabertura.
  6. Após seguir os passos acima, monitore o status do evento para confirmar que ele foi reaberto com sucesso. Se o problema persistir, entre em contato com o suporte técnico.


**Dicas e recomendações:**

Para evitar problemas similares no futuro, sempre verifique se está utilizando a versão mais recente do software. Manter o software atualizado garante compatibilidade com as exigências do governo e melhora a eficiência dos processos.
