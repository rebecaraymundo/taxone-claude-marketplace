# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Como corrigir o preenchimento incorreto do campo COD_MODELO_COTEPE na SAFX07 durante a carga de dados

## Problema

Durante a validação da carga de dados no sistema, alguns documentos de contas de consumo estão retornando um código de modelo inconsistente no campo **COD_MODELO_COTEPE** da SAFX07, mesmo estando corretamente preenchidos no Oracle. O erro impede a correta integração e processamento dos dados na interface.

## Causa Raiz

A origem do problema está na ausência de uma regra de conversão (de-para) para modelos de documentos específicos (como NFS, NFEE e NTSC) na configuração do sistema. Isso faz com que o campo **COD_MODELO_COTEPE** seja preenchido de forma incorreta durante o processamento dos dados pelo Interdados, gerando inconsistências em relação à Tabela 4.1.1 do SPED (documentos fiscais do ICMS).

## Resolução

  1. Acesse o menu **Cadastro > Regras por Descrição > Regra 12 Modelo Docto**.
  2. Inclua uma definição de regra de-para para os modelos de documentos afetados. Exemplo:
     * **NFEE** (Nota Fiscal de Energia Elétrica) → código do modelo = **66**
     * Realize o mesmo procedimento para os demais modelos identificados no seu cenário, como **NFS** e **NTSC**.
  3. Referencie a regra no layout do procedimento **SAFX07** :
     1. Acesse **Procedimentos** e localize o procedimento 7.
     2. Clique em **Mais > Layout**.
     3. Localize o campo **229** (correspondente ao **COD_MODELO_COTEPE**).
     4. Dê duplo clique no campo e preencha o campo **Regra De-Para** com a **regra 12**.
     5. Salve as alterações.
  4. Reprocese o **PST_ID** gerado no Interdados ou realize uma nova carga de dados.

## Próximos Passos

  * Após realizar o procedimento de configuração e reprocesar a carga, verifique se o campo **COD_MODELO_COTEPE** está sendo preenchido corretamente para todos os modelos de documentos afetados.
  * Valide a integração e o processamento dos dados na interface, garantindo que não haja mais inconsistências.
  * Caso o problema persista, reúna evidências e entre em contato com o suporte, informando as ações realizadas.


## Dicas e Recomendações

  * Sempre mantenha atualizadas as regras de de-para para todos os modelos fiscais utilizados no processo.
  * Consulte a [Tabela 4.1.1 da EFD-ICMS/IPI](<https://www.gov.br/receitafederal/pt-br/acesso-rapido/legislacao/consulta-legislacao/instrucoes-normativas-e-atos-declaratorios-executivos/efd-icms-ipi/tabelas-da-efd/tabela-4-1-1-documentos-fiscais-do-icms>) para garantir a correta correspondência dos códigos de modelos fiscais.
  * Realize testes em ambiente de homologação/UAT antes de aplicar em produção.
  * Documente as alterações realizadas para fácil rastreio em futuras manutenções ou auditorias.
  * Caso novos modelos sejam adicionados ao seu processo, lembre-se de atualizar as regras de-para conforme necessário.
