### **Cancelamento de NFe (XML) via OBI: Procedimento de Integração (SAFX07)**

**Problema:** O usuário está enfrentando dificuldades em integrar informações de **Notas Fiscais Eletrônicas canceladas** via OBI, pois não sabe como utilizar o XML do evento de cancelamento para realizar o procedimento no Tax One.

**Causa Raiz:** O sistema Tax One não processa o XML de evento de cancelamento isoladamente para efetuar a baixa da nota. Para cancelar um documento, é exigida uma nova carga completa da **SAFX07** , com todos os campos preenchidos e o campo **SITUACAO** alterado para **"S"** (Situação de Cancelado).

**Resolução:** Para integrar o cancelamento de uma nota fiscal, é necessário desenvolver uma rotina que **unifique os dados** do **XML de Autorização** da nota com o **XML do Evento de Cancelamento**. Esse XML com as informações deve ser integrado com o OBI para que a SAFX completa seja gerada, alterando o campo **SITUACAO** para **"S"**.

**Dicas e Recomendações:**

  * Acompanhe nosso **Roadmap** e **Informativos de Versão**. Existe uma evolução em nosso backlog para tratar cancelamentos de nota via XML de NF-e no OBI, sem a necessidade da integração do XML completo da autorização.
  * Enquanto a nova funcionalidade não é liberada, o desenvolvimento do merge de XML é o procedimento obrigatório para garantir a correta baixa do documento no Tax One.
