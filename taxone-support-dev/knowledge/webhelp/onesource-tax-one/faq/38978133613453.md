# Como Evitar a Duplicidade de Tarefas ao Criar Fluxos no Tax Automation

###

Ao criar fluxos no Tax Automation, os usuários relatam que as tarefas foram multiplicadas para cada estabelecimento, gerando duplicidade de tarefas vinculadas ao mesmo fluxo.

## Causa Raiz

Esse problema ocorre quando o fluxo é criado com o parâmetro "Estabelecimentos = Todos" ao acessar o Tax Automation. Como resultado, o sistema vincula múltiplas rotinas dos estabelecimentos dentro do mesmo fluxo, gerando duplicidade.

## Resolução

Siga os passos abaixo para evitar a duplicidade de tarefas nos fluxos:

  1. Acesse o Tax Automation e, ao criar um novo fluxo, certificando-se de que está logado em apenas um estabelecimento único.
  2. Ao criar o fluxo, não selecione "Todos os Estabelecimentos". Escolha apenas um estabelecimento para a vinculação inicial.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40181166968845)

3\. Após criar o fluxo com uma rotina fiscal, clique em "Próximo Passo" para vincular os demais estabelecimentos ao fluxo.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40181166972301)

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40181192299789)

4.Salve o fluxo após realizar a vinculação com os estabelecimentos necessários.

Importante: Caso o fluxo já criado apresente tarefas duplicadas, inative-o e crie um novo fluxo seguindo as orientações acima.

## Próximos Passos

Após criar ou ajustar o fluxo:

  * Execute o fluxo para testar se as tarefas vinculadas estão corretamente parametrizadas sem gerar duplicidade.
  * Monitore o fluxo durante a execução para validar se as obrigações fiscais são geradas corretamente para cada estabelecimento.
