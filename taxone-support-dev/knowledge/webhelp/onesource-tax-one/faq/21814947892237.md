Para saber como proceder em casos de teste de latência da aplicação Tax One e quando perceber problemas de latência, lentidão ou performance, siga as orientações:


Se o Kit de performance para teste de latência não estiver operacional, os clientes devem realizar um teste de ping nos seguintes endereços:

  * ping prod01.taxone.thomsonreuters.com
  * ping uat01.taxone.thomsonreuters.com


Em situações em que o cliente relata problemas de latência, lentidão ou performance, deve-se seguir o fluxo abaixo:

  1. Verificar se há algum alerta de incidente no sistema. Se houver, proceder com as tratativas específicas para o caso.
  2. Caso não existam incidentes, questionar se o problema ocorre de forma geral ou em módulos ou processos específicos.
  3. Avaliar a necessidade de realizar um acesso remoto para observar o usuário utilizando o sistema e identificar possíveis problemas.
  4. Se for confirmado um problema de lentidão ou performance, abrir uma demanda de PERFORMANCE para avaliação pela equipe de tecnologia.
  5. É importante documentar todas as evidências dos testes realizados e, se possível, realizar um acesso remoto para gravar os testes. O link da gravação deve ser anexado à demanda.



**Nota:** É essencial seguir todos os procedimentos estabelecidos para a abertura de demandas.
