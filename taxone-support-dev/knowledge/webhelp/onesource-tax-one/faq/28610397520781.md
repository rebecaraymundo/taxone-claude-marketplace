Caso encontre dificuldades para gerar a PERDCOMP de uma filial específica no sistema Tax One, pois o arquivo TXT não é criado, siga as orientações:

  1. Verifique o log para a seguinte informação: "Não existe movimento para o trimestre. O arquivo não será gravado. (PKG_PERDCOMP_MM_FPROC)". Se esta mensagem estiver presente, o arquivo não será criado devido à falta de movimento para o trimestre.
  2. Avaliações necessárias no ambiente:
     * Certifique-se de que as informações necessárias para o período analisado foram importadas para a SAFX48 Tabela das Operações de Exportação. Isso pode ser verificado em: Básicos > Datawarehouse > Manutenção > Exportação/Importação > Operações de Exportação. A informação de Reintegra é gerada a partir desses dados.
     * No registro, além dos dados obrigatórios, é necessário informar o campo de código NBM e inserir valores no campo 'Valor Reintegra'.
     * No módulo IPI, é necessário parametrizar a mesma NBM em: Meio Magnético > IND 320/03 PER/DCOMP > Parametrização de NBM.


**Atenção:**

1\. Após realizar as alterações e gerar o arquivo PERDCOMP, é importante verificar se o arquivo foi gerado corretamente e se todas as informações estão completas. Caso encontre problemas, entre em contato com o suporte técnico para uma revisão mais detalhada do processo.

2\. É recomendável revisar regularmente as configurações de estabelecimentos centralizadores para garantir que não ocorram problemas futuros em gerações de arquivos fiscais. Mantenha sempre uma comunicação clara com a equipe de suporte para resolver quaisquer dúvidas ou problemas que possam surgir.
