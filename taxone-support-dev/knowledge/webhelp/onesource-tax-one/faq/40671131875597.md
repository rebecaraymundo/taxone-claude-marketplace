# [Básicos][OBI - OneSource Brasil Integration] ONESOURCE Campos Gerados como "@" (Nulo) na Importação do OBI

**Problema:** Durante a importação de dados via OBI, campos que deveriam conter informações são gerados no sistema Tax One com o valor fixo **"@"** (nulo), mesmo havendo dados válidos no arquivo de origem.

**Causa Raiz:** O problema ocorre nos **processamentos padrões de SAFX para SAFX**. A configuração do processamento no OBI está definindo um valor fixo para o campo no Perfil de Saída, sobrescrevendo o dado que veio do arquivo de origem. A presença de **FIXO = "@"** no cadastro do campo é comum em versões mais antigas de processamentos padrão, onde essa regra era utilizada para forçar um valor nulo.

**Resolução:** Para solucionar o problema, é necessário corrigir ou atualizar o processamento no OBI:

  1. **Ajuste Manual:**
     * Acesse o cadastro dos campos do processamento no OBI.
     * Localize os campos que estão sendo gerados como **"@"**.
     * **Remova a configuração "FIXO = "@"** e salve o processamento.
  2. **Atualização de Processamento (Recomendado):**
     * Se você estiver usando um processamento padrão antigo, o ideal é **excluir o processamento atual** e realizar o download e a importação do pacote de processamentos mais recente disponível na Central de Ajuda. As novas versões já possuem as configurações corrigidas.
