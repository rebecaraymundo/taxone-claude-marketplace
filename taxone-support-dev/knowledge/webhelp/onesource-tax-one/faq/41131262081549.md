## Problema

Identificado erro nos saldos iniciais dos créditos da EFD Contribuições após a importação do arquivo SAFX274. Os registros importados exibem valores incorretos e, em alguns casos, são substituídos por dados com chaves diferentes.

## Causa Raiz

O arquivo SAFX274 realiza apenas a inclusão de dados, sem substituir valores existentes. Caso os créditos já tenham sido utilizados ou configurados incorretamente antes da nova importação, isso pode gerar inconsistências nos saldos iniciais.

## Resolução

  1. Confirme se os valores incorretos ainda não foram utilizados.
  2. Exclua os registros incorretos no sistema, utilizando o módulo **Ferramentas**.
  3. Realize uma nova importação do arquivo SAFX274, garantindo que os valores corretos sejam carregados.
  4. Após a importação, verifique os saldos iniciais dos créditos para confirmar que os dados estão corretos.


## Próximos Passos

  1. Realize testes para garantir que os valores importados estão corretos e podem ser utilizados normalmente.
  2. Monitore os próximos períodos da EFD Contribuições para confirmar que o problema foi resolvido e não há mais inconsistências.


## Dicas e Recomendações

  * Antes de realizar uma nova importação, certifique-se de que os registros existentes não estão sendo utilizados. Caso estejam, avalie se é necessário realizar ajustes manuais.
  * Utilize o módulo **Ferramentas** para gerenciar e excluir registros incorretos antes de importar novos dados.
  * Documente qualquer ajuste realizado no sistema, incluindo os passos seguidos e as evidências para facilitar futuros acompanhamentos.
  * Consulte o manual da EFD Contribuições e os materiais de suporte disponíveis para evitar erros de configuração.
