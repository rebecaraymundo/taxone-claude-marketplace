Se você estiver enfrentando problemas na declaração do Reinf no evento R-4020, especificamente em relação ao valor do Imposto de Renda (IR) que não está sendo registrado corretamente, mesmo com todas as telas e códigos devidamente preenchidos, a causa raiz pode ser uma discrepância entre o Código de Natureza de Rendimento e o Código de Receita configurado no sistema. Essa configuração não estava alinhada com a tabela MOR apropriada para o evento R-4020. Para resolver, siga as orientações:

  1. Acesse o módulo de Básicos > Ferramentas > Tabelas Internas > Manter > Tabelas.
  2. Abra a tabela "Código de Receita".
  3. Localize o Código de Natureza de Rendimento atualmente configurado e ajuste para o Código de Receita correto conforme a tabela MOR para o evento R-4020.
  4. Salve as alterações e realize uma nova geração prévia do movimento para o R-4020 para verificar se o problema foi resolvido.
  5. Após ajustar o código de receita, monitore os próximos eventos de R-4020 para garantir que o IR está sendo corretamente reportado. Se o problema persistir, entre em contato com o suporte técnico para uma análise mais detalhada.


**Importante:** Verifique as tabelas MOR periodicamente para garantir que todos os códigos utilizados estejam atualizados e em conformidade com as normas vigentes. Isso ajuda a prevenir discrepâncias e erros em futuras declarações.
