Dúvida

Necessidade de documentação WSDL para iniciar o projeto de migração MastersafDW x Tax One utilizando o módulo PI do SAP para enviar informações via Web Services.

Causa Raiz

A causa raiz é a falta da documentação WSDL que é necessária para a criação do serviço no SAP PI como uma definição externa.

Resolução

Acesse o Portal do suporte Thomson Reuters , vai a ONESOURCE TAX ONE na sequencia a seção FAQ >Básicos> Job Servidor:

Localize os seguintes documento os link de cada documento esta logo abaixo de cada nome.

  * WebService TAXONE CARGA DE SAFX VIA JSON POR LINHA

[WebService TAXONE CARGA DE SAFX VIA JSON POR LINHA](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/4420817442573--WebService-TAXONE-WEBSERVICE-CARGA-DE-SAFX-VIA-JSON-POR-LINHA>)

  * Webservice Carga SAFX

[Webservice Carga SAFX](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Tecnicos/Webservice_Carga_SAFX.htm>)

  * Webservice Carga SAFX Importação

[Webservice Carga SAFX Importacao](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Tecnicos/Webservice_Carga_SAFX_Importacao.htm>)


Próximos Passos

Após enviar a documentação para a equipe relevante, mantenha uma comunicação frequente para resolver qualquer dúvida que possa surgir durante o desenvolvimento da integração.

Caso haja problemas adicionais, entre em contato com o suporte da Thomson Reuters abrindo um chamado que deverá ser direcionado ao time Tax One.

Dicas e Recomendações

Verifique sempre se a versão de qualquer documento baixado é a mais recente disponível. Manter a documentação atualizada pode prevenir problemas futuros de compatibilidade e implementação.
