**Objetivo** : Solucionar o problema de cargas que não são enviadas ao Tax One, e no Log do Interdados apresenta o erro “Erro ao obter NumLote: 404 null”.

Passo a passo para resolver o erro "Erro ao obter NumLote: 404 null" ao enviar cargas ao Tax One:

  1. Após realizar a carga de qualquer SAFX e ela não ser enviada ao Tax One, solicite ao cliente o arquivo **Wrapper.log** para análise.
  2. No arquivo Wrapper.log, verifique se há a mensagem "Erro ao obter NumLote: 404 null".


Exemplo:

INFO|2704/0|Service yajsw|24-05-21 23:54:20|2024-05-21 23:54:20.516 ERROR 2704 --- [rtToTaxOneRoute] c.t.i.e.taxone.service.WebServiceTaxOne : [PST_ID: XXXX] **Erro ao obter NumLote: 404 null**
INFO|2704/0|Service yajsw|24-05-21 23:54:20|2024-05-21 23:54:20.516 ERROR 2704 --- [rtToTaxOneRoute] c.t.i.e.taxone.routes.SafxToQueueRoute : [PST_ID: XXXX] Erro obtendo Lote. Reenviando para Status Q
INFO|2704/0|Service yajsw|24-05-21 23:54:20|2024-05-21 23:54:20.531 DEBUG 2704 --- [rtToTaxOneRoute] c.t.i.e.taxone.routes.SafxToQueueRoute : [PST_ID: XXXX] StatusODI alterado para E
INFO|2704/0|Service yajsw|24-05-21 23:54:20|2024-05-21 23:54:20.531 INFO 2704 --- [rtToTaxOneRoute] c.t.i.e.taxone.routes.SafxToQueueRoute : [PST_ID: XXXX] Quantidade de registros: 0
INFO|2704/0|Service yajsw|24-05-21 23:54:20|2024-05-21 23:54:20.531 INFO 2704 --- [rtToTaxOneRoute] c.t.i.e.taxone.routes.SafxToQueueRoute : [PST_ID: XXXX] Tempo total de processamento: 7 segundos"

  1. No arquivo **Wrapper.conf**, localize a linha:
     * “wrapper.java.additional.8 = -DsafxToTaxOne.envioAutomatico=false”.
  2. Verifique se essa linha está comentada com #. Exemplo:
     * #wrapper.java.additional.8 = -DsafxToTaxOne.envioAutomatico=false
  3. Caso esteja comentada, remova o # e realize uma nova rodada de testes.
