Para resolver a duplicação de Dados no Datamart após a limpeza, siga as orientações:

Caso tenha registros gerados na EFD Contribuições mesmo após a limpeza do Datamart resultando em dados duplicados como os do Bloco 0 e F, isso pode ocorrer devido ao processo de equalização/limpeza do Datamart que foca apenas nos documentos fiscais que impactam os blocos A, C e D. Para resolver, siga as orientações:

  1. N módulo EFD Contribuições na seção de Manutenção.
  2. Verifique os registros nos Blocos 0 e F e realize a limpeza manual dessas entradas, se necessário.
  3. Para registros do Bloco M, revise a Apuração do PIS e Cofins e faça os ajustes necessários.
  4. Após realizar essas alterações, execute novamente a geração do EFD Contribuições para confirmar que os registros duplicados foram removidos.
  5. Após realizar as correções, monitore os próximos ciclos de geração do EFD Contribuições para assegurar que o problema da duplicação de dados não se repita. Caso o problema persista, entre em contato com o suporte técnico para uma análise mais aprofundada.


**Importante:** Regularmente revise e limpe os registros manuais ou importados nos módulos do sistema para evitar acumulação de dados que não são afetados pela limpeza automática do Datamart. Isso ajudará a manter a integridade e a precisão dos dados na geração do EFD Contribuições.
