# [Interfaces] [SAP – Namespace/OBTI] [Bloco K] Configuração para materiais com múltiplas classes de avaliação

Problema

O cliente Identificou no SAP múltiplas classes de avaliação para um mesmo tipo de material, a extração via Interface OBTI esta considerando considera uma, gerando divergências na EFD Fiscal (principalmente Bloco K) no campo **CLAS_ITEM** da **SAFX2013** (origem: /MSAFX/SAFTB013-CLAS_ITEM).

Causa Raiz

Essas inconsistências ocorrem porque o OBTI, não está parametrizado para lidar com múltiplas classes de avaliação associadas ao mesmo tipo de material desta forma o **CLAS_ITEM** do material específico, não está saindo de acordo com o que necessitam.

Resolução

Siga os passos abaixo para configurar o OBTI de forma que ele considere múltiplas classes de avaliação

  1. Acesse a transação`/N/MSAFX/SAFTR085` no sistema SAP (abaixo print exemplo da tela de parametrização)

  2. Configure os seguintes campos chave para segmentação adequada:

     * **Tipo do Material**

     * **Centro**

     * **Utilização do Material** (normalmente, encontrado no campo _MBEW-MTUSE_).


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40476504019981)

Próximos Passos

  * Executar uma extração de dados via Interface OBTI para validar as configurações realizadas.

  * Monitorar os dados extraídos nos próximos ciclos, garantindo que não existam divergências no **CLAS_ITEM**.


Dicas e Recomendações

  * Padronize o cadastro de materiais no sistema SAP, minimizando a variação entre classes de avaliação e tipos de materiais.

  * Realize revisões periódicas nas configurações de parametrização do OBTI para garantir que estão atualizadas conforme as mudanças no processo fiscal.

  * Considere documentar internamente os critérios utilizados para parametrização dos materiais, facilitando futuras análises e configurações.
