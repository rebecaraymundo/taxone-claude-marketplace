# [Interfaces] [SAP – Namespace/OBTI] [Web Service] Parametrização de Espaços Duplos na extração via WebServices

Problema

Diferença de tratamento de espaços em branco entre integrações via TXT e via WebService (JSON) campos descritivos chegam com espaços duplos preservados no TXT, enquanto o WebService remove os espaços duplos. Essa inconsistência impacta diretamente a qualidade e a padronização das informações recebidas, podendo gerar divergências entre o que é enviado e o que é efetivamente carregado no Tax One.

Causa Raiz

Há comportamento diferente no tratamento de espaços em branco entre os dois tipos de integração:

  * No TXT, os espaços duplos são preservados.

  * No WebService, os espaços duplos são removidos no processamento, causando inconsistências nas cargas de dados.


Trata-se de uma inconsistência no padrão de tratamento de espaços em branco entre os dois tipos de integração, afetando cargas de dados que dependem da integridade exata dos campos descritivos.

Resolução

Houve uma alteração relacionada ao tratamento de espaços duplos no processo de geração de arquivos JSON.
Essa alteração foi implementada na versão **4105** da Interface OBTI, ajustando o comportamento com relação ao espaço duplo durante a geração e o consumo dos arquivos JSON utilizados pelos WebServices.
Conforme descrito na documentação de versão (carta), a mudança passou a atuar especificamente no processo de geração de arquivos JSON, criando um parâmetro que permite alinhar o comportamento da integração via WebService com o do TXT.

  * Para que os espaços duplos **não sejam removidos** , é necessário que o parâmetro esteja **marcado**.

  * Caso o parâmetro esteja **desmarcado** , o tratamento se manterá como antes, com a **remoção** dos espaços duplos na extração via WebServices.


Próximos Passos

  * Confirmar se o parâmetro responsável pela remoção de espaços duplos em arquivos JSON está devidamente marcado.

  * Realizar testes de integração via TXT e via WebService com os mesmos dados, comparando os campos descritivos no Tax One.

  * Registrar o resultado dos testes e, se estiver de acordo, homologar o comportamento ajustado junto às áreas responsáveis.


Dicas e Recomendações

  * Manter a interface OBTI atualizada, garantindo que as correções e melhorias de tratamento de dados (como espaços em branco) estejam aplicadas.

  * Incluir cenários com espaços duplos em campos descritivos nos roteiros de teste sempre que houver atualização de versão ou mudança na integração.
