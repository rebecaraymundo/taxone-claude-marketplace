Problema

Durante o processo de atualização da versão da Interface OBTI, foi aplicado **equivocadamente** o pacote de **Instalação** , em vez do pacote de **Atualização**.

Como consequência, algumas tabelas de controle da interface (tabelas **/MSAFX**) tiveram seus dados zerados.

Causa Raiz

No portal são disponibilizados dois tipos de pacotes para a Interface OBTI, conforme demonstrado no print exemplos abaixo:

  * Instalação:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/41212945772173)

  * Atualização:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/41212945773453)

A diferença entre esses pacotes está relacionada à Request Customizing:

  * Pacote de Instalação: ela é responsável por criar as tabelas de controle da Interface OBTI e carregar as configurações iniciais.

  * Pacote de Atualização: ela é responsável apenas por atualizar as tabelas já existentes, preservando os dados de controle e parametrizações de Namespace previamente configurados.


Quando, por engano, o pacote de Instalação é aplicado em um ambiente em que a Interface OBTI já está operacional, as tabelas de controle existentes são recriadas ou sobrescritas, resultando em limpeza (zeramento) das informações nelas contidas.

Resolução

Para corrigir o problema, pode ser adotado o seguinte procedimento:

  1. Restauração dos dados

     * Realizar o upload/restauração do backup das tabelas**/MSAFX** através da transação `/N/MSAFX/SAFTR503`.

     * Após essa restauração, a Interface volta a exibir a versão antiga na transação `/N/MSAFX/SAFTR001A` e pode ser que ocorra de ser exibida a seguinte mensagem no rodapé da transação "Webservice – Request Customizing incompatível com a versão pacote WS”

  2. Correção da Request de Customzing

     * Em seguida, aplicar a Request Customizing correspondente ao pacote de **Atualização** da Interface OBTI.

     * Dessa forma, as tabelas de controle são atualizadas corretamente, preservando os dados restaurados e ajustando a Interface para a versão desejada.


Próximos Passos

  * Validar se todas as tabelas de controle **/MSAFX** foram restauradas corretamente (comparar com ambiente de referência, se disponível).
  * Verificar na transação `/N/MSAFX/SAFTR001A` se a versão da Interface OBTI esta consistente.
  * Criar um checklist de atualização da Interface OBTI, incluindo a confirmação explícita do tipo de pacote antes da importação.


Observação: O procedimento aqui descrito serve como referência para orientar a correção de um procedimento aplicado de forma incorreta pelo cliente/consultoria. Nem todas as tabelas permitem a realização de backup e podem ser necessários ajustes manuais adicionais para restaurar alguma parametrização.

Caso não tenha sido feito o backup das tabelas **/MSAFX** , não há como utilizá-lo.

Dicas e Recomendações

  * Sempre conferir atentamente a descrição do pacote no portal antes de fazer o download e aplicar (verificar se é **"Instalação"** ou **"Atualização"**).

  * Restringir o uso do pacote de Instalação apenas a novos ambientes, onde a Interface OBTI ainda não foi configurada.

  * Manter backups das tabelas de controle da Interface **/MSAFX** antes de qualquer atividade de atualização de versão.

  * Testar previamente a aplicação dos pacotes em ambiente de homologação ou qualidade, validando o comportamento das tabelas de controle antes de aplicar em produção.
