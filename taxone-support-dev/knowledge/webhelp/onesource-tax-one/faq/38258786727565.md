### **Configuração de Exceção ao SSO para Usuários de Integração SAFX via API (Webservice)**

**Problema:** A rotina de integração automática de SAFX via API pela Namespace OBTI (SAP) parou de funcionar após a habilitação do Single Sign-On (SSO) no ambiente Tax One.

**Causa Raiz:** O problema ocorre porque o SSO é incompatível com integrações que dependem de acesso direto via API (machine-to-machine). Após a habilitação do SSO, o sistema exige uma autenticação que o serviço de integração não pode fornecer. Portanto, o usuário utilizado pela API deve ser configurado como uma exceção para poder ignorar a tela de SSO e autenticar-se diretamente.

**Resolução:** Siga os passos abaixo para configurar uma exceção de login via SSO para o usuário de integração:

  1. Acesse o módulo **Administration** no Tax One.
  2. Vá até o menu **Company Info** e clique em **Groups**.
  3. Crie um novo grupo com um nome descritivo (exemplo: **BYPASS_SSO**) e associe o usuário de integração a esse grupo.
  4. No menu **Setup Single Sign-On Authentication** , localize a seção **Single Sign-On Exemptions**.
  5. Mova o grupo criado (por exemplo, "BYPASS_SSO") da lista da esquerda para a direita, incluindo-o como exceção.
  6. Para confirmar, clique na aba **Users**. Todos os usuários associados ao grupo de exceção devem aparecer nessa aba.
  7. Finalize a configuração e teste a autenticação da sua rotina de integração.


**Próximos Passos e Recomendações:**

  * Após concluir as configurações, realize um teste para verificar se a autenticação via API está funcionando corretamente.
