## Problema:

Cliente enfrenta dificuldades para liberar um usuário de integração necessário para a implementação do sistema Interdados V17 ao Oracle Cloud, devido a preocupações de segurança com as permissões que este usuário receberia.

## Causa Raiz:

O cliente está preocupado com o acesso amplo que o usuário de integração teria às APIs do Oracle Cloud, o que poderia comprometer a segurança ao permitir um acesso menos privilegiado do que o desejado.

## Resolução:

  * Verificar a documentação "Serviços Interface Oracle Cloud.pdf" que detalha as roles e privilégios mínimos requeridos para a comunicação com o ERP Oracle Cloud;


  * Solicitar ao cliente que revise a documentação da Oracle disponível em "Doc ID 2291252" e "Doc ID 2588752.1" no My Oracle Support para entender melhor as roles necessárias.


  * Caso persistam dúvidas, abrir um chamado na Oracle (SR) para esclarecimentos adicionais sobre os privilégios e roles atribuídos ao usuário de integração.


## Próximos Passos:

Após a revisão da documentação e o esclarecimento das dúvidas com a Oracle, o cliente deve proceder com a criação do usuário de integração conforme as roles e privilégios acordados. Monitorar a integração para garantir que não ocorram problemas relacionados à segurança ou funcionalidade.
