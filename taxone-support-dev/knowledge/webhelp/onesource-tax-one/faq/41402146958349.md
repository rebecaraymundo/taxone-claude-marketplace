# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] COD_CLASS_DOC_FIS não sendo preenchido na SAFX07E

## Problema

Alguns documentos fiscais apresentam "@" no campo **cod_class_doc_fis** durante o processamento na rotina SAFX07E.

## Causa Raiz

O campo "COD_CLASS_DOC_FIS" na SAFX07E captura a informação a ser preenchida das TAGs <PRODUCT_TYPE> dentro da <G_LINES> e seguindo a seguinte regra:

**"Se houver somente itens GOODS, então "1"; se houver somente itens "SERVICES", então "2"; se houver ambos, então "3"; se não houver itens, então "1"**
**Para os casos em que o COD_CLASS_DOC_FIS for definido como "2", o campo NORM_DEV receberá o valor "1933", para que seja possível defini-lo como serviço"**

Conforme a descrição de regra, são esperadas apenas as informações "SERVICES" ou "GOODS" nas TAGS, então caso alguma dessas informações não estejam de acordo com o esperado, o campo não será preenchido corretamente.

## Resolução

Para resolver o erro em questão, é necessário acionar a equipe Oracle Cloud da empresa para ajustar as informações enviadas nas TAGS "PRODUCT_TYPE" de acordo com o [mapeamento técnico do Interdados V17](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Interfaces/Interface_Oracle_V17R01_Cloud/Webhelp_Interface_Oracle_V17R01_Cloud/Manuais_portugues/manuais_pdf_excel_ppt/TR_INTERFACE_INTERDADOS_V17_MAPEAMENTO_TECNICO_V8.4_EXTRATOR_ORACLE_CLOUD.zip>) para a SAFX07E e conforme cartilha da Oracle do extrator "LACLS_ExtractApSpedData_Incremental"
