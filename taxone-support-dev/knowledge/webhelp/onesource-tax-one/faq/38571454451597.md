Atualmente o Interdados utiliza a conexão via SOAP para a conexão com o Oracle Cloud, usando quatro APIs distintas, dependendo do processamento sendo executado, sendo eles:

HOST + /xmlpserver/services/v2/ScheduleService?wsdl - Extrator e BIP

HOST + /xmlpserver/services/v2/SecurityService?wsdl - BIP

HOST + /xmlpserver/services/v2/CatalogService?wsdl - BIP

HOST + /xmlpserver/services/PublicReportService?wsdl - BIP

A informação referente ao HOST é configurado manualmente no Interdados no campo "**Cloud.Host"** diretamente nas opções do sistema. Para a configuração,, Vá até _Opções_ → _Propriedades_.

Insira o host no campo "**Cloud.Host** ". Exemplo de configuração:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40838606987533)

Caso estejam enfrentando algum problema de integração junto ao Oracle Cloud, é recomendado tentar acessar o Host conforme configurado em Cloud.Host, caso retorna alguma mensagem de erro, é um indicador de bloqueio de acesso ao Oracle Cloud, sendo necessário acionar o suporte Oracle Cloud para avaliação.

Dicas e Recomendações:

Certifique-se de que as configurações de rede estão alinhadas com os requisitos de comunicação descritos. Consulte a [documentação oficial](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Interfaces/Interface_Oracle_V17R01_Cloud/Webhelp_Interface_Oracle_V17R01_Cloud/Manuais_portugues/Manuais_funcionais/Extratores_Oracle.htm>) para mais detalhes.

Garanta que o protocolo HTTPS esteja liberado na porta 443. Esta configuração é essencial para o funcionamento do SOAP.

Revisite regularmente os pré-requisitos e documentações de suporte da integração Oracle Cloud para evitar problemas futuros.
