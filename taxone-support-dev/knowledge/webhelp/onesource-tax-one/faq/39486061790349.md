# Como adicionar upload manual de arquivos via SFTP no diagrama de arquitetura

## Problema

O diagrama de arquitetura fornecido ao cliente não contempla o upload manual de arquivos via SFTP para o usuário, conforme solicitado.

## Causa Raiz

O diagrama de arquitetura apresentado na análise do sistema não incluía explicitamente o componente de upload manual via SFTP, o que gerou dúvidas na interpretação do processo.

## Resolução

  1. Acesse o diagrama de arquitetura original fornecido ao cliente.
  2. Inclua uma representação clara do processo de upload manual via SFTP no diagrama, destacando sua funcionalidade e integração ao sistema.
  3. Utilize os links e documentos técnicos compartilhados para assegurar que a integração do SFTP está corretamente detalhada:
     * [Manual Técnico - Upload/Download de Arquivos](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Tecnicos/Upload_Download_Arquivos.htm>)
     * [Visualização Geral da Integração](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Interfaces/Onesource_Brasil_Tax_Integration/WebHelp_OBTI/assets/ONESOURCE_BRASIL_TAX_INTEGRATION/Visualizacao_Geral/OBTI_visualizacao_Vol9.htm>)
  4. Revise o diagrama para garantir clareza e conformidade com as solicitações do cliente.
  5. Compartilhe o diagrama atualizado com o cliente por meio de um canal oficial.


## Próximos Passos

Solicite ao cliente que revise o diagrama atualizado. Caso necessário, agende uma reunião de alinhamento para esclarecer dúvidas e validar as alterações realizadas. Monitorar o status do ticket para acompanhar o feedback durante o prazo de resolução.

## Dicas e Recomendações

  * Certifique-se de documentar todas as mudanças realizadas no diagrama de arquitetura para futuros suportes e atualizações.
  * Reforce com o cliente a possibilidade de realizar testes no ambiente para validar a integração do upload manual via SFTP.
  * Utilize reuniões virtuais, como o Teams, para resolver dúvidas mais complexas de forma colaborativa.
  * Mantenha os diagramas atualizados com a inclusão de todos os fluxos do sistema para evitar solicitações semelhantes no futuro.
