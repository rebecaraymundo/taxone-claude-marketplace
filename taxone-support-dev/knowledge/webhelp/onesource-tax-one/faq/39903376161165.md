##

## Dúvida

O cliente necessita de orientações sobre como proceder para atualizar a versão da Interface OBTI. Ao analisar os pacotes disponíveis, o mesmo identificou que agora existem três Requests SAP, incluindo uma Request de WebService, que não fazia parte do processo anterior, quando ainda era utilizado o Mastersaf DW.

##

## Causa Raiz

O link acessado para visualizar os pacotes de atualização apontou para o repositório que já inclui a nova Request de WebService, a qual não fazia parte do processo antes da migração para o Tax One. No entanto, ainda existe outro link disponível em que é possível encontrar apenas as duas Requests originalmente utilizadas no processo anterior.

##
Resolução

O cliente possui duas opções para atualização da Interface OBTI. Siga as orientações abaixo:

  1. Atualização utilizando o pacote do ONESOURCE BRASIL TAX INTEGRATION – **DW**

Acesse o link: [[Interfaces] Interface SAP - OBTI – Atualização de Versão/Patches - Thomson Reuters](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/27512075618573--Interfaces-Interface-SAP-OBTI>)

Ordem de aplicação das Requests:

     1. Request Principal (TRQ)
     2. Request Customizing (TDD)

  2. Atualização utilizando o pacote do ONESOURCE BRASIL TAX INTEGRATION –**TAX ONE (Web Service)**

Acesse o link: [[Interface SAP] OBTI – Atualização de Versão/Patches - Thomson Reuters](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/27513286162829--Interface-SAP-OBTI>)

Ordem de aplicação das Requests:

     1. Request Principal (TRQ)
     2. Request Customizing (TDD)
     3. Request de WebServices (TRQ)

O pacote referente ao Web Service inclui exatamente a nova Request de WebService mencionada na dúvida.
Entretanto, essa Request **não deve ser aplicada** em ambientes que:

     * não estejam integrados ao Web Service do Tax One; ou
     * utilizem versões inferiores ao **SAP Basis 731 SP 16**


##

## Próximos Passos

Após aplicar os requests na ordem especificada, é importante realizar os testes de integração no ambiente, especialmente nas funções relacionadas ao WebService, caso ele esteja ativo.

##
Dicas e Recomendações

  * Antes de aplicar os requests, crie um backup do ambiente para evitar perda de dados em caso de problemas..
  * Mantenha seu ambiente SAP atualizado com os patches de segurança e estabilidade recomendados pela SAP.
