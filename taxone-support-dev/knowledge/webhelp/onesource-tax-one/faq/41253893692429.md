# Erro no Cadastro de Municípios para Clientes e Fornecedores Estrangeiros no SPED

###

Erro no cadastro de clientes e fornecedores estrangeiros no SPED, onde o código do município aceita apenas 5 caracteres, mas o SPED exige 7 caracteres. O preenchimento automático do Tax One utiliza "00" como complemento, quando o correto seria "99" para municípios no exterior.

##

## Resolução

  1. Acesse o módulo **Básicos Data Warehouse** e navegue até **Manutenção Cadastros Município**.
  2. Localize o cadastro do município configurado como "99999" com a descrição "Exterior".
  3. No campo _Cód. Município p/ U.F._ , preencha com o valor "99".
  4. Certifique-se de que o campo _Nº. U.F. p/ Município_ também esteja configurado corretamente.
  5. Se necessário, crie um novo cadastro com essas configurações e atualize os dados da Pessoa Física/Jurídica vinculada.
  6. Após as alterações, proceda com a equalização do período e gere novamente o meio magnético.


Exemplo de configuração no cadastro:

![Exemplo de configuração de município](https://atendimentotr.zendesk.com/attachments/token/OY4pjzVl4JrPCK9TR67VksKIg/?name=inlineImage.png)

##

Após realizar as configurações indicadas:

Teste a geração do SPED para confirmar que o código do município está sendo preenchido corretamente com 7 caracteres.
