## Problema

Usuários estão enfrentando dificuldade em compreender quais parâmetros devem ser utilizados para realizar extrações na API Tax Data Management, tanto para extrações completas quanto incrementais (delta).

## Causa Raiz

A confusão se dá devido à diversidade de combinações de parâmetros disponíveis, cada qual retornando diferentes conjuntos de dados. Além disso, houve dúvidas sobre a utilização e comportamento de parâmetros como `updatedDateStart` e `updatedDateEnd`.

## Resolução

  1. **Para realizar a extração completa dos dados:**
     * Utilize os parâmetros `taxDateStart` e `taxDateEnd`. Esses parâmetros retornam todos os registros cujo campo `data_fiscal` esteja entre as datas informadas. É recomendado utilizar uma ampla janela temporal, como 2 anos entre as datas.
     * Exemplo de configuração de carga inicial:
       * `taxDateStart = 2000-01-01`
       * `taxDateEnd = 2100-01-01`
  2. **Para realizar a extração incremental (delta):**
     * Use o parâmetro `updatedDateStart`. Esse parâmetro retorna registros cujo campo `data_alteracao` é atualizado no período especificado. É adequado para extrações diárias de dados recentes.
     * O comportamento do `updatedDateStart` na API pode ser observado como:
       * Para registros novos sem alterações, o campo corresponde à data de criação.
       * Para registros alterados, o campo corresponde à data de alteração.
  3. **Comportamento de combinações de parâmetros:**
     * Quando os parâmetros `updatedDateStart` e `updatedDateEnd` são usados juntos, os registros retornados possuem `data_alteracao` entre (between) o intervalo especificado.
     * Quando apenas o parâmetro `updatedDateStart` é utilizado, ele retorna registros cujo campo `data_alteracao` é igual (equals) à data informada.
     * Quando os parâmetros `taxDateStart` e `taxDateEnd` são usados juntos, os registros retornados possuem `data_fiscal` entre (between) o intervalo especificado.
     * Combinar os parâmetros `taxDateStart`, `taxDateEnd`, `updatedDateStart`, e `updatedDateEnd` retorna registros que possuem simultaneamente `data_fiscal` entre o intervalo de `taxDate` e `data_alteracao` entre o intervalo de `updatedDate`.
