O BOT USER ACCOUNT é um usuário não humano, é utilizado para realizar tarefas automáticas. É possível criar um usuário, flegar a opção BOT USER ACCOUNT e utilizá-lo no Web Service. Lembrando que após criar o usuário no Tax One você deve atualizar a mesma senha nas configurações feitas no Webservice para que ocorra a conexão de forma correta.

Para auxilio segue link do Manual de Utilização do módulo Administration dentro do Help no Tax One.
Neste manual é possível verificar a indicação da possibilidade de adicionar usuários humanos (Usuários convencionais) e usuários BOT (Automáticos) no sistema.
[Add a user (thomsonreuters.com)](<https://www.thomsonreuters.com/en-us/help/onesource-platform/administration/user/add-a-user.html>).

Nota: A configuração padrão de usuários no sistema Tax One inclui expiração de senha e necessidade de renovação periódica, o que pode interromper serviços automatizados como a integração via webservice.

Seguir as orientações abaixo para criar uma conta BOT USER ACCOUNT

1\. Acesse o módulo Administrador no Tax One.

2\. Clique em "Adicionar Usuário".

3\. Insira as informações necessárias para o novo usuário.

4\. Marque a opção “BOT USER ACCOUNT” para definir o usuário como não humano.

5\. Salve as alterações.

6\. Atualize as configurações no webservice para usar a nova senha gerada.

7\. Após a criação e configuração do BOT USER ACCOUNT, monitore o funcionamento da integração webservice para garantir que não existam interrupções devido a problemas de autenticação ou expiração de senha.

Obs:. A opção de não expirar a senha dos usuários não está disponível no sistema. Com isso é mandatório realizar a atualização da senha após o prazo configurado no Administration do Tax One.

A explicação técnica é que o Tax One fica em uma plataforma global da Thomson Reuters e nesta plataforma possuem outros produtos, a criação, edição e manutenção de todos os usuários é controlada pelo time que gerencia essa plataforma global e eles ainda está avaliando quais serão os impactos se deixar os usuários sem tempo mínimo de alteração de senhas devido a lei LGPD.

Com isso ainda estamos avaliando todas as competências legais referente a esse assunto e analisando quais impactos seriam causados nos outros produtos que ficam na mesma plataforma.

De momento, a opção do BOT USER ACCOUNT pode ser marcada apenas para diferenciar os usuários que não são utilizados por um humano, ou seja, esses usuários podem ser destinados a tarefas automáticas como Webservice e outras.

**Atenção:** É recomendável não usar o BOT USER ACCOUNT como administrador do ambiente do Tax One, e sim mantê-lo exclusivamente para tarefas automáticas. Para mais informações sobre a criação e gestão de usuários no Tax One, consulte o [Manual de Utilização do Módulo Administration](<https://support.thomsonreuters.com>) disponível no Help do Tax One.
