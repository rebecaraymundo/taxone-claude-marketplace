# Resolução para Inclusão de Notas Fiscais com Data Fiscal Diferente

## Problema:

O usuário relatou que o sistema não está bloqueando ou alertando sobre a inclusão de notas fiscais duplicadas com a mesma NF, série, fornecedor, mas datas de escrituração diferentes.

## Causa Raiz:

A ferramenta classifica registros com datas fiscais diferentes como distintos, mesmo que outros dados sejam idênticos. Se os períodos anteriores não estiverem fechados no sistema, ele permite a importação de novos dados nesses períodos.

## Resolução:

  * Acesse Ferramentas Menu Rotinas Acessórias Fechamento para fechar os períodos anteriores.


  * Verifique o manual do sistema na seção [Ferramentas_Menu_Rotinas_Acessorias_Fechamento](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Funcionais/Basicos/Basicos_Ferramentas/Menu_Rotinas_Acessorias/Fechamento.htm>) para detalhes sobre como realizar este procedimento.


  * Consulte o relatório de documentos duplicados para verificar a existência de duplicidades. Este relatório pode ser encontrado em [Basicos_DW_Menu_Rela_Confere](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Funcionais/Basicos/Basicos_Data_Warehouse/Basicos_DW_Menu_Relatorios/Basicos_DW_Menu_Rela_Confere.htm>).


  * Para mais informações, consulte o FAQ disponível em [faq_bas_ferramentas](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/FAQs/Basicos_Ferramentas.htm>)
