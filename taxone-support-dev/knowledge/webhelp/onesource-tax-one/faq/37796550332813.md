### **Erro de Tamanho de Campo Excedido na Carga (Delimitador Incorreto)**

**Problema:** Durante a carga dos dados, o sistema retorna uma falha indicando que o tamanho de um campo foi excedido (ex: **"Field size larger than allowed - max(70) - length(424)"**). Esse erro impede o processamento dos dados.

**Causa Raiz:** O erro ocorre porque o arquivo SAFX gerado pelo **OBI** não está utilizando o delimitador correto. Para a integração via **OBI** , o **Tax One exige o delimitador PIPE** (`|`). Se o perfil de saída estiver configurado para gerar o separador **TAB** (que é o padrão para integração de TXT via SFTP), o sistema interpreta todo o conteúdo do registro como pertencente ao primeiro campo, excedendo seu limite de caracteres.

**Resolução:** Para resolver este problema, o perfil de saída utilizado pelo OBI deve ser ajustado para o delimitador PIPE:

  1. Acesse o OBI e navegue para as configurações do **Perfil de Saída** utilizado no processamento.
  2. Localize a configuração do delimitador de colunas.
  3. Altere o delimitador de **TAB** (ou qualquer outro) para o caractere **PIPE** (`|`). Lembre-se que o **PIPE** é o padrão para cargas que se originam no OBI.
  4. Salve as alterações e execute um novo teste de importação.


**Próximos Passos e Recomendações:**

  * **Diagnóstico de Conteúdo:** Se o erro persistir após a correção do delimitador, o problema pode estar no conteúdo gerado em algum campo. Realize o **download do arquivo SAFX** gerado pelo OBI (na tela de **Histórico do Processamento**) e verifique se o conteúdo de cada campo está de acordo com os limites máximos de tamanho especificados no Manual de Layout SAFX do Tax One.
