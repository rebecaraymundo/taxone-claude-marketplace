Passo a passo para identificar e corrigir o problema de cargas finalizadas no Interdados V11R01 que não enviam informações para o Tax One:

  1. Verifique o arquivo **wrapper.log** que fica dentro da pasta Interdados.

  2. Abra o arquivo wrapper.log e vefifique as três primeiras linhas, que mostram informações da instalação do Interdados, versão do Sistema Operacional e versão do Java.

  3. No exemplo a seguir é identificado que a versão do Java instalada é o Java Corretto 11, que não é homologado pela Thomson Reuters e não é compatível com o Interdados:


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/20168800966413)

  1. Caso seja o Java Corretto 11, o cliente deve:
     * Desinstalar o Java Corretto 11
     * Apagar todas as pastas criadas na instalação correspondentes ao Java Corretto 11
     * Fazer a instalação do Java 8 JDK (versão mais completa)
  2. Para instalar o Java 8 JDK 32 bits, baixe o arquivo jdk-8u202-windows-i586.exe no link: [https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html](<https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html> "Hyperlink \[https://www.oracle.com/br/java/technologies/javase/javase8-archive-downloads.html\]")

  3. Após a instalação do Java 8 JDK, realizer uma nova carga e verifique se as informações foram enviadas corretamente para o Tax One.


Esse procedimento ajuda a identificar e corrigir o problema de cargas finalizadas no Interdados que não enviam dados para o Tax One, causado pela incompatibilidade da versão do Java instalada.
