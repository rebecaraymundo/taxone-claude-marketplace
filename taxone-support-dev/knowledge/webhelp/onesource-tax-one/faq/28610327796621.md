Quando um usuário enfrenta dificuldades para enviar um DUMP do banco de dados muito volumoso, pois o tamanho excede o limite permitido para arquivos compactados em um único arquivo .zip ou .gzip. Normalmente, isto ocorre devido ao limite de tamanho para arquivos compactados que podem ser enviados. Alguns aplicativos de compactação podem limitar o tamanho dos arquivos dificultando a criação de um único processamento para envio. Para resolver, siga as orientações:

Nota: O sistema SFTP da Thomson Reuters suporta o recebimento do DUMP compactado e dividido em tamanhos menores para facilitar o envio, mas é necessário que os arquivos sejam enviados em uma única remessa.

1\. Particione o arquivo DUMP em partes menores. Isso pode ser feito usando ferramentas de particionamento de arquivos ou softwares de compactação que permitem a criação de volumes menores.

2\. Compacte cada parte do arquivo DUMP usando os formatos .zip ou .gzip, conforme especificado no manual técnico.

3\. Envie todos os arquivos compactados em uma única remessa para garantir que todos os dados necessários estejam disponíveis para análise.

4\. Antes de gerar o DUMP, pode ser realizada a limpeza de tabelas que contenham apenas dados temporários, conforme o manual de boas práticas.

5\. Após enviar os arquivos, aguarde a confirmação de recebimento por parte do suporte. Monitore o processo de upload para garantir que não ocorram falhas na transferência dos dados.

**Atenção:** Considere utilizar ferramentas de compactação que suportem a criação de volumes menores para facilitar o envio de arquivos grandes no futuro. Mantenha a documentação atualizada e consulte o manual de boas práticas regularmente para evitar problemas similares.
