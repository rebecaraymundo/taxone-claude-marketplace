### **Guia de Autenticação na API Gateway do OBI (CIAM Credentials)**

**Problema:** O usuário precisa configurar a autenticação na API Gateway do OBI para uso em uma aplicação customizada, OBI Sender ou Namespace OBTI (SAP), mas tem dúvidas sobre como obter as credenciais CIAM e quais parâmetros utilizar.

**Causa Raiz:** As credenciais de autenticação (Client ID e Client Secret) são geradas durante o processo de licenciamento do módulo OBI – Onesource Brasil Integration. A principal dificuldade do cliente é confirmar o status de licenciamento do módulo para saber se as credenciais já existem ou se precisam ser solicitadas.

**Resolução:** Siga a ordem de verificação abaixo para obter as credenciais e realizar a autenticação.

#### **1\. Verificação do Licenciamento e Obtenção das Credenciais**

O primeiro passo é verificar se o módulo OBI está licenciado e ativo em seu ambiente:

  1. Acesse o Tax One e navegue até **Básicos > OBI - Onesource Brasil Integration**.


  * **Cenário A: Módulo Não Licenciado**
    * **Se for exibida a mensagem "Usuário não autenticado":** Isso indica que o OBI não está licenciado (ou seja, o módulo não está criado).
    * **Ação:** Crie um ticket para o time de suporte solicitando o licenciamento do OBI. As credenciais CIAM serão geradas e enviadas ao responsável cadastrado durante esse processo (o licenciamento do OBI não tem custo adicional).
  * **Cenário B: Módulo Licenciado**
    * **Se o OBI abrir:** Isso significa que o módulo está ativo e as credenciais CIAM já foram enviadas anteriormente (durante o licenciamento inicial) ao responsável cadastrado para receber informações de licença.
    * **Ação:** Verifique com o responsável qual é o **Client ID** e o **Client Secret** para utilizá-los na autenticação.


#### **2\. Parâmetros e Requisição de Autenticação**

Com as credenciais CIAM em mãos, a autenticação segue um padrão fixo para todos os ambientes e clientes:

  * **Endpoint de Autenticação:** A URL para solicitar o token é `**https://auth.thomsonreuters.com/oauth/token**`. Essa URL é a mesma, tanto para ambientes **UAT** quanto para **PROD**.
  * **Parâmetros Mandatórios:** Na requisição de autenticação, você deve utilizar os seguintes valores fixos:
    * `**grant_type**`**:** Deve ser preenchido **sempre** com o texto `**client_credentials**`.
    * `**audience**`**:** Deve ser preenchido **sempre** com o valor padrão `**39220446-4dcd-4694-9246-0bf60b548716**`. Este é o identificador padrão da API para todos os clientes e ambientes.


O token de autenticação gerado será válido para acessar a API tanto do ambiente PROD quanto do UAT.

**Próximos Passos e Recomendações:**

  * Garanta que as credenciais CIAM sejam mantidas em segurança.
