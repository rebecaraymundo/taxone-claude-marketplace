Correção de Erro na Geração de XML para o Campo 'indCPRB' no Evento R2010

Problema:

Erro na geração do XML no campo 'indCPRB' para o evento R2010, impede o envio correto das informações ao sistema.

Causa Raiz:

A falta de dados ou dados incorretos no campo 'indCPRB' resultam em erro, pois o sistema espera um valor determinado baseado nas regras de negócio da aplicação, especificamente na alíquota de INSS relacionada à notas fiscais enviadas.

Resolução:

Verifique se a alíquota de INSS nas notas fiscais é igual a 3,5%. Se for, o sistema preenche automaticamente o campo 'indCPRB' como ‘1’.

Caso a alíquota seja diferente, é necessário ajustar as informações da alíquota nas notas fiscais relevantes.

Primeiro, localizar nas configurações em Parâmetros > Dados Iniciais> "Desconsiderar NFs sem Valor de Retenção informado" para evitar o processamento de NFs que não possuem retenção de INSS.

Realize uma equalização do período e uma nova geração prévia dos eventos R2010.

Conferir se o problema persiste. Se sim, pode ser necessário verificar individualmente as notas fiscais ou realizar ajustes adicionais na parametrização das regras no sistema.

Dicas e Recomendações:

Mantenha sempre atualizadas as tabelas de alíquota de INSS e informações de prestação de serviços, garantindo assim a correta geração de dados para os eventos do sistema. Regularmente revise as regras de negócio aplicadas no sistema para assegurar sua adequação às normas atuais.
