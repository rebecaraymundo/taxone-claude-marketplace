Objetivo deste artigo é informar sobre a utilização do SSO em conjunto com o Interdados.

  1. **Conheça a compatibilidade do SSO com a Oracle Cloud:**
     * De acordo com as informações providas pela Oracle Cloud, não existem incompatibilidades na conexão via web services para bases ERP Cloud habilitadas com SSO, independentemente do serviço de autenticação externo utilizado.
  2. **Saiba como configurar usuários de integração:**
     * Para assegurar a integração eficaz, os usuários de integração não devem ser cadastrados no autenticador externo. Eles devem ser mantidos somente no ERP Cloud, com as permissões (roles) adequadas. Isso evita redirecionamentos para autenticação externa durante chamadas via API ou web services (WS).
  3. **Acesse a documentação relevante:**
     * A documentação "LACLS_FISCAL_SOLUTION_PATH_V1.pdf" foi fornecida pela Oracle Cloud para auxiliar na configuração correta.
  4. **Crie usuários de integração corretamente:**
     * Os usuários de integração devem ser criados como Traditional Cloud Accounts. Isso significa que eles não devem ser registrados no sistema de autenticação externa.
  5. **Consulte a documentação para criação de usuários:**
     * Para mais informações sobre a criação de Traditional Cloud Account User, consulte a documentação oficial da Oracle Cloud no seguinte link: [Create a Traditional Cloud Account User](<https://docs.oracle.com/en/cloud/get-started/subscriptions-cloud/csgsg/create-traditional-cloud-account-user.html> "Hyperlink \[https://docs.oracle.com/en/cloud/get-started/subscriptions-cloud/csgsg/create-traditional-cloud-account-user.html\]")
