# Criação e Gerenciamento de Usuários Administradores no Powerlock

## Problema:

O usuário necessita criar um administrador nominal no módulo Powerlock para atender às exigências de uma auditoria de tecnologia, garantindo rastreabilidade e históricos das manutenções realizadas.

## Causa Raiz:

A plataforma atual utiliza um usuário administrador geral, o que não permite a rastreabilidade individual das ações realizadas, não atendendo aos requisitos de auditorias que exigem identificação individual dos administradores.

## Resolução:

  * Acesse o módulo Powerlock.


  * Navegue até **Segurança Maintenance Administrators**.


  * Clique em **Criar novo usuário administrador**.


  * Forneça os detalhes necessários e salve o novo usuário.


  * Para atribuir funções ao novo administrador, vá em **Action Administrator Assignment**.


  * Selecione todas as funções na coluna da direita e arraste para o novo usuário administrador.


  * Clique em salvar para aplicar as configurações.


  * Note que o novo administrador não terá permissões para criar ou excluir outros usuários administradores.


## Próximos Passos:

Após a criação e configuração do novo usuário administrador, realize testes para garantir que todas as funções necessárias estão sendo executadas corretamente. Monitore o uso do novo administrador para confirmar a eficácia das configurações de rastreabilidade.

## Dicas e Recomendações:

É recomendável revisar periodicamente as permissões dos usuários administradores para garantir que eles continuem alinhados com as políticas de segurança da empresa. Considere a utilização de um cofre de senhas para gerenciar de forma segura as credenciais dos administradores.
