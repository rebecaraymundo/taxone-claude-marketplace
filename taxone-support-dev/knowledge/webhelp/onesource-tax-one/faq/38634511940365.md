# [Básicos][Report Fiscal] TAX ONE -  Como Gerenciar Relatórios de Validação de Sequência de Documentos para Evitar Divergências

**Problema:**

Usuários enfrentam desafios com a geração de relatórios de 'Validação da Sequência de Documentos’ que inclui numerações inutilizadas/denegadas indevidamente, afetando a precisão dos dados.

**Causa Raiz:**

Os registros de Notas Fiscais Denegadas/Inutilizadas estão com data_ref divergente do ano de emissão das demais notas, e algumas notas de entrada emitidas por terceiros estão sendo incluídas indevidamente no relatório de sequência.

**Resolução:**

Ajuste o campo data_ref das notas fiscais denegadas/inutilizadas para o mesmo ano dos demais documentos registrados.

Para notas fiscais de entrada emitidas por terceiros, certifique-se de que o campo movto_e_s esteja ajustado para representar somente as notas emitidas pelo estabelecimento (ex: movto_e_s 2, 3, 4, 5 ou 9).
