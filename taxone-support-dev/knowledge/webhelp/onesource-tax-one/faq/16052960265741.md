Este artigo tem por objetivo orientar quanto ao fluxo e procedimentos para restaurar a base do Tax One.

Hoje os clientes podem exportar dados que foram importados pelo módulo do JOB Servidor. Essa seria uma alternativa para o usuário gerar um export em formato SAFX.

Obs:. "Apenas dados."

O cliente também pode reimportar os dados gerando uma nova extração na origem. Neste caso, se foi realizada alguma alteração no Tax One na primeira carga, ele terá que realiza-las novamente.

Caso realmente seja necessário realizar Backup de todo o ambiente para restaurar suas configurações deve se atentar as seguintes orientações:

**- > Deve ser realizado contato diretamente com o CS e AM solicitando a validação contratual para confirmar se já existe uma contratação prévia para o processo ou se será necessário gerar uma nova proposta<-**

1 - Para esse processo é extremamente necessário ter o consentimento do Cliente por escrito via chamado para seguir com a solicitação;

2 - O Procedimento possui custo adicional, por este motivo deve ser solicitado via CS e AM;

2.1 - O Procedimento não é realizado pela equipe de Suporte e por isso o Ticket será escalado para o Time de Desenvolvimento que é o responsável pelo procedimento;

3 - Após o início, o procedimento pode ter uma duração aproximada de 7 a 10 dias para ser finalizado;

4 - Após o início da execução não é possível interromper o processo;

5 - Após o início, o sistema pode continuar a ser utilizado, mas todo procedimento realizado será perdido ao final do processo de BackUp/Restore.

6 - Durante a etapa final do processo, será realizado o export da base recuperada e a reimportação para o ambiente do Tax One e por este motivo o sistema poderá ficar sem acesso;

**Observação Importante:.** O BackUp/Restore só recupera dados de até 7 dias anteriores a data de aprovação da solicitação, ou seja, todos os passos para a aprovação devem ser feitos o quanto antes para que seja viável recuperar a data da desejada.
