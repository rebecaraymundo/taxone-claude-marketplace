# [Básicos][Report Fiscal] TAX ONE - Como Resolver a Não Consideração dos CFOPs 1933 e 2933 no Relatório do TaxOne

Problema:

O relatório gerado pelo caminho "Obrigações Acessórias ICMS Resumos do ICMS p/ DataMart Cód. Fiscal de Operação - Report Server" no TaxOne não está considerando os CFOPs 1933 e 2933.

Causa Raiz:

O problema ocorre devido à falta de parametrização correta no sistema TaxOne para incluir os CFOPs 1933 e 2933 nas notas "conjugadas" do relatório de ICMS.

Resolução:

  * Acesse **Estadual ICMS Manutenção Parâmetros por UF** e marque a opção 64.
  * Em seguida, vá para **Estadual ICMS Manutenção Parâmetros para serviços de NFs conjugadas**.
  * Ajuste os CFOPs para itens de serviços de acordo com o ajuste SINIEF, marcando os CFOPs 1933 e 2933.
  * Salve as modificações realizadas.
  * Realize uma nova equalização do período relevante.
  * Gere o relatório novamente para verificar se as notas com os CFOPs ajustados estão sendo apresentadas corretamente.
