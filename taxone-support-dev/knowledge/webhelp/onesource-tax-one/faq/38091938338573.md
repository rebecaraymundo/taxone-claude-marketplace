### **Problemas recorrentes de autenticação com a API de Carga SAFX (Webservice) do Tax One**

**Problema:** O usuário de integração da API de Carga SAFX sofre bloqueios recorrentes, interrompendo o processo automático, mesmo após o desbloqueio manual e a troca da senha.

**Causa Raiz:** O bloqueio é disparado quando o limite de tentativas de login com senha incorreta é atingido. Como a autenticação é unificada na plataforma, a falha pode vir de duas fontes distintas, mas ambas impactam o mesmo User ID global:

  * **Conflito de Sistema:** Outro ambiente de integração ou serviço automatizado (em outro ambiente ou sistema) está configurado com a **senha antiga/incorreta** e está fazendo tentativas contínuas na API, bloqueando o usuário.
  * **Tentativa Manual:** Um usuário está tentando, de forma manual, acessar o portal do Tax One com o mesmo User ID e a **senha incorreta** , atingindo o limite de tentativas.


**Resolução:** Para solucionar o problema, é essencial eliminar a fonte de tentativas incorretas.

  1. **Diagnóstico Imediato:** Após o desbloqueio, verifique todos os ambientes e sistemas de integração que utilizam este User ID.
  2. **Correção da Senha:** Corrija a senha em **todos** os pontos de uso (scripts, agendadores, ou sistemas de terceiros) ou instrua o usuário a utilizar a senha correta no portal.


**Ação Definitiva:** Se não for possível identificar a origem das tentativas incorretas, a solução mais rápida e definitiva para restaurar o serviço é:

  1. **Criar um novo User ID (login)**.
  2. **Configurar a API de Carga SAFX** para usar esse novo login.


**Próximos Passos e Recomendações:**

  * **Priorize IDs Exclusivos:** Se possível, utilize um User ID diferente para cada serviço ou ambiente de integração. Isso isola o impacto, garantindo que o bloqueio de um serviço não derrube todos os outros.
  * Mantenha um registro seguro e atualizado de qual User ID está associado a cada sistema.
