Ao gerar uma carga da SAFX82, os dados não são gravados, mesmo após a verificação e aplicação do Patch da Oracle para tratamento dos caracteres inválidos. Uma vez que o Patch da Oracle para tratamento dos caracteres inválidos foi verificado e aplicado, é impostante revisar a estrutura do XML que está sendo enviada para o Interdados em busca de inconsistências

Passo a passo para resolver o problema de não gravar registros na SAFX82:

  1. Verifique se no extrator LACLS_ExtractSpedAssetDeprnPA não existe a TAG G_ASSET_SUMMARY, dentro do caminho /DATA_DS/G_PARAM/G_ASSET_ID/, conforme evidencia abaixo. Nessa tag G_ASSET_SUMMARY deve ter as informações que servem de parâmetro da nota como livro, invoice_id, line, unit.
  2. Analise a query do **Extrator** assim como o **SETUP** do **ORACLE CLOUD** em relação ao ativo e entenda o porquê esse grupo de TAGS G_ASSET_SUMMARY não estão sendo geradas.
  3. Portanto é preciso verificar a geração desta tag na origem:
