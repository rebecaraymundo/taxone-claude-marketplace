## Problema

Usuário relata que, apesar das variantes SAFx desenvolvidas estarem funcionando normalmente para extração de dados, estas não estão sendo transferidas de forma automática como outras extrações.

## Causa Raiz

A causa identificada está relacionada à configuração das variantes na interface SAP, especificamente nos campos "P_AUTO" e "P_TPGRV".

## Resolução

  * Acessar a interface SAP e localizar a variante específica mencionada.
  * Verificar se o campo "P_AUTO" está marcado. Esse campo é responsável por habilitar a importação automática.
  * Confirmar a configuração do campo "P_TPGRV". Se estiver configurado como "T", a variante está definida para operar via TXT e não via WebService, o que impacta no método de transferência de dados.
  * Modificar as configurações conforme necessário para que correspondam aos requisitos de transferência automática desejados.


### Próximos Passos

Após ajustar as configurações, realize testes para verificar se as variantes estão agora corretamente programadas para efetuar as transferências de dados de forma automática. Continue monitorando as importações nos dias subsequentes para assegurar que o funcionamento está conforme o esperado.

### Dicas e Recomendações

Recomenda-se revisar periodicamente as configurações das variantes na interface SAP para evitar problemas similares no futuro. Se necessitar de assistência adicional, contate o suporte técnico.
