Passo a passo para corrigir o **Erro 91323** \- O campo Indicador de Cliente sem Identificação deve ser nulo, S ou N:

  1. Informe o campo******IND_CLIENTE_SID** (Indicador de Cliente SID), conforme o [manual de layout](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/Help/WebHelp/Manuais_Tecnicos/Manual_Layout.htm>):

     1. S – Se o cliente é sem identificação;

     2. N – Se o cliente tem identificação (CPF ou CGC).


Usado somente pela IN-86 para preenchimento do Item 3 (CPF_CGC) – Arquivo 4.9.1., com a expressão “SID”, para os clientes sem identificação (consumidor final sem CPF/CGC) e SPED Contribuições, registro A100 (campo 04 - COD_PART).

Nota: O problema ocorre pois o campo **IND_CLIENTE_SID **não possui algum dos valores definidos no manual de layout. Para corrigir o problema, basta alterar o conteúdo do campo para um dos valores citados acima.
