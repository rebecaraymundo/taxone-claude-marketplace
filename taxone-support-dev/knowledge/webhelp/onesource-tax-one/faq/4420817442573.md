# Introdução

Este artigo tem a finalidade de demonstrar a execução de uma carga de safx a partir de consumo de json.

Será utilizado para exemplo o aplicativo Postman, porem o cliente pode utilizar outro aplicativo de consumo/produção de json e envio de requisição REST, de sua preferência, considerando o as parametrizações dispostas no manual [Webservice para Carga de SAFX](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/Help/WebHelp/Manuais_Tecnicos/Webservice_Carga_SAFX.htm>).

# Passo a Passo

## AUTENTICAÇÃO

Para que seja possível consumir os demais serviços, é necessário um token válido. O token é gerado através do serviço de autenticação, onde deverá ser informado um login e senha.

Endpoints de Homologação e Produção

Homologação: https://safx.uat.taxone.thomsonreuters.com/api/authentication

Produção: <https://safx.prod.taxone.thomsonreuters.com/api/authentication>

![mceclip1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4420839269645)

## CRIAÇÃO DE LOTES

A integração através de Webservices deve sempre considerar o envio particionado das informações quando estas representarem um grande volume de dados, desta forma, nossa recomendação é que os registros enviados estejam agrupados em no máximo 10 mil registros por mensagem. Em virtude destas quebras, este serviço foi criado para permitir a criação de um lote que irá englobar todas as mensagens relacionadas a uma mesma SAFX, facilitando assim o processo de consulta e conferência do status das várias mensagens enviadas.

Endpoints de Homologação e Produção

Homologação: https://safx.uat.taxone.thomsonreuters.com/api/carga/geraNumLote

Produção: <https://safx.prod.taxone.thomsonreuters.com/api/carga/geraNumLote>

![mceclip2.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4420839341069)

## CARGA VIA JSON POR LINHA

Esse é o serviço para carga dos dados de uma tabela SAFX, através de um JSON, onde todos os campos da SAFX devem separados por pipe (|).

Endpoints de Homologação e Produção

Homologação: https://safx.uat.taxone.thomsonreuters.com/api/carga/2

Produção: https://safx.prod.taxone.thomsonreuters.com/api/carga/2

Na requisição, os parâmetros abaixo devem ser enviados no header:

token: token válido obtido através do serviço de autenticação.

numLote: número do lote obtido através do serviço de criação do número do lote (opcional e utilizado apenas caso tenha sido gerado o número do lote).

O body deve conter um JSON com as informações relativas as linhas da SAFX com os campos preenchidos seguindo o formato abaixo.

## CONSULTA DE STATUS DO LOTE

Esse é o serviço para consultar o status de uma requisição, através do número do lote e com isso verificar o status de todos os protocolos gerados.

Endpoints de Homologação e Produção

Homologação: https://safx.uat.taxone.thomsonreuters.com/api/carga/consultaLote

Produção: https://safx.prod.taxone.thomsonreuters.com/api/carga/consultaLote

Na requisição, os parâmetros abaixo devem ser enviados no header:

token: token válido obtido através do serviço de autenticação.

numLote: número do lote gerado obtido através do serviço de criação do lote, e que foi usado para a realização das cargas em lote.
