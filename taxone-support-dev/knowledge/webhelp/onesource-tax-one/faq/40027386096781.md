#

## Problema:

O problema reportado envolve a geração incorreta de códigos de DARF no módulo MIT. Códigos que deveriam ser gerados não são exibidos, e outros que estão marcados para ser "desprezados" na DCTF são gerados indevidamente.

## Causa Raiz:

A origem do problema está na configuração inadequada de códigos de DARF no sistema Mastersaf DW/TAX ONE. Alguns códigos que não pertencem ao MIT não estavam marcados corretamente para serem "desprezados" da DCTF.

## Resolução:

  1. Acesse o sistema Mastersaf DW/TAX ONE.
  2. Navegue até a seção 'Básicos Data Warehouse Manutenção Códigos'.
  3. Verifique todos os códigos de DARF que são exclusivos de outros módulos como REINF ou DIRF.
  4. Marque os códigos que não pertencem ao MIT como "Desprezar da DCTF".
  5. Salve as configurações e faça um novo teste de geração de DARFs no MIT.


Obs. O que é listado na tela da Ficha de Débito é o Código de Receita. Então no movimento onde código de DARF (exemplo 5434) o código de receita 597907. E para o movimento do código de DARF (exemplo 5442) o código de receita: 596007. Nesse caso se entender estar errado, ajustar no documento do Controle de Tributos.


## Próximos Passos:

Após ajustar as configurações no sistema, gere novamente os DARFs através do MIT e verifique se os Códigos são exibidos corretamente e que outros códigos indevidos não são gerados. Monitore de perto as próximas gerações para garantir a precisão dos dados e estações.

## Dicas e Recomendações:

Fique atento às configurações de códigos exclusivos para cada módulo; a correta configuração dos "Códigos de Desprezo" é crucial para a precisão dos dados gerados e submetidos nas declarações. Revisões periódicas das configurações podem evitar erros futuros em declarações fiscais.
