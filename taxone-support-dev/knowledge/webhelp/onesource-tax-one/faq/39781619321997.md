## **Problema:**

Ao gerar o arquivo municipal de Blumenau (F007), o campo "cd_regime_especial_tributacao" (campo 07), obrigatório, está aparecendo em branco.

## **Causa Raiz:**

A ausência de valores no campo "cd_regime_especial_tributacao" está relacionada à configuração incorreta ou incompleta dos parâmetros do município. Esse campo é dependente de ajustes específicos nos dados de classificação das pessoas físicas/jurídicas e serviços cadastrados.

No manual de layout consta as seguintes opções:

![](/attachments/token/ZY9xwfaWYyl9Hm0LIKFcRQF6C/?name=image.png)

## **Resolução:**

  1. Acesse o sistema para realizar os ajustes necessários.
  2. Navegue para o módulo **Municipal Parâmetros por Município Parâmetros Classificação de Serviços.**

**![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40704014424973)**

  3. Localize o município de Blumenau e revise os parâmetros cadastrados.
  4. Nesta parametrização vamos utilizar o código "07 - Tributação por faturamento (Variável)" certifique que está corretamente atribuído ao município correspondente.
  5. Caso necessário, configure os serviços associados às notas fiscais para que correspondam aos parâmetros esperados pelo manual de layout.
  6. Salvar as alterações realizadas e gerar novamente o arquivo para validação.


## **Próximos Passos:**

Após realizar os ajustes descritos, teste novamente a geração do arquivo municipal e valide os dados utilizando o validador oficial. Certifique-se de que o campo "cd_regime_especial_tributacao" está preenchido corretamente.
