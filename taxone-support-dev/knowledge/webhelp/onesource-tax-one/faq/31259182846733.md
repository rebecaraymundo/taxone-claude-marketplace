O objetivo deste artigo é guiar a consultoria ou cliente na consulta e análise das extrações geradas no BIP dentro do Oracle Cloud, quando uma carga é executada no Interdados.

Após a execução de uma carga no Interdados, a consultoria ou cliente pode acessar e analisar os dados no Oracle Cloud utilizando o PST_ID.

Passo a passo para consultar e analisar cargas BIP do Interdados no Oracle Cloud:

  1. Acesse o Oracle Cloud através da URL que possui e acrescente a palavra "**xmlpserver** ".
  2. Dentro do Oracle Cloud, você deve ter em mãos o **PST_ID** da carga em questão. Obtenha o PST_ID da carga que deseja analisar no Interdados, clicando em **LOG > Consulta Processamento por Status**. A primeira coluna com o nome "ID" é o PST_ID da carga.
  3. No Oracle Cloud, clique em "CATÁLOG" e acesse a pasta configurada para o processo. Se não souber a pasta, consulte o caminho dentro do Interdados:
     * Clique em _Opções > Configuração-Propriedade_s.
     * No filtro, digite CLOUD e serão exibidas as configurações de Chave e Valor a seguir:
       * **CHAVE: CLOUD_BIP.CORE_FOLDER_BASE**

VALOR: Corresponde ao caminho no Oracle Cloud onde a SAFX é gravada. Este é o valor padrão para SAFXs com Cod.Exec = CORE.

       * **CHAVE: CLOUD_BIP.CUSTOM_FOLDER_BASE**

VALOR: Corresponde ao caminho no Oracle Cloud onde a SAFX é gravada. Esta chave é utilizada para SAFXs com Cod.Exec = CTM, ou seja, customizadas, onde o caminho na coluna Valor pode ser diferente do caminho para SAFXs com Cod.Exec = CORE.

  4. No entanto, dentro do Oracle Cloud, sempre acesse **Catálog\Pastas Compartilhadas.** Depois siga o caminho do valor da chave no Interdados:

     * Exemplo de uma Chave CORE: Catálog/Pastas/Compartilhadas/Custom/THOMSON/INTF/msafi_nomecliente/01-BASICOS e aqui estará o número do PST_ID.

     * Clicando no PST_ID, você terá acesso aos objetos criados no Oracle Cloud para a extração da carga executada no interdados.

     * Dentro da pasta existem quatro objetos onde, o nome do objeto é sempre o PST_ID, depois CTM (Caso seja customizado) e KEDXXX, onde o XXX é o número da SAFX da Carga. Exemplo: 12345_CTM01_KED0004_20_RPT

     * Sobre os quatro objetos, temos:

       1. **12345_CTM01_KED0004_20_RPT_CHAVE** – Objeto no qual traz quantos registros retornaram na carga. As informações como empresa, estabelecimentos, livro GL, etc., estarão dentro de DATA_DS e a TAG G_1, terá a tag <PST_LEVEL_QTD>QTDREGITROS</PST_LEVEL_QTD> no qual mostra a quantidades de registros nesta carga.

       2. **12345_CTM01_KED0004_20_MDL_CHAVE** – Query da chave com o data model onde você pode acessar a query, clicando no conjunto de dados e depois clicando no Lápis para copiar a query e trabalhar nela.

       3. **12345_CTM01_KED0004_20_MDL** – Neste objeto você pode editar a query gerada, clicando em Editar, basta copiar a query e colar em um Notepad++, por exemplo. A edição pode ser realizada dentro a r_prog_start que é onde você encontra todos os valores passados como parâmetros. Depois terão as sub-queryes que podem ser editadas e realizadas algumas análises.

       4. **12345_CTM01_KED0004_20_RPT** – Resultado principal com os 11 Registros, onde baixando o relatório, você terá acesso ao XML. Clicando em Histórico do Job, acesse o XML contendo o DATA_DS e acesse os parâmetros da query chave. Dentro do DATA_DS, temos as TAGs de <G_1>. Cada bloco de <G_1> corresponde a um registro do XML, sendo assim, teremos 11 <G_1> neste XML. Dentro de cada <G_1>, temos os dados da SAFX em questão e estes registros, podem ser consultados no Mapeamento técnico correspondente a SAFX. Caso precise consultar a query para alguma análise, retorne ao Oracle, edite e execute a query novamente, pasta editar no objeto 3 - 12345_CTM01_KED0004_20_MDL.


**Dicas e Recomendações:**

  * Familiarize-se com os manuais técnicos e participe de treinamentos oferecidos para melhor compreensão do sistema Oracle.
  * Mantenha o sistema atualizado e consulte regularmente a documentação disponível no site para prevenir problemas futuros.
