## Problema:

Usuários enfrentaram dificuldades ao realizar consultas de documentos no TDM, devido à configuração incorreta da URL e ausência de cadastro válido de estabelecimentos no módulo OBI.

## Causa Raiz:

A questão está relacionada a duas principais causas: a utilização de uma URL incorreta nas requisições e o cadastro incompleto de estabelecimentos no módulo OBI, que impossibilita o retorno de dados pelo TDM.

## Resolução:

  1. Para garantir que as requisições sejam feitas corretamente, ajuste a URL base (`baseURL`) no Postman para uma das URLs abaixo, de acordo com o ambiente de cada cliente: [`https://uat01.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
[`https://uat02.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
[`https://uat03.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
[`https://prod01.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
[`https://prod02.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
[`https://prod03.tdm.thomsonreuters.com`](<https://uat01.tdm.thomsonreuters.com>)
  2. Verifique os parâmetros das requisições. Certifique-se de desativar o parâmetro `{{queryString}}` disponível na Collection padrão e inserir manualmente os seguintes valores básicos:
     * `taxDateStart`: Data Fiscal inicial
     * `taxDateEnd`: Data Fiscal final
     * `page`: Página (padrão 0)
  3. Adicione parâmetros para filtrar por CNPJ ou código do estabelecimento (se necessário):
     * `codEnterprise`: Código do estabelecimento
     * `cnpjEstablishment`: CNPJ do estabelecimento
  4. Certifique-se de que todos os estabelecimentos relevantes estão devidamente cadastrados no módulo OBI do Tax One, com os campos de empresa (CNPJ) e código do estabelecimento preenchidos.
  5. Após realizar cadastro ou ajustes nos estabelecimentos, aguarde a próxima execução do extrator, que ocorre diariamente à 01:00, para que os dados fiquem disponíveis para consumo via API.


## Próximos Passos:

  1. Realize uma nova consulta no TDM utilizando as configurações e parâmetros revisados.
  2. Caso os dados não sejam retornados para o período ou parâmetros, verifique novamente o cadastro no OBI.
  3. Caso persistam problemas para períodos específicos, abra um novo ticket para análise detalhada.


## Dicas e Recomendações:

  * Realize o cadastro dos estabelecimentos no módulo OBI o mais cedo possível para garantir que os dados sejam processados no próximo extrator.
  * Fique atento às melhorias anunciadas pela Thomson Reuters, como a funcionalidade futura de cadastro de empresas via planilha.
  * Para mais orientações, consulte a documentação oficial da API TDM na Central de Ajuda.
