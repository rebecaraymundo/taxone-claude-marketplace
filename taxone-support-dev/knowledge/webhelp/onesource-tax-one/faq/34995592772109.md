O objetivo deste artigo é orientar o consultor\cliente na solução do problema referente ao envido de lotes de importação do Taxone.

## Problema:

O envio de lotes para o módulo de importação do TaxOne não está sendo realizado corretamente, resultando na ausência do lote no sistema, mesmo após o log indicar que foi enviado.
A configuração no arquivo wrapper.conf estava desabilitando o envio automático de lotes para o TaxOne.

## Resolução:

Para solucionar este problema, siga os passos abaixo:

  * Abra o arquivo wrapper.conf localizado na pasta de configurações do sistema.
  * Localize a linha: wrapper.java.additional.8 = -DsafxToTaxOne.envioAutomatico=false
  * Altere esta linha para: wrapper.java.additional.8 = -DsafxToTaxOne.envioAutomatico=true
  * Salve o arquivo e reinicie o serviço para que a alteração tenha efeito.
  * Realize um novo teste de envio do lote para verificar se o problema foi resolvido.


## Próximos Passos:

Após realizar a alteração e reiniciar o serviço, monitore os próximos envios de lote para garantir que o problema foi completamente resolvido e que os lotes estão aparecendo corretamente no módulo de importação do TaxOne.

## Dicas e Recomendações:

É recomendável verificar periodicamente as configurações no arquivo wrapper.conf para assegurar que nenhuma alteração inadvertida impeça o funcionamento correto do sistema. Mantenha o sistema sempre atualizado conforme as recomendações do fornecedor para evitar problemas de compatibilidade ou funcionamento.
