Se o banco de dados Oracle foi atualizado de 12c para Oracle 19c, existem mapeamentos com configurações novas que impedem a conexão com a Base, conforme descrito no erro.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/18950252250125)

Passo a passo para corrigir o "Erro no teste de conexão com BASE" após atualização do Oracle 12c para 19c:

**Opção 1:**

  1. Certifique-se de que o parâmetro **"SEC_CASE_SENSITIVE_LOGON"** está definido como "**FALSE** " no Oracle. No Oracle 19c, esse parâmetro é "**TRUE** " por padrão.
  2. Se ainda assim estiver true, faça o seguinte:

     * 01 - ALTER SYSTEM SET SEC_CASE_SENSITIVE_LOGON = FALSE;

     * 02 - Reinicie o banco de dados;

     * 03 - Altere a senha de TODOS os usuários (MSAF, ONESOURCE e DFE) colocando a mesma senha antiga (isso deve ser feito para surtir efeito).​​​


**Opção 2:**

  1. Altere o arquivo **SQLNET.ORA** do diretório dbhome para SQLNET.ALLOWED_LOGON_VERSION=8.

  2. Reinicie o banco de dados Oracle.


Após seguir um desses procedimentos, o erro "Erro no teste de conexão com BASE" deve ser corrigido, permitindo a conexão com o banco de dados Oracle 19c no Interdados.
