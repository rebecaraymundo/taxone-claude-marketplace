Referente a produção TECO ate o momento temos a informação que a namespace não realiza este tipo de operação na geração das SAFX 108/109.

Temos como sugestão para ser realizado esta tratativa e a fim de obterem uma solução para o assunto em pauta:


  * Ordens de Produção que são geradas no período posterior com a quantidade zerada, pois, por algum quesito interno, essas ordens são reabertas, mesmo após o fechamento/conclusão.


Existem duas opções de pontos de EXIT no código fonte, onde podem ser implementado os tratamentos para exclusão das ordens que, segundo a vossa análise, não deveriam sair na extração.
![Image](https://dev.azure.com/tr-ggo/9464d7d1-c63b-4af4-9399-dc57bf983384/_apis/wit/attachments/56d70dbf-d654-4d76-ad62-809cce0b25f6?fileName=image.png)
Dentro desse contexto, se nos permite uma SUGESTÃO, vocês poderiam fazer o tratamento através do segundo ponto de EXIT (490), fazendo a exclusão dos dados indesejados, na tabela interna h1_trab_108.
Avaliem os quesitos e, se porventura pudermos auxiliá-los em algum outro ponto, estaremos à disposição.
