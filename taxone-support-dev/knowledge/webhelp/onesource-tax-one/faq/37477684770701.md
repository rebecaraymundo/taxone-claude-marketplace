## O objetivo deste artigo é orientar os usuários em relação à descontinuação do ODI no Interdados V11.

## Problema:

Usuários necessitam de informações sobre a descontinuação do uso do Oracle Data Integrator (ODI) nas integrações do Interdados com o ERP Oracle, bem como orientações para atualização de patches.

## Causa Raiz:

A descontinuação do uso do ODI em cenários do produto Interdados é devido à decisão da Oracle que também parou de desenvolver e oferecer suporte para esses cenários.

## Resolução:

  * Uma opção é válida principalmente para clientes que estão com Patch muito antigo, como por exemplo, Patch 35, ao invés de realizar a atualização Patch a Patch, o cliente pode fazer backup de todas suas customizações e realizar uma nova instalação do Patch 047 sem ODI em ambiente de Homologação, QA e depois PROD.
  * Caso o cliente esteja com um Patch mais atual como Patch045, por exemplo, pode continuar trabalhando com o mesmo ou para trabalhar com o Patch047 (mais atual) sem ODI, o mesmo deverá realizar uma instalação nova.
  * Lembrando que as atualizações de Patch do Interdados não são acumulativas, sendo assim o cliente terá que atualizar Patch a Patch.


## Próximos Passos:

Após realizar as atualizações ou instalações conforme as instruções acima, monitore o sistema para verificar se todas as integrações estão funcionando conforme o esperado.

## Dicas e Recomendações:

Esteja ciente de que as atualizações de patch não são acumulativas; portanto, será necessário atualizar cada patch sequencialmente, a menos que faça uma nova instalação. Sempre teste as alterações em um ambiente de teste antes de aplicar em produção.
