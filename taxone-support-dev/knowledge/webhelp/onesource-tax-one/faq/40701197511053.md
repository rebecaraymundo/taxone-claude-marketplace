## Problema:

O sistema está gerando o registro 0220 para um produto que possui a mesma unidade de medida cadastrada no sistema, o que não deveria ocorrer conforme as configurações do usuário.

## Causa Raiz:

O problema ocorre devido à presença de um registro de inventário com uma unidade de medida diferente para o produto no momento da geração do arquivo EFD ICMS IPI. Isso faz com que o sistema gere o registro 0220, mesmo que a unidade de medida principal esteja correta no cadastro do produto.

## Resolução:

  1. Acesse o sistema MasterSAF-DW.
  2. Navegue até a seção 'Básicos Data Wharehouse Manutenção Cadastros Conversão de Unidades de Medida Por Produto'.
  3. Revise as configurações de conversão de unidades para o produto em questão, verificando se as unidades listadas estão corretas.
  4. Verifique os registros de inventário para a data de referência usada na geração do EFD ICMS IPI e assegure que as unidades estejam alinhadas com o cadastro do produto.
  5. Se necessário, ajuste as informações de unidade de medida no inventário ou nos parâmetros de conversão de unidades.
  6. Regenere o arquivo EFD ICMS IPI e verifique se o problema foi resolvido.


## Próximos Passos:

Após realizar os ajustes, monitore as próximas gerações do arquivo EFD ICMS IPI para garantir que o registro 0220 não seja mais gerado indevidamente. Caso o problema persista, entre em contato com o suporte técnico para uma análise mais detalhada.

## Dicas e Recomendações:

Mantenha sempre os cadastros de produtos e as tabelas de conversão de unidades atualizados no sistema para evitar inconsistências nos registros gerados nos relatórios fiscais. Em caso de dúvidas ou necessidade de assistência, não hesite em contatar o suporte.
