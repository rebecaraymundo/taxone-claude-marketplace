Resolução de Erro na Geração do XML: Informação Sobre o Campo 'indCPRB'

Problema:

O usuário enfrenta erro na geração do XML relacionado à falta da tag `indCPRB` no registro R-2010, impedindo o sucesso na transmissão de dados para o sistema do governo.

Causa Raiz:

O sistema espera um valor para a tag `indCPRB` que indica se a alíquota do INSS na nota fiscal é de 3,5%, situação que deve ser marcada como ‘1’ no XML gerado. Caso contrário, é marcado como ‘0’. No caso de alíquotas de 11% ou 20% não havia preenchimento adequado para essa tag devido à falta de parâmetros claros no sistema ou à ausência de informação de retenção de INSS.

Resolução:

Siga os passos abaixo para resolver o problema:

Verifique se a nota fiscal possui alíquota de INSS informada com 3,5%.

Se sim, marque o campo `indCPRB` como '1' na geração do XML.

Para notas com alíquotas de 11% ou 20%, deixe o campo `indCPRB` em branco ou como '0' e depois refaça os testes.

Execute a equalização do período novamente e faça uma nova geração prévia do evento R2010.

Após a geração prévia gere e envie o XML conforme indicado, submeta novamente ao sistema do governo.

Próximos Passos:

Monitore a aceitação e verifique se o problema persiste. Se o erro continuar, repita o processo de verificação e equalização, ou entre em contato com o suporte técnico.

Dicas e Recomendações:

Regularmente revisite as configurações dos registros dos eventos para garantir que todas as informações estejam corretas e atualizadas, minimizando erros de transmissão.
