O objetivo deste artigo é mostrar ao consultor\cliente qual o processo a ser tomado quando se deparar com uma situação apresentada a seguir.

### Problema:

O Oracle Fusion está gerando um PST_ID duplo para o relatório LACLS_ExtractTaxWithholdingData, o que não é o comportamento esperado, já que deveria emitir um único PST_ID para cada configuração de relatório.
O problema ocorreu especificamente após execução da SAFX53 no Interdados e o Oracle Cloud mostra duas cartas idênticas para o PST_ID em questão.
![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39293313479565)

### Resolução:

  1. Acesse o Oracle Fusion e verifique os parâmetros configurados para os extratores AP e Withholding.
  2. Confirme se ambos os extratores estão parametrizados corretamente
  3. Verifique se o PST_ID foi cancelado com sucesso para evitar congestões na fila de processamento.
  4. Caso o problema persista, um ticket deverá ser aberto ao Suporte Técnico da Thomson Reuters com todas as evidências.
  5. Uma possível instabilidade no Oracle Cloud relacionada ao Extrator AP pode ter feito com que o sistema assumisse erroneamente o extrator Withholding, resultando na emissão dupla.
  6. Contudo, contate o suporte Oracle Cloud para uma investigação mais detalhada da infraestrutura caso o problema persista.


### Próximos Passos:

Após realizar as verificações acima, monitorize o sistema para garantir que os PST_IDs estejam sendo gerados corretamente. Caso o problema se repita, contate novamente o suporte.

### Dicas e Recomendações:

É recomendável manter a documentação dos extratores atualizada e revisar periodicamente as configurações para evitar possíveis inconvenientes. Além disso, manter comunicação ativa com o suporte Oracle pode assegurar uma resolução mais ágil de instabilidades.
