Se os campos "200 - IND_NAT_BASE_CRED" da safx08 e "254 - IND_NAT_BASE_CRED" da safx07 não estão sendo carregados no TXT do SPED Contribuições apesar de estarem populados no sistema, o problema pode estar relacionado com a configuração dos parâmetros iniciais. O sistema tem duas configurações possíveis para determinar a natureza da base de crédito: uma baseada nos documentos fiscais e outra no mapeamento "CFOP x Natureza da Base de Crédito". Para resolver, siga as orientações:

  1. Acesse os Dados Iniciais do sistema.
  2. Verifique o estado do parâmetro "Considerar a Natureza da Base de Crédito a partir dos Documentos".
  3. Se estiver marcado, o sistema deverá gravar o conteúdo do campo "Natureza da Base de Crédito" informado no documento fiscal.
  4. Se estiver desmarcado, o sistema usará o código da natureza da base de crédito que consta no mapeamento "CFOP x Natureza da Base de Crédito", de acordo com o CFOP informado no documento fiscal.
  5. Corrija o parâmetro conforme necessário para garantir que os campos sejam carregados corretamente no TXT do SPED Contribuições.
  6. Após ajustar os parâmetros, gere novamente o TXT do SPED Contribuições e verifique se os campos estão sendo carregados corretamente. Se persistirem erros, entre em contato com o suporte técnico.


**Importante:** Revise regularmente as configurações dos parâmetros iniciais para garantir que estejam alinhadas com as necessidades atuais de sua empresa. Manter uma documentação precisa sobre a configuração desses parâmetros pode ajudar a evitar problemas futuros.
