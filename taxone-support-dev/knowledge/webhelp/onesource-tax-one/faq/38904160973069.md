## **Solução de Problemas em Consultas de Documentos no TDM**

**Problema:** Não é possível realizar consultas de documentos no TDM, mesmo utilizando as URLs e configurações da collection, pois os dados esperados não são retornados.

**Causa Raiz:** A ausência de dados nas consultas do TDM geralmente ocorre devido a duas causas principais:

  * **Configuração incompleta no OBI:** Os cadastros de empresas e estabelecimentos não estão corretos ou o parâmetro de integração com o TDM não foi ativado para os estabelecimentos.
  * **Requisição incorreta na API:** A URL base ou os parâmetros obrigatórios e opcionais nas requisições estão incorretos, o que impede o retorno dos dados.


**Resolução:** Para solucionar o problema, siga um processo de verificação em duas etapas:

  1. **Verificação no OBI**
     * No módulo de empresas do OBI, certifique-se de que cada estabelecimento esteja corretamente cadastrado, com todos os campos preenchidos, incluindo o código do estabelecimento.
     * Para cada estabelecimento, ative o parâmetro de TDM. Caso a ativação não seja possível de imediato, salve o cadastro do estabelecimento antes de habilitar o módulo.
     * Após a configuração, é necessário aguardar a execução do extrator, que ocorre diariamente às 01:00, para que os dados sejam disponibilizados para consumo na API.
  2. **Verificação das requisições realizadas na API do TDM**
     * Utilize a URL base correta para o seu ambiente (ex: `https://uat01.tdm.thomsonreuters.com` para o ambiente UAT01).
     * Configure os parâmetros obrigatórios na requisição:
       * `taxDateStart` \- Data fiscal inicial.
       * `taxDateEnd` \- Data fiscal final.
       * `page` \- Página a ser recuperada pela requisição. A página inicial é sempre 0.
     * Se a consulta for específica, adicione os parâmetros opcionais:
       * `codEnterprise` \- Código do estabelecimento.
       * `cnpjEstablishment` \- CNPJ do estabelecimento.
     * Caso haja persistência de erros, realize um teste manual utilizando apenas os parâmetros básicos e desabilitando qualquer script pré-configurado na collection.


**Próximos Passos e Recomendações:** Se o problema persistir após todos os procedimentos, abra um ticket de suporte com uma descrição detalhada e o exemplo da requisição que está falhando para que possamos realizar uma análise mais aprofundada.
