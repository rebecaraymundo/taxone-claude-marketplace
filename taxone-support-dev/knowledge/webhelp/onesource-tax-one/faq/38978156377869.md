# Como gerar o campo " Situação" para o indicador :

# “3” – Isenta/ “0” – Tributada / “4” – Retida

## **Problema**

O indicador de isento/tributado/retido não está sendo gerado no arquivo de meio magnético do município de São Manuel, impossibilitando a importação do arquivo na prefeitura.

## **Causa Raiz**

O problema ocorre devido à ausência de parametrização cadastrada para o município do prestador na nota fiscal, enquanto outros municípios que possuem parametrizações corretas geram o indicador com sucesso.

## **Resolução**

Siga os passos abaixo para corrigir o problema:

  1. Acesse a tela de **Parâmetros – Classificação de Serviços**.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39259438154765)

  1. Certifique-se de que o município do prestador está devidamente parametrizado utilizando o código COD_PARAM correspondente ao indicador desejado:

     * **“3” – Isenta:** O serviço e o município do prestador devem estar parametrizados com o código COD_PARAM = “433”.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39259465973773)

     * **“0” – Tributada:** O serviço e o município do prestador devem estar parametrizados com o código COD_PARAM = “495”.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39259465976973)
     * **“4” – Retida:** Verifique o local da prestação do serviço (campo COD_MUNIC da tabela DWT_DOCTO_FISCAL). Caso não esteja preenchido, será considerado o município do estabelecimento. Consulte as condições para o campo IND_TP_RET e outras variáveis conforme descrito na regra.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39259438165645)
  2. Aplique as parametrizações apropriadas para o município do prestador e salve as alterações.
  3. Gere novamente o arquivo de meio magnético e valide os indicadores gerados.


## **Próximos Passos**

Após realizar as parametrizações e gerar o arquivo com sucesso:

  1. Teste a importação do arquivo atualizado na prefeitura.
  2. Monitore o processo para garantir que todos os municípios necessários estão parametrizados corretamente.
  3. Se houver outros erros ou a necessidade de adicionar dados de obras, abra um novo ticket com as evidências detalhadas.
