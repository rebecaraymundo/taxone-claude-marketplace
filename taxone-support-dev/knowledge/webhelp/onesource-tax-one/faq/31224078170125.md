# Como corrigir o erro "Inscrição do Prestador especificada no arquivo não confere com o Prestador selecionado" no validador ISS RJ

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/35643207678733)

As **Observações Gerais** do Manual determina que se necessário, o campo "numérico" deverá ser preenchido com zeros à esquerda até completar seu tamanho máximo.

![](https://atendimentotr.zendesk.com/attachments/token/v839piSwVrDTIvNcgMMGYWgVT/?name=image.png)

## Problema:

Usuário enfrenta erro ao gerar a Inscrição Municipal no arquivo para a prefeitura do Rio de Janeiro, especificamente um erro de validação que indica que a Inscrição do Prestador especificada não confere com o Prestador selecionado.

## Causa Raiz:

O problema ocorre devido a uma discrepância entre o formato dos dados fornecidos no arquivo gerado pelo sistema e os requisitos específicos do validador da prefeitura, relacionados à formatação de campos numéricos com zeros à esquerda.

## Resolução:

  * Verifique o layout do arquivo gerado pelo sistema Tax One e compare com o Manual de Layout da prefeitura do Rio de Janeiro.


  * Assegure-se de que todos os campos numéricos estejam preenchidos corretamente conforme especificado no manual, incluindo zeros à esquerda quando necessário.


  * Se o arquivo continuar sendo rejeitado, ajuste manualmente os dados conforme a orientação do manual e tente a validação novamente.


  * Se o problema persistir, entre em contato com a Prefeitura para entender possíveis divergências entre a regra do manual e a validação realizada.


## Próximos Passos:

Após ajustar o arquivo conforme as especificações, realize novos testes de envio. Se o erro for corrigido e o arquivo aceito sem problemas, continue monitorando os próximos envios para garantir que não ocorram mais erros. Se o erro persistir mesmo após ajustes, considere solicitar uma revisão detalhada das especificações técnicas junto à Prefeitura.
