# Resolução de Problemas na Geração de Relatório de Conferência das NFs de Entrada na PER/DCOMP.

###

## Problema:

O usuário enfrenta dificuldades na geração do Relatório de Conferência das NFs de Entrada após realizar a Geração das NFs na PER/DCOMP. O arquivo não aparece no diretório esperado, mesmo após ser consumido pelo sistema.

## Causa Raiz:

O problema ocorre porque o arquivo estava sendo colocado na pasta incorreta no SFTP. O arquivo precisa ser colocado no diretório **raiz** e **não estar zipado**. Se o arquivo for colocado incorretamente ou estiver zipado, ele não é listado para importação.

## Resolução:

  1. Acesse o SFTP e navegue até o diretório raiz, que é a primeira pasta que aparece ao acessar o sistema.
  2. Transfira o arquivo .txt para o diretório raiz do SFTP.
  3. Aguarde alguns minutos até que o arquivo seja consumido pelo sistema, o que causará o desaparecimento visual do arquivo na interface.
  4. Caso o arquivo não apareça onde deveria, verifique a pasta "Rejeitados" pois é possível que o arquivo tenha sido movido para lá.
  5. Garanta que o arquivo não esteja em formato zipado.


## Próximos Passos:

Após seguir os passos acima, o relatório deve ser gerado corretamente. Continue monitorando o comportamento do sistema e, em caso de novas inconsistências, entre em contato com o suporte técnico para investigação adicional.

## Dicas e Recomendações:

Para evitar futuros problemas, sempre verifique se está enviando os arquivos para a pasta correta no SFTP e se os arquivos não estão zipados. Além disso, manter uma organização das pastas no SFTP pode facilitar a identificação e correção de problemas semelhantes no futuro.
