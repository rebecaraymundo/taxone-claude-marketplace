# Correção de Inconsistências no Relatório de Saldos x Lançamentos

## Problema:

O usuário relatou que existe uma diferença entre os valores reportados nos saldos e lançamentos no relatório de conferência do Tax One, o que não condiz com as evidências apresentadas.

## Causa Raiz:

A causa raiz do problema é a forma como o campo "Período" é preenchido ao gerar o relatório. Uma divergência é notada quando o campo é preenchido via seleção no calendário comparada ao preenchimento manual.

## Resolução:

  * Acesse a tela de parâmetros de geração do relatório.


  * Insira o período manualmente no formato MM/AAAA (exemplo: 04/2023).


  * Confirme a entrada de dados e gere o relatório.


Nota: Se optar por usar o calendário para definir a data, sempre selecione o **dia 01** do respectivo mês.

## Próximos Passos:

Após seguir as instruções de resolução, gere o relatório novamente e verifique se a divergência persiste. Se o problema continuar, entre em contato com o suporte técnico para uma análise mais detalhada.

## Dicas e Recomendações:

Para evitar problemas futuros em relatórios, sempre prefira digitar manualmente o período. Isso ajuda a evitar a geração de relatórios com advertências ou inconsistências na representação de dados.
