## Problema

O cliente identificou que os valores da receita bruta apresentados nos registros M210/M610 estavam duplicados, mesmo após a correção de uma duplicidade anterior no registro 1900. O sistema estava considerando informações do bloco F550 e do bloco A170, gerando inconsistências nos registros.

## Causa Raiz

A duplicidade ocorreu devido ao preenchimento simultâneo de dados consolidados e dados de movimentação no sistema. Além disso, havia informações sendo consideradas nos registros X183, que estavam sendo geradas automaticamente, mesmo sem parâmetros de apresentação do bloco A no perfil do cliente.

## Resolução

  1. Acesse o módulo **Sped Contribuições** ou o caminho **Ferramenta Rotinas Acessórias Inicialização Exclusão de Tabelas Definitivas Movimentos** para excluir os registros X183.
  2. Certifique-se de trabalhar exclusivamente com dados de movimentação e evitar a inclusão de dados consolidados na tabela de presumido.
  3. Para corrigir a duplicidade nos registros M210/M610, siga estas etapas:
     * Revise os parâmetros de geração do bloco A no perfil do cliente e desmarque a opção de apresentação, caso esteja ativa.
     * Remova o preenchimento da data de lançamento dos itens de serviço no sistema para que os documentos não sejam considerados no cálculo.
  4. Realize a equalização após qualquer alteração nos parâmetros de movimentação ou exclusão de registros.


## Próximos Passos

  1. Após aplicar as alterações, gere novamente o relatório "Demonstrativo da Apuração por Tipo de Contribuição" para validar se os valores apresentados nos registros M210/M610 estão corretos.
  2. Certifique-se de que o cliente compreendeu o procedimento e está utilizando os parâmetros adequados para evitar novas duplicidades.
  3. Monitore as próximas entregas e mantenha o cliente informado sobre qualquer ajuste necessário.


## Dicas e Recomendações

  * Evite o preenchimento simultâneo de dados consolidados e de movimentação para prevenir inconsistências futuras.
  * Utilize os relatórios disponíveis no sistema para verificar os parâmetros configurados e corrigir eventuais problemas antes do fechamento da obrigação.
  * Revise o uso do bloco A no perfil do cliente e ajuste as configurações para garantir que apenas os registros necessários sejam considerados.
  * Consulte o manual do sistema e outros artigos relacionados na base de conhecimento para entender melhor as regras aplicáveis aos blocos M210/M610 e F550.
