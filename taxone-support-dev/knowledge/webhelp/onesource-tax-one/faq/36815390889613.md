# [SPED][EFD - REINF]TAX ONE - SPED - EFD - REINF  - Alocação dos Valores de CSLL, PIS e COFINS no Código DARF 5952 no Relatório de Conferência de Eventos

### Problema:

Valores de CSLL estão sendo alocados na aba "Valor Agregado" ao invés do campo "VLR_CSLL" no relatório de Conferência dos Eventos, utilizando o código DARF 5952.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39449733092493)

### Causa Raiz:

O código DARF 5952 é utilizado para o agrupamento dos impostos PIS, COFINS e CSLL à taxa de 4,65%, por isso os valores são apresentados na seção "Valor Agregado" nos relatórios. Quando os impostos são tratados separadamente com códigos DARF individuais para cada categoria de imposto, eles são detalhados nos campos específicos dentro do relatório.

### Resolução:

Verifique o código DARF utilizado durante a configuração ou geração do relatório de Conferência dos Eventos.

Para uma visualização detalhada dos valores, considere utilizar os códigos DARF específicos para cada tipo de imposto se desejar que eles sejam listados nos campos apropriados: PIS (5979), COFINS (5960), CSLL (5987).

Regere o relatório após ajustar as configurações de acordo com a necessidade de detalhamento dos impostos.

### Próximos Passos:

Após implementar a resolução, certifique-se de verificar o relatório para confirmar que os valores estão sendo apresentados corretamente. Em caso de dúvidas ou problemas adicionais, entre em contato com o suporte técnico.

### Dicas e Recomendações:

É importante sempre verificar o código DARF utilizado para garantir que a apresentação dos valores no relatório de Conferência de Eventos esteja de acordo com as necessidades fiscais da empresa. Revisar periodicamente as configurações pode evitar inconsistências e facilitar a análise financeira e fiscal.
