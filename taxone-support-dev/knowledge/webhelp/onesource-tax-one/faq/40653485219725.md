## Problema:

O usuário enfrenta dificuldades em como resolver o erro de não gerar os valores do F100 no relatório sintético da tela de manutenção da apuração do PIS/COFINS no módulo do SPED Contribuições.

## Causa Raiz:

O problema ocorre porque o relatório sintético emitido no ícone de "impressora" na tela de manutenção da apuração de PIS/COFINS não segue a mesma regra dos valores apurados demonstrados em tela.

Para demonstrar os valores do F100 no relatório sintético é necessário que o estabelecimento utilizado no lançamento tenha Dados Iniciais criado, mesmo que já tenha Dados Inicias para o estab. centralizador.

## Resolução:

  1. Acesse a tela de Dados Inicias em Módulo EFD Contribuições> Parâmetros> Dados Iniciais. É necessário ter Dados Inicias cadastrado para o estabelecimento que foi efetuado o lançamento do F100


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40653463368973)

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40653463373325)

2\. Acesse a tela de apuração de Pis e Cofins em Módulo EFD Contribuições> Obrigações> Manutenção> Apuração Geral PIS/COFINS

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/40653485213325)

Com os Dados Inicias criado para o estabelecimento com lançamentos do F100 será gerado a receita no relatório sintético.
