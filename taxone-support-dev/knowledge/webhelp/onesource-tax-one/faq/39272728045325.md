O objetivo deste artigo orientar o consultor\cliente sobre possível solução para o erro ao enviar dados ao TaxOne após alterar a senha do mesmo.

## Problema:

Ao alterar a senha do TaxOne e depois tentar realizar o processamento de uma carga no Interdados, mesmo que também tenha alterado a senha no Interdados, o consultor\cliente se depara com a mensagem no Consulta Processamento por Status dentro do Interdados: "ERRO AO ENVIAR DADOS PARA O TAXONE" conforme a imagem a seguir.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39272728041485)
Com isso as cargas não têm seus dados enviados ao TaxOne.

## Resolução:

  * Verifcar se a senha realmente foi alterada no Interdados, onde o consultor\cliente pode acessar o arquivo msafi_msfts_config, identificando o USUÁRIO_TAX_WS e a SENHA_TAX_WS, se é realmente a nova senha que foi alterada.
  * Se na verificação acima os dados estiverem corretos, o consultor\cliente pode executar uma nova carga e depois analisar o arquivo interfaces-erp-taxone.log, onde irá mostrar no log da carga, tanto a URL de envio ao taxone, como também o Usuário e Senha. Assim, ele poderá validar novamente se a senha está correta.
  * Caso a senha que aparecer seja a anterior, é um sinal de que a senha possa ter sido alterada, mas o Cache do Browser possa não ter sido atualizado, mantendo a senha anterior.
  * Desta forma, basta limpar o cache, fechar o browser e se possível, reiniciar o computador.
Assim que os passos acima forem realizados, executar uma nova carga e verificar se o problema foi resolvido.


## Próximos Passos:

Se o problema ainda persistir, verifique novamente no log do arquivo interfaces-erp-taxone.log e do wrapper.log, se a senha nova é a que consta.
Se a senha nova estiver correta nos dois arquivos, o consultor\cliente deverá abrir um ticket para o time de TaxOne analisar, pois para o Interdados as senhas estão atualizadas.

## Dicas e Recomendações:

Antes de realizar quaisquer migrações ou modificações significativas no sistema, é crucial consultar a documentação do produto e buscar suporte técnico profissional para evitar problemas de compatibilidade e perda de dados.
