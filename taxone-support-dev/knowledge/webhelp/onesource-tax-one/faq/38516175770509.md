## Problema

Usuário enfrenta problemas de interface automática entre o SAP e o Tax One, causando a necessidade de carga manual de extrações para disponibilizar as informações no Tax One.

## Possíveis causas

  * URLs informadas incorretamente
  * Ausência ou informação incorreta do FirmD
  * Falha de conexão devido bloqueio no firewall


## Resolução

  * Para verificar as URLs configuradas, navegue até a tela de parâmetros gerais, aba "WebServices".
    * Revise as URLs conforme as indicações do manual: [Webservice_Carga_SAFX_Importacao](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Tecnicos/Webservice_Carga_SAFX_Importacao.htm>)
    * Salve as alterações e certifique-se de que as novas URLs estão liberadas no firewall pelo administrador de sistemas para permitir a comunicação.
    * Verifique novamente se a comunicação via web service está funcionando corretamente.

  * Para verificar o FirmID, acesse o Tax One e confirme se a empresa faz uso do FirmId
    * Navegue até a tela de parâmetros gerais, aba "WebServices" e clique em "Cadastrar FirmID".
    * Realize o cadastro, correção ou apague o registro conforme necessário.
    * Salve as alterações e verifique se o envio da carga é realizado corretamente.


### Próximos Passos

Após realizar as mudanças, monitorar as importações automáticas diárias para verificar se as cargas começam a ser recebidas no Tax One automaticamente. Caso contrário, repetir a verificação dos parâmetros de importação e consultar novamente o suporte técnico.

### Dicas e Recomendações

Mantenha a documentação de configuração da interface sempre atualizada e verifique periodicamente as configurações para evitar descontinuidades no processo de importação automática. Certifique-se de ter acesso administrativo para realizar ajustes necessários ou entre em contato com seu suporte técnico para assistência.
