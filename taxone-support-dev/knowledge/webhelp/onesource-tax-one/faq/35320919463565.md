# [Básicos][Job Servidor] TAX ONE - Resolvendo Problemas de Importação com Campos DAT_INI_APUR e DAT_FIM_APUR no PVA do SPED

**Problema:** Durante a importação de dados no arquivo do PVA do SPED, o programa exige preenchimento dos campos "DAT_INI_APUR" e "DAT_FIM_APUR", mesmo tratando-se de campos não obrigatórios em certas situações.

**Causa Raiz:** A origem do problema está na maneira como o sistema de importação está configurado para tratar esses campos, interpretando-os como necessários mesmo quando não são obrigatórios para o registro K270.

**Resolução:** Preencha os campos "DAT_INI_APUR" e "DAT_FIM_APUR" com a data '19000101', que representa uma data-padrão não-real, para que o sistema aceite a importação. Isso deve ser feito durante a edição ou inclusão dos registros no arquivo. Após essa alteração, reimporte o arquivo e verifique se o problema persiste.

**Próximos Passos:** Após implementar a solução, verifique no sistema se os dados foram importados corretamente. Se o problema persistir, entre em contato com o suporte técnico forneçendo detalhes adicionais para um diagnóstico mais preciso.

**Dicas e Recomendações:** Sempre valide os dados antes de importar para o sistema, e confirme se os campos estão de acordo com os requisitos do arquivo importado. Para evitar problemas futuros, mantenha um registro das configurações que resultaram em importações bem-sucedidas.
