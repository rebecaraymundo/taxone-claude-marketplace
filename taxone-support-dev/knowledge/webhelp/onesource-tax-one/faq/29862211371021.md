# [SPED][EFD - Contribuições]TAX ONE - SPED - EFD Contribuições - Como resolver divergências na Geração de Códigos de Material no EFD Contribuições?

Caso o código de material para um serviço que gera crédito de PIS e COFINS é exibido corretamente na tela de itens no perfil Data Warehouse, mas ao gerar o arquivo EFD Contribuições o sistema gera um valor diferente. O problemas pode estar na parametrização dos Dados Iniciais no sistema, onde o código do serviço está sendo substituído pelo código da natureza do serviço sem que o campo "Código da Natureza do Serviço" esteja devidamente preenchido. Para resolver, siga as orientações:

  1. Acesse o sistema com seu login e senha.
  2. Navegue até EFD Contribuições > Parâmetros > Dados Iniciais.
  3. Verifique se o parâmetro 'Considerar o Indicador no Código do Item' está marcado.
  4. Assegure-se de que o campo 'Código da Natureza do Serviço' está preenchido corretamente.
  5. Se necessário, ajuste as configurações para que o sistema não substitua o código do serviço pelo código da natureza do serviço automaticamente.
  6. Após realizar as alterações, gere novamente os registros e o arquivo magnético.
  7. Após implementar a solução, monitore os próximos arquivos gerados para garantir que o problema foi completamente resolvido. Caso o problema persista, entre em contato com o suporte técnico para uma análise mais aprofundada.


**Nota:** Para evitar problemas semelhantes no futuro, é recomendável revisar regularmente as parametrizações no sistema e certificar-se de que todos os campos necessários estão corretamente preenchidos. A manutenção de registros precisos e a verificação das configurações podem prevenir inconsistências na geração de dados fiscais.
