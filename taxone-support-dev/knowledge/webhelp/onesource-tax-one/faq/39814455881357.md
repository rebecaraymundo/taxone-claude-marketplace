# SAFX16, SAFX17 e SAFX18

###

## Problema:

O usuário relata que não estão sendo gerados insumos para os produtos acabados, apesar das informações estarem chegando ao extrator.

## Causa Raiz:

A Oracle introduziu um novo campo que precisa ser mapeado para a geração adequada da SAFX17 no Interdados. A falta desta configuração estava impedindo a geração correta dos dados.

## Resolução:

  1. Atualizar o extrator LACLS_ExtractBillOfMaterials para a última versão disponível.
  2. Acessar o Interdados e ir até a seção de parametrização do extrator.
  3. Para a SAFX16 inclua os seguintes parâmetros:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39832754779917)
  4. Para a SAFX17 inclua os seguintes parâmetros:![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39832740761229)
  5. Para a SAFX18 inclua:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/39832754783245)
  6. Validar as alterações em ambiente de teste para garantir que os insumos para produtos acabados estão sendo gerados corretamente.
  7. Após a aprovação no ambiente de teste, aplicar as correções no ambiente de produção.


## Próximos Passos:

Após aplicar a correção, realizar novos testes e validar se os dados estão corretos. Monitorar o sistema por alguns dias para garantir que o problema não retorne.

## Dicas e Recomendações:

Mantenha-se atualizado com as mudanças e atualizações, pois podem impactar diretamente nos processos internos. Certifique-se de testar qualquer nova atualização em um ambiente controlado antes de movê-la para a produção para evitar interrupções nos negócios.
