O objetivo deste artigo é orientar o cliente como alterar o endereço URL do ambiente Oracle Cloud no Interdados.

**Problema:**

Um usuário precisa alterar o caminho de extração do ambiente de homologação no sistema Interdados para configurar o endereço correto.

**Causa Raiz:**

O caminho de extração no parâmetro `CLOUD.HOST` estava configurado para um endereço incorreto (`https://ibrwjb-dev1.fa.ocs.oraclecloud.com`). Isso pode ocorrer devido a mudanças no ambiente ou configurações anteriores que não foram ajustadas.

**Resolução:**

Siga o passo a passo abaixo para alterar o caminho de extração no ambiente de homologação:

  * Acesse o sistema Interdados.
  * Navegue até a aba **Opções >** **Propriedades**.
  * Localize o parâmetro denominado `CLOUD.HOST`.
  * Altere o valor atual para o endereço desejado: `https://ibrwjb-test.fa.ocs.oraclecloud.com`.


Salve a alteração.

Segue um exemplo visual para localizar e alterar o parâmetro:

![Visualização de alteração do parâmetro CLOUD.HOST](https://atendimentotr.zendesk.com/attachments/token/eOIpVmaIupZZZ2p0KVvnTZ705/?name=image.png)

**Próximos Passos:**

Após realizar a alteração, recomendamos:

  * Testar a conexão para confirmar que o novo caminho foi configurado corretamente.
  * Verificar se o ambiente de homologação está funcionando conforme o esperado.
  * Documentar a alteração internamente para evitar confusões futuras.


**Dicas e Recomendações:**

Para evitar problemas similares no futuro, siga essas melhores práticas:

  * Mantenha um registro atualizado da configuração do ambiente, incluindo alterações feitas nos parâmetros.
  * Estabeleça um processo de revisão para garantir que as configurações estejam corretas antes de realizar homologações ou testes.
  * Considere testar as alterações em um ambiente isolado para limitar potenciais impactos.
