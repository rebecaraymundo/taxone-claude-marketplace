Se o serviço TR Transfer Files estiver inoperante e a tentativa de parar ou reiniciar o serviço não está sendo possível, isso pode acontecer devido a versão do Java utilizada não ser compatível com a aplicação. Para resolver, siga as orientações:

  1. Acesse a tela de Serviços do Windows como Administrador.
  2. Localize os serviços 'TR Ftp' e 'TR Ftp Nginx' e verifique se eles estão parados. Se algum estiver ativo, tente pará-lo.
  3. Exclua todo o conteúdo da pasta LOG em _C:\Program Files (x86)\TR Transfer Files\log_.
  4. Tente iniciar os serviços novamente e observe se o problema persiste.
  5. Se o erro continuar, verifique a versão do Java instalada usando o comando `java -version` no CMD.
  6. Instale o Amazon Corretto 17, que é compatível com a aplicação.
