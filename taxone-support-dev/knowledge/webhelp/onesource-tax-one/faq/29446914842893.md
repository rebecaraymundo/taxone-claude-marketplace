Este artigo tem por objetivo orientar a correção para o problema de cargas relacionado à SAFX53, quando o campo COD_SERVICO_NAT_REND não está sendo populado no Interdados.

A SAFX53 é parametrizada como Extrator do Oracle Cloud. Portanto, inicialmente, após realizar a carga, devemos verificar se o XML foi gerado seguindo os passos abaixo:

  1. Abra o Interdados, clique em Log e depois em Consulta Processamento por Status.
  2. Procure a carga em questão, clique em Log e, em seguida, clique em Download XML EXTRATOR.
  3. Abra o XML e procure as tags SERVICE_LIST_CODE.
  4. De acordo com o Mapeamento Técnico, a tag ITEM_NUMBER será preenchida se a tag SERVICE_LIST_CODE estiver vazia e se a Regra 105 estiver aplicada no campo COD_SERVICE_NAT_REND na Regra De-Para do Layout da SAFX53.
  5. Portanto, se o COD_SERVICE_NAT_REND não estiver sendo preenchido no arquivo JSON do Interdados, é provável que a regra esteja sendo aplicada.
  6. Para verificar se o arquivo JSON contém essa informação, o cliente ou consultoria deve abrir o Interdados, encontrar o PST_ID correspondente à carga necessária, clicar em Log e, na coluna MENSAGENS, encontrar a mensagem "SAFX CONVERTIDA PARA JSON".
  7. Clique nos três pontinhos da coluna Info e, ao abrir a tela do arquivo, verifique se o campo 103 dentro do arquivo está com a informação desejada.

Neste caso então devemos verificar outro ponto do sistema, as parametrizações fiscais. Para isso, siga os passos abaixo:

  1. Abra Parametrização Dados de Origem e clique em Tipo de Imposto.
  2. Verifique como está parametrizado PARA do Tipo de Imposto LACLS_IRPF_WT.
  3. Se estiver diferente de IR-IR, deverá alterar para tal.
