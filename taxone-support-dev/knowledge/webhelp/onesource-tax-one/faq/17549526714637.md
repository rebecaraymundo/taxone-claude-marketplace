Objetivo: auxiliar na resolução do problema de incompatibilidade quando é realizado a atualização da versão do ambiente SAP.

  1. Após realizar a atualização do ambiente, quando for efetuar a verificação da versão, pode ser que ocorra de ser exibida a seguinte mensagem no rodapé da transação /N/MSAFX/SAFTR001A: "Webservice – Request Customizing incompatível com a versão pacote WS”.


![download.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/17549541281421)

  1. Caso esta mensagem ocorra inicialmente, verifique se as versões que estão informadas estão corretas.
     1. Exemplo: na imagem acima a versão referente a request principal está com 4045, porém a request customizing está com 4037, e no rodapé da tela a versão é incompatível com o Webservice.


  1. Para solucionar este caso, inicialmente verifique se a Request Customizing foi aplicada para todos os mandantes no ambiente, se está utilizando diretamente a request fornecida pela TR, e se a mesma foi aplica sem erros impeditivos. Também verifique se durante o transporte foram assinalados todos os campos, conforme exemplo:


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/17549419081357)

  1. Após fazer estas verificações e ter realizado a aplicação em todos os mandantes sem apresentar erro no decorrer do processo, entre na transação /MSAFX/SAFTR057ATU, verifique se tem alguma informação para ser atualizada e clique em Atualização Versão / Campo, mesmo que as atualizações nos campos abaixo estejam zeradas.


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/17549472347917)

  1. Após realizar este processo, entre novamente na transação /N/MSAFX/SAFTR001A e verifique se as versões estão corretas. Neste ponto é possível verificar que tanto a versão da Interface (request Principal), como a namespace (request customizing) foram aplicadas corretamente e estão na mesma versão, porém a mensagem continua no rodapé.


  1. Referente a mensagem:


  * Verifique se o ambiente/cliente utiliza WebService. Caso seja utilizado, verifique se foi aplicado e/ou aplique a request para Webservice.
  * Se o cliente não utilizar o WebService e em algum momento foi aplicado a request de forma indevida, abra um chamado solicitando uma request de remoção do WebService. Para verificar se foi aplicada a request, entre na transação através da SE16n da tabela /MSAFX/SAFTB001.
  *


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/17549452427533)

  * Outra maneira de consultar é acessando a transação SE80, e verificar se consta o pacote aplicado, conforme abaixo:
  *


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/17549485746701)

  * Caso seja exibida alguma informação referente a WebService (WS), tanto na SE16n, ou na SE80, conforme os exemplos, significa que foi aplicada a request no passado, e por isto é informado que a request está incompatível, pois a mesma não foi aplicada agora atualizar.
