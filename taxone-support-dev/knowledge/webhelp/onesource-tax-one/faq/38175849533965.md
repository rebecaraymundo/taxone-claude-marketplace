# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Resolução de Geração Incorreta de Notas Fiscais Modelo 57 - CTE na Interface SAFX09.

O objetivo deste artigo é orientar o consultor\cliente sobre a geração incorreta de NF Modelo 57 - CTE na inteface SAFX09.

### Problema:

Notas fiscais modelo 57 - CTE estão sendo geradas incorretamente na interface SAFX09 ao invés da interface SAFX07 destinada para este propósito.

### Resolução:

O problema ocorreu devido à configuração incorreta no sistema, onde as notas fiscais modelo 57 - CTE, foram associadas à interface SAFX09. Essa configuração errônea causou a geração de notas no local errado durante o processo de extração de dados.

O Consultor\cliente deverá realizar o procedimento a seguir para resolver o problema.
1 - Acessar o Interdados e clicar em "Procedimentos".
2 - Clicar no ícone do lápis ao lado da SAFX09 para editar as configurações.
3 - Em editar procedimento, no combo "Modelos não contemplados", adicione o Modelo 57 à lista.
![](/attachments/token/fKHrFKrfTpRdOrix1mPoWzvgv/?name=image.png)

4 - Salvar as alterações realizadas.
5 - Realizar uma nova carga para verificar se o problema foi efetivamente resolvido.

### Próximos Passos:

Após realizar os passos acima, monitore o sistema para garantir que as notas fiscais modelo 57 - CTE estão sendo geradas corretamente na interface SAFX07. Caso o problema persista, entre em contato com o suporte técnico para assistência adicional.

### Dicas e Recomendações:

Revise periodicamente as configurações das interfaces para garantir que todos os modelos de notas fiscais estão sendo gerados nos locais designados adequadamente. Utilize relatórios de auditoria para rastrear quaisquer discrepâncias e corrija-as prontamente para evitar problemas recorrentes.
