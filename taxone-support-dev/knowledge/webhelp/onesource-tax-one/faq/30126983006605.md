# Resolução de Erro na Importação de Registros F550 - SAFX183 para Notas de Serviço

## Problema:

Erro **"ORA-01403: no data found"** relacionado ao **Código de Produto** ao realizar carga dos registros F550 - Demais documentos e Receitas – Lucro Presumido (SAFX183) de notas fiscais de serviço, onde o sistema não considera os códigos de serviço informado.

## Causa Raiz:

O problema ocorre devido existir na carga de dados da SAFX183 a informação de código de serviço no campo 16 "Código de Produto" onde a informação não é necessária para documentos de serviço.

## Resolução:

Para resolver o problema, siga os passos abaixo:

  * Acesse a SAFX183 e localize o campo 16 "Código do Produto".
  * Para notas de serviço, onde não há um código de produto específico, utilize o caractere "@" neste campo para indicar que se trata de um serviço.
  * Realize uma nova importação da SAFX183 com essa alteração.
  * Verifique se a importação foi concluída com sucesso e se os registros F550 foram gerados corretamente.


## Próximos Passos:

Após realizar os passos acima, continue realizando testes para garantir que as futuras importações de notas de serviço sejam registradas corretamente nos registros F550. Monitore o sistema para quaisquer inconsistências futuras e relate imediatamente qualquer discrepância.

## Dicas e Recomendações:

Regularmente verifique e atualize os cadastros de produtos e serviços para garantir que as informações estejam corretas e atualizadas. Isso evitará problemas similares no futuro. Também é recomendável revisar periodicamente as configurações do sistema para assegurar que estejam de acordo com as necessidades atuais e regulamentos vigentes.

Recomendamos também observar no Manual de Layout de SAFX, as instruções de preenchimento de campos. Neste caso, o campo 16 da SAFX183 a informação na íntegra segue abaixo.

SAFX183 | 16 | Código do Produto | COD_PRODUTO | Informar o Código do Produto/Mercadoria, de acordo com as solicitações da dos Convênios ICMS, RIPI e IN SRF, devidamente registradas na Tabela de Produto (SAFX2013). Esta codificação de mercadorias/produtos deve representar os códigos dos registrados nos sistemas corporativos e nos sistemas de faturamento (Billing). Assim, para simples ilustração podemos citar que para empresas industriais e comerciais, devem ser informados os códigos internos, para telecomunicações, os serviços que especificam os tipos de ligações, etc. Para o registro do “Resumo Diário” e Outros Resumos referentes a outros Modelos de Documentos Fiscais, não existe a exigência de Códigos específicos, bem como para registrar aquisições de Bens do Ativo e/ou as Alienações. Por isso, para cada Tipo de Modelo de Documento Fiscal, deve ser definido os critérios e os códigos a serem utilizados e analisar se as Legislações requerem ou não Códigos para o Atendimento a Escrituração, Meios Magnéticos e Obrigações Acessórias. Obs.: O preenchimento dos campos 13 e 14 exclui o preenchimento dos campos 15 e 16, e vice-versa.
