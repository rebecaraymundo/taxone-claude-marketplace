# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Divergência na geração do campo COD_OPERACAO

O objetivo deste artigo é orientar o usuário referente à possíveis causas que ocasionam erro na geração do campo COD_OPERACAO.

## Problema:

O campo COD_OPERACAO no layout SAFX53 está retornando o valor numérico **184** ao invés da descrição esperada **COFINS** para o código de DARF 5960.

## Causa Raiz:

Este comportamento ocorre devido à parametrização na tela "Retenção de Impostos". O sistema busca o valor parametrizado na tela "Retenção de impostos" COD_OPERACAO, quando COD_TRIBUTO = TAX_TYPE_CODE e diferente de IRPJ ou IR, e busca o valor parametrizado na regra 6.

## Resolução:

Siga os passos abaixo para corrigir o valor no campo COD_OPERACAO:

  1. Acesse a tela "Retenção de Impostos" e revise as parametrizações para a coluna Cód.Operação
  2. Se necessário atualize o valor configurado
  3. Revise a regra De Para de número 6 e verifique se há parametrização para o valor enviado
  4. Reprocesse o PST_ID associado do layout SAFX53 informado no chamado.


Verifique se após o reprocessamento o valor no campo COD_OPERACAO foi ajustado corretamente.

## Próximos Passos:

Após concluir o ajuste e reprocessar o PST_ID:

  * Execute um teste no layout SAFX53 para garantir que o campo COD_OPERACAO está configurado corretamente.
  * Monitorar possíveis inconsistências em outras parametrizações relacionadas ao campo COD_OPERACAO e corrigi-las, se necessário.


## Dicas e Recomendações:

  * Certifique-se de realizar revisões regulares das parametrizações das telas de "Retenção de Impostos" para evitar problemas futuros.
  * Documente e mantenha registros das regras aplicadas no layout SAFX53, facilitando a identificação de inconsistências.
  * Consulte a documentação oficial do layout SAFX53 para verificar boas práticas relacionadas à parametrização dos campos.
  * Caso tenha dúvidas sobre mapeamentos de campos específicos, entre em contato com o suporte.
