## Problema

O campo **COD_FIS_JUR** na SAFX04 não está sendo preenchido corretamente, embora as informações estejam presentes no extrator.

## Causa Raiz

O problema ocorre porque os valores enviados à interface no campo **REPORTING_TYPE_CODE** não correspondem às expectativas da interface (espera-se 'ORA_BR_CNPJ' ou 'ORA_BR_CPF'), como o **COUNTRY_CODE** é BR. Não podendo ser aplicada as regras para preenchimento do campo, resultando no preenchimento vazio do campo **COD_FIS_JUR**.

## Resolução

  1. Verifique os cadastros na origem (Oracle Cloud) para garantir que os valores esperados pela interface estão sendo enviados corretamente.
  2. Para **COUNTRY_CODE** igual a BR, certifique-se de que o campo **REPORTING_TYPE_CODE** está mapeado como 'ORA_BR_CNPJ' ou 'ORA_BR_CPF' para os registros relevantes.
  3. Realize uma análise comparativa com outros cadastros que possuem o campo preenchido corretamente
  4. Corrija os cadastros que estão enviando valores incompatíveis com o esperado pela interface.
  5. Teste novamente o preenchimento do campo **COD_FIS_JUR** após as correções.


## Próximos Passos

Após implementar as correções nos cadastros na origem:

  1. Execute o processamento novamente e verifique se o campo **COD_FIS_JUR** está sendo preenchido corretamente.
  2. Monitore os próximos envios para garantir que o problema foi resolvido de forma definitiva.
  3. Entre em contato com o suporte caso o problema persista.


## Dicas e Recomendações

  * Mantenha os cadastros atualizados no Oracle Cloud para evitar inconsistências nos campos.
  * Certifique-se de que as informações mapeadas para a interface estão alinhadas às especificações documentadas.
  * Considere realizar uma auditoria periódica nos cadastros para identificar possíveis erros antes de gerar os arquivos.
  * Consulte a documentação oficial da interface para entender os valores esperados em cada campo.
