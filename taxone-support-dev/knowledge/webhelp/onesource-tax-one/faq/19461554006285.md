O procedimento abaixo é para resolver o problema quando uma determinada carga apresenta no log uma mensagem como: 'Campo TRIB_ICMS mapeado na Laydb, não existe na tabela SAFX09'. Este é apenas um exemplo, mas outros campos como IND_ORIGEM_CRED mapeado na Laybd, não existe na tabela SAFX147, ou o COD_EX mapeado na Laybd, não existe na tabela SAFX49, também podem ocorrer.

Para solucionar este problema, realize o procedimento abaixo:

  1. Baixe o arquivo **interfaces-erp-taxone-V11.2023.03.jar** no [link](<https://webhelp.thomsonreuters.com.br/Site/download/interfaces/oracle/interfaces-erp-taxone-V11.2023.03.jar.zip>).
  2. Pare o serviço de cargas do Interdados.
  3. Substitua o **arquivo .jar novo** que foi baixado, renomeie o arquivo existente na pasta **Interdados\interfaces-erp-taxone.jar** para **.old** e renomeie o interfaces-erp-taxone-V11.2023.03.jar para **interfaces-erp-taxone.jar**
  4. Suba o serviço de cargas novamente e realize um teste.
