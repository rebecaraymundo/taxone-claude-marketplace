## **Requisito de Versão para Migração do DW para o Tax One**

**Problema:** O processo de validação da base de dados (**schemadiff**) e a migração do sistema DW para o Tax One não podem ser iniciados devido a versões antigas do DW.

**Causa Raiz:** A execução desses processos tem como pré-requisito que o ambiente do Data Warehouse (DW) esteja atualizado em uma das **últimas 3 major releases** do sistema. O setor responsável pela validação e migração tem essa regra interna para garantir a compatibilidade e a integridade dos dados durante o processo. Se a versão do sistema estiver fora desse padrão, a solicitação é barrada e não é executada.

Como exemplo, se o seu ambiente estiver na versão 305.0.2, o processo não poderá ser iniciado, pois a versão mínima exigida é, por exemplo, a major release **307.x.x**.

**Resolução:** Para que o processo de **schemadiff** e migração possa ser realizado, é necessário que o sistema esteja em uma versão compatível.

  1. Verifique a versão atual do seu DW.
  2. Se o sistema estiver desatualizado, siga o manual de migração do DW para o Tax One e realize a atualização para a versão mais recente.
  3. Após a atualização, **abra um ticket de suporte** para que o processo de validação da base de dados e a migração possam ser iniciados.


**Dicas e Recomendações:** Manter o sistema DW sempre atualizado é fundamental para garantir a compatibilidade e evitar atrasos em processos internos. É altamente recomendável estar sempre na versão mais recente para aproveitar as últimas correções e melhorias.
