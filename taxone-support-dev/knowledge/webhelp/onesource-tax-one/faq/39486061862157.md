## Problema

Ao tentar realizar ajustes de créditos (M110, M115, M510, M515) na seção de Apuração Geral do PIS/COFINS, o sistema permitia preencher os dados, mas apresentava erro ao tentar salvar. Por conta disso, não era possível validar o arquivo SPED .txt.

## Causa Raiz

O erro no salvamento dos ajustes foi relacionado a inconsistências nos parâmetros dos períodos de apuração ou regras específicas que não estavam sendo corretamente aplicadas no sistema.

## Resolução

  1. Primeiramente, verifique as permissões de acesso do usuário em questão. Certifique-se de que o acesso é completo (Full Access) para execução dos ajustes no sistema.
  2. Realize um teste de ajuste utilizando o procedimento paliativo enviado ao cliente, baseado na importação de arquivos de exemplo como M110.
  3. Caso o erro persista, informe os detalhes do erro (com prints) para o suporte técnico, além do período de apuração onde o problema ocorre.
  4. Agende, se necessário, uma reunião para demonstrar o cenário ao time técnico, garantindo que todas as informações pertinentes do banco de dados sejam verificadas.
  5. Após o diagnóstico e correção feita pela equipe de desenvolvimento, execute novos testes e valide o ajuste diretamente na obrigação SPED EFD Contribuições.


## Próximos Passos

Após realizar a validação do ajuste e confirmar que os erros foram resolvidos:

  * Finalize o processo de apuração do período no sistema.
  * Confirme que o arquivo SPED .txt seja gerado corretamente e valide-o no programa oficial EFD Contribuições.
  * Informe à equipe de suporte caso quaisquer outros eventos inesperados ocorram durante os testes adicionais.


## Dicas e Recomendações

  * Certifique-se de atualizar sempre o sistema para a última versão disponível, garantindo que todas as correções técnicas sejam aplicadas.
  * Documente os passos seguidos no processo e mantenha um histórico de screenshots e arquivos utilizados, para referência futura.
  * Consulte sempre os manuais fornecidos pela Thomson Reuters, disponíveis na Central de Ajuda, para garantir que todos os parâmetros de apuração estejam corretos.
  * Busque comunicar a equipe de suporte imediatamente caso novos erros sejam identificados.


## Conclusão

O problema foi resolvido com sucesso após diagnóstico detalhado e intervenção da equipe técnica. O ajuste de créditos foi finalmente salvo, e o arquivo .txt validado no SPED. Este procedimento pode servir como referência para casos similares no futuro.
