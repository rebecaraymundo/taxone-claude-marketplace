## Problema

Ao gerar o EFD-Contribuições para a Sócia Ostensiva, o sistema está incluindo os registros 1800 da Sociedade em Conta de Participação (SCP), o que não atende os requisitos do Guia Prático da EFD-Contribuições.

## Causa Raiz

O problema ocorre porque o sistema está configurado para levantar automaticamente os registros 1800 da SCP no arquivo da Sócia Ostensiva, mesmo quando esta deve contemplar somente suas operações próprias. Isso está relacionado à configuração atual de perfis no sistema.

## Resolução

Para corrigir o problema, siga os passos abaixo:

  1. Acesse o sistema e revise os perfis configurados para geração dos arquivos EFD-Contribuições.
  2. Crie dois perfis separados, um para a Sócia Ostensiva e outro para a SCP, garantindo que o perfil da Sócia Ostensiva não esteja marcado para incluir os registros 1800 da SCP.
  3. Confirme as configurações com base nas orientações legais disponíveis no Guia Prático da EFD-Contribuições.
  4. Teste a geração dos dois arquivos separadamente e verifique se o problema foi resolvido.


## Próximos Passos

Após realizar as configurações, é recomendado que o usuário monitore as próximas entregas de arquivos EFD-Contribuições para garantir que os dados estão sendo gerados corretamente. Caso necessário, entre em contato com o suporte para validação técnica.

## Dicas e Recomendações

  * Certifique-se de consultar regularmente atualizações no Guia Prático da EFD-Contribuições, pois mudanças legais podem impactar a geração dos dados.
  * Centralize a documentação de perfis e configurações para evitar inconsistências em futuras alterações no sistema.
  * Considere implementar uma rotina de validação antes de transmitir os arquivos ao governo.
  * Consulte nossa [Central de Ajuda](<URL-Ficticio-Central-de-Ajuda>) para informações adicionais sobre o módulo EFD-Contribuições.
