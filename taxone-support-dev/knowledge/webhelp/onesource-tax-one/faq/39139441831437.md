Este artigo tem como objetivo demonstrar a geração do campo “Modelo” no arquivo ISS para o Validador eGoverno.

Usaremos como exemplo o município de São Gonçalo, que adota esse validador para a validação dos arquivos.

No link a seguir, você terá acesso ao [ROTEIRO DE UTILIZAÇÃO - ARQUIVO ISS - VALIDADOR EGOVERNO](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Conteudo/Demais_Empresas/Onesource_Tax_One/WebHelp/mergedProjects/Webhelp_ONESOURCE_TAXONE/Manuais_Funcionais/Municipal/Validadores/egoverno.htm>)

**CADASTRO DO ESTABELECIMENTO GERADOR** : Básicos > Parâmetros > Preliminares > Empresa/Estabelecimento

![KBA EGOVERNO - Campo MODELO - PRINT 1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139453617421)
![KBA EGOVERNO - Campo MODELO - PRINT 2.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139453618317)


**CADASTRO DA PESSOA FÍSICA/JURÍDICA DA NOTA** : Básicos > Data Warehouse > Manutenção > Cadastros > Pessoas Físicas/Jurídicas
O terceiro do nosso documento, pertence ao município de Igrejinha:

![KBA EGOVERNO - Campo MODELO - PRINT 3.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441822605)
![KBA EGOVERNO - Campo MODELO - PRINT 4.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441824653)

**DOCUMENTO FISCAL** : Básicos > Data Warehouse > Manutenção > Documento > Doc. Fiscal de Mercadoria e Serviço
O foco no documento fiscal, será a capa, para demonstrar qual o modelo da NF selecionado.
Número da nota: 1149010
Modelo Doc.: 70 - NOTA FISCAL DE SERVIÇO.
Veja que o documento está dentro do período 04/2024, e que se trata de nota de serviço:

![KBA EGOVERNO - Campo MODELO - PRINT 5.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441825165)

**EQUALIZAÇÃO** : Básicos > Data Warehouse > DATA MART > Equalização/Limpeza
Devemos sempre lembrar de equalizar o período da nota

![KBA EGOVERNO - Campo MODELO - PRINT 6.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441825677)

Para a correta geração do campo modelo, é necessário que seja cadastrado o parâmetro do Modelo Msaf x Modelo:

**PARÂMETRO MODELO MSAF X MODELO** : Municipal > Parâmetros Para Município > Parâmetros > Cadastro das Séries/Modelos das Obrigações > Modelo Msaf x Modelo

![KBA EGOVERNO - Campo MODELO - PRINT 7.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441826701)

Veja que o modelo 70, que foi inserido na nota, foi parametrizado:

![KBA EGOVERNO - Campo MODELO - PRINT 8.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441827213)

Por fim, vamos a geração do arquivo:

**GERAÇÃO DO MEIO MAGNÉTICO** : Municipal > São Leopoldo > Obrigações > Meio Magnético

![KBA EGOVERNO - Campo MODELO - PRINT 9.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441827469)

Na aba "conferência do meio magnético", é possível ver que a nota foi gerada no arquivo:
![KBA EGOVERNO - Campo MODELO - PRINT 10.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139441828749)
Com o arquivo aberto, podemos visualizar o campo modelo gerado:

![KBA EGOVERNO - Campo MODELO - PRINT 11.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/39139453642125)
