**PROBLEMA:**

Este artigo tem como objetivo orientar os Analistas da Thomson como atender as solicitações dos clientes quando for solicitado o procedimento de criação de Chave Pública/Privada para conectar no SFTP.

Atualmente no SFTP do Tax One, existem duas principais formas de autenticação disponíveis:

1\. Autenticação por senha: Cada usuário possui suas credenciais pessoais de acesso, que incluem um nome de usuário e senha para a conta SFTP. Essa é a forma mais comum de autenticação.

2\. Autenticação por chave pública: Sim, o protocolo baseado em chave pública está disponível. Esse método envolve a geração de uma chave pública e privada, onde a chave pública é usada para autenticar a conexão SFTP. Você deve configurar seu cliente SFTP com a chave privada correspondente e registrar a chave pública no sistema do Tax One.

Ambos os métodos são configuráveis e podem ser utilizados para assegurar a comunicação de acordo com as políticas de segurança da sua organização.

Por padrão a Thomson Reuters somente irá disponibilizar as credenciais (Usuário e Senha) para acessos ao diretório de SFTP.

Caso o cliente opte pela Chave Pública, ela deverá ser gerada de forma independente pelo próprio cliente, nós só realizamos a configuração da permissão de acesso no diretório.

Por este motivo não possuímos Manual Explicativo para o procedimento, mas podemos orientar o cliente que é necessário solicitar ao seu Time de T.I ou de DBA para gerar a chave e somente após a geração ele deve nos enviar os dados para ser feita a liberação internamente.

Quando o cliente gerar a CHAVE, na verdade será criado um par de chaves (1 Pública e 1 Privada) é importante saber que cada par de chaves deverá ser configurada para um único usuário. Ou seja, será necessário criar um par de chaves para cada usuário que for utilizar esta modalidade de autenticação no SFTP.

Em Resumo:

  * Para cada usuário SFTP o cliente deve criar um par de chaves.
  * Por questões de segurança não é possível “reaproveitar” a mesma chave privada em mais de um usuário.
  * O arquivo com a chave pública o cliente deve enviar via chamado da para que o Time de TechOps da Thomson possa configurar no servidor (authorized_keys desse usuário no SFTP).
  * A chave privada ele deve utilizar e importar no seu aplicativo client de preferência (FileZilla, WinSCP, etc.). Este procedimento é independente e deve ser feito pelo Time de TI ou DBA do Cliente pois não prestamos suporte para estas aplicações não homologadas pela TR.


Após o cliente enviar o Arquivo da chave e indicar para qual usuário a chave será configurada o chamado deve ser direcionado para o Time de TechOps realizar a liberação.

-> O Cliente pode ter usuários utilizando chaves e credenciais ?

Resp: Sim, ele pode ter um usuário configurado com chaves e outro que faz a conexão via credenciais. Como os diretórios do FTP são individuais, uma conexão não interfere na outra.
