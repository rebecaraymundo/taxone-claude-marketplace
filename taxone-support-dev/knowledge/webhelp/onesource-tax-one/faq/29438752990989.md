# [SPED][EFD - Contribuições]TAX ONE - Erro de formato inválido no campo Alíquotas PIS/COFINS no registro A170 do EFD Contribuições, como resolver?

Nos casos em que o arquivo do EFD Contribuições é rejeitado pelo validador do PVA devido a críticas relacionadas ao formato das alíquotas de PIS/PASEP e COFINS, o problema pode ser causado pelo uso de uma versão desatualizada do PVA. Versões antigas do programa podem não reconhecer o formato com quatro casas decimais após a vírgula, resultando em erros de validação.

Conforme orientação da versão 5.1.0 do programa da EFD Contribuições, os campos ALIQ_PIS (Alíquota do PIS/PASEP) e ALIQ_COFINS (Alíquota do COFINS) no Registro A170, foram tratados conforme a tabela 4.3.17 de alíquotas diferenciadas, passando a considerar quatro casas decimais após a virgula.

Para resolver, siga as orientações:

  1. Acesse o link para atualizar a versão do PVA: [Atualização do PVA](<https://www.gov.br/receitafederal/pt-br/assuntos/orientacao-tributaria/declaracoes-e-demonstrativos/sped-sistema-publico-de-escrituracao-digital/efd-contribuicoes/programa-validador-da-escrituracao-fiscal-digital-das-contribuicoes-incidentes-sobre-a-receita-efd-contribuicoes-2>).
  2. Instale a versão mais recente do PVA.
  3. Refaça a geração e validação do arquivo no novo PVA.
  4. Após a atualização e validação do arquivo, verifique se todas as alíquotas estão sendo aceitas sem erros. Se persistirem problemas, entre em contato com o suporte técnico para mais assistência.


**Nota:** Para evitar problemas futuros, é recomendável sempre verificar se você está utilizando a versão mais recente do software antes de iniciar a geração de arquivos importantes. Mantenha-se atualizado com as versões e orientações disponibilizadas no site oficial da Receita Federal.
