### **Uso de Proxy no OBI Sender**

**Problema:** Arquivos enviados para o Tax One via OBI Sender não são processados. O problema é causado pela política de rede do cliente, que exige o uso de um proxy para todas as saídas de internet.

**Causa Raiz:** O OBI Sender não suporta configurações de proxy. Como a aplicação não consegue realizar a conexão de saída através do proxy obrigatório da rede, a comunicação com os endpoints do Tax One é bloqueada.

**Resolução:** Se o uso de proxy for mandatório na infraestrutura, há duas alternativas:

  1. **Liberação de URLs (Alternativa preferível):**
     * Solicite à equipe de infraestrutura que libere as URLs específicas utilizadas pelo OBI para que a comunicação possa ser feita sem a necessidade de passar pelo proxy.
  2. **Desenvolvimento Customizado (Obrigatório se o Proxy for mandatório):**
     * Se o uso de proxy for inegociável, será necessário o **desenvolvimento de uma aplicação customizada** pelo cliente. Essa aplicação deve gerenciar as integrações e requisições diretamente para o **OBI Gateway (API do OBI)** , onde a lógica de configuração do proxy pode ser implementada dentro do código customizado.
