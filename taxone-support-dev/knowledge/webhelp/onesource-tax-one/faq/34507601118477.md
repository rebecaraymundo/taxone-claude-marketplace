# Corrigindo a Geração do Campo IND_OPER no Registro F100 para o EFD Contribuições

## Problema:

O campo IND_OPER do registro F100 no EFD Contribuições não está sendo gerado corretamente no arquivo TXT, aparecendo em branco e impedindo a importação do arquivo para validação no PVA.

## Causa Raiz:

A causa raiz do problema está na configuração incorreta das parametrizações de geração do campo IND_OPER no sistema. Esse campo depende de regras específicas que devem ser seguidas para ser preenchido corretamente.

## Resolução:

Para preenchimento desse campo deve observar as seguintes regras selecionadas em Dados iniciais:


Se o parâmetro “Utilizar a parametrização do Tipo do Documento X CST/Operação/Nat. Base de Crédito” estiver marcado (nos dados iniciais): - Buscar o código do tipo de operação associado ao Tipo do documento (campo da SAFX147), através da parametrização no módulo: SPED > EFD – Escrituração Fiscal Digital – PIS/PASEP – COFINS, menu: Parâmetro > Registros F à Tipo de Documento X (CST/Operação/ Nat. Base de Crédito.

Se o parâmetro ‘’Utilizar a parametrização da Conta Contábil/Centro de Custo X CST/Operação/Nat. Base de Crédito’’ estiver marcado (nos dados iniciais): - Buscar o código do tipo de operação associado a Conta Contábil (campo 17 da SAFX147) ou ‘’Conta Contábil e Centro de Custo’’ (campo 18 da SAFX147), através da parametrização no módulo abaixo: SPED à EFD – Escrituração Fiscal Digital – PIS/PASEP – COFINS, menu: Parâmetro à Registros F a Conta Contábil/Centro Custo X CST/Operação/ Nat. Base de Crédito (SAFX176).

  * Acesse o módulo SPED EFD – Escrituração Fiscal Digital – PIS/PASEP – COFINS.


  * Selecione o menu Parâmetro Registros F - Tipo de Documento X (CST/Operação/ Nat. Base de Crédito).


  * Verifique se o parâmetro “Utilizar a parametrização do Tipo do Documento X CST/Operação/Nat. Base de Crédito” está marcado. Se estiver, confira se o código do tipo de operação está associado corretamente ao Tipo do documento.


  * Caso o parâmetro “Utilizar a parametrização da Conta Contábil/Centro de Custo X CST/Operação/Nat. Base de Crédito” esteja marcado, confira se o código do tipo de operação está associado corretamente à Conta Contábil ou à combinação Conta Contábil e Centro de Custo.


## Próximos Passos:

Após realizar as correções nas parametrizações, gere novamente os registros e meio magnético e tente importá-lo para o PVA. Certifique-se de que o campo IND_OPER esteja sendo preenchido corretamente. Monitore o sistema para garantir que não ocorram mais erros na geração deste campo em futuras operações.

## Dicas e Recomendações:

Verifique regularmente as configurações de parametrização no sistema para evitar erros de preenchimento nos registros do EFD Contribuições. Mantenha-se atualizado com as normativas fiscais para garantir a conformidade dos arquivos gerados.
