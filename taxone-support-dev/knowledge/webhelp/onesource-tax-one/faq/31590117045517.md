# Gerenciamento de Expiração de Senhas para Usuários Bot em Ambientes Integrados

## Problema:

Usuários bot configurados para integrações automáticas estão enfrentando expiração de senha, o que não deveria ocorrer, uma vez que são destinados a operações integradas contínuas.

## Causa Raiz:

A expiração de senha para usuários bot ainda é aplicada devido a políticas de segurança global que estão sendo reavaliadas sob a legislação da LGPD. A plataforma global que gerencia esses usuários ainda não diferenciou completamente os usuários bot das contas regulares em termos de políticas de expiração de senha.

## Resolução:

  * Acesse o módulo Administration do sistema.


  * Verifique os parâmetros de expiração de senha e ajuste para o máximo permitido de 90 dias para garantir operacionalidade contínua até a próxima necessidade de renovação.


  * Para acompanhar a implementação de uma política diferenciada para usuários bot, acesse [Customer Influencer](<http://customerinfluencer.thomsonreuters.com/>) e crie uma solicitação de melhoria.


## Próximos Passos:

Após ajustar as configurações de expiração de senha, monitore regularmente as notificações do sistema para renovar as senhas conforme necessário e mantenha-se atualizado sobre as mudanças nas políticas através do portal de influência do cliente.

## Dicas e Recomendações:

Considere marcar as datas de renovação de senha no seu calendário para evitar interrupções nas operações automatizadas. Além disso, participe ativamente no portal Customer Influencer para ajudar a moldar as futuras políticas de segurança de acordo com as necessidades dos usuários bot.
