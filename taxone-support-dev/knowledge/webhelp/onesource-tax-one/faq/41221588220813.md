#

## Problema

O campo **VlrUnitarioNfi** está sendo gerado com valor “0” no arquivo, tanto nos registros quanto nos logs de erro.

## Causa Raiz

O valor do campo **VlrUnitarioNfi** é proveniente do campo **VLR_UNIT** da tabela **DWT_ITENS_SERV** (campo 20 da SAFX09). Se esse campo estiver zerado na base de dados, o arquivo gerado também apresentará “0” neste campo.

## Resolução

Acesse a base de dados do sistema.

Localize a tabela **DWT_ITENS_SERV**.

Verifique o valor do campo **VLR_UNIT** (campo 20 da SAFX09) referente ao documento fiscal em questão.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/41541283094413)

Se o valor estiver zerado, atualize para o valor unitário correto de acordo com a regra de negócio da empresa.

Após a correção, gere novamente o arquivo para conferir se o campo **VlrUnitarioNfi** está preenchido corretamente.

## Próximos Passos

  * Após realizar a atualização do campo **VLR_UNIT** , gere um novo arquivo de exportação e valide se o campo **VlrUnitarioNfi** está com o valor esperado.
  * Monitore os próximos arquivos gerados para garantir que o preenchimento está correto em todos os documentos fiscais.
  * Se o problema persistir, entre em contato com o suporte informando os detalhes do documento fiscal e do campo.
