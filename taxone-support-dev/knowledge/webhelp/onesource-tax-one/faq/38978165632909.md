## **Procedimento para Carga de Base Histórica do DW no Tax One**

**Problema:** Durante o envio de uma base histórica do DW para o Tax One, não é possível fornecer o código do ambiente (tenant code) de destino, pois a base histórica não está visível no sistema.

**Causa Raiz:** A base histórica é um serviço contratado à parte. Eventualmente, o ambiente de destino pode não ser criado previamente, pois sua provisão é feita automaticamente durante o processo de carga do dump da base de dados histórica. Nesses casos, o ambiente não existirá para ser visualizado antes do início da importação.

**Resolução:** Para iniciar o processo de forma correta, siga os passos abaixo:

  1. **Confirme o Contrato:** Primeiro, entre em contato com o nosso time de **CS - Customer Success** para confirmar se o serviço de Base Histórica foi contratado para seu ambiente.
  2. **Abra o Ticket de Migração:** Após a confirmação, abra um ticket para o time responsável pela migração, informando que o ambiente de destino é uma nova base histórica que deve ser criada durante a importação, pois ainda não está visível em seu sistema.


O time de migração será responsável por criar o ambiente e realizar o processo de carga. O código do ambiente (tenant code) será fornecido a você após a conclusão da importação.

**Próximos Passos:** Depois de abrir o ticket e confirmar o processo, o novo ambiente de base histórica estará disponível para acesso. Monitore a comunicação do ticket para receber as informações de acesso.

**Dicas e Recomendações:** Sempre verifique com o nosso time de CS - Customer Success se o serviço de base histórica foi contratado. Essa é a primeira e mais importante etapa para garantir que o processo ocorra sem atrasos e que sua base seja criada corretamente.
