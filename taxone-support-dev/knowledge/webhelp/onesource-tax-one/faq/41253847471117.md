# [Interfaces] [ORACLE Interdados - V17R01 Cloud] [Geral] Erro nos campos COD_MUNICIPIO_ORIG e COD_MUNICIPIO_DEST na SAFX07 de saídas

## Problema

Ao tentar importar os dados da SAFX07, o erro "Código de município de origem do serviço não cadastrado para o estado correspondente". Foi identificado que os campos COD_MUNICIPIO_ORIG e COD_MUNICIPIO_DEST estão cortados, apresentando inconsistências no layout.

## Causa Raiz

O problema ocorre devido ao tamanho do campo no layout da SAFX07, que é de 5 posições, enquanto os códigos possuem 7 posições. Além disso, a direção de extração dos campos está configurada incorretamente, realizando a leitura da esquerda para a direita, o que inclui o código do estado junto ao código do município. Exemplo: 3304524 (Rio das Ostras), sendo 33 o código do estado (RJ) e 04524 o código do município.

## Resolução

  1. Acesse o procedimento da SAFX07 Saídas no menu Procedimentos.
  2. Dentro do procedimento, selecione **Mais > Layout**.
  3. Localize os campos **COD_MUNICIPIO_ORIG** e **COD_MUNICIPIO_DEST** e clique duas vezes sobre eles.
  4. Na tela de parametrização do campo, localize a opção **Direção extração** e altere de **Esquerda** para **Direita**.
  5. Repita o procedimento para ambos os campos.
  6. Reprocesse o PST_ID ou gere uma nova carga para validar as alterações realizadas.


## Próximos Passos

Após realizar as alterações descritas acima, o usuário deve:

  1. Processar um novo PST_ID ou realizar uma nova carga para verificar se o problema foi corrigido.
  2. Monitorar as importações subsequentes para confirmar que os campos estão sendo preenchidos corretamente.


## Dicas e Recomendações

  * Certifique-se de que as alterações realizadas correspondem ao procedimento correto (SAFX07 de Saídas).
  * Segregue os procedimentos de entradas e saídas para evitar confusões futuras.
  * Mantenha documentação atualizada sobre a configuração dos campos no SAFX07, facilitando o entendimento para outros usuários ou equipes.
  * Se necessário, consulte a equipe de suporte técnico para validar as configurações e evitar erros relacionados ao layout.
