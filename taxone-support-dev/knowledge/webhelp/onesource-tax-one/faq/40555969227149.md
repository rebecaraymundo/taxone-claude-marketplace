## Problema:

Usuários enfrentam problemas com o campo obrigatório de movimento entrada/saída que não está sendo preenchido automaticamente durante a carga de saídas.

## Causa Raiz:

A causa raiz do problema é a ausência de um valor De-Para preenchido/cadastrado para o valor enviado a tag G_LINES campo TRX_TYPE_DETERMINANT no XML, nesse caso o valor enviado era "GOOD_SHIPMENT" na regra 11 do mapeamento, o valor De-Para estava vazio, ocasionando o valor nulo no momento do processamento.

## Resolução:

  1. Acesse o mapeamento do procedimento SAFX08 Saídas.
  2. Verifique o campo 04 Movimento Entrada/Saída e sua respectiva regra De/Para (regra 11).
  3. Se o valor "GOOD_SHIPMENT" não estiver preenchido, você pode:
     * Inserir manualmente o valor De-Para na regra 11 ou
     * Processar a carga 1136 responsável por buscar os valores cadastrados no Oracle Cloud.
  4. Insira o valor PARA, para os campos enviados à tag G_LINES campo TRX_TYPE_DETERMINANT.
  5. Realize uma nova carga de teste para confirmar se o problema foi resolvido.


## Próximos Passos:

Após seguir os passos acima, valide se o campo está sendo preenchido corretamente. Se o problema persistir ou surgirem novas dúvidas, entre em contato com o suporte para mais assistência.

## Dicas e Recomendações:

Mantenha sempre o mapeamento de layout e as regras De/Para atualizados para evitar problemas de preenchimento automático de campos. Consulte regularmente a documentação técnica disponível e participe de treinamentos oferecidos para entender melhor o processo e as ferramentas utilizadas.
