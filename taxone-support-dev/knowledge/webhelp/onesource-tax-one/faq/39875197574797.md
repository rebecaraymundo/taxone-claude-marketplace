## Problema

Há inconsistências recorrentes no tratamento de códigos NBM/NCM, com perda de dígitos, gerando divergências em relatórios e conciliações, exigindo ajustes manuais.

## Causa Raiz

Este erro ocorre devido à configuração incorreta do parâmetro "Pos. Inicial NBM", que está configurado para ignorar os primeiros dígitos do NBM/NCM.

## Resolução

  1. Acesse a transação **/n/MSAFX/PARAMETROS** no sistema SAP.
  2. Configure o campo "Pos. Inicial NBM" na aba "Configurações" de acordo com o necessário, caso a SAFX esteja extraindo o código incompleto, garanta que o campo esteja vazio.
  3. Salve as alterações realizadas.
  4. Realize novamente a extração da SAFX e valide se todos os códigos NCM estão completos.


### Próximos Passos

Após implementar a solução:

  * Verifique se a SAFX08 e a SAFX2013 estão apresentando os códigos NCM completos, garantindo que os relatórios de CAPA X ITEM e conciliações estejam consistentes.
  * Realize testes adicionais com diferentes notas fiscais para assegurar que o problema foi resolvido em outros casos também.


###

### Dicas e Recomendações

  * Evite modificar parâmetros ou configurações do sistema sem uma análise prévia para identificar possíveis impactos em outros processos.
  * Realize uma revisão periódica das configurações de tratamento no SAFX para evitar falhas semelhantes.
  * Considere manter backups dos arquivos de configuração antes de realizar qualquer modificação.
