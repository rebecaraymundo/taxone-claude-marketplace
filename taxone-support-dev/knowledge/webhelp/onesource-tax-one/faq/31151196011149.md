Ao executar uma carga para SAFX82 via extrator da Oracle e ela finalizar com o status “Sem dados a Enviar”, realize os passos a seguir:

  1. Verifique se o XML da Oracle Cloud gerou as informações necessárias.
  2. Havendo informações, verifique se mesmo com a carga finalizada com o Status de “Sem dados a enviar”, o XML foi gerado.
  3. No Interdados, acesse **Log\Consulta Processamento por Status**.
  4. Localize o PST_ID da carga em questão e clique em "**DOWNLOAD XML EXTRATOR** ".
  5. Abra o extrator e verifique se o arquivo XML "LACLS_ExtractApSpedData_Incremental.xml" existe. Abra o XML em um editor de texto como o Notepad++ e clique em "**Localizar** " (Esta SAFX tem três extratores).
  6. Dentro da campo localizar, coloque os caracteres [^0-9A-Za-z\t\n\r\<\>\|\$\?\ \"\=\:\/\\-\\+\%\@\\*\\#\\(\\)\;_,.], marque a opção **Expressão regular** e depois clique no botão **Localizar** no documento atual.
  7. Verifique se houve retorno de caracteres como \n, e caso positivo, se estão na TAG <EFD_PROD_ADD_INFO>.
  8. Se estiver nesta TAG, existe um Patch da Oracle para tratar especificamente esta TAG.
  9. Sendo assim, aplique o patch abaixo, e uma nova rodada de extração da SAFX82 deve ser realizada. Se possível com as mesmas informações.


**Informações do Patch: Patch Oracle:** 36627743 AP EXTRACTOR - EFD_PROD_ADD_INFO TAG HAS INVALID CARACTERS.

Se, mesmo após a aplicação do patch, o problema persistir, abra um chamado no Service Desk para análise.
