Este artigo tem como objetivo explicar o motivo da notificação de atenção ao parametrizar o Tipo Docto Msaf em relação ao Tipo Documento: "Validade é obrigatória". Essa notificação aparece na tela de Parâmetros do Tipo Docto Msaf x Tipo Documento.





Módulo |  Municipais > Parâmetros Para Município
---|---
Menu: |  Parâmetros > Tipo Docto Msaf x Tipo Documento


![Revisão KBA - PRINT 2.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/27624048145805)



Observação: A validade é considerada "a partir de" uma data específica. Essa data serve para identificar o grupo do Tipo de Documento. No exemplo fornecido, foi informada a data de Validade 01/01/1990, mas ao acessar a tela de Tipo de Documento:





Módulo |  Básicos > Data Warehouse
---|---
Menu: |  Manutenção > Códigos > Tipo de Documento


Ao verificarmos o Tipo de Documento utilizado na Parametrização Tipo Docto Msaf x Tipo Documento, vemos que o início da Validade é 01/01/1900

![Revisão KBA - PRINT 1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/27624048146573)

Portanto a data a ser inserida deve ser maior que parametrizada

Quando inserimos a data 02/01/1900, é possível salvar a parametrização sem nenhum erro ou impedimento:

![Revisão KBA - PRINT 3.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/27624048147469)

A data de validade a ser informada não é a data em que foi cadastrado o tipo de documento e sim a data que vai começar a valer o parâmetro.
