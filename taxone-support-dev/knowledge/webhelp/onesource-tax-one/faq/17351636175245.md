**Nota:** Este problema ocorre, pois em alguns casos, a Interface carrega o campo 191 - COD_MUNIC_ARMAZ_ORIG, mas sem carregar o campo 145 - UF_ARMAZ_ORIG, causando a mensagem na importação:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/25175165101837)

Para resolver esse problema, é necessário verificar se os campos estão como não tratados, caso estejam, realize o procedimento abaixo:

  1. Acesse o 'Mapeamento de Campoa' através da transação **/n/MSAFX/SAFTR057** ou através da transação **/n/MSAFX/MASTERSAF**.
     1. Selecione a **SAFX07** pelo processo **ZSAFE026**.
  2. Localize o campo **145**.
     1. Na coluna de “Tratamento”, se estiver como N, altere o tratamento de N para **S** .
     2. Verifique se o campo **148 – UF_ARMAZ_DEST** está tratado como N, caso esteja, altere o tratamento dele para **S**.
  3. Após alterar o tratamento de N para **S** e clique em salvar.
