O Objetivo deste artigo é orientar a correção de divergência de dados com o período informado para geração das cargas via Extrator.

Ao realizar as cargas das SAFX07 e SAFX09 via Extrator, podem ocorrer problemas com as datas geradas, diferente do período informado para extração das cargas no Interdados.

Existe o parâmetro **LAST_UPDATE** onde o mesmo determina o período que estará extraindo, podendo ser este D-1, dessa forma, ao gerar a carga no Interdados solicitando qualquer período, o XML sempre virá com um dia a menos, tanto na **LAST_UPDATE_FROM** , quanto na **LAST_UPDATE_TO.**

Desta forma, é necessário verificar a configuração do SYSDATE no Oracle Cloud.
