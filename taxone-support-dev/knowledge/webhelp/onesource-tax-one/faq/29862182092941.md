Caso identifique que os valores do ISS Retido estão sendo preenchidos incorretamente nos campos de integração para o Registro A100 na EFD Contribuições, não correspondendo ao esperado conforme a legislação vigente. Isso pode ocorrer porque os campos configurados para receber o Valor do ISS na integração para o Registro A100 são diferentes dos campos usados para o ISS Retido. Por exemplo, os campos 189 e 58 estão sendo utilizados em vez dos campos 46 e 33.

Para resolver, siga as orientações:

  1. Acesse o sistema de integração de dados.
  2. Navegue até as configurações dos campos de mapeamento para o Registro A100.
  3. Modifique os campos de Valor do ISS para que o campo 46 - VLR_ISS (SAFX07) ou 33 - VLR_ISS (SAFX08) sejam usados para o registro do Valor do ISS, conforme a legislação do Ato COTEPE 70/05.
  4. Salve as alterações e execute um teste de integração para garantir que os valores estejam sendo preenchidos corretamente.
  5. Após realizar as alterações sugeridas, é recomendável monitorar os próximos envios de dados para a EFD Contribuições para assegurar que os valores do ISS sejam reportados corretamente no Registro A100. Em caso de discrepâncias, entre em contato com o suporte técnico.


**Importante:** Revisar periodicamente as configurações de integração e os mapeamentos de campo para garantir a conformidade com as atualizações legislativas. Manter a documentação técnica atualizada e realizar verificações regulares pode prevenir problemas futuros e garantir a precisão dos dados fiscais reportados.
