# O objetivo deste artigo é orientar o usuário a identificar possíveis causas da divergência de tipos de documentos na SAFX53

## Problema:

Tipo de documento incorreto para o IRRF

## Causa Raiz:

A causa raiz do problema foi identificada como um erro na aplicação das regras De/Para para o código. O código 39 foi reconhecido como NFSE enquanto o código 55 como NFE, devido a regras pré-estabelecidas no sistema interdados.

## Resolução:

Para resolver o problema, recomenda-se a alteração da regra De Para no Interdados para os códigos, alinhando os tipos de documento para ambos os extratores:

  * Acesse o Interdados e navegue até a aba Cadastros;
  * Clique em Regras por Descrição e em seguida Tipo de documento;
  * Localize a regra relacionada ao código divergente e altere a regra De Para para o desejado;
  * Faça uma reimportação da Nota Fiscal para verificar se as alterações foram efetivadas corretamente.


## Próximos Passos:

Após realizar a alteração sugerida, monitore as próximas importações para assegurar que o mapeamento está funcionando como esperado. Caso o problema persista, entre em contato com nosso suporte para análise detalhada.

## Dicas e Recomendações:

Mantenha as regras de mapeamento do sistema regularmente revisadas para evitar inconsistências futuras. Consulte a documentação técnica fornecida ou entre em contato com o suporte sempre que tiver dúvidas sobre processos de mapeamento e importação de documentos.
