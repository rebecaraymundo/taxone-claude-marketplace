Objetivo: demonstrar como solucionar a falha durante o aplicação da request de atualização.

  1. Após realizar a aplicação da Request Principal (WorkBench), ela é finalizada com o retorno:


![imagem](https://suporte.thomsonreuters.com.br/hc/article_attachments/15801802308621)

Encerrado com código de retorno: ==> 12 <==.

  1. Neste caso está ocorrendo falha ao atualizar a tabela /MSAFX/SAFTB082, conforme mensagem acima. Durante a aplicação da request principal, não está conseguindo ativar a tabela /MSAFX/SAFTB082 no Dicionário de Dados.


Para resolver este problema:

  * Exclua e recrie a tabela mencionada no log de erro através da transação SE14 e DB02. Com este processo o Dicionário de dados irá entender que a tabela existe e está ativa.
  * Após realizar este processo, reaplique a request principal, sendo finalizada com sucesso dê continuidade com o processo de atualização.
