# Criação Correta de Arquivo do SIMPLISS no Município de Volta Redonda

###

## Problema:

O erro na geração do arquivo do SIMPLISS para o município de Volta Redonda foi identificado devido à parametrização incorreta.

## Causa Raiz:

O erro decorreu da seleção errada do valor no campo **Es_Documento_tipo**. Foi parametrizado o valor “SE”, o qual não é compatível com as configurações permitidas pelo layout do sistema SIMPLISS, que apenas aceita os valores "1 – Nota Fiscal", "3 – Cupom Fiscal" e "4 – Recibo".

## Resolução:

  1. Acesse o módulo Municipal > Parametros para Município e verifique os tipos de Documentos, precisa estar cadastrado com os valores aceitos: "1 – Nota Fiscal", "3 – Cupom Fiscal" e "4 – Recibo".

  2. Verifique também em Municipal > Parametros para Município > Parametros > Cadastros dos Tipos de Documento das Obrigações > Tipo Docto. Msaf x Tipo Docto. Deverá escolher apenas 1, 3 ou 4.

  3. Salve as configurações e reinicie o processo de geração do arquivo.


## Próximos Passos:

Após efetuar a correção, gere novamente o arquivo e verifique se o erro persiste. Caso o problema continue, entre em contato com nosso suporte técnico para uma avaliação mais detalhada.

## Dicas e Recomendações:

Revisar periodicamente as configurações estabelecidas para garantir que estejam em conformidade com os manuais operacionais disponíveis para o SIMPLISS. Evite alterações nas parametrizações sem a orientação adequada ou desconhecimento das implicações técnicas dessas mudanças.

Consulte o [Manual de Layout do SIMPLISS](<https://manual.layout.simpliss.com>) para outras dúvidas.
