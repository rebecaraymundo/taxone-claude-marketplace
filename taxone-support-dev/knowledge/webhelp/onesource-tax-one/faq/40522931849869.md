## Este artigo tem como objetivo orientar os analistas sobre a forma de validação do erro de sistema fechar a tela em uso.

## Obs:. Manter este artigo visível apenas para usuários internos

## Problema

O sistema Tax One está fechando automaticamente a tela de inclusão de novo documento fiscal, mesmo enquanto está em uso, causando interrupções no fluxo de trabalho dos usuários.

## Causa Raiz

O problema está relacionado com uma instabilidade no ambiente de programação do sistema, e não está relacionado com intermitências e incidentes de performance na plataforma. Com base nas análises já realizadas, não é possível identificar uma causa específica para estes casos e por isso necessita de monitoramento em tempo real para validação.

## Resolução

Siga os passos abaixo para resolver o problema:

  1. Realizar agenda técnica com o cliente e gravar o problema acontecendo em tempo real.
  2. Realize testes de performance na conexão e latência usando os comandos abaixo:
     * **Ping:** Execute `ping prod01.taxone.thomsonreuters.com`. Caso o ping esteja bloqueado, utilize o comando `curl -w "\ndnslookup: %{time_namelookup} | connect: %{time_connect} | appconnect: %{time_appconnect} | pretransfer: %{time_pretransfer} | redirect: %{time_redirect} | starttransfer: %{time_starttransfer} | total: %{time_total} | size: %{size_download}\n" -so /dev/null https://prod01.taxone.thomsonreuters.com`.
  3. Realize os mesmo testes na base Tax One do Suporte, cuide para testar no mesmo ambiente utilizado pelo cliente (UAT, UAT02, PROD, PROD02, PROD03).
  4. Abra uma demanda para avaliação do Time Técnico (MFS) Certifique-se de que todas as evidências estejam anexadas na demanda e dê as permissões de acesso para o vídeo da gravação.
