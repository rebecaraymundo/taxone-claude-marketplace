O objetivo deste artigo é orientar o consultor\cliente como corrigir o problema referente a inserção de dados na tabela da SAFX53 após atualização da versão V17.009.8.16.



## Problema

Após a atualização do Interdados para a versão V17.009.8.16, os dados da tabela SAFX53 estão sendo inseridos diretamente na tabela SAFX53_DW em vez do sinônimo SAFX53, que faz o reapontamento para o Data Warehouse (DW). Como consequência, o DW não consegue importar os dados corretamente.

##

## Resolução

A funcionalidade de extração via Serviço Java foi desenvolvida para atender exclusivamente à integração com o TaxOne (T1) em ambientes SaaS. No caso de ambientes On Premise, o Serviço Java não está configurado para realizar o reapontamento necessário entre SAFX53 e sua tabela correspondente no DW.

Para mitigar o problema, recomendamos os seguintes passos:

  1. Utilize o modo de execução da carga de dados **SEM o Serviço Java** , que está configurado para realizar o reapontamento automaticamente para o sinônimo SAFX53.
  2. Certifique-se de que o procedimento armazenado (PLSQL) está sendo utilizado como alternativa para importação de dados diretamente no DW.
  3. Em casos de maior necessidade de performance, considere ajustar os períodos de carga para minimizar o impacto do tempo de execução.


## Próximos Passos

Após implementar a solução paliativa:

  1. Acesse o Interdados, ir em Procedimentos e clicar no Lápis da SAFX53.
  2. Desmarque a opção “Serviço Java” e clique em Salvar. Assim será realizada a configuração para realizar o reapontamento automaticamente para o sinônimo SAFX53.
  3. Certifique-se de que o procedimento armazenado (PLSQL) está sendo utilizado como alternativa para importação de dados diretamente no DW.
  4. Em casos de maior necessidade de performance, considere ajustar os períodos de carga para minimizar o impacto do tempo de execução.


## Dicas e Recomendações

  * Para evitar problemas semelhantes, mantenha um controle rigoroso sobre as atualizações de versões e analise previamente o impacto em seu ambiente On Premise.
  * Considere migrar para a versão SaaS, que possui suporte completo para integração com as funcionalidades modernas do Interdados.
  * Esteja atento ao processo de descontinuação do DW Solução Fiscal, que está previsto para 2025. Planeje a migração para soluções mais modernas e de longo prazo.
  * Consultem a documentação oficial da versão V17 para melhor compreensão das funcionalidades e limitações.
