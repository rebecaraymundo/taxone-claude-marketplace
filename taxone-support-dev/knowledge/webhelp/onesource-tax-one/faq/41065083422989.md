## Problema

O usuário precisa confirmar as URLs de conexão para o Webservice no ambiente de PRD.

## Causa Raiz

A causa raiz é a necessidade de identificar o ambiente correto de PRD (Produção) para a utilização das URLs específicas de conexão do Webservice.

## Resolução

  1. Acesse o Tax One do cliente.
  2. Clique no ícone de informação no canto superior direito.
  3. Identifique o ambiente:
     * Se aparecer "OSMastersaf", o ambiente é 01.
     * Se aparecer "OSMastersaf02", o ambiente é 02.
  4. Utilize as URLs correspondentes ao ambiente identificado:
     * Para o ambiente 01, use as URLs que começam com `https://uat01.taxone.thomsonreuters.com` ou `https://prod01.taxone.thomsonreuters.com`
     * Para o ambiente 02, use as URLs que começam com`https://uat02.taxone.thomsonreuters.com` ou `https://prod02.taxone.thomsonreuters.com`

### Próximos Passos

Após implementar as URLs corretas na interface, teste a conexão para garantir que está funcionando conforme esperado. Em caso de problemas, contate o suporte técnico.
