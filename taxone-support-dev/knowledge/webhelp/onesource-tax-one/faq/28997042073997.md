O objetivo deste artigo é mostrar como o consultor\cliente que enfrenta dificuldades especificamente no módulo Interface Oracle V17, devido ao volume de dados que precisa ser processado manualmente e que os dados necessitam da parametrização da Regra 105 – Código de Serviço.

O cliente\consultoria utiliza no parâmetro de Mapeamento para campo COD_SERVICO a opção ITEM_NUMBER na qual necessita que o De-Para tenha que ser preenchido o “Para” no parâmetro e pelo fato de serem muitos dados a sererm preenchidos o cliente solicitou alguma forma de realizar o procedimento automaticamente.

Para este caso, o interdados não tem uma opção para realizar o preenchimento do “PARA” através de importação ou processo automático.

Neste caso a orientação é realizar a solicitação de melhoria no sistema fia Customer Incfluencer.

Após submeter a solicitação, o cliente\consultoria monitore o status da mesma através do portal do do Customer Influencer. Se necessário, entre em contato através do e-mail Customer_Influencer_Tax@TRTEN.onmicrosoft.com para esclarecimentos adicionais.



## Dicas e Recomendações:

Antes de enviar a solicitação, verifique se a melhoria já não está disponível ou em processo de desenvolvimento. Isso pode economizar tempo e esforço. Mantenha-se atualizado com as atualizações do sistema para maximizar a utilização das funcionalidades disponíveis.
