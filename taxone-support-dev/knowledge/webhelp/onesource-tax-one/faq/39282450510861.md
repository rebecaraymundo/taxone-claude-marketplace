# Como corrigir falhas na integração de cargas Interdados x TaxOne

## O objetivo deste artigo é orientar o usuário sobre as possíveis causas que ocasionam problemas na importação das cargas do Interdados no TaxOne.

## Problema:

Durante a carga de dados do Oracle EBS para o TaxOne, arquivos JSON não estão sendo gerados corretamente, impedindo a integração e importação de lotes. Esse problema também afeta múltiplos procedimentos SAFX.

## Causa Raiz:

A causa do problema está associada à configuração incorreta da coluna `TAB_DEST_INTERF` na tabela `MSAFI_MSFTS_LAYBD`. A definição dos procedimentos SAFX utiliza sufixos como `_DW`, que não estão alinhados ao padrão esperado para integração com o TaxOne, interferindo na geração adequada dos arquivos JSON.

## Resolução:

Siga os passos abaixo para corrigir as configurações e garantir a geração correta dos arquivos JSON:

  * Verifique a configuração atual:


  1. Execute o seguinte comando em seu banco de dados para verificar os valores de `TAB_DEST_INTERF` associados aos procedimentos SAFX:


sql select * from MSAFI_MSFTS_LAYBD where cod_procedimento = (código do procedimento)

  * Atualize os valores de `TAB_DEST_INTERF`:


  1. Substitua os valores incorretos que contém sufixos `_DW` pelo nome correto do procedimento.


Por exemplo:

sql update MSAFI_MSFTS_LAYBD set TAB_DEST_INTERF = '' where COD_PROCEDIMENTO = ; COMMIT; ```

  * Reinicie o serviço de carga:


  1. Após a atualização, reinicie os serviços relacionados à carga no Oracle EBS.


  * Realize testes:


\- Execute uma nova extração e verifique a geração do arquivo JSON. Utilize o comando SQL abaixo para validar: ```sql SELECT INFO_MESSAGE FROM MSAFI_MSFSY_PROC_LOG WHERE PST_ID = ? AND MENSAGEM LIKE 'TRANSMITINDO DADOS TAXONE WS'; ```

  * Repita o processo para todos os procedimentos SAFX:


  1. Confirme que todos os procedimentos utilizam o nome correto sem sufixos (_DW). Caso contrário, aplique o processo descrito acima individualmente para cada procedimento impactado.


## Próximos Passos:

Após a implementação da resolução:

  * Monitore a geração de arquivos JSON para os próximos lotes e certifique-se de que estão de acordo com os padrões definidos no manual técnico.
  * Teste todos os procedimentos SAFX integrados ao TaxOne para garantir que as alterações aplicadas resolvam o problema para todos os cenários.
  * Informe a equipe de suporte ou consultoria técnica caso outros problemas sejam identificados durante os testes.


## Dicas e Recomendações:

  * Confirme se o manual técnico do TaxOne está acessível e atualizado para que os dados gerados sigam os padrões de integração.
  * Estabeleça como prática revisar as configurações de tabelas e parâmetros durante processos de Go Live para evitar impedimentos que afetam cronogramas do projeto.
  * Documente qualquer ajuste técnico realizado, incluindo scripts SQL e resultados obtidos, para facilitar futuras auditorias e manutenções.
  * Caso nenhum JSON seja gerado mesmo após a correção, revise o parâmetro `BACKEND.WRITE.LOG` na tabela `MSAFI_MSFTS_CONFIG_PROP` e confirme que ele está configurado como 'S'.


Para mais informações, recomendamos consultar os recursos adicionais sobre parametrizações e layouts na plataforma da empresa.
