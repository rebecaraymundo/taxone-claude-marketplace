### **Exemplo de Requisição 'Envio Completo' na API Gateway do OBI**

**Problema:** Dúvidas sobre a estrutura correta (JSON Body) para realizar a requisição de envio completo de um arquivo de integração via API Gateway do OBI.

**Resolução:** O endpoint de **Envio Completo** permite a submissão de um ou mais arquivos de integração de uma só vez, com os parâmetros de processamento necessários.

#### **Estrutura JSON da Requisição**

A requisição deve ser estruturada com os metadados no corpo principal e o conteúdo do arquivo dentro do array `arquivos`.

Campo | Descrição | Exemplo
---|---|---
`**cnpjEmpresa**` | CNPJ da empresa/matriz a que se refere o lote. | `"99999999999999"`
`**codProcessamentos**` | Código do processamento (SAFX) que deve ser executado. | `"SAFX2013"`
`**dataIni**` | Data de início do período que o processamento deve cobrir. Formato: AAAAMMDD. | `"20240101"`
`**dataFim**` | Data de fim do período que o processamento deve cobrir. Formato: AAAAMMDD. | `"20240430"`
`**arquivos**` | Array contendo os metadados e conteúdo do(s) arquivo(s). | `[...]`

#### **Exemplo de Envio Completo (JSON Body)**

JSON


    {
      "cnpjEmpresa": "99999999999999",
      "codProcessamentos": "SAFX2013",
      "dataIni": "20240101",
      "dataFim": "20240430",
      "arquivos": [
        {
          "nome": "SAFX2013.txt",
          "bytesBase64": "NAkwMDAwMDAwMDAwMTA2MTM3MzkgI..."
        }
      ]
    }


**Próximos Passos e Recomendações:**

  * **Codificação:** O conteúdo do arquivo de texto (ou SAFX) deve ser **codificado em Base64** antes de ser inserido no campo `bytesBase64`.
  * **Processamento:** O OBI usará os parâmetros (`dataIni`, `dataFim`, `codProcessamentos`) para processar o conteúdo do arquivo enviado.
  * Após o envio, monitore o Painel de Integralções (Tax Automation) para verificar se os dados foram transmitidos e processados corretamente no Tax One.
