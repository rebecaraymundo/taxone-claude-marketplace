# Como corrigir o preenchimento incorreto do campo ESP_TRIBUTO na importação SAFX53 do TaxOne

## Problema

Ao realizar a importação do arquivo SAFX53 no TaxOne, o campo **ESP_TRIBUTO** foi preenchido com o valor “nu”, enquanto o campo na nota fiscal estava preenchido. O valor esperado não foi importado corretamente.

## Causa Raiz

O problema ocorreu devido ao tamanho do campo utilizado para preenchimento do campo **ESP_TRIBUTO** parametrizado na Regra 19. O sistema utiliza uma regra de-para (Regra 19) que depende do valor enviado ao campo **TAX_TYPE_CODE** no XML da nota fiscal. Se a configuração ou o valor enviado não estiverem corretos, o campo pode ser preenchido incorretamente.

## Resolução

  1. Verifique o valor enviado ao campo **TAX_TYPE_CODE** no XML da nota fiscal que está sendo importada.
  2. Acesse o sistema Interdados e revise a configuração da regra de-para (Regra 19) para que os valores mapeados estejam corretos e condizentes com o esperado para o campo **ESP_TRIBUTO**.
  3. Se necessário, ajuste o mapeamento na Regra 19 para contemplar corretamente os valores enviados ao **TAX_TYPE_CODE**.
  4. Após realizar as alterações, execute uma nova importação do arquivo SAFX53 e verifique se o campo **ESP_TRIBUTO** está sendo preenchido conforme o esperado.
  5. Caso outros campos apresentem erro (exemplo: **COD_TRIBUTO**), realize o mesmo procedimento de revisão e ajuste da regra de-para específica (Regra 20).
  6. Se persistirem dúvidas ou problemas, entre em contato com o suporte informando os detalhes do erro e os ajustes realizados.


## Próximos Passos

  * Após ajustar as regras de-para e realizar nova importação, valide o preenchimento dos campos diretamente no sistema para garantir que os dados foram carregados corretamente.
  * Monitore as próximas importações para assegurar que os ajustes realizados solucionaram o problema de forma definitiva.


## Dicas e Recomendações

  * Mantenha as regras de-para sempre atualizadas conforme alterações nos layouts de XML ou na legislação vigente.
  * Documente todas as alterações realizadas nos mapeamentos para facilitar futuras manutenções e auditorias.
  * Antes de realizar grandes importações, execute testes com notas fiscais de exemplo para validar o correto preenchimento dos campos.
  * Consulte o manual do produto ou base de conhecimento para entender o funcionamento dos campos obrigatórios e suas regras de preenchimento.
  * Em caso de dúvidas, acione o suporte técnico e forneça informações detalhadas sobre o erro, incluindo exemplos de XMLs e screenshots das regras de-para.
