Nota: Este dump pode ocorrer por conta do calendário de fábrica do SAP não estar mais na data vigente ou se o calendário utilizado não for BR, pois na **SAFX53** é verificado se o calendário é BR, e quando não é ocorre o dump.

Atenção: Orientamos que em alguns casos em que o cliente não utiliza o calendário brasileiro, que altere o tratamento do campo DATA_VENCTO de S para N no mapeamento pela transação /n/MSAFX/SAFTR057, pois esse campo não é chave e com isso não gera erro de importação para o cliente.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/25176120466701)

  1. Para verificar o **calendário de Fábrica do SAP** , acesse a transação standard: **SCAL**.
     1. Selecione “**Calendário Fábrica** ” e clique no botão lápis.
  2. Verifique se as datas para o calendário não estão vencidas.
Ex.: BR – Calendário Brasileiro
  3. Caso o calendário esteja vencido, é necessário alterar:

     1. Selecione o ID e clique no botão lápis para modificar.

  4. Modifique o campo “válido até” e salve.
