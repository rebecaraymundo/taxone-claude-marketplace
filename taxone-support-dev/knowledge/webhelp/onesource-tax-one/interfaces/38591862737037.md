O objetivo deste artigo é orientar a consultoria\cliente, a parametrizar e analisar o SEGMENT referente ao Livro GL.


Em alguns casos a consultoria\cliente se depara com um problema relacionando ao SEGMENT de conta ou o COD_CONTA de uma SAFX.
O COD_CONTA no caso, busca o Segmento da conta parametrizada na tela do “Livro GL” do interdados.

No XML do Oracle Cloud, o Campo XML é o CONCATENATED_SEGMENTS da TAG XML G_LINES.

No Interdados, o SEGMENT é parametrizado no Livro GL que fica em Parametrização Dados Origem\Livro do GL.
Neste parametrização do Livro GL, a consultoria\cliente tem a Empresa, Livro GL, Descrição Livro do GL, Conta Contábil, Cento de Custo e Delimitador.

O Centro de Custo é o SEGMENT e o Delimitador é a separação dos SEGMENTS que o consultor\cliente encontrará no XM e que em uma análise é fundamental.
Sempre parametrizar o Delimitador conforme está no XML, evitando assim futuros problemas.

**Analisando o Campo XML CONCATENATED_SEGMENTS para entender se o SEGMENT está correto:**
Se o COD_CONTA for preenchido com valor 5 e o cliente questionar informando que deveria ser 2 por exemplo, o cliente poderá verificar se o XML está trazendo as informações correta.

Exemplo: No XML, está vindo o <CONCATENATED_SEGMENTS>4000000.57.01050.00000.28001001005.526210001.0000000.0.0</CONCATENATED_SEGMENTS> e no Interdados está parametrizado o Centro de Custo (SEGMENT) 5, então o COD_CONTA será 28001001005.

O SEGMENT é identificado conforme o Centro de Custo (SEGMENT) e o Delimitador neste caso é importante justamente para realizar a contagem e entender qual é o SEGMENT.

**A contagem é feita conforme abaixo.**
Temos o <CONCATENATED_SEGMENTS>4000000.57.01050.00000.28001001005.526210001.0000000.0.0</CONCATENATED_SEGMENTS>
Cada range de números é separado pelo delimitador, onde neste caso é o ponto (.), então neste Campo XML, podemos dizer que temos 9 SEGMENTS, conforme podemos ver abaixo:
SEGMENT1 = 4000000
SEGMENT2 = 57
SEGMENT3 = 01050
SEGMENT4 = 00000
SEGMENT5 = 28001001005
SEGMENT6 = 526210001
SEGMENT7 = 526210001
SEGMENT8 = 0
SEGMENT9 = 0

Problemas que possam causar diferença de COD_CONTA.
1 – Delimitador parametrizado no Interdados, diferente do que está no Campo XML – Corrigir o Delimitador deixando igual está no XML e executar uma nova carga da SAFX correspondente.
2 – COD_CONTA mostrado no JSON diferente do que está parametrizado no Interdados. – Neste caso, deve ser aberto um chamado para o Suporte Técnico da Thomson Reuters.
3 – COD_CONTA está vindo nulo. – Verificar se o Livro GL está cadastrado.

**Dicas e Recomendações:**
Verifique regularmente as parametrizações no sistema para garantir que todas estejam corretas e atualizadas, evitando problemas nos processos de carga e relatórios. Em caso de dúvidas ou problemas recorrentes, não hesite em entrar em contato com o suporte.
