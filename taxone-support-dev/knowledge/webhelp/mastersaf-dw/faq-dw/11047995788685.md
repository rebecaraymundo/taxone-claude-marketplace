## Introdução

O objetivo deste artigo é demonstrar e auxiliar na resolução do erro 90384: "O Campo Código de Atividade Econômica não está cadastrado".

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/11047705183501)

## Passo a passo

Este erro ocorre quando o campo 22 - COD_ATIVIDADE é alimentado por algum código que o Mastersaf DW não encontrou na base. De acordo com o LOG, é necessário importar a TACES01 entretanto, é muito possível que o código que está sendo criticado não exista na TACES01 também.

Sendo assim, as soluções para esse erro seriam importar a TACES01 caso seja um código que exista na mesma. Para os casos de código que não conste na tabela, é necessário inseri-la na TACES01 e importá-la, ou realizar o cadastro no seguinte caminho:

**BÁSICOS > PARÂMETROS > Preliminares > Atividade Econômica**
