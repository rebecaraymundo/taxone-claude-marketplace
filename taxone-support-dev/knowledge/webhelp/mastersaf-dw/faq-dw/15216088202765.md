# Introdução

Este artigo visa demonstrar como resolver o problema de eventos gerados na versão incorreta por meio de exclusão.

# Passos

Para resolução da situação seguir passos abaixo.

  * Caso tenha feito a geração dos eventos incorretamente na versão 2.1.1 e os mesmos já estão rejeitados, deve efetuar uma geração prévia dos movimentos na versão 1.5.1 e verificar se serão criados os eventos originais.
  * Caso as notas fiscais que precisa entregar são de prestadores que já tem evento no painel, então deve reprocessar o evento para criar a retificação.
  * Além disso, recomendamos fazer uma nova equalização do Datamart do período.


Para excluir tudo da sua base do DW que foi gerado na versão 2.1.1 pode rodar o script abaixo:


    delete from reinf_pger_apur where cod_versao = 'v2_01_01'1

**Informativo:** O REINF na versão 2.1.1 foi postergado para 09/2023, portanto qualquer evento gerado deverá ser na versão 1.5.1
