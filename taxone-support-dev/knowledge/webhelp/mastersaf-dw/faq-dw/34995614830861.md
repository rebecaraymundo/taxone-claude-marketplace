# Resolução de Erros no Reprocessamento de Tributos Após Correção no Controle de Tributos

**Problema:** Usuário encontrou um erro ao tentar reprocessar o tributo 0422 após ter corrigido a base no Controle de Tributos. A flag de reprocessamento estava ativada, mas o sistema não gerou o evento para envio à Receita Federal do Brasil (RFB).

**Causa Raiz:** O erro ocorreu devido algum problema no documento, com isso a necessidade de excluir o movimento no controle de tributos antes de conseguir reprocessar e gerar o evento para o envio após a correção.

**Resolução:**

  * Excluir o movimento no Controle de Tributos relacionado ao tributo 0422.


  * Realizar a geração do evento R-9000 de exclusão.


  * Após a exclusão, adicionar novamente o movimento no Controle de Tributos com os valores corrigidos.


  * Realizar a geração prévia do movimento para o tributo R-4020.


  * Enviar o evento corrigido para a RFB.


  * Fazer o fechamento do período com o evento R-4099.


**Próximos Passos:** Após seguir os passos anteriores para a correção, monitorar a situação para qualquer outro erro. Caso persistam dificuldades, entrar em contato com o suporte.

**Dicas e Recomendações:** É crucial conferir todos os movimentos e valores no Controle de Tributos antes de iniciar o processo de geração de quaisquer eventos. Manter o sistema sempre atualizado para evitar incompatibilidade de versões e outros problemas técnicos.
