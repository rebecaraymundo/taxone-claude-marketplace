# Introdução:

Este artigo tem por objetivo demonstrar como corrigir o erro 91323: O campo Indicador de Cliente sem Identificacao deve ser nulo, S ou N

## Passo-a-Passo

1\. Erro

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414660551437/mceclip0.png)

2\. O problema ocorre porque o CEP que está no arquivo pertence a uma UF diferente da que está informada no arquivo.

![mceclip1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414668924045/mceclip1.png)

3\. Existem 2 maneiras de ajustar o problema:

* Se o CEP estiver correto, verificar na tela BÁSICOS > DATAWAREHOUSE > Manutenção > Cadastros > CEP qual a UF que ele pertence;

* Se a UF estiver correta o CEP deve ser ajustado no arquivo;

No exemplo o CEP estava incorreto, então foi realizado o ajuste;
