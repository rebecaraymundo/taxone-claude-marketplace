**Objetivo desta FAQ é ensinar como realizar a emissão do Livro Batch**

Neste menu o usuário pode fazer a programação, execução e emissão de relatório para a geração dos Livros Fiscais P1 (entradas), P2 (saídas), P9 (apuração) entre outros.

Esta geração dos livros é feita através da programação de processo que sejam disparados através de schedule (sem a interferência do usuário).

_Obs. É necessário ter cadastrado antes o calendário com as datas de apuração do livro correspondentes ao período desejado._

Neste passo a passo iremos aprender a;

**1.** Fazer a programação batch dos tipos de livros fiscais.

**2.** Fazer a execução da programação batch criada no passo anterior.

**3.** Fazer a emissão do histórico da programação batch (relatório).

**Programação**

Permite ao usuário fazer a programação da geração dos livros disponibilizados na listagem “Tipos de Livro” informando os dias em que a geração é realizada para cada estabelecimento.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21703182526477)

Dados a serem informados:

**Programação:** Apresenta automaticamente para o usuário, o número da programação.

**Descrição:** Informar a descrição que para a programação.

**Programação Válida a partir de: **Informar a data em está sendo feita a programação. A programação somente pode ser executada a partir dessa data.

**Periodicidade:** Informar a Periodicidade com que esta programação é executada, podendo ser:

_Executar Programação em Qualquer Dia:_ Programação não será executada em um ou alguns dias específicos do mês. Pode ser executada em qualquer dia

_Gerar nos dias:_ Programação a ser executada em dias específicos do mês.

**Obs.:** Este campo somente fica habilitado se a opção escolhida no campo Periodicidade for Gerar nos dias.

_Eventual:_ Programação para um período de data específica. É executada apenas no período contido no campo Período.

_Dias Específicos:_ Programação a ser executada em dias específicos.

**Período:** Informar o período para a programação, com a data inicial e final.

**Obs.:** Este campo somente fica habilitado se a opção escolhida no campo Periodicidade for Eventual.

**Prog. Dias Específicos:** Informar os dias específicos que deseja fazer a programação, tendo a opção de escolha UDM (Último Dia do Mês).

**Obs.:** Este campo somente fica habilitado se a opção escolhida no campo Periodicidade for Dias Específicos.

**Estabelecimentos:** Informar o(s) estabelecimento(s) que deve(ão) constar nesta programação.

No exemplo, a programação ficou como valida a partir do dia 01/10/2023 a Periodicidade com data fixa no dia 30. Selecionei o Estabelecimento de geração e o tipo de livro; 108 - Registro de Apuracao do ICMS - P9 , 108 - Registro de Apuracao do ICMS ST - P9 e Ajuste SINIEF-ST.Para saber tipos de livros disponíveis para o procedimento acima acesse o [Manual Operacional - Livro Batch](<https://webhelp.thomsonreuters.com.br/Site/HELPDW/V2R010/Estadual/icms/oper_safic04.htm#menu_batch>)

**Execução**

Para utilizá-la, é preciso informar a data equivalente ao dia atual ou a que deixou de ser executada,

para que esta seja comparada com as datas programadas.

Dados a serem informados:

**Código da Programação:** Informar o código da programação (podendo ser um ou todos) para a execução.

**Data Processamento:** Preencher com a data atual ou data em que a programação deixou de ser executada.

A programação somente é executada se os seguintes critérios forem atendidos:

\- Data Processamento igual ou superior à data informada na Programação, campo “Programação Válida a partir de”;

\- Para programação onde a opção “Gerar nos dias” foi selecionada, o dia da Data Processamento deve ser um dos dias

programados (vide campo Prog. Dias Específicos).

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21703233175821)

Relatório pemiti verificar o histórico da programação.

**Histórico**

Emitir o relatório do histórico dos processos batch gerados, informando o período de apuração dos livros, conforme o calendário fiscal e o número da programação que são utilizados como filtro para emissão do relatório.

Nesta rotina Emitir o relatório do histórico dos processos batch gerados, informando o período referente as datas de execução e o número da programação, que são utilizados como filtro para emissão do relatório.

Esta rotina possui os seguintes campos:

**Período:** Informar o período de geração do livro, conforme calendário fiscal, para a verificação do histórico.

**Programação:** Informar o código da programação para a visualização do histórico.
