---

Isso pode ocorrer quando o arquivo está no formato UTF-8.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/26775893475213)
Para corrigir este problema, basta converter o arquivo para ANSI e se certificar que o delimitador de fim de linha que está LF, esteja como CRLF no formato windows ANSI:
![](https://suporte.thomsonreuters.com.br/hc/article_attachments/26775872703501)

Obs: Conforme manual, o formato padrão de codificação utilizado na montagem dos arquivos “.txt” é** “ANSI”.**
