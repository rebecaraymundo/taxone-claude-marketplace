# [Básicos][Segurança] DW - Após configuração do AD, erro de permissão de usuário.

Este erro ("Não foi possível acessar a base de dados de permissão do usuário") pode ser causado por várias razões, incluindo:

  1. O servidor LDAP não está em execução ou não está acessível a partir do cliente. Isso pode ser devido a problemas de rede, como um firewall bloqueando a conexão, ou o servidor pode estar desligado ou ter falhado.
  2. As credenciais fornecidas pelo cliente são inválidas. Se o cliente está tentando se autenticar no servidor LDAP, ele precisará fornecer um nome de usuário e senha válidos. Se essas credenciais estiverem incorretas, a conexão falhará.
  3. A URL ou porta do servidor LDAP especificada pelo cliente está incorreta. O cliente precisa saber onde o servidor LDAP está localizado para se conectar a ele. Se a URL ou porta estiverem incorretas, o cliente não conseguirá encontrar o servidor e a conexão falhará.


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/26776288199565)

Para corrigir esse erro, você precisará identificar a causa exata do problema. Isso pode envolver a verificação da configuração do servidor LDAP, a verificação da configuração do cliente, a verificação da rede entre o cliente e o servidor, e a verificação das credenciais usadas para autenticação.
