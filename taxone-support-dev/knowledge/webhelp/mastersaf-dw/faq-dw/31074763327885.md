# **REGRAS PARA SEREM SEGUIDAS PARA O ENVIO DE ONETIME-USER DO SERVIÇO DE SFTP PARA TESTE DOS CLIENTES.**

Assunto:

Este Artigo tem como objetivo instruir o Time de Suporte quando for solicitado usuário do serviço de SFTP para testes de performance da taxa de transferência.

Causa:

Ao realizar a transmissão do DUMP é percebida baixa performance interna na rede do cliente. Após ação de correção do DBA precisam testar a taxa de transmissão para validar se a houve correção através das ações que foram aplicadas.

Solução:

\- A Solicitação deve ser feita pelo cliente através de Ticket para que o usuário onetime-user seja enviado para realizar teste performance.

**Observações Importantes:**

1 - Orientar os clientes que este usuário deve ser utilizado exclusivamente para testes e não para transmissão completa do arquivo DUMP ou outros fins.

2 - Orientar que após a conclusão dos testes, deve enviar os detalhes (Qtd de arquivos, Nome, Tamanho) do que foi enviado através do ticket aberto.

3 - Após receber os detalhes, deve se fazer a abertura de Demanda ADO ao time de Techops com todas as informações solicitando a exclusão dos dados do servidor da TR afim de que esses arquivos não sejam importados e nem fiquem parados no servidor.

4 - Indicar na Planilha de SFTP- Users Support que o usuário foi disponibilizado para testes - utilizar a descrição: **Envio de usuário SFTP para teste da Taxa de Transferência (Após teste arquivos serão deletados)**
