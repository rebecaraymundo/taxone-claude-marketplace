## PROBLEMA

Erro na tabela "DATA_ESCR_CONT" durante o processo de importação da SAFX53

## SOLUÇÃO PROPOSTA

É comum informar em campos DATA a informação "00000000", como no exemplo abaixo:

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21827021949325)

Porém, de acordo com nosso manual de Layout de SAFX, este campo não aceita os caracteres "00000000", isso porque ao inserir no sistema, essa data é Inválida, para que seja importada com Sucesso a SAFX é preciso ajustar para @, após esse ajuste é possível importar normalmente a SAFX53 sem erros nesta tabela.

![](https://suporte.thomsonreuters.com.br/hc/article_attachments/21827028184845)

Qualquer dúvida, devem abrir chamado na Thomson Reuters.
