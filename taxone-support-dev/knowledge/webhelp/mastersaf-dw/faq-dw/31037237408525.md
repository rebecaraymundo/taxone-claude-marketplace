## Problema:

Usuários não conseguem gerar relatórios na máquina local quando executáveis estão configurados para rodar via compartilhamento de rede a partir de um servidor.

## Causa Raiz:

A configuração do arquivo msafdw.properties está correta no servidor, porém, quando o executável é usado através de um compartilhamento de rede, o sistema não reconhece a máquina local como destino para salvar os relatórios, entendendo que deve ser salvo apenas no servidor.

## Resolução:

  * Verifique se o arquivo msafdw.properties está configurado corretamente no servidor.


  * Opte por uma das duas soluções a seguir:


  * Instale o software diretamente em cada máquina local dos usuários e configure o arquivo msafdw.properties individualmente.


  * Mapeie uma pasta no servidor de rede acessível a todas as máquinas locais para que os relatórios sejam salvos nessa localização e acessados por todos os usuários.


## Próximos Passos:

Após implementar a solução escolhida, verifique se os relatórios estão sendo gerados e salvos corretamente na localização desejada. Realize testes com diferentes usuários para garantir a funcionalidade em diferentes máquinas.

## Dicas e Recomendações:

Considere a possibilidade de realizar uma formação ou criar um documento de orientação para os usuários finais, explicando como acessar os relatórios na nova configuração. Isso pode evitar confusões e chamados futuros relacionados ao mesmo problema.
