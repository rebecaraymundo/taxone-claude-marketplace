Para a utilização desta rotina é necessário:

  * Ter o Java instalado no servidor da aplicação (não precisa estar instalado no servidor do banco de dados). As versões compatíveis estão no manual de instalação, segue o link para consulta: https://suporte.thomsonreuters.com.br/hc/pt-br/articles/360053084951-Manuais-T%C3%A9cnicos
  * O Informe de Rendimentos (Grandes Volumes) NÃO necessita a instalação do PDF Writer. A geração do PDF é feita pelo Java.


  * O arquivo msafdw.properties, deve ser alterado com o IP do Banco de Dados do cliente (bd.server), o nome do Banco do cliente (bd.database), a informação se utiliza ou não a autenticação SSL no Servidor de email do cliente (email.usaSSL), a informação se utiliza ou não autenticação no Servidor de email do cliente (email.indAuth). Esse arquivo somente é utilizado na opção de Grandes Volumes, portanto, configura-lo para o outro item de menu não surte efeito algum.
  * NÃO utilizar a opção: “Envio usando o Outlook”;
  * Utilizar a opção: “Envio usando o Winsock”. Configurando as informações de SMTP, Porta, De:, Usuário e Senha que são do ambiente do cliente.


  1. bd.server=< Número do IP do banco de dados >
  2. bd.port=< Porta do banco de dados >
  3. bd.database=< Utilizar o que está informado no arquivo TNSNAMES em SERVICE_NAME >


*Validações importantes para correto funcionamento no ambiente *

1 - Na última linha do 'msafdw.properties' insera mais isso:
email.sslProtocols=TLSv1.2

2 - Verifique se o JAVA_OPTS está assim. (Digite na barra do Windows: "Variáveis de ambiente")

![](https://atendimentotr.zendesk.com/attachments/token/o6Ejvx6DeVhKWLWIBK6KrBTEa/?name=inlineImage.png)

\- Caso as configurações estejam devidamente parametrizadas e não for enviado o e-mail, será necessário que o cliente verifique diretamente com a sua Infra se chegaram as solicitações no servidor de e-mail, ou seja, se o servidor de e-mail recebeu a requisição para o envio do e-mail, ou se ocorreu algum erro no servidor ou não.
