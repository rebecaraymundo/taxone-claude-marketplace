**Artigo: Resolvendo o Erro 90713 na Gravação do Datamart de Documentos Fiscais**

O erro 90713, um aviso indicando um erro não tratado na gravação do Datamart de documentos fiscais, pode surgir devido a inconsistências ou problemas nos dados presentes nas tabelas temporárias fundamentais para a geração correta do Datamart. Identificamos que uma solução eficaz para solucionar esse problema é realizar uma limpeza nas tabelas temporárias DWT_DOCTO_FISCAL, DWT_ITENS_MERC e DWT_ITENS_SERV, seguida pela reimportação dos dados que estão causando esse aviso.

**Passos para Resolver o Erro 90713:**

  1. **Backup dos Dados:** Antes de iniciar qualquer procedimento, é altamente recomendável fazer um backup completo das tabelas envolvidas. Isso garante a segurança dos dados atuais e permite restaurá-los caso algo inesperado ocorra durante o processo de limpeza e reimportação.

  2. **Limpeza das Tabelas Temporárias:**

     * DWT_DOCTO_FISCAL (tabela temporária)
     * DWT_ITENS_MERC (tabela temporária)
     * DWT_ITENS_SERV (tabela temporária)

Execute a exclusão dos dados presentes nessas tabelas temporárias. Certifique-se de seguir os procedimentos adequados para a limpeza dos registros e assegure-se de remover apenas os dados indesejados.

  3. **Reimportação dos Dados Afetados:** Após a limpeza das tabelas temporárias, é necessário reimportar os dados que estavam causando o erro 90713 no Datamart de documentos fiscais. Utilize os registros corretos e verificados para garantir a precisão das informações.

  4. **Verificação pós-importação:** Após a reimportação dos dados nas tabelas temporárias mencionadas, realize uma verificação minuciosa para garantir que o processo foi concluído com sucesso e que o erro 90713 não está mais sendo gerado durante a gravação do Datamart de documentos fiscais.


**Importante:** Execute esses passos com atenção, assegurando-se de seguir os procedimentos corretos para evitar a perda de dados importantes. Se necessário, envolva a equipe de TI ou profissionais responsáveis pelo gerenciamento do banco de dados para realizar essas operações.

A correção dessas inconsistências nas tabelas temporárias DWT_DOCTO_FISCAL, DWT_ITENS_MERC e DWT_ITENS_SERV, seguida pela reimportação dos dados, deve resolver o aviso de erro 90713 que você está enfrentando ao gravar o Datamart de documentos fiscais.

Caso persistam dúvidas ou necessite de suporte adicional durante esse processo, estamos à disposição para ajudar e fornecer orientações adicionais.
