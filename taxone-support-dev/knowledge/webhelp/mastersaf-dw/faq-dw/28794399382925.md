# [SPED][EFD - REINF]DW - Documentos sem retenção considerados nos eventos do REINF

## Problema:

Documentos sem base, alíquota ou imposto de INSS estão sendo incorretamente considerados na geração prévia, resultando em erros no envio dos eventos.

## Causa Raiz:

A causa do problema está relacionada à configuração do parâmetro "Desconsiderar NFs sem Valor de Retenção informado" no módulo REINF. Esse parâmetro precisa ser verificado e ajustado conforme necessário.

## Resolução:

  * Acesse o módulo REINF do sistema.


  * Navegue até os Dados Iniciais.


  * Localize o parâmetro "Desconsiderar NFs sem Valor de Retenção informado".


  * Verifique se o parâmetro está ativo. Caso não esteja, ative-o para desconsiderar as Notas Fiscais sem valor de retenção.


  * Salve as alterações e gere novamente a prévia dos documentos.


## Próximos Passos:

Após ajustar o parâmetro e regerar a prévia, verifique se o erro persiste. Se o problema continuar, entre em contato com o suporte técnico para uma análise mais detalhada.

## Dicas e Recomendações:

Regularmente verifique se há atualizações ou patches novos disponíveis para o sistema, pois eles podem conter correções importantes para problemas conhecidos. Além disso, sempre confira as configurações dos parâmetros após realizar atualizações para garantir que tudo está configurado corretamente.
