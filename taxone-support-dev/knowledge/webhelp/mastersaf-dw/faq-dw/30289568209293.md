# Solução para a Não Atualização de Eventos no DW Após Retorno do Onesource

## Problema:

O usuário reportou que o evento R4099 não está retornando do Onesource para o Data Warehouse (DW), mesmo após tentativas de retorno para a solução fiscal e reinicialização do integrador.

## Causa Raiz:

A análise indica que o JOB responsável pela leitura dos eventos retornados não está sendo executado automaticamente. Isso pode ser devido a uma configuração inadequada na quantidade máxima de JOBS em paralelo no banco de dados, que está configurada para um valor abaixo do recomendado.

## Resolução:

  * Verifique se o usuário de integração está ativo.


  * No Onesource REINF, selecione o evento e clique no ícone de três pontos (...) e escolha 'Retornar para solução fiscal'. Aguarde alguns instantes para ver se o status no painel do DW é atualizado.


  * Se o problema persistir, execute o seguinte comando no banco de dados do DW para um teste manual:






begin

reinf_processa_retorno.executar;

end;

  * Se o comando acima atualizar o status dos eventos, entre em contato com o DBA do banco para verificar por que o JOB `DWT_PROCESSA_STG_IN`, que automatiza o comando, não está sendo executado automaticamente.


## Próximos Passos:

Após a execução do comando e a verificação com o DBA, monitore o sistema para garantir que os eventos estão sendo atualizados conforme esperado. Se o problema for resolvido, não são necessárias mais ações. Caso contrário, entre em contato com o suporte técnico para mais assistência.

## Dicas e Recomendações:

Regularmente verifique as configurações de execução de JOBS no banco de dados para garantir que estão conforme as práticas recomendadas e ajustadas para as necessidades atuais do sistema. Isso pode prevenir problemas futuros com a execução e atualização de eventos.
