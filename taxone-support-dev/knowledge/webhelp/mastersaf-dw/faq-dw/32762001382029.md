# Criação de Conta Diferente para Aplicar Patches em Ambiente de Banco de Dados

## Problema:

Usuários questionam a possibilidade de criar uma conta diferente da conta MASTER para aplicar patches em um ambiente de banco de dados, para atender a auditorias e aumentar a segurança.

## Causa Raiz:

A necessidade surgiu de uma recomendação de auditoria para aumentar a segurança nas operações de aplicação de patches, propondo a criação de uma conta com privilégios limitados e rotacionamento de senha.

## Resolução:

  * Verificar todas as permissões necessárias que a conta atual possui para aplicação de patches.


  * Criar uma nova conta de usuário no banco de dados com essas permissões.


  * Implementar políticas de rotacionamento de senha para esta nova conta.


  * Utilizar esta nova conta para a aplicação de patches, garantindo que ela tenha acesso restrito conforme as necessidades de segurança.


## Próximos Passos:

Após a criação da nova conta, monitorar as primeiras sessões de aplicação de patches para garantir que tudo está funcionando como esperado. Regularmente revisar as permissões e a política de segurança para ajustes necessários.

## Dicas e Recomendações:

Manter a documentação atualizada sobre as contas de usuários e suas permissões é crucial para a segurança e auditoria do sistema. Considere realizar auditorias periódicas para garantir que as políticas de segurança estão sendo seguidas corretamente.
