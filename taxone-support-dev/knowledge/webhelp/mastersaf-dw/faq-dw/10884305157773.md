## Introdução

O objetivo desse artigo é como tratar o erro "Um processo de Equalização/Importação está sendo executado por outro usuário no momento", que pode ocorrer tanto na importação quanto na equalização:

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/10275194363149)![](http://www.mastersaf.com.br/HELPDW/V2R010/Conteudo/Demais_Empresas/Mastersaf_DW/DW/KBA_FAQ/FAQ/FAQ_-_Impossibilidade_de_login_na_aplica%C3%A7%C3%A3o_\(Citrix\)_1_00.gif)

## Causas

No Mastersaf DW, existe uma funcionalidade que é acionada sempre que um usuário executa um processo concorrente (Ex.: importação/equalização). Esta, se comporta como uma trava de segurança, já que processos concorrentes trabalham com o mesmo tipo tabela (definitivas "X" do documentário fiscal) em que um pode impactar no outro.

Seu funcionamento consiste em registrar o IND_UTILIZACAO como 'S' no banco de dados (PRT_IDENT_DMART) quando determinada tabela estiver em uso, então é alterado automaticamente para 'N' ao final do processo. Isso impede que dois ou mais usuários utilizem a mesma tabela enquanto uma importação/equalização estiver ocorrendo.

## Solução

Se de fato existe outro usuário utilizando a(s) tabela(s), deve-se aguardar o fim do processo.

Caso não haja usuários concorrendo pelas mesmas tabelas, o mais provável é que o ambiente/rede passou por instabilidades enquanto um processo ocorria, impedindo a troca do indicador ao fim do processo, travando o registro como se estivesse em uso indefinidamente. Neste caso, é possível alterar o status manualmente, através do menu: **Ferramentas > Parâmetros do Sistema > Status de Concorrência**. Nessa tela, deve-se alterar o indicador de utilização (IND_UTILIZACAO) para 'N' (não).

Caso sejam muitos registros, se o usuário preferir, é possível executar um UPDATE na tabela PRT_IDENT_DMART, conforme abaixo:


    UPDATE PRT_IDENT_DMART SET IND_UTILIZACAO = 'N' WHERE IND_UTILIZACAO <> 'N'

Obs.: Se o ambiente/rede está instável e os usuários utilizam o DATAMART Automático, é bem provável que seja necessário desativá-lo até corrigir o ambiente, pois o problema pode ser muito frequente, já que o DATAMART seria executado automaticamente a cada documento fiscal importado/alterado.
