# [Básicos][Job Servidor] DW -  Mastersaf DW - Job Servidor - Importação batch, o sistema não identifica arquivos cujo nome contém sufixos como "_V1"

**Problema**

Usuários reportaram que, ao realizar uma importação batch para carregar arquivos do Bloco K, o sistema não identifica arquivos cujo nome contém sufixos como "_V1", "_V2" ou "_V3". No entanto, ao remover esses sufixos, a importação ocorre corretamente, indicando uma possível restrição nos formatos aceitos.

**Causa Raiz**

O problema ocorre porque a programação do processo de importação batch possui um tratamento específico que busca arquivos no diretório apenas com um formato padrão especificado no manual de layout. O sistema aceita apenas o sufixo de data no formato esperado, sem combinações de outros textos, como "_V2". Esta configuração está associada à funcionalidade "Nome de arq. com data", que precisa estar devidamente marcada.

**Resolução**

Siga os passos abaixo para adequar os nomes dos arquivos e garantir que a importação batch ocorra corretamente:

Certifique-se de que o nome dos arquivos contém apenas o prefixo correspondente à empresa (ex.: CB01 ou CB02) seguido pelo nome do layout da SAFX (ex.: SAFX109).

Evite utilizar sufixos como "_V1", "_V2" ou "_V3" nos nomes dos arquivos, pois o sistema não os reconhecerá.

Caso seja necessário utilizar datas como sufixo para diferenciação, habilite a opção "Nome de arq. com data" no módulo de programação batch.

Consulte o manual do layout, se necessário, para verificar as especificações completas. O manual pode ser acessado no seguinte link: [Módulo Job Servidor](<https://suporte.thomsonreuters.com/module-job-servidor>).
