## Problema:

Aviso de vulnerabilidade detectada - Sensitive Information Disclosure

## Resolução:

A aplicação MasterSAFDW é uma aplicação client-server utilizando Powerbuilder e Oracle-client.

Uma sugestão para criptografar os pacotes de comunicação entre o MasterSAF-DW e o Oracle Client para garantir a segurança dos dados durante a transmissão. Isso geralmente é feito configurando o Oracle Net Services para usar criptografia e integridade de dados.

Essa configuração, no caso do MasterSAF-DW, por ser on premise, deve ser verificada pelo seu DBA.

Aqui estão os passos básicos para configurar a criptografia:

Configurar o Oracle Net Listener: Edite o arquivo listener.ora no servidor Oracle para incluir as configurações de criptografia. Você pode especificar o método de criptografia desejado e a exigência de criptografia.
Configurar o Oracle Client: Edite o arquivo sqlnet.ora no cliente Oracle (que o PowerBuilder usa para se conectar ao banco de dados) para incluir as configurações de criptografia compatíveis com as definidas no servidor. Isso inclui especificar o método de criptografia e se a criptografia é exigida ou solicitada.
Reiniciar o Listener: Após configurar a criptografia no listener.ora, você precisa reiniciar o Oracle Net Listener para que as novas configurações tenham efeito.
Testar a Conexão: Após configurar tanto o servidor quanto o cliente, teste a conexão do PowerBuilder com o Oracle para garantir que a criptografia está funcionando conforme esperado.

Um exemplo de configuração no sqlnet.ora para exigir criptografia seria:

Plaintext
SQLNET.ENCRYPTION_SERVER = required
SQLNET.ENCRYPTION_TYPES_SERVER= (AES256)
SQLNET.ENCRYPTION_CLIENT = required
SQLNET.ENCRYPTION_TYPES_CLIENT= (AES256)
SQLNET.CRYPTO_CHECKSUM_SERVER = required
SQLNET.CRYPTO_CHECKSUM_CLIENT = required
SQLNET.CRYPTO_CHECKSUM_TYPES_CLIENT = (SHA1)
SQLNET.CRYPTO_CHECKSUM_TYPES_SERVER = (SHA1)


Este exemplo configura o Oracle para usar AES256 para criptografia e SHA1 para checksums de integridade, tanto no servidor quanto no cliente.
Lembre-se de que a configuração exata pode variar dependendo da versão do Oracle e dos requisitos específicos de segurança da sua organização. É importante consultar a documentação oficial do Oracle para as versões específicas que você está usando e considerar a consulta com um especialista em segurança de banco de dados para garantir que a configuração atenda às necessidades de segurança da sua aplicação.
