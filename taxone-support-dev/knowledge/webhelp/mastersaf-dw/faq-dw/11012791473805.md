_Problema:_

_Na listagem dos Créditos Extemporâneos está gerando em duplicidade. EX:

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/11012677383437)

_

_Solução:_

_Rodar o Select:

_

SELECT APT_CALC_BEM_EXTEMP.COD_EMPRESA,
APT_CALC_BEM_EXTEMP.COD_ESTAB,
APT_CALC_BEM_EXTEMP.COD_BEM,
APT_CALC_BEM_EXTEMP.COD_INC,
EMPRESA.RAZAO_SOCIAL,
ESTABELECIMENTO.RAZAO_SOCIAL,
ESTABELECIMENTO.CGC,
REGISTRO_ESTADUAL.INSCRICAO_ESTADUAL,
APT_CALC_BEM_EXTEMP.DATA_APURACAO,
X13_BEM_ATIVO.DESCRICAO,
APT_CALC_BEM_EXTEMP.DATA_INI,
APT_CALC_BEM_EXTEMP.DATA_FIM,
APT_CALC_BEM_EXTEMP.NUM_PARCELA,
APT_CALC_BEM_EXTEMP.VLR_PARC_APROP,
APT_CALC_BEM_EXTEMP.VLR_TOT_SAIDA,
APT_CALC_BEM_EXTEMP.VLR_SAIDA_TRIB,
APT_CALC_BEM_EXTEMP.VLR_PARCELA,
X233_BEM_EXTEMP.NUM_PARCELA QTD_PARCELAS,
X233_BEM_EXTEMP.DATA_ATIV_BEM,
NVL(X233_BEM_EXTEMP.VLR_ICMS,0) VLR_ICMS,
NVL(X233_BEM_EXTEMP.VLR_ICMSS, 0) VLR_ICMSS,
NVL(X233_BEM_EXTEMP.VLR_ICMS_FRETE,0) VLR_ICMS_FRETE,
NVL(X233_BEM_EXTEMP.VLR_ICMS_DIFAL, 0) VLR_ICMS_DIFAL

FROM APT_CALC_BEM_EXTEMP,
X13_BEM_ATIVO,
X233_BEM_EXTEMP,
APT_ESTAB,
ESTABELECIMENTO,
EMPRESA,
REGISTRO_ESTADUAL

WHERE ( APT_CALC_BEM_EXTEMP.COD_EMPRESA = 'xxx') and

( ( APT_CALC_BEM_EXTEMP.COD_ESTAB IN ('xxxx')) ) and
( APT_CALC_BEM_EXTEMP.DATA_APURACAO = to_date( to_char('xxxxxxxx', 'YYYYMMDD'))) and
APT_CALC_BEM_EXTEMP.COD_BEM = 'xxxxxxx' and
( registro_estadual.cod_empresa (+) = estabelecimento.cod_empresa) and
( registro_estadual.cod_estab (+) = estabelecimento.cod_estab) and
( registro_estadual.ident_estado (+) = estabelecimento.ident_estado) and

(( X233_BEM_EXTEMP.COD_EMPRESA = APT_CALC_BEM_EXTEMP.COD_EMPRESA ) and
( X233_BEM_EXTEMP.COD_ESTAB = APT_CALC_BEM_EXTEMP.COD_ESTAB ) and
( X233_BEM_EXTEMP.DATA_APURACAO = APT_CALC_BEM_EXTEMP.DATA_APURACAO ) and
( X233_BEM_EXTEMP.COD_BEM = APT_CALC_BEM_EXTEMP.COD_BEM ) and
( X233_BEM_EXTEMP.COD_INC = APT_CALC_BEM_EXTEMP.COD_INC )) and

( ESTABELECIMENTO.COD_EMPRESA = APT_ESTAB.COD_EMPRESA ) and
( ESTABELECIMENTO.COD_ESTAB = APT_ESTAB.COD_ESTAB ) and

( EMPRESA.COD_EMPRESA = ESTABELECIMENTO.COD_EMPRESA ) and

( APT_CALC_BEM_EXTEMP.COD_EMPRESA = ESTABELECIMENTO.COD_EMPRESA ) and
( APT_CALC_BEM_EXTEMP.COD_ESTAB = ESTABELECIMENTO.COD_ESTAB )

and (X13_BEM_ATIVO.COD_EMPRESA = APT_CALC_BEM_EXTEMP.COD_EMPRESA)
and (X13_BEM_ATIVO.COD_BEM = APT_CALC_BEM_EXTEMP.COD_BEM)
and (X13_BEM_ATIVO.COD_INC = APT_CALC_BEM_EXTEMP.COD_INC)
and (X13_BEM_ATIVO.COD_ESTAB = APT_CALC_BEM_EXTEMP.COD_ESTAB)

ORDER BY APT_CALC_BEM_EXTEMP.COD_EMPRESA ASC,
APT_CALC_BEM_EXTEMP.COD_ESTAB ASC,
APT_CALC_BEM_EXTEMP.COD_BEM ASC,
APT_CALC_BEM_EXTEMP.COD_INC ASC,
APT_CALC_BEM_EXTEMP.NUM_PARCELA ASC

_
O resultado do select Identificou que tem dois registros com acentuação diferente na descrição, o que esta causando a duplicidade:
