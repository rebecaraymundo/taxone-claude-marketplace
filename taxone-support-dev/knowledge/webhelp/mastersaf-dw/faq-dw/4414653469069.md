# Introdução:

Este artigo tem por objetivo demonstrar como corrigir o erro 91323: O campo Indicador de Cliente sem Identificacao deve ser nulo, S ou N

## Passo-a-Passo

1\. Erro

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414653456525/mceclip0.png)

2\. O problema ocorre pois o campo IND_CLIENTE_SID não possui algum dos valores pré-definidos no manual de layout:

Informar o Indicador de Cliente SID, de acordo com:
S – Se o cliente é sem identificação;
N – Se o cliente tem identificação (CPF ou CGC).
Usado somente pela IN-86 para preenchimento do Item 3 (CPF_CGC) – Arquivo 4.9.1., com a expressão “SID”, para os clientes sem identificação (consumidor final sem CPF/CGC) e SPED Contribuições, registro A100 (campo 04 - COD_PART).

![mceclip1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414660495757/mceclip1.png)

3\. Para solucionar o problema basta informar o valor "S" ou "N", no exemplo acima foi informado o valor "2"

![mceclip2.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414660497677/mceclip2.png)
