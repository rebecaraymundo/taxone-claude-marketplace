## Problema:

O usuário está enfrentando dificuldades com a soma dos valores de notas fiscais existentes com novas notas fiscais durante o processo de retificação, resultando em valores incorretos no sistema.

## Causa Raiz:

A causa raiz do problema parece estar relacionada com a necessidade de uma equalização correta do DATAMART, além de um reprocessamento adequado dos eventos no sistema para a inclusão correta das notas fiscais no arquivo XML de retificação.

## Resolução:

  * Assegure-se de realizar uma equalização do DATAMART para o período correto, entre 01/2024 (data emissão) a 05/2024. (data fiscal)


  * Acesse o painel de controle de eventos no sistema.


  * Localize o evento R-2010 que foi recebido com sucesso para este prestador específico.


  * Clique na opção 'ações' e selecione 'reprocessar evento'.


  * Na janela subsequente, selecione a versão 2.1.2 e marque a opção "Desconsiderar NF's fora da competência (R-2010)" como NÃO.


  * Informe o período específico, neste caso, 05/2024, para gerar um XML de retificação com as novas informações.


  * Antes de enviar, verifique através da opção REINF conferência dos movimentos se a nota fiscal foi corretamente incluída no XML de retificação.


## Próximos Passos:

Após realizar os passos acima, monitore o sistema para confirmar que as notas fiscais estão sendo corretamente acumuladas e refletidas nos relatórios finais. Em caso de discrepâncias, repita o processo de reprocessamento ou contate o suporte técnico.

## Dicas e Recomendações:

Mantenha sempre os parâmetros de configuração do sistema atualizados e verifique regularmente se os procedimentos de retificação estão sendo executados conforme as instruções para evitar problemas de soma de valores em notas fiscais futuras. Consulte o manual do usuário ou entre em contato com o suporte técnico para assistência adicional.
