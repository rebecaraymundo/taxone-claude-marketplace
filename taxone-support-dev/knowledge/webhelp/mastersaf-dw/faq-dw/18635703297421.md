## INTRODUÇÃO


O objetivo desse artigo é auxiliar os clientes na correção da mensagem 'Já existe registro com a chave informada'



## PROBLEMA

Ao cadastrar um novo usuário, ocorre a mensagem do print abaixo:


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/18760044261645)


Esta mensagem ocorre, pois há divergências do campo USR_KEY da tabela PL_USR com a tabela PL_KEYS


![](https://suporte.thomsonreuters.com.br/hc/article_attachments/18759989769613)



## SOLUÇÃO

Para correção realizar o update abaixo:


    UPDATE PL_KEYS
    SET USR_KEY = (SELECT MAX(USR_KEY) FROM PL_USR);
    COMMIT;
