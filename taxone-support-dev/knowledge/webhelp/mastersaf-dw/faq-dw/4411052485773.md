## Introdução

_Este artigo tem por objetivo mostrar as principais causas da geração dos eventos R-2010 e R-2020 não considerarem algumas notas fiscais._

## Passo-a-passo

**Importante** : Utilize o relatório de Validação das Retenções Previdenciárias, do módulo REINF, para conferir se a nota fiscal será considerada no módulo REINF. Você pode utilizar o seguinte artigo, para conferir o passo-a-passo de geração:[ Utilizando o relatório "Validação das Retenções Previdenciárias"](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/4411052470029>)

Se a nota não for exibida no relatório, prossiga com as validações dos passos abaixo. Se ela for exibida, provavelmente já existe algum evento autorizado para este prestador e deve ser gerado um evento do tipo Retificação. Para verificar como gerar este tipo de evento, acesse o artigo [Como gerar eventos do tipo Retificação](<https://suporte.thomsonreuters.com.br/hc/pt-br/articles/4411053118861>).

**1)** O valor de retenção do INSS é carregado nos eventos, através das informações dos itens das notas fiscais. Acesse o cadastro dos itens das notas e verifique se os campos estão preenchidos. Caso não esteja, efetue o preenchimento e salve o cadastro da nota.

Módulo: Data Warehouse > Básicos
Caminho: Manutenção > Documento Fiscal > Novo Documento Fiscal > Doc Fiscal de Serviço (Versão Anterior) ou Doc Fiscal de Mercadoria (Versão Anterior)**![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4411038808717/mceclip0.png)**

**2)** Verifique se o **código de serviço** utilizado no cadastro da nota fiscal, está parametrizado para geração dos eventos do REINF.
Módulo: SPED > EFD-REINF
Caminho: Parâmetros > Retenções Previdenciárias > Identificador do Tipo de Serviço > Por código de Serviço ou Por código de Produto.![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4411039979533/mceclip0.png)



**3)** Verifique a data de entrada das notas e equalize o período no Datamart. Sugerimos que seja equalizado todo o período de entrada de notas que será utilizado na geração. Supondo que há notas cadastradas para os prestadores/tomadores entre 01/01/2021 à 31/03/2021, efetue a equalização do mesmo período.

Módulo: Data Warehouse > Básicos
Caminho: DATA MART > Equalização / Limpeza.![mceclip1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4411038864653/mceclip1.png)
