# Introdução:

Este artigo tem por objetivo demonstrar como corrigir o erro 91684: O campo Indicador de Simples Nacional deve ser nulo, @ , S ou N .

## Passo-a-Passo

1\. Erro

![mceclip0.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414660720013/mceclip0.png)

2\. O problema ocorre pois o campo IND_SIMPLES_NAC não possui algum dos valores pré-definidos no manual de layout:

Informar se é enquadrado como Simples Nacional. Este campo foi criado para atender a DIM São Luis.

A informação deste campo também é utilizada no Módulo Ressarcimento de ICMS-ST – Paraná, para identificar as pessoas física/jurídicas optantes pelo Simples Nacional. Esta identificação é necessária, para verificar o conteúdo a ser preenchido no campo CST_CSOSN das operações de entrada.

Quando o emitente da nota for optante pelo Simples Nacional, o campo CST_CSOSN é preenchido com o Código CSOSN, e quando não, é utilizado o Código de Situação Tributária da Tabela B.

Preencher com um dos valores abaixo:
@ - Nulo;
S – Sim;
N - Não.

![mceclip1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/4414660726541/mceclip1.png)

3\. Para solucionar o problema basta informar o valor "S" ou "N", no exemplo abaixo foi informado o valor "2"
