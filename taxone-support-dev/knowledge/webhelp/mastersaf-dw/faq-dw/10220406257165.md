## Introdução

Esse documento tem por objetivo mostrar como sanar o erro causado por duplicidade de chave para o código de participante no registro 0150.

## Passo-a-passo Duplicidade Registro

1 - Erro na validação do arquivo no validador do SPED Contribuições:

**Duplicidade de ocorrência da chave COD_PART**

![1.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/10220089372557)

1.2 - Esse erro ocorre quando o cliente tem um mesmo código de cadastro pessoa Física/ Jurídica, um com indicador cliente e outro com indicador fornecedor.

**Básicos > Data Warehouse> Manutenção > Cadastros > Pessoas Físicas/Jurídicas**![2.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/10220075137293)

2 - Para corrigir este erro, basta acessar os Dados Iniciais do EFD Contribuições, marcar o parâmetro 'Considerar o Indicador no Código do Participante' e após realizar uma nova geração para o erro de duplicidade do registo 0150 ser sanado.

**SPED >EFD – Escrituração Fiscal Digital das Contribuições >****Parâmetros >Dados Iniciais**

**![3.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/10220089570317)**
