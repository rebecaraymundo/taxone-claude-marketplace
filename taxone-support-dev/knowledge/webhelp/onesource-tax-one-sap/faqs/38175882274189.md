Resolução de Erro no Encerramento da Apuração de PIS/COFINS com Ajustes Manuais Incorretos

**Problema:**

O usuário não consegue encerrar a apuração do PIS/COFINS devido a um erro que é apresentado durante esse processo:

Erro: cannot insert NULL or update to NULL
tax4b.piscof::PR_TF_APURACAO
TrexColumnUpdatee failed on table 'SAPABAPTDF/TMF/D_AJ_DET_P' with error: constraint NOT NULL violation, checkString COLUMN(): found NULL on pos 0, column 'COD_CAT', table 'SAPABAPTDF:/TMF/D_AJ_DET_P'

**Causa Raiz:**

O problema ocorre devido a ajustes manuais estarem inseridos incorretamente e, como resultado, populam de forma errada o campo COD_CTA, o que impede o encerramento da apuração corretamente.

É possivel observar no exemplo abaixo que os ajuste grifados em azul estão populando os campos de forma incorreta.

![Ajuste_COD_CTA.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/38220224252557)

**Resolução** :

Acesse a configuração dos ajustes na apuração de PIS/COFINS.

Verifique se os ajustes manuais estão cadastrados corretamente, principalmente no preenchimento do campo COD_CTA.

Corrija quaisquer inconsistências encontradas nos ajustes manuais.

Após as correções, tente realizar novamente o encerramento da apuração de PIS/COFINS.

**Próximos Passos:**

Após realizar as correções necessárias, monitorar se o encerramento da apuração de PIS/COFINS pode ser concluído sem erros. Se o problema persistir, entre em contato com o suporte técnico para mais investigações.

**Dicas e Recomendações:**

É recomendável verificar regularmente as parametrizações dos ajustes utilizados na apuração para garantir que todos os campos estejam corretamente preenchidos, evitando futuros problemas ao realizar o encerramento da apuração. Consulte a documentação disponível em [suporte](<https://support.company.com>) para mais detalhes sobre a configuração correta de ajustes.
