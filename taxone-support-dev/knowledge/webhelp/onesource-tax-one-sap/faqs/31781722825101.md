Ao gerar a DIME SC, ocorria um erro de validação devido à configuração do código de receita. O código foi inicialmente configurado com 6 caracteres (como 100099), mas o sistema processou apenas as 4 primeiras posições (1000). A causa raiz desse problema é que o campo 'Código de Receita' no registro tipo 33 deve ter exatamente 4 caracteres. O excesso de caracteres configurados fez com que apenas as primeiras 4 posições fossem consideradas, resultando em um erro de estrutura durante a validação do arquivo.

Passo a passo para resolver o erro de código de receita e classe de vencimento inválido no registro 33 ao gerar a DIME SC:

  1. Acesse **Apuração do ICMS-ST**.
  2. Localize a configuração do '**C****ódigo de Receita** ' para o registro tipo 33. Esse código deve ter apenas 4 caracteres.
  3. Altere o **código de receita** configurado atualmente (ex: 100099) para um código válido com 4 posições (ex: 1000).
  4. Salve as alterações realizadas.
  5. Gere novamente o arquivo da DIME SC.
  6. Verifique se o arquivo gerado agora está sendo validado corretamente, sem apresentar erros relacionados ao código de receita.
  7. Caso ainda persista algum erro, entre em contato com o suporte técnico para uma análise mais detalhada da situação.


É importante sempre consultar o manual de orientação e especificações da obrigação fiscal para preencher corretamente todos os campos exigidos, evitando erros de validação.
