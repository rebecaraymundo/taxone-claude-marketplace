Passo a passo para tratar o erro que ocorre ao clicar no botão **Atualizar Saldo **na tela de apresentação do IRPJ:

**_"Erro ao processar a requisição:_**

**_Erro ao atualizar o buffer: Error while EXECUTE statement WITH PARAMS SOL error: insuficient privilege: search table error: [29501 exception 306003: Authorization failed in Calculation Engine. User is not authorized to 'SELECT' "_**

As tabelas abaixo devem possuir privilégios para a execução do sistema.

  1. Verifique se as roles TAX4B_DATASOURCE_ACCESS e TAX4B_DATASOURCE_ACCESS_GRANTABLE**** possuem privilégios nos objetos que retornam do select abaixo (Substituir o valor <**TDFSCHEMA** > pelo schema TDF do ambiente):


**SELECT **CURRENT_SCHEMA, TABLE_NAME **FROM **TABLES **WHERE **SCHEMA_NAME = '<TDFSCHEMA>' **AND **TABLE_NAME **LIKE **'/TMF/D_%';

  1. Para conceder os privilégios, execute os comandos abaixo para cada tabela que não possui acesso (Substituir o valor <**TDFSCHEMA** > pelo schema TDF do ambiente e <**TABLE** > pela tabela que será concedido acesso):


**GRANTSELECT** ,**UPDATE** ,**INSERT** ,**DELETEON **"<TDFSCHEMA>"."<TABLE>" **TO **TAX4B_DATASOURCE_ACCESS;

**GRANTSELECT** ,**UPDATE** ,**INSERT** ,**DELETEON **"<TDFSCHEMA>"."<TABLE>" **TO **TAX4B_DATASOURCE_ACCESS_GRANTABLE **WITHGRANTOPTION** ;

  1. Outra opção é conceder o acesso ao schema TDF como um todo, não sendo necessário efetuar o script para todos os objetos.


Nesse caso conceder acesso de SELECT, UPDATE, INSERT, DELETE para o schema do TDF para as roles TAX4B_DATASOURCE_ACCESS e TAX4B_DATASOURCE_ACCESS_GRANTABLE.

Na role TAX4B_DATASOURCE_ACCESS_GRANTABLE é necessário selecionar `Yes` para a opção `Grantable to Others`.
