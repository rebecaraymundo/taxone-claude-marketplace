1. **Caso seja reportado um erro de atualização ou instalação de objetos do produto TAX ONE FOR SAP deve ser avaliado se foram enviados os LOGs.**


Caso não tenha sido enviado deve ser solicitado conforme orientação a seguir:

Digite o comando **xs mta-ops.**

Localize o ID referente ao objeto que apresentou erro.

Serão listados apenas os deploys que apresentaram erro.

![1- id.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/29714022488845)

Na sequencia digite o camando **xs dmol -i [Infome o ID do objeto que deu erro]**

O downalod sera realizado.

![2- download.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/29714022491021)

,

**2- Avaliação de LOG**

Para avaliar os objetos que deram erro no deploy deverá ser solictado o camando a seguir ao qual deve ser informado o nome do objeto que deu erro:

**xs logs [infome objeto que deu erro] --last 300 >> logs.log**

**ex** : xs logs tax4b-gia_sc-db --last 300 >> logs.log

A aplicação do objeto poderá dar erro nos casos em que o ambiente do cliente possui EXIT(CUSTOMIZAÇÕES) para isso é possível identificar se existe EXIT através do log do deploy .

Geralmente o log do deploy indica se possui EXIT conforme exemplo abaixo, onde caso este possua informação abaixo deverá ser solicitado ao cliente a devida avaliação.![3 id.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/29714069297805)
