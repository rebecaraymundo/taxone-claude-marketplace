# Correção da Geração do Registro E531 no SPED Fiscal - Módulo IPI

## Problema

O cliente identificou que o registro **E531** , relacionado aos ajustes automáticos na apuração do IPI, não estava sendo gerado corretamente no SPED Fiscal, mesmo com o ajuste automático **101.**

## Causa Raiz

O problema ocorreu devido ao sistema não considerar adequadamente o campo **Origem do Processo (IND_DOC)** nas informações complementares dos ajustes automáticos, além de inconsistências na manipulação das sombras que armazenam os dados do ajuste.

## Resolução

Para corrigir o problema de geração do registro E531, foi realizada a atualização dos objetos relacionados ao módulo de Apuração de Impostos Indiretos IPI:

  * **cfg_filtro-backend_tax4b** \- Versão `1.150.1`

  * **cfg_filtro-ui_tax4b** \- Versão `1.150.1`


Passos seguidos durante a resolução:

  1. Realizar o deploy das novas versões dos componentes.

  2. Excluir e recriar as configurações de ajuste automático relacionadas à apuração do IPI.

  3. Reabrir apuração, salvar novamente as configurações e encerrar o processo.

  4. Validar os dados gerados nas sombras `tax4b.tdf::/TMF/D_AJUS_IPI` e `tax4b.tdf::/TMF/D_AJ_IPI_AD`.

  5. Executar os testes no ambiente de qualidade e verificar a consistência dos registros E530 e E531 no arquivo SPED Fiscal gerado.


## Próximos Passos

Após a atualização dos componentes e configuração dos ajustes automáticos:

  * Realizar novos testes em ambiente de qualidade para garantir que o comportamento foi corrigido.

  * Implementar os mesmos passos no ambiente de produção do cliente.

  * Monitorar os dados gerados no SPED Fiscal por um período inicial para garantir que o problema não se repita.


## Dicas e Recomendações

  * Antes de realizar qualquer atualização de componentes, sempre realize o deploy em um ambiente de qualidade para identificar possíveis problemas antes de ir para produção.

  * Mantenha os objetos e configurações documentados para facilitar revisões futuras em casos de problemas similares.

  * Considere realizar auditorias sistemáticas nas apurações fiscais para garantir a consistência dos dados gerados.

  * Certifique-se de que todas as licenças e acessos ao ambiente do cliente estejam regularizados antes de realizar testes cruciais.


## Conclusão

Com a atualização realizada nas versões `1.150.1` dos componentes **cfg_filtro-backend_tax4b** e **cfg_filtro-ui_tax4b** , o problema relatado foi corrigido e o sistema agora gera os registros E530 e E531 corretamente. No entanto, é importante sempre seguir as melhores práticas de validação e monitoramento para evitar problemas futuros.
