# Erro no Aplicativo de Licença TAX14SS: Problemas ao Salvar Configurações

## Problema

Após aplicar o XML de licenciamento no ambiente de produção (PRD), a lista de municípios não estava sendo exibida. Além disso, ao tentar salvar as configurações, a tela congelava e o sistema não conseguia salvar os dados. Também foi identificado um erro relacionado ao tamanho do arquivo MTAR backend, que excedia o limite permitido.

## Causa Raiz

O problema ocorreu devido a uma falha no mapeamento de entidades no banco de dados e na configuração do servidor. Além disso, houve uma limitação no tamanho permitido para o arquivo MTAR backend, que foi excedida, causando falhas durante o processo de atualização.

## Resolução

  1. Realize o seguinte SELECT para verificar os dados na tabela de municípios:

         SELECT * FROM SLT_S4H.J_1BTXJURT WHERE SPRAS = 'P' AND COUNTRY = 'BR';

  2. Confirme se a tabela J_1BTXJURT está sendo corretamente replicada no schema DBADMIN. Caso contrário, revise a configuração interna do sistema.
  3. Verifique a configuração do mandante utilizando o comando:

         select * from "tax4b.koine::CD_UTIL.PARAM".

Certifique-se de que o valor do mandante está configurado corretamente (por exemplo, 400 em vez de 200).

  4. Execute o seguinte SELECT para validar os mandantes no cadastro das ORGSTR:

         SELECT
           "MANDT",
           "CENTRAL_EFD",
           "VIG_DE",
           "VIG_ATE",
           "MANDT_MAIN",
           "EMPRESA_MAIN",
           "FILIAL_MAIN",
           "MANDT_BP",
           "EMPRESA_BP",
           "FILIAL_BP",
           "VIG_DE_MAIN",
           "VIG_ATE_MAIN",
           "VIG_DE_BP",
           "EH_MAIN"
         FROM "tax4b.wrapper::EMPRESA_CENTRAL_EFD"
           (placeholder."$$P_DT_INI$$"='20251001',
           placeholder."$$P_DT_FIN$$"='20251031')

  5. Caso o erro persista, atualize os componentes listados abaixo para as versões corrigidas:
     * **soter-db_tax4b** \- versão 1.150.1
     * **soter-ui_tax4b** \- versão 1.150.2
     * **soter-backend_tax4b** \- versão 1.150.1
  6. Para resolver o problema de tamanho do arquivo MTAR backend, implemente o componente atualizado:
     * **soter-backend_tax4b** \- versão 1.150.2


## Próximos Passos

Após executar as correções e atualizar os componentes, teste novamente o salvamento das configurações e verifique se a lista de municípios está sendo exibida corretamente. Caso o problema ainda persista, entre em contato com o suporte técnico e forneça os logs gerados durante o processo.

## Dicas e Recomendações

  * Certifique-se de que todas as configurações do ambiente estão corretamente ajustadas, especialmente os parâmetros relacionados ao mandante.
  * Mantenha os componentes do sistema sempre atualizados para evitar problemas de compatibilidade e desempenho.
  * Considere documentar os procedimentos internos e realizar treinamentos com a equipe para garantir a correta aplicação de licenças e configurações.
  * Em caso de dúvidas ou problemas, entre em contato com o suporte técnico utilizando os canais apropriados.
