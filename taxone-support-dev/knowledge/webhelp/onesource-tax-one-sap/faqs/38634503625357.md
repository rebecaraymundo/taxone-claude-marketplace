# INTRODUÇÃO

Resolução de Duplicidade de documentos fiscais no arquivo FINTELISS gerado.

## Problema

O problema se refere à duplicidade de documentos fiscais identificada no arquivo gerado no sistema TaxOne for SAP.

## Causa Raiz

A duplicidade ocorre devido a existencia de dois ou mais itens do documento fiscal que compartilham o mesmo código no campo "NUM_REF_LANCTO" na view "NF_ITEM", referenciando estes itens a um único lançamento de retenção na view "NF_IMPOSTO_RETIDO", causando duplicidade no arquivo.

## Resolução

Para resolver a duplicidade de documentos fiscais no arquivo, siga os passos abaixo:

  * Revise os itens do documento fiscal listados e identificar quais possuem o mesmo código no campo "NUM_REF_LANCTO" da view "NF_ITEM".

  * Editar os itens identificados e realizar a correção garantindo que cada lançamento possua um único código referenciando lançamentos de retenção, conforme necessidade.

  * Reenvie o arquivo para processamento para verificar se a duplicidade foi resolvida.


#### Próximos Passos

Após realizar as correções sugeridas, monitore o sistema para verificar se o problema de duplicidade não se repete em outros arquivos ou documentos fiscais.

Caso o problema persista, considere abrir um chamado de suporte para uma análise mais profunda.

#### Dicas e Recomendações

Para evitar futuras ocorrências de duplicidades em notas fiscais, é recomendável fazer uma revisão periódica dos códigos utilizados no campo "NUM_REF_LANCTO" para garantir que eles sejam únicos para cada item dentro de uma mesma nota.

Adicionalmente, considere validar os arquivos antes de seu envio final para identificar e corrigir quaisquer inconsistências imediatamente.
