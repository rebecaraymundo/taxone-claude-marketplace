# SCANC: Novas Regras de Cálculo para os Campos 22 e 23 no Registro Tipo 45

## Problema

Erro no preenchimento dos campos 22 e 23 do registro tipo 45 no módulo Obrigações Oil & Gas - SCANC Contribuinte. O erro ocorre ao tentar atribuir valores para notas fiscais de entrada (DIRECT = 1) com produto BXD.

## Causa Raiz

A ausência de regras específicas para o cálculo dos campos 22 e 23 no registro tipo 45 do SCANC contribui para o preenchimento incorreto ou incompleto dessas informações para notas fiscais de entrada (DIRECT = 1) com produto BXD.

## Resolução

Foram implementadas novas regras de cálculo conforme as especificações técnicas do SCANC:

  * **Campo 22:** Para notas fiscais de entrada (DIRECT = 1) e produto igual a 'BXD', o valor é calculado multiplicando a quantidade do campo 19 (registro tipo 45) pela alíquota do campo 21 (registro tipo 45).
  * **Campo 23:** Para notas fiscais de entrada (DIRECT = 1) e produto igual a 'BXD', o valor é calculado multiplicando a quantidade do campo 20 (registro tipo 45) pela alíquota do campo 21 (registro tipo 45).


Os componentes necessários foram atualizados na versão 1.149.1 do módulo **scanc-db_tax4b**.

## Próximos Passos

Após a implementação das regras, realize os seguintes passos:

  1. Verifique se a atualização para versão **scanc-db_tax4b 1.149.1** foi realizada com sucesso.
  2. Realize testes em uma base de homologação para garantir que os cálculos estão sendo executados de acordo com as regras descritas.
  3. Confirme que os campos 22 e 23 estão sendo preenchidos corretamente para todas as notas fiscais com DIRECT = 1 e produto BXD.
  4. Faça o monitoramento durante os próximos dias para identificar possíveis inconsistências ou erros.


## Dicas e Recomendações

  * Utilize uma base de homologação para validar as atualizações antes de aplicá-las em produção.
  * Certifique-se de que os operadores responsáveis pelo sistema estão cientes das mudanças implementadas.
  * Mantenha o contato com o suporte em caso de dúvidas ou necessidade de análise adicional.
