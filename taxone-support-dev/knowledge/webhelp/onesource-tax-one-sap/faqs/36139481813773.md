## Identificando problemas na Performance do Aplicativo IRPJ Durante o Encerramento Fiscal ou navegação no aplicativo.

#### Problema:

Usuários estão enfrentando lentidão excessiva ao utilizar o aplicativo IRPJ durante o encerramento fiscal, ou navegação no aplicativo tornando sua execução inviável.

#### Causa Raiz:

Para avaliação mais detalhada do cenário, por gentileza, responda as seguintes informações:

  * Foi efetuada alguma análise pela infraestrutura do ambiente (banco de dados e avaliação de ambiente)? Nos envie as evidencias técnicas realizadas que indicam erro de performance no produto TAX ONE FOR SAP;
  * Nos envie o consumo das views, FUNCTIONS ou Procedures relacionadas ao módulo que apresenta o cenário e seus respectivos planos de execução;
Informar volumetria de dados
  * Cenário ocorreu a partir de alguma atualização de objetos? Se, sim quais objetos e enviar deploy dos objetos atualizados;
  * Cenário ocorre para um **período** e ou **empresa** em específico ou para todas empresas e períodos?
Existem novas empresas/organizações inseridas no sistema? Se, sim foi reavaliado o SIZING do ambiente?

**Ponto de Atenção:**
É relevante destacar os fatores que afetam o desempenho das nossas apurações:
O próprio código, sob responsabilidade da TR.
O ambiente, gerenciado pelo cliente (embora eu não saiba quem administra o banco de dados de cada cliente).
O volume de dados.

Vale ressaltar que o código do IRPJ já é otimizado e utiliza buffers para melhorar a eficiência. No entanto, a origem dos dados é dos lançamentos contábeis, especificamente das tabelas ACDOCA e BSEG, disponibilizadas pelas views SALDO_CONTABIL e LANCAMENTO_CONTABIL. Essas views são lentas e onerosas para o ambiente, e a SAP recomenda o particionamento dessas tabelas para melhorar o desempenho (consulte a nota 3325787). Realizamos análises nos clientes, e constatamos que a performance do IRPJ é fortemente impactada pela performance dessas views.

Portanto, é essencial analisar a base de dados, fazer a higienização dos dados e aplicar o particionamento recomendado pela SAP. Apenas ajustes no código não serão suficientes para alcançar a performance ideal, não sendo possível garantir a performance sem esse particionamento.

SAP NOTE:3325787 - Partition-Distribution of Financial Data in SAP S/4HANA - SAP for Me


#### Resolução:

Verifique se o ambiente esta atualizado com as ultimas versões disponibilizadas.

Acompanhe as liberações realizadas pelo Produto, através dos Informativos de versão na central de ajuda do Zendesk - Webhelp.

#### Próximos Passos:

Continue monitorando a performance do aplicativo, especialmente durante períodos de alta carga como o encerramento fiscal. Reporte qualquer inconsistência para o suporte técnico.

#### Dicas e Recomendações:

Considere a possibilidade de realizar o particionamento das tabelas conforme recomendado pela SAP para melhorar a performance geral do sistema. Mantenha o ambiente de banco de dados otimizado e monitorado regularmente.
