## Problema:

O campo 'Município e UF' no processo de contabilização do ISS retido apresentava o mesmo município diversas vezes para seleção e não gravava o município selecionado corretamente.

## Causa Raiz:

A origem do problema pode ocorrer devido a inconsistências nos dados da tabela J_1BTXJURT do sistema SAP, especialmente na formatação do campo TAXJURCODE.

Observou-se que os códigos dos municípios eram formatados como "**UF** **Código IBGE** " (por exemplo, SP 3550308) com um espaço entre a UF e o código IBGE, enquanto para o cenário impactado, os campos estavam sendo apresentados juntos (por exemplo, **PE2607901**).

E possível observar que no cenário impactado o SELECT retorna com o campos **Código Município** com uma "?" ao invés de estar com o código da UF.
Esse código de UF e utilizado no sistema para gravar a informação de forma correta.

![interrrogacao.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/40179629253773)

Em um cenário correto o SELECT retornou com o campo**Código Município** preenchido com a UF + Codigo do município.

![UF e codido municipio.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/40187804150925)![J1BTX.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/40187804152717)

## Resolução:

  1. Realize uma análise da tabela J_1BTXJURT para verificar como os municípios estão sendo registrados.
  2. Utilize o seguinte SELECT no container correspondente para consultar os dados da tabela:
`SELECT SUBSTRING_REGEXPR('[^\s]+' IN "TAXJURCODE" FROM 1 OCCURRENCE 1) as "UF", SUBSTRING_REGEXPR('[^\s]+' IN "TAXJURCODE" FROM 1 OCCURRENCE 2) AS "CODIGO", "TEXT" AS "MUNICIPIO", "CLIENT" FROM "tax4b.wrapper::J_1BTXJURT" WHERE "SPRAS" = 'P' AND "COUNTRY" = 'BR'`
  3. Caso os dados estejam com a formatação incorreta (UF e código IBGE juntos), corrija os registros utilizando o formato padrão "**UF** **Código IBGE** " . Entre em contato com o cliente para alinhar possíveis ajustes.
  4. Execute o SELECT abaixo no container db-contab para verificar se os municípios estão retornando corretamente após a correção:
`SELECT TOP 1000 "UF", "CODIGO", "MUNICIPIO" FROM "FA8938F9409441D698E1CDE563F34AC5"."tax4b.contab::CV_LOOKUP_MUNICIPIO" GROUP BY "UF", "CODIGO", "MUNICIPIO"`


## Próximos Passos:

Após implementar a solução, revise o comportamento do sistema em relação ao problema relatado e observe se o campo 'Município e UF' apresenta os municípios corretamente e grava as seleções sem erros. Reforce ao cliente a importância de utilizar o formato correto para os registros.

## Dicas e Recomendações:

  * Certifique-se de que todos os dados da tabela J_1BTXJURT estão padronizados e formatados corretamente.
  * Documente internamente os passos para correção, permitindo que futuros problemas semelhantes sejam resolvidos com maior agilidade.
  * Em caso de dúvidas entre em contato com o Suporte.
