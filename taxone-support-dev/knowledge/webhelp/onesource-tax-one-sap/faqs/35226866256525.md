# Fluxo botão Documento Bloqueado para Pagamento

## Problema:

Ao utilizar o módulo Tax One For SAP para contabilização de pagamento no SAP ECC, os documentos de pagamento, como uma guia de ICMS, não são incorporados no fluxo do Workflow do SAP ECC após o lançamento pela FB60.

## Causa Raiz:

O Botão "Documento bloqueado para pagamento" em **Contabilização Automática** serve para indicar se a guia contabilizada no S/4 será bloqueada para pagamento, ou seja, para pagamento da guia , alguém precisará aprovar a operação, e cairá no workflow de aprovação do financeiro.

![1-cont.png](https://suporte.thomsonreuters.com.br/hc/article_attachments/37429981539853)

## Resolução:

Se o parâmetro "Documento bloqueado para pagamento" é igual a **SIM** , mandamos 'A - Bloqueado p/ pgto.'

Se o parâmetro for **NÃO** , mandamos 'vazio' que significa 'Autorizado p/ pgto'

A opção para envio do status "A (Pagamento Bloqueado)" é uma definição estabelecida durante o desenvolvimento do sistema de contabilização. Caso o cliente desejar implementar outros indicadores nesse campo para melhor atender aos seus processos, é necessário a abertura de uma sugestão de melhoria através do portal Customer Influencer para que a solicitação seja devidamente avaliada pelo time de Produto.

## Próximos Passos:

Após ajustar as configurações ou solicitar a mudança no parâmetro, monitorar os próximos lançamentos para assegurar que estão sendo corretamente integrados ao Workflow do SAP ECC. Se persistirem problemas, entre em contato com o suporte técnico.

## Dicas e Recomendações:

Verifique regularmente as configurações de parâmetros no Tax One For SAP, assegurando que elas estejam alinhadas com os processos internos de sua empresa. Para uma gestão eficaz, mantenha-se atualizado com as últimas versões e melhorias oferecidas pelo fornecedor do software.
