# Como solucionar problemas na exibição de histórico de contabilizações e estornos no Log de Lançamento Automático

## Problema:

O histórico de contabilizações e estornos dos documentos extemporâneos não estava sendo exibido no Log de Lançamento Automático. O comportamento era correto apenas para documentos de arrecadação do mês.

## Causa Raiz:

O problema ocorreu devido ao comportamento inesperado dos componentes responsáveis pela exibição dos dados no sistema, que não processavam corretamente os documentos extemporâneos ao gerar o histórico.

## Resolução:

Siga os passos abaixo para corrigir o problema:

  1. Atualize os seguintes componentes no ambiente do cliente:

     * `pcc-ui_tax4b` \- versão `1.149.1`

     * `irrf-ui_tax4b` \- versão `1.149.1`

  2. Após a atualização, realize testes no sistema para verificar se o histórico de contabilizações e estornos está sendo exibido corretamente para documentos extemporâneos.

  3. Se o problema persistir, agende uma reunião para análise detalhada no ambiente do cliente.


## Próximos Passos:

Após implementar a solução, confirme se o problema foi resolvido realizando os testes necessários no módulo "Apuração de Impostos Retidos - IRRF". Caso o comportamento não seja o esperado, solicite suporte adicional junto à equipe responsável.

## Dicas e Recomendações:

  * Certifique-se de sempre usar as versões mais recentes dos componentes do sistema, proporcionando maior estabilidade e correções adicionais.

  * Documente os procedimentos realizados e mantenha o histórico de atualizações do ambiente para facilitar futuras análises e suporte.

  * Reforce os testes em ambientes de desenvolvimento antes de realizar alterações no ambiente de produção.

  * Consulte a documentação oficial do sistema para entender melhor a integração entre os módulos.
