# Correção no arredondamento do campo "Alíquota" no arquivo da obrigação municipal ISS-NET

## Problema

Os valores no campo "Alíquota" do arquivo da obrigação municipal ISS-NET estavam sendo gerados com divergências entre o valor informado e o cálculo esperado. Essa discrepância poderia causar erros operacionais ou inconsistências nos documentos fiscais.

## Causa Raiz

A lógica de arredondamento utilizada pelo sistema não estava considerando adequadamente as duas casas decimais necessárias para o preenchimento do campo "Alíquota". Isso gerava valores imprecisos, mesmo que os arquivos fossem aceitos pelo validador.

## Resolução

Para resolver o problema, foi implementada uma atualização no sistema que ajusta o arredondamento do campo "Alíquota", considerando adequadamente duas casas decimais. Agora, valores como 3,4999% são arredondados corretamente para 3,50%.

A correção foi entregue com o seguinte objeto liberado: **tax4b.iss_net-db_1.145.1.mtar**.

## Próximos Passos

Após aplicar a atualização, o cliente deve validar os novos arquivos gerados para garantir que o campo "Alíquota" está sendo preenchido corretamente. Recomendamos sua revisão interna e, caso os arquivos sejam aprovados, prosseguir com o envio para os órgãos reguladores.

Caso algum erro persistir, abrir um novo ticket para suporte detalhado.

## Dicas e Recomendações

  * Utilize sempre as versões mais recentes do sistema para evitar erros conhecidos.

  * Certifique-se de que todos os dados utilizados no cálculo das alíquotas estão devidamente configurados no sistema.

  * Realize validações regulares nos arquivos gerados antes de enviá-los aos órgãos municipais.

  * Mantenha registros de todas as interações com o sistema, incluindo a evidência dos cálculos para auditorias futuras.


Para mais informações, consulte o manual da obrigação municipal e entre em contato com o suporte, caso necessário.
