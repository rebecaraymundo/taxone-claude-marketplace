# Correção do Preenchimento do Campo "Cidade" no Arquivo SPEED GOV

## Problema

O campo "Cidade" no arquivo SPEED GOV estava sendo gerado incorretamente com o código do município do tomador do serviço ou do local de prestação, em vez do município do prestador do serviço.

## Causa Raiz

O erro foi causado por um comportamento incorreto, onde o sistema utilizava informações do tomador ou do local de prestação para preencher o campo "Cidade" no arquivo, desconsiderando o município do prestador do serviço conforme estabelecido no layout oficial da obrigação.

## Resolução

Foi implementada uma correção técnica nos componentes do sistema SPEED GOV para garantir o correto preenchimento do campo "Cidade". A partir das versões abaixo, o sistema passa a utilizar o código IBGE do município do prestador do serviço, em conformidade com o layout oficial:

  * **tax4b-speed_gov-db** - versão 1.149.1
  * **tax4b-speed_gov-ui** - versão 1.149.1


## Próximos Passos

Após a atualização das versões do sistema acima mencionadas, confirme que o campo "Cidade" está sendo gerado corretamente, testando diferentes configurações e revisando os arquivos criados. Caso os problemas persistam ou seja necessário algum ajuste adicional, entre em contato com nossa equipe de suporte.

## Dicas e Recomendações

  * Realize uma validação periódica dos arquivos gerados pelo sistema para assegurar conformidade com os layouts oficiais e evitar penalizações fiscais.
  * Em caso de atualizações ou correções nos componentes, sempre revise as notas de versão para identificar melhorias que possam impactar sua operação.
  * Documente alterações significativas realizadas no sistema para facilitar o controle interno e garantir alinhamento com sua equipe.


Para mais informações sobre o layout oficial do arquivo SPEED GOV ou outras funcionalidades do módulo Obrigações Municipais, consulte nossa documentação técnica ou entre em contato com o Suporte Soluções Fiscais.
