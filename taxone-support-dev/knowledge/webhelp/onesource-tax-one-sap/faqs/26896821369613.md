Ao acessar a tela principal de**Controle Remessa e Retorno** , o sistema apresenta mensagem de erro: **"Erro durante atualização dos registro. Ver erros: Erro Unique Constraint Violated PR:FILL_NOTAS_PARAMETRIZADAS"**

Para corrigir o erro de duplicidade de dados durante a atualização dos registros no aplicativo **Gestão de Dados > Controle de Mercadoria Remessa e Retorno**, siga os passos abaixo:

1\. Revise as **Configurações** do CFOP.​​​​​​

2\. Identifique os CFOPs que estão configurados tanto como **operação própria** quanto como **operação de terceiros** , como o exemplo do CFOP 1902AA mencionado.

3\. Para cada CFOP duplicado, decida se ele será utilizado apenas como operação própria ou apenas como operação de terceiros. Não é possível manter a mesma configuração de CFOP para ambos os tipos de operação.

Exemplo:

  * **CFOP 1902AA está configurado como "Operação Próprio:****REMESSA PARA INDUSTRIALIZAÇÃO",** ou seja, o cliente enviou uma mercadoria (Remessa) para outra empresa, essa outra empresa quando devolveu (Retorno), o cliente escriturou com o CFOP 1902AA. Neste caso a mercadoria pertence ao cliente.
  * **CFOP 1902AA está configurado como "Operação Terceiro:****RECEBIMENTO VASILHAME/SACARIA",** ou seja, o cliente recebeu uma mercadoria (Remessa) de outra empresa, o cliente escriturou com o CFOP 1902AA, e o cliente precisa realizar o retorno, pois a mercadoria pertence a outra empresa. Neste caso a mercadoria pertence a outra empresa.


![1-controle remessa.PNG](https://suporte.thomsonreuters.com.br/hc/article_attachments/26896823703821)

4\. Diante das configurações que foram realizadas, o docnum "0009882438" por exemplo, consta como terceiro e próprio, o que está errado.

![2-controle remessa.PNG](https://suporte.thomsonreuters.com.br/hc/article_attachments/26896823707917)

5\. Nas configurações já disponibilizamos filtros adicionais para distinguir Cód. Participante, NCM e Cód. do item, desta forma devem ser revistas.
