# [Internal Bug] Ambiente DW: AC e RC | Básicos - JS OL - [Teste 164] Campo SOLUCAO_ERRO está sendo gravado vazio na LOG_DET_PROC para o COD_ERRO = 91067 (TFIX22)

## Passos de Reproducao

Credenciais de acesso para reprodução da falha:
 
- Tennant / Database: AC e RC 
- User | Password: acesso próprio  
  Testes que falharam: 
-   

 

Resultado esperado versus Resultado obtido:  

 

- 
  
 

Evidência da falha:
 

Em atendimento ao ticket 120445, a demanda 736013 efetuou a retificação da mensagem de Solução do erro de TFIX37 para TFIX34 (verdadeira responsável pelos Códigos de Recolhimento do INSS). O teste 164 força o cenário de erro de tal forma que o log grava que o usuário necessita realizar o seguinte: "Realize a importacao do arquivo TFIX34.TXT".  

 

Ao consultar a CAT_ERRO (que é a TFIX22 no banco de dados), vemos que os campos pertinentes ao COD_ERRO = 91067 estão devidamente preenchidos.  

- select * from cat_erro where cod_erro = 91067;
 
- 
   

No entanto, ao realizar a importação ONLINE da safx07, o campo SOLUCAO_ERRO da LOG_DET_PROC está vazia.  

 

- select *
  from log_det_proc
 where cod_erro = 91067
   and num_processo = ((Select max(num_processo)
                          from log_processo
                         where cod_empresa = '076'
                           and cod_estab = 'REINF1'
                           and ind_processo = 'IMP'
                           and cod_usuario = 'suite'
                           and descricao = 'IMPX07'));
 
- 
   

Como a query do arquivo obtido grava exatamente o que está nesse campo desta tabela, o teste finaliza com erro.
