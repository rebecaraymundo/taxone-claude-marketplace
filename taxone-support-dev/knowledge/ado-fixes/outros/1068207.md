# TKT175260 - CAOA - ONESOURCE TAX ONE - SPED - Contribuições - Inconsistência no C100/C170: Itens com CST 06

## Descricao

Continuação da demanda https://dev.azure.com/tr-ggo/Mastersaf%20Fiscal%20Solutions/_workitems/edit/1064883

O cliente possui um item que utiliza com CST 06 e, no mesmo item, a base de substituição tributária de PIS/COFINS com CST 05. 

No Registro C170: Para o CST 06 (Normal), este lançamento possui apenas a base de cálculo, sem alíquotas e impostos. No entanto, nos arquivos, os campos estão sendo preenchidos com as mesmas informações do CST 05, o que inclui: 

- Campo 36: VL_COFINS (Valor da COFINS) 
- Campo 33: ALIQ_COFINS (Alíquota do COFINS em percentual) 
- Campo 30: VL_PIS (Valor do PIS/PASEP) 
- Campo 27: ALIQ_PIS (Alíquota do PIS em percentual)  
Isso está causando inconsistências nos dados.

## Root Cause

No package EpcDF_Dados.pck, a condição que copia valores de Substituição Tributária (ST) para o registro C170 verificava apenas se cod_sit_pis_st = '05', sem checar o CST normal. Quando o CST normal era 06 (Alíquota Zero), os valores de alíquota e imposto eram zero, satisfazendo a condição de cópia (vlr_aliq_pis_st > 0 AND vlr_aliq_pis = 0), e os valores ST eram incorretamente copiados.
