# [KLABIN] - Ticket_173799 - Importação SAFX235 e SAFX236 - Registros Ignorados_PRD e UAT

## Descricao

Boa tarde, 
 Cliente reportou erro no processo de importação da Tabela SAFX236 conforme segue: 
 
- SAFX236 - o log de processo de importação mostra que os registros foram lidos e ignorados.  
- Os registros ignorados não reflete o resultado, pois deveriam ser importados para a base de dado. 
- Quando consultamos os registros, não temos nada carregado para ser ignorado.  
- Evidências: Todas as evidências do processo estão anexadas a essa demanda, arquivo [Klabin] - EVIDÊNCIAS DO PROCESSO DE IMPORTAÇÃO SAFX236.docx 
- Ambientes: 
- TaxOne (UAT) base do Cliente - Empresa: 007-Klabin - Estab: PI01 - Período: 02/2026 
- ONESOURCE: TAX ONE
  
- TaxOne (QA) base TR de Testes - Empresa: 076-TSL - Estab: 4992-2 - Período: 02/2026 
- ONESOURCE: TAX ONE
  
- MastersafDW (base Local): Os mesmo teste foi realizado com sucesso. Ou seja, o mesmo número de registros lidos, é o mesmo que foi importado para a SAFX236.   
 Essa demanda é para atendimento aos registros do Bloco K, conforme legislação vigente.

## Root Cause

A coluna CHAVE_SAFX236 da tabela SAFX236, não esta levando em consideração o cod_produto_ins e ind_produto_ins, deixando a chave de itens identica entre registros. Com isso na importção os registros que possuem chaves iguais estao sendo desconsiderados na importação, ficando com status de Ignorados.

## Release Notes

Módulo: Job Servidor Menu: Importação > Importação > Execução 
 Ajuste na importação da SAFX236 permitindo a inclusão de insumos diferentes para a mesma ordem de produção e produto acabado.
