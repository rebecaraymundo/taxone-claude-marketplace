# [PROLEC] TKT126973 - TAX ONE - BÁSICOS - JOB SERVIDOR - Verificar/alterar regra da SAFX218 para poder importar com o campo VLR_AJUSTE zerado

## Descricao

TKT126973 - PROLEC - ONESOURCE TAX ONE - BÁSICOS - JOB SERVIDOR - Verificar/alterar regra da SAFX218 para poder importar com o campo VLR_AJUSTE zerado
 
Estamos com um problema, onde se faz necessário criar os registros C197 e E113 para validar a GIA mesmo que a entrada de mercadoria importada não seja tributada. Neste caso, o problema que temos é na geração das informações do registro E113 por meio da SAFX218. Mais especificamente, em relação ao campo “VLR_AJUSTE” da tabela, o qual não aceita para importação valor igual a zero (0), pois é um campo chave. 

Atualmente, acabamos gerando a tabela com o valor 0,01 para cada NFe não sujeita ao ICMS. No entanto, é necessário a correção deste valor no txt para a validação do SPED. O problema é que são mais de 100 linhas para a correção, e, a tendência é o aumento desta demanda com o aumento das importações. 

Portanto, teriam alguma solução para isso? Alguma forma de não deixar esse campo como obrigatório?
