# TKT175397 - AGROTERENAS - ONESOURCE TAX ONE - BÁSICOS - DATA WAREHOUSE - Sistema não esta gravando valor no campo Base Outras ICMS-S, mesmo com mensagem de item salvo com sucesso. 

## Descricao

Sistema não esta gravando valor no campo Base Outras ICMS-S, mesmo com mensagem de item salvo com sucesso.  

Evidencias em arquivo anexo.

## Root Cause

Variavel morta BASE_ICMSS_W em PKG_TMP_DOCTO_FISCAL_FPROC.GravarNotaFiscal (linha 7375).

Na derivacao ICMS-S (linhas 7825-7871), a variavel BASE_ICMSS_W era calculada corretamente a partir de VLR_BASE_ICMSS_2/3/4, mas nunca inserida no INSERT do TMP_EXEC_SAF_OL_DW (linha 7889+). Apenas BASE_ICMSS_REDU_W (alimentada exclusivamente por VLR_BASE_ICMSS_4, linha 7881) era propagada via CAMPO_100.

Consequencia: Quando o usuario preenchia VLR_BASE_ICMSS_3 (Base Outras) ou VLR_BASE_ICMSS_2 (Base Isento), o valor era salvo com sucesso na TMP_X08_ITENS_MERC (mensagem "Item salvo com sucesso"), mas ao Gravar Nota Fiscal o valor era perdido no fluxo: GravarNotaFiscal -> TMP_EXEC_SAF_OL_DW (CAMPO_100=null) -> saf_ol_dw_x08 -> OL_IMP_X08 -> GR_X08_BASE_MERC (nao chamado pois NVL(BASE_REDU_ICMSS,0) = 0).

Comparacao com ICMS: Para ICMS (nao-S), BASE_ICMS_W E inserido no INSERT (linha 7969). Para ICMS-S, BASE_ICMSS_W era variavel morta - calculada mas descartada.

Fix: Adicionar bloco IF apos derivacao ICMS-S para sobrescrever BASE_ICMSS_REDU_W com BASE_ICMSS_W, propagando o valor base do tributo primario. O mecanismo AUX2 ja trata o tributo secundario nos casos de dois tributos simultaneos.

Arquivo: artifacts/sp/msaf/basico/dw/PKG_TMP_DOCTO_FISCAL_FPROC.pck
PR: https://github.com/tr/taxone_dw/pull/12260
