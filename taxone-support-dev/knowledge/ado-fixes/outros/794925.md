# TKT129755 - IPIRANGA - ONESOURCE TAX ONE - ESPECÍFICOS - I-SIMP - Gerando indevidamente o código de município para operação de estoque 3030004 (em trânsito) com código de instalação 2 preenchido

## Descricao

Cliente:
Erro "CRT19 - Informar o campo Código Instalação 2 OU os campos Identificação do Terceiro,  Código do Município e Código da atividade econômica com dados válidos." está sendo apresentado para a operação 3030004, vinda da SAFX10;
 Ou seja, se a instalação 2 estiver preenchida, não pode preencher Identificação do Terceiro,  Código do Município e Código da atividade econômica, somente se a instalação 2 não estiver preenchida, deve preencher Identificação do Terceiro,  Código do Município e Código da atividade econômica.
 
 Suporte:
Cliente realiza a movimentação de estoque na SAFX10 com a operação 3030004 e código de instalação 2 preenchido da SAFX04, porém mesmo com o código da instalação 2 preenchido a informação do código do município esta sendo carregada no meio magnético apresentando criticas.

## Root Cause

Carta do Patch – DW e Taxone: 

Módulo: Específicos > i-SIMP
Menu: Geração > Meio Magnético DPMP 

     Alteração da regra de geração de preenchimento do Código do Município, passando a preencher com zeros quando o código de Operação ANP for igual a “3030004”, para a geração baseada em Movimento de Estoque.
