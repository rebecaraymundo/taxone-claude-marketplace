# [BUNGE] TAX ONE - JOB SERVIDOR - Erro processo de importação safx248

## Descricao

Prezados, bom dia. Estamos com problemas na safx248, onde a mesma é gerada através de um processo customizado. No momento da importação, via job servidor, ele está ignorando os dados da safx. Para a safx248, podemos ter mais de um lançamento para o mesmo COD_FISJUR. Ele está lendo 1 para 1, com a safx247. Segue todas as evidências e safx relacionadas.
 
 Bunge Alimentos Prod02 - C7Z

## Root Cause

O sistema está configurado para tratar a relação entre as tabelas SAFX247 e SAFX248 como 1 para 1, quando deveria ser 1 para muitos. Isso acontece porque existe uma restrição de unicidade no campo COD_FISJUR na tabela SAFX248, impedindo que múltiplos lançamentos de rubricas sejam registrados para um mesmo código. Durante a importação, o processo agrupa ou elimina registros duplicados por COD_FISJUR, fazendo com que apenas um lançamento seja importado mesmo quando existem vários no arquivo de origem.
