# [HEINZ] TKT95975 - TAXONE - MUNICIPAL - NEROPOLIS - O valor de ISS retido não está sendo considerado quando é retido pelo Tomador.

## Descricao

**​MUNICIPAL - NEROPOLIS - O valor de ISS retido não está sendo considerado no arquivo magnetico, quando é retido pelo Tomador.** 

 
 
 Cliente 
 Ao tentarmos gerar a obrigação, não vem os valores o ISS retido.
 
 Suporte 
 O campo – Valor Retenção ISS não está sendo gerado no arquivo magnetico, quando é retido pelo Tomador. Na regra o valor de retenção de ISS é considerado quando o campo indicador de Tipo de retenção estiver com preenchimento com "2 - Prestador". Porém a retenção de ISS no cliente é pelo "1 - Tomador". 
 
 
 
 
 
 Segue simulação na aba analysis

## Release Notes

Informações p/a Carta do Patch: DW e TAX ONE 

Objetivo: Ajuste de regra, para que seja
considerado na geração do meio magnético, o valor de ISS retido, uma vez que o
indicador de retenção, estiver preenchido com a opção 1 (Tomador)
