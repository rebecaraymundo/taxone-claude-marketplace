# [OLAM DO BRASIL] TKT171007 - ONESOURCE TAX ONE - SPED -  - Revisar a regra 0100 sped contribuições

## Descricao

De acordo com a regulamentação, é possível gerar mais de um registro 0100 por arquivo. No entanto, atualmente nossa regra está configurada para gerar apenas um registro 0100

## Root Cause

O sistema gera apenas um único Registro 0100 (dados do contador responsável) por arquivo do SPED EFD Contribuições, mesmo quando existem múltiplos estabelecimentos cadastrados com contadores distintos. Isso ocorre porque a lógica de geração desse registro não considera a parametrização individual de contador por estabelecimento, tratando todos como se tivessem um único responsável. Como o Guia Prático da obrigação exige um Registro 0100 para cada estabelecimento com contador diferente, o arquivo gerado fica em desacordo com as regras da obrigação fiscal.

SaaS

## Release Notes

Carta do Patch - DW e TAX ONE:  
   
 
Módulo impactado: SPED > EFD – Escrituração Fiscal  Digital das Contribuições  

 
 
Menu: Parâmetros  > Dados Iniciais  
 
Inclusão de um novo parâmetro na tela de Dados Iniciais para permitir configurar a geração de um registro 0100 para cada estabelecimento centralizado: “Considerar todos os estabelecimentos centralizados na geração do registro 0100.”  
 
   
 
Menu: Obrigações > Geração dos Registros  
 
Alteração de regra para gerar um registro 0100 para cada estabelecimento centralizado caso o parâmetro “Considerar todos os estabelecimentos centralizados na geração do registro 0100.” esteja habilitado.
