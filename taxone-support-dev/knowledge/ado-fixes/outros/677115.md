# [CENTRO DE INTELIGENCIA EM CONSTRUCAO E ADMINISTRACAO LTDA] TKT107492 - TAXONE - MUNICIPAL - SOROCABA - DFSNET - Inconsistência na geração do campo 28 - "Código do município do prestador de serviços" - Layout Nota Joseense

## Root Cause

Causa: Para o município de Sorocaba o campo referente ao código do município pretação (cod_municipio_NF) quando menor de 5 caracteres não estava sendo tratado.

 Solução:  Atualizado para Sorocaba para colocar zeros a esquerda quando o campo cod_municipio_NF tiver menos de 5 caracteres.
 
 Ações preventivas/Boas práticas:
 N/A

## Release Notes

Módulo: MUNICIPAL - SOROCABA - DFSNET

 Resumo da Alteração: Ajuste no campo do meio magnético para o Município de Sorocaba referente a informação de município prestação que deve conter impreterivelmente 5 caracteres e caso não possua deve ser preenchido com zeros a esquerda.
 
 Descritivo das Alterações: Ajuste no campo do meio magnético para o Município de Sorocaba referente a informação de município prestação que deve conter impreterivelmente 5 caracteres e caso não possua deve ser preenchido com zeros a esquerda.
 
 Funcionalidade Alterada: MUNICIPAL - SOROCABA > OBRIGAÇÕES > MEIO MAGNÉTICO
