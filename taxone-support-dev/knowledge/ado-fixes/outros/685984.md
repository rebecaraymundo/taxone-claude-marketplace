# [LASA]TKT109398 - DW - ESTADUAL - RESSARCIMENTO ICMS-ST PR - Cálculo do Ressarcimento / Complemento desconsiderando documento fiscal de saída quando existe devolução correspondente

## Descricao

Cliente:
Suporte, bom dia!No processamento do Ressarcimento ICMSST - Paraná, identificamos que, por algum motivo, o sistema está desconsiderando notas fiscais de venda a consumidor final quando ocorre uma devolução.
 Pela análise que fizemos, aparentemente foi desconsiderada porque o sistema entendeu que a venda estava gerando estoque negativo. Entretanto, analisando as ultimas entradas do material, não era para ter sido considerado negativo.
 
 Suporte: Após análise da situação reportada foi possível simular o problema conforme relatado pelo cliente.  No Cálculo do Ressarcimento / Complemento está sendo desconsiderado o documento fiscal de saída correspondente a devolução, causando erro em sua validação.
Segue simulação na aba analysis.

## Release Notes

Carta do
Patch: DW e TAX ONE

Menu: Estadual, módulo Ressarcimento de ICMS-ST - PR 

          Geração
> NPF 003/20 > Cálculo do Ressarcimento / Complemento 

    Alteração na regra de recuperação das notas fiscais
para tratamento do parâmetro “Desconsiderar movimentos de saída cuja quantidade
não é acobertada pelas entradas”. Essa alteração vai permitir trazer as notas fiscais
de saída que não estavam vindo no arquivo, mas as notas fiscais de devolução estavam
no arquivo.
