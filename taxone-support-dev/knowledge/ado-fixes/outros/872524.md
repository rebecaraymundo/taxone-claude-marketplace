# TKT144474 - TAX ONE - MUNICIPAL - MANAUS NFSE ABACO - "Erro de tag <?xml version="1.0" encoding="utf-8"?>"

## Descricao

Ao importar o XML no sistema da Prefeitura de Manaus, foi identificado erro de formatação. A mensagem de erro apresentada foi: 
"0- Dados não puderam ser carregados do XML, verifique formato do arquivo (ANSI)!"
 

 

📌 Detalhes Técnicos 

- O arquivo gerado pelo sistema continha a seguinte declaração XML:  
 <?xml version="1.0" encoding="utf-8"?> 

 

 

A prefeitura lhe orientou em editar o arquivo XML de <?xml version="1.0" encoding="urf-8"?> OU <?xml version="1.0" encoding="ANSI"?> para <?xml version="1.0"?>.  

Após o ajuste o arquivo xml de envio foi processado com sucesso em ambiente interno de teste conforme segue: 

  

 

 

 

De acordo com o retorno da Base tributária não tivemos alteração de layout recentemente.  

 

Na nossa regra consta que será gerado dessa forma :  

 

- A prefeitura solicitou a alteração para:  
<?xml version="1.0"?> 

- Mesmo com o encoding corrigido para ANSI, o validador da prefeitura não aceitava a tag com o atributo encoding.

## Root Cause

Alteração da regra de negócio para gerar a tag xml version.

SaaS
