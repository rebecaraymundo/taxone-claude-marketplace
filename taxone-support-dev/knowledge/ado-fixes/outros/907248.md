# [INX] TKT150001 - TAX ONE - MUNICIPAL - DES XML - Cabreúva - Revisão da regra dos Campos Alíquota e Base de Cálculo

## Descricao

TKT150001 - INX - TAX ONE - MUNICIPAL - DES XML - Cabreúva - Revisão da regra dos Campos Alíquota e Base de Cálculo 

 

Atualmente a geração destes tem como origem os campos Valor Base ISS e Alíquota ISS, porém o cliente só preenche estes campos quando o ISS é devido. Na situação em que o ISS é Retido, os campos que ele preenche são: Valor Base ISS Retido e Alíquota ISS retido, mas não temos na regra esses campos. Neste caso é necessário ler os dois campos, tanto os campos de ISS Devido, quanto de ISS Retido. 

 

Regra: https://trten.sharepoint.com/:w:/r/sites/REQ_MTZ/_layouts/15/Doc.aspx?sourcedoc=%7B1A8CE743-AD29-4AF8-8F41-B2DF1C22B6AF%7D&file=MTZ_DESXML_Geracao_do_Meio_Magnetico.docx&action=default&mobileredirect=true

## Root Cause

Alteração de regra de negócio.

## Solucao

SaaS

## Release Notes

Carta do
Patch – DW e TAXONE: 

Módulo
Municipal > Cabreúva, 

Menu
Obrigações > Meio Magnético 

 
  Alteração da regra de preenchimento das tags <BaseCalculo> e <AliqISS>
para o município de Cabreúva.  Os valores de base de cálculo de ISS e de
alíquota retidos serão considerados apenas quando os campos de base de cálculo e alíquota
de ISS não estiverem preenchidos.
