# TKT166044 - SANTISTA - ONESOURCE TAX ONE - BÁSICOS - DATA WAREHOUSE - Erro na alteração quantidade de um item de uma ordem de produção

## Descricao

Prezados, 

 

Cliente está tentando alterar o campo "Qtd Utilizada" de um item de produção em determinada OP. 

 

Cenário: 

 

Cliente quer editar o campo citado no item 3. 

 

Ele se dirige até o item clicando na seta "Pág +"; 

Ao chegar no item e clicar no campo que quer editar, o sistema retorna para o item 1; 

Ao clicar novamente na seta "Pág +", o sistema leva para o item 4, gerando assim, determinada perda de tempo. 

 

Os vídeos de exemplificação tanto na base do cliente quanto na base de suporte estarão em anexo.

## Root Cause

O problema ocorre porque ao editar o campo "Qtd Utilizada" de um item na tela de Ordem de Produção, o sistema realiza uma atualização interna que reposiciona o cursor automaticamente para o primeiro registro da lista, perdendo a referência do item que estava sendo editado. Com isso, o controle de navegação (Pág +) passa a considerar o item 1 como ponto de partida, fazendo com que a próxima página exiba registros a partir de uma posição incorreta. Esse comportamento indica que o mecanismo responsável por manter o posicionamento atual do registro não está preservando corretamente o item selecionado após a edição do campo.
