# Servidor de Relatórios - TAXONE - Básicos - Report Fiscal - Aplicar tratamento SEM MOVIMENTO no Report Server  (QA)

## Descricao

Tratamento do período SEM MOVIMENTO p/ relatórios migrados para o Report Server  Localização: Módulo: Básicos >> Report Fiscal   Menu:  1) Obrigações Acessórias >> ICMS IPI p/ Datamart >> Relatório de NF p/ CFOP 2) Obrigações Acessórias >> ICMS >> Resumos do ICMS p/ DataMart >> Cód. Fiscal de Operação - Report Server

 Ramo: MFS666566 Descrição: Aplicar tratamento do período sem movimento para o relatório.  Tratamento para Período sem Movimento:
 
- Arquivo PDF incluir texto “SEM MOVIMENTO” no
cabeçalho. Cabeçalho deve conter os dados de estabelecimento, período como
ocorre para um período sem movimento. 

- Arquivo EXCEL incluir a coluna
“PERIODO_COM_MOVIMENTO” e preencher com “NÃO” para períodos sem movimento. 

 ***** A coluna “PERIODO_COM_MOVIMENTO” deve ser incluída ao final do relatório!! *******

## Release Notes

Carta do Patch – TAXONE APENAS: 

  

Módulo: Report Fiscal 

  

Resumo da alteração: Melhoria nos relatórios migrados para o Servidor de Relatórios para aplicar tratamento em períodos sem movimento. 

 

Módulo Básicos >> Report Fiscal
 

Menu:  

 
1) Obrigações Acessórias >> ICMS IPI p/ Datamart >> Relatório de NF p/ CFOP 2) Obrigações Acessórias >> ICMS >> Resumos do ICMS p/
DataMart >> Cód. Fiscal de Operação - Report Server 
No caso de não existir movimento para o estabelecimento e período apurado, os relatórios emitidos em formato PDF, apresentarão o texto "SEM MOVIMENTO" no cabeçalho. Quando emitidos em formato Excel, a última coluna denominada "PERIODO_COM_MOVIMENTO" apresentará a informação "não".
