# [RAIZEN] TKT142806 - DW - SPED Fiscal -  Registro E113 - Atendimento para modelo de documento = 28

## Descricao

TKT142806 - COSAN RAIZEN - DW - SPED Fiscal -  Registro E113 - Atendimento para modelo de documento = 28 para o SPED Fiscal - AVALIAÇÃO REGRA 

O cliente precisa gerar o modelo 01 na CAT 79, e modelo 28 no Registro E113 do SPED Fiscal, quando se tratar de Nota Fiscal/Conta de Gás. 

Recentemente, foi tratado esta situação na demanda: User Story 839303: [RAIZEN] TKT137260 - DW - SPED - EFD - Registro E113 - Gerar modelo de documento 28, quando modelo igual a 01, porém o cliente não concordou, e diz que a regra: "Para identificar a nota fiscal de modelo 28, será necessário cadastrar a Pessoa Física/Jurírica da nota como “Fornecedor de Gás Canalizado” na parametrização Pessoas Física /Jurídica por Ramo de Atividade (Básicos > Data Warehouse > Manutenção > Cadastro Parâmetros >  Pessoas Física /Jurídica por Ramo de Atividade)" não condiz com a situação, uma vez que o estabelecimento é o fornecedor do gás e não o contrário. 

Com base nisso, abrimos esta demanda para alteração de regra.

## Solucao

On-Premise

## Release Notes

Carta do
Patch - DW e TAX ONE: 

Módulo:
SPED > EFD - Escrituração Fiscal Digital 

Menu:
Geração > Meio Magnético > Gerar o SPED 

 
    Alteração na regra de geração do campo 03 - COD_MOD do
Registro E113 (Informações adicionais dos ajustes da apuração do ICMS –
Identificação dos documentos fiscais).  A alteração visa adequar o
tratamento para Notas Fiscais de saída (Gás), que utilizam o modelo 01 na nota
fiscal, mas devem ser registradas como modelo 28 no SPED-EFD 

     Para
identificar a nota fiscal de modelo 28, será utilizado o campo 06-Atividade
Econômica da SAFX2064 (Cadastro de Estabelecimento).
