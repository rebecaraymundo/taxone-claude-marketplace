# [HEINZ] TKT142029 - ONESOURCE TAX ONE - BÁSICOS - REPORT FISCAL - Relatório esta trazendo ### em alguns campos quando selecionado impressão em excel

## Descricao

Bom dia, 

Houve alguma alteração no relatório de serviços? Pois alguns campos estão puxando ###### como exemplo NF 5811 na parte da descrição campo "desc_serv "e "desc_nat_serv" não estão vindo a descrição que tenho dentro do SAP. 

Consegue verificar por favor? 

Segue evidencias.

## Root Cause

Causa: Relatório para os campos de descrição de COD_SERVICO e COD_NAT_SERV quando vazios na base apresentavam o valor '####' quando estes eram exportados para o Excel via Salvar Como.
 Solução:  Retirada da string '####' para os campos supra sitados e substituídos por ' ' (espaço) no SQL do relatório.

## Solucao

SaaS
