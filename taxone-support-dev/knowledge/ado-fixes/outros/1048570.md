# [MRV] TKT 172120 - ONESOURCE TAX ONE - FEDERAL - OBRIGAÇÕES DE TRIBUTOS FEDERAIS - JSON MIT - Tag CNPJIncorporação gerando com 6 digitos finais quando SCP deve gerar 14 digitos completos

## Descricao

Relato Cliente 

  

Prezados, boa noite 

  

Temos um gap para o código Darf 106807-RET SCP, onde na tag RetPagamentoUnificado / ListaDebitos / "CodigoDebito": "106807","CnpjIncorporacao": "000120", o CNPJ gerado nesta tag entendemos que está incompleto causando erro ao importar o arquivo no validador. 

Erro do validador é : Debitos.RetPagamentoUnificado.ListaDebitos[id 8].CnpjIncorporacao: valor informado inválido. 

  

Se avaliarmos o manual tag CnpjIncorporacao : 

Sim, para o objeto RetPagamentoUnificado, para cada ocorrência de objeto em ListaDebitos e em ListaDebitosAposEvento, se código do débito for de incorporação ou de SCP. 

Últimos 6 dígitos (número de ordem + DV) do CNPJ da incorporação, para débitos de incorporação, ou CNPJ completo da incorporação com 14 dígitos, para débitos de SCP do grupo RET/PAGAMENTO UNIFICADO. Exemplos: “000100” e "12345678000195" 

No nosso cenário seria a 2 opção "CNPJ completo da incorporação com 14 dígitos, para débitos de SCP do grupo RET/PAGAMENTO UNIFICADO", mas ao observar no arquivo foi gerado o CNPJ últimos 6 dígitos apenas. 

Pedimos avaliação o que pode estar causando este erro, pois para o momento temos apenas esta falha e não está sendo possível efetuar o upload do arquivo e finalizarmos os testes. 

  

Anexo seguem as evidencias + arquivo gerado Jason.

## Root Cause

O problema ocorre porque, ao gerar o arquivo MIT para débitos do tipo SCP (DARF 106807), o sistema busca e utiliza apenas os 6 dígitos finais do CNPJ (sufixo) para preencher o campo "CnpjIncorporacao", quando deveria utilizar o CNPJ completo com 14 dígitos. Isso indica que a lógica de montagem do arquivo JSON está recuperando ou formatando o CNPJ de forma incompleta para esse tipo específico de débito, desconsiderando os 8 dígitos iniciais (raiz do CNPJ). Como o manual do MIT exige o CNPJ completo para débitos do grupo RET/Pagamento Unificado, o arquivo gerado fica em desacordo com as regras do órgão receptor.

## Release Notes

O problema relacionado ao CNPJ na tag CnpjIncorporacao estava incompleto, foi resolvido. Agora, o CNPJ é gerado corretamente com 14 dígitos completos, conforme exigido para débitos de SCP do grupo RET/PAGAMENTO UNIFICADO. 
Com essa correção, o arquivo pode ser importado sem erros no validador, permitindo que os testes sejam finalizados com sucesso.
