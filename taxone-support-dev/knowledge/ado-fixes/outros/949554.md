# [CA PESQUISA] TKT156303 - INFOR - ONESOURCE TAX ONE - BÁSICOS - DATA WAREHOUSE - ORDEM DE PRODUÇÃO - Erro em alteração

## Descricao

Ao abrir o menu de ordem de produção para realizar qualquer alteração, é possível salvar normalmente a primeira modificação efetuada. No entanto, ao tentar editar uma segunda ordem diretamente pelo mesmo menu já aberto, a alteração não é salva, e o sistema retorna um erro informando que os campos "Período/Referência" não estão preenchidos — mesmo quando já estão visivelmente preenchidos. 

 

Para conseguir alterar uma segunda ordem de produção, é necessário fechar o menu e abri-lo novamente. Esse procedimento precisa ser repetido a cada nova edição: ou seja, para alterar uma terceira ordem, é preciso fechar e reabrir o menu mais uma vez. 

 

As evidências do cliente no ambiente PROD02 e do suporte em UAT01 estarão em anexo.

## Root Cause

Causa: Ao solicitar uma segunda gravação de registros sem sair da janela a mesma não mantém a persistência dos valores e retorna uma crítica de campo que já esta preenchido.

 Solução:  Ajuste no processo de persistência de dados para validação e gravação.
