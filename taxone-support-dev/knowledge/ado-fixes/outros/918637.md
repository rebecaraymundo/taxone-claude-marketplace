# [MRV] TKT152949 - TAX ONE - MUNICIPAL - Catanduva   ISS WEB XML - Alteração na Regra de Preenchimento da Inscrição Municipal eventual para Geração do arquivo no Ramo da Construção Civil

## Descricao

Catanduva - validador ISS WEB XML 

 

Para a geração do arquivo XML no ramo de construção civil, é necessário que seja informada a inscrição municipal cadastrada no Data Warehouse > Manutenção > Cadastros > Inscrições Municipais 

 e não a inscrição do cadastro da empresa. Nesse caso, deve-se considerar a inscrição eventual, conforme configurado nos módulos básicos. 

 

 

 

Atualmente verifiquei que o que esta sendo levado é conforme a regra :  

 

O preenchimento da TAG <InscricaoMunicipal> esta seguindo as seguintes regras:

 

Para o Tomador:

- Utilizar o campo INSC_MUNICIPAL da tabela ESTABELECIMENTO, referente ao estabelecimento que está gerando o arquivo magnético.
- Formato: Alfanumérico
- Tamanho: 15 caracteres
- Pontuação é permitida. Não é necessário adicionar espaços ou zeros para completar o campo.
- Tratamento: Caso o campo não esteja preenchido, será emitida uma mensagem padrão de log.
- ⚠️ Atenção: O cliente informa que precisa que seja considerado a IM  do caminho Data Warehouse > Manutenção > Cadastro > Inscrições Municipais para buscar essa informação.

- 
 

Para o Prestador:

- Utilizar o campo INSC_MUNICIPAL da tabela X04_PESSOA_FIS_JUR (campo 09 da SAFX04), referente ao prestador cadastrado na nota fiscal.
- Formato: Alfanumérico
- Tamanho: 15 caracteres
- Não é necessário complementar com espaços ou zeros.
- Tratamento: Caso o campo não esteja preenchido, será emitida uma mensagem padrão de log.
- 
 

Situação já foi conversada com Rose para analisar a alteração da regra
