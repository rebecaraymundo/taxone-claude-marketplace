# [HUGHES] TKT177717 - TAX ONE - SPED - EFD CONTRIBUIÇÕES - Reavaliação de regra: Registro D600 não gerando para Item Informativo

## Descricao

Cliente informa: 

Identificamos um problema na geração do bloco D600, no SPED Contribuições, os campos "Valor Acumulado das Prestações de Serviços Tributados pelo ICMS" e "Valor Acumulado dos Serviços não-tributados pelo ICMS" estão gerando com os valores incorretos, que não correspondem ao documento fiscal. 

O campo "Valor Acumulado das Prestações de Serviços Tributados pelo ICMS", deveria gerar somente com os valores tributados pelo ICMS, mas está vindo o valor total do serviço.
O campo "Valor Acumulado dos Serviços não-tributados pelo ICMS", deveria gerar com os valores não tributados pelo ICMS, mas está vindo zerado.

## Root Cause

Nas funcoes valInclusaoRegistro_D601 e valInclusaoRegistro_D605 do package EpcUT_RegraDicionario.pck, o filtro ind_tp_registro <> 1 (Item Informativo) era aplicado para todos os modelos (21, 22 e 62). Conforme especificacao RNG-D600, esse filtro deve ser aplicado apenas para modelos 21 e 22. Para modelo 62, itens informativos devem ser incluidos na geracao do registro D600. Correcao: adicionar condicao cod_mod = 62 para bypass do filtro.

## Solucao

SaaS

## Release Notes

Carta do Patch:  DW e TAXONE 
 Módulo: SPED > EFD – Escrituração Fiscal Digital das Contribuições Menu: Obrigações > Geração dos Registros 
 Alteração nas regras de geração dos registros D600, D601 e D605 para o modelo 62, visando considerar os itens informativos na composição desses registros.
