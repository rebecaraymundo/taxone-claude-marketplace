# [CLARO] TKT142062 - DW - BASICOS - DW - Falha na recuperação da descrição do cod_class na tela dos itens da X43 e X08

## Descricao

DESCRIÇÃO CLIENTE: 
 Estamos carregando as Notas da NFCom no ambiente de desenvolvimento para homologação dos dados do projeto e nos deparamos com este problema na importação dos dados.

 Para as safx08 campo 268 e safx43 o campo 124, a informação vem preenchida nos arquivos para todos os itens e carrega para o banco de dados, mas a descrição que deveria carregar automaticamente para o campo de descrição só preenche para o item 1 de cada NFCom, para os demais itens a descrição fica vazia.
 
 Apenas lembrando, esta tabela é a TACES113.
 
 
 
 DESCRIÇÃO SUPORTE:
 
 A tela de item fiscal de mercadoria X08 e item de telecom X43 está apresentando problemas na recuperação da descrição do COD_CLASS. Mesmo as informações estando corretas na TACES113 e no banco, a tela apenas traz a descrição do primeiro item. A Descrição dos outros itens não é recuperada. 
 Simulação: 
 
 
 
Informações da NF no banco de dados: 

 
Visualização da NF em tela (Capa): 

 

 
Visualização do primeiro item na aba de itens, Descrição do
código de classificação é exibida normalmente: 

 
 
Ao avançar para os demais itens, a informação da descrição
não é exibida, apenas o código de classificação: 

 
 
 Caso seja feito a seleção do mesmo código de classificação,
daí a descrição aparece, ou seja, aparentemente o problema está nos
executáveis, na parte de carregar as informações dos demais itens da NF.

## Root Cause

Causa: Rotina de recuperação da descrição do "Código de Classificação" foram implementados depois do fechamento do loop de carregamento dos dados dos itens.

 Solução: Mover a recuperação da descrição do "Código de Classificação" para dentro do loop de itens da janela.
