# [YAMAHA] TKT141910 - TAX ONE - ESTADUAL - ICMS - Relatório de Conferência - GNRE Emitida por Grupo/Convênio não agrupa os valores dos convênios

## Descricao

Olá, 

Solicitamos apoio para analisar a situação relatada: Cliente em projeto de migração do DW para o TaxOne identificou divergência na apresentação do Relatório de Conferência - GNRE Emitida por Grupo/Convênio onde o TaxOne  esta segregando os convênios em mais linhas ou páginas. 

TaxOne (print do chamado inicial encaminhado pelo cliente antes das orientações do dia 22/07) 

 

DW (print do chamado inicial encaminhado pelo cliente) 

 

 
Teste realizado na base do cliente em UAT (Q8M) e avaliado durante as calls realizadas.

Relatório não agrupa os valores dos convênios ( traz as informações por páginas em separado), a exemplo da UF do CE onde apresenta valores na página 1 e página 6 em separado.

Tela de parâmetros do relatório no módulo ICMS.  

 
 Os parâmetros utilizados no teste acima são os mesmos utilizados pelo cliente.

## Root Cause

Causa: Datawindow com barra de divisão de grupo 4 e 3 utilizam uma motangem de data para período para agrupamento que  o TaxOne não suporta.

 Solução:  Criado o campo de período dentro do SQL para ajustar os grupos que a utilizam na datawindow.
