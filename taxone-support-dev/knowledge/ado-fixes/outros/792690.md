# [TIGRE] TKT126915 - TAX ONE - ESTADUAL - ICMS - Apuração > Guias de Recolhimento > Guias Recolhimento Tributos Estaduais (GNRE Online) "Geração de Chave de Acesso quando código é o 102 de Campo Extra para XML em Goiás"

## Descricao

Cliente entrou em contato conosco informando que quando é preenchido o campo Extra com o valor = 102 a Chave de acesso para a UF de GO não é gerada, porém ocorre erro no validador da GNRE solicitando que a chave seja preenchida.  

 

Quando colocamos manualmente o código 102  

 

  

Ele valida com sucesso :  

 

Só da o erro de data mas que naõ vem ao caso no momento.  

 

Ao realizarmos um teste no sistema quando ajustamos  o Campo Extra 1 para "17", a chave de acesso foi gerada corretamente no arquivo XML para o estado de Goiás.  

No entanto, um erro de validação ocorreu , indicando a necessidade de informar o código 102 .  

Erro de validação - Portal : 

Código 243 

Campo : campos/Extras/campoExtra [item:1] 

Descrição: O campos Extra 'Chave de Acesso da NFe ou do CTE/CTE-OS' (Código:'102') deve ser informado! 

 

 

Um link fornecido (https://www.gnre.pe.gov.br:444/gnre/portal/consultarTabelas.jsp) oferece acesso às tabelas de campos adicionais aceitos pelo validador de Goiás.  

 

 

Estramos em contato com nosso time de Base Tributária para esclarecer esta questão e providenciar uma solução e o correto seria realizar o preenchimento da chave de acesso para o código 102.  

 

Hoje em nossa regra não localizei nenhum parâmetro especifico para o código 102 para o Código de receita 100102.  

Somente esse
