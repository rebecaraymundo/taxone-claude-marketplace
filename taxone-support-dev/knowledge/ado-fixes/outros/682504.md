# [UNIVAR] TKT105701 - TAX ONE - ISIMP - Campo 08 - QUANTIDADE DO PRODUTO 

## Descricao

[UNIVAR SOLUTIONS BRASIL LTDA] TKT105701 - TAX ONE - ISIMP - Campo 08 - QUANTIDADE DO PRODUTO OPERADO NA UNIDADE DE MEDIDA OFICIAL ANP com quantidade = 0 ocasionando erro na validação do arquivo

Este cliente emite nota de amostra, sendo uma quantidade muito pequena, e após a conversão de unidade de medida, o valor não atinge 0,51 para que assim suba no arquivo com quantidade, pelo contrário, no campo 08 - QUANTIDADE DO PRODUTO OPERADO NA UNIDADE DE MEDIDA OFICIAL ANP, a quantidade levada é = 0.

Quando validado o arquivo o mesmo apresenta erro/crítica:

"CRT03 - Os campos Quantidade Unitária Medida Anp e Quantidade de Produto em Kg devem ser preenchidos".

Deixei em anexo a cartilha do ISIMP - MANUAL-DO-I-SIMP-PRODUTORES-E IMPORTADORES DE LUBRIFICANTES

Campo 08:

A planilha SIMP - Operações x Regras 1 3
O arquivo do cliente, ao qual apresenta o erro, na linha 202 (28899_DMP0824)
O arquivo com o erro (2a. inconsistência mapa tax One não validado)

Realizado uma reunião com o cliente, onde ficou alinhado a criação de um parâmetro conforme sugerido pelo time de informação:

"Observação Interna: Considerando os cenários envolvidos e outros clientes, dada a ausência de orientação explícita no manual neste sentido, sugerimos aplicar na solução uma parametrização de forma que observada uma operação cujo resultado do cálculo seja entre 0,XXXX e 0,9999 não comporá o arquivo, ressalvada exclusiva responsabilidade do usuário fiscal."

Lembrando que o TAX ONE segue hoje uma regra de arredondamento:
