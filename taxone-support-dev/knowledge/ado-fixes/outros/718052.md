# [MAGALU] TKT117832 - DW - ESTADUAL - ATIVO PERMANENTE - Calculo do Crédito não alimenta a tabela Cálculo Geral dos Créditos quando é utilizado livro modelo C.

## Descricao

Cliente: 

Filiais que nunca tiveram movimentação no CIAP, e
possui créditos extemporâneos a calcular, estão apresentando logs de erro, e
apesar de executarmos o cálculo mensal dos meses apontados, não está gerando os
dados dos cálculos, que consultamos em Ativo Permanente>Manutenção Dados de
Cálculo>Modelo C>Cálculo Geral dos Créditos.  Valores que inserimos
no momento do cálculo (Menu Movimento >> Cálculo do Crédito – ICMS).
Nesta tela já informamos os valores que serão utilizados no coeficiente das
saídas para todo o período extemporâneo, então a Thomson já não poderia fazer
uma melhoria e incluir estas informações na tabela: APT_DEM_CR_MENSAL. 

  

Suporte: 

Cliente precisar calcular o crédito extemporâneo, porem nunca teve movimento de CIAP. Ao realizar o calculo do extemporâneo é apresentado erro no log informando que não existe coeficiente de saídas e assim não é possível realizar o calculo. Ao realizar o cálculo do crédito do ICMS livro modelo
C, o sistema não está alimentando a tabela "Cálculo Geral dos
Créditos" no caminho: Movimento > Manutenção Dados do Cálculo >
Modelo C > Cálculo Geral dos Créditos. Isso impede a realização do cálculo
do crédito extemporâneo quando não há movimentação no período. 

Ajustar o sistema para que, no momento do cálculo do
crédito do ICMS modelo C, a tela "Cálculo Geral dos Créditos" seja
devidamente alimentada. Isso permitirá a realização correta do cálculo do
crédito extemporâneo, mesmo na ausência de movimentação no período. 

 

Situação acontece para o credito extemporâneo parcial e integral

## Release Notes

Menu: Movimento >> Manutenção Dados do Cálculo >> Modelo C >> Cálculo Geral dos Créditos 
 Correção na geração do Cálculo Geral dos Créditos, quando utilizado Modelo C.
