# [CLARO] - Ticket 162176 - DW - BASICOS - JOB SERVIDOR - Problema de performance na importação batch da SAFX334

## Descricao

A importação batch diária da SAFX334, no ambiente da Claro, com aproximadamente 1.500.000 registros, está levando 2 a 3  horas. 
 
 
 Anexo trace com as duas primeiras horas de processamento. 
 A importação batch de outras tabelas com esse tamanho leva poucos minutos, devido à possuir processamento por thread. A SAFX334 não possui opção de processamento por thread. 
 Segue comentário do DBA Eli Dias, alocado na Claro: 
 "Sobre o trace da 334, identifiquei um único ponto, a tabela X334_FUND_TRIB_FED_NFE.
 Ela sofre insert linha a linha, mesma situação que vimos na X07, para terem ideia, durante as 2 horas de trace, tivemos as seguintes estatísticas: 
 
- 447.218 leituras físicas  
- 13.257.436 (escritas em buffer)  
- 814.459 execuções   
 O aumento de threads é o mais indicado para a atual situação. 
 Não sei se existe uma package vinculada a este processo, mas se houver seria interessante validar a possibilidade de incluir um hint de APPEND na query para que o Oracle realizasse a carga via batch. Esta opção pode ser um paliativo."
