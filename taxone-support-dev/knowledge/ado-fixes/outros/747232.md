# [TEX COURIER] TKT120847 - TAXONE - MUNICIPAL - ISS.NET - Brasília - Campo "Unidade Econômica" não gera = 0 - NÃO

## Descricao

TKT120847 - TEX COURIER S.A. - TAX ONE - MUNICIPAL - ISS.NET - Brasília - Campo "Unidade Econômica" não gera = 0 - NÃO 

 

O cliente precisa gerar no campo "Unidade Econômica" a informação "0 - NÃO", porém o TAX ONE gera somente 1 - SIM 

 

De acordo com o manual de layout (https://www.notacontrol.com.br/static-issnetonline/Ajuda/html/index.html?ajuda_importacao_consulta.html), este campo tem as duas opções, segue trecho com a descrição: 

 
"Carácter que indica a utilização da Unidade Econômica, representando "0" para NÃO e "1" para SIM." 

 
 

De acordo com a regra (https://trten.sharepoint.com/:w:/r/sites/REQ_MTZ/_layouts/15/Doc.aspx?sourcedoc=%7B062A8C8A-F280-4071-A3E5-092CE1361F58%7D&file=MTZ-ISSNET-Geracao_do_Meio_Magnetico.doc&action=default&mobileredirect=true), o campo em questão é gerado da seguinte forma: 

 

"Regras p/ o campo Unidade Econômica Caracter que indica a utilização da Unidade Econômica, representando “0” para NÃO e “1” para SIM. 

Preencher com “1” Sim (fixo) 
Campo Numérico 1 posição"

## Root Cause

Houve necessidade de alteração da regra de negócio, visto que na regra havia a especificação para a recuperação de zero para o campo sempre.
  Passando agora a recuperar a informação com zero ou um dependendo da regra de negócio.
