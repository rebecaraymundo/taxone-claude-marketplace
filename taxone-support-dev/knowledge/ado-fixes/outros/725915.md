# [IDEMIA DO BRASIL] TKT119526 - TaxOne - Municipal - Cotia - Validador NFSE_GIAP_PRESCON - Manual de layout consta tag <LocalPrestacao> avaliar para considerar no arquivo meio magnético.

## Descricao

Cliente: 

Conforme referenciado em anexo via imagem 001 e 002
correspondente ao local da prestação do serviço em alguns códigos específicos,
exemplo há códigos de serviço como o 11.04 - Armazéns Gerais que se realizado
dentro de Cotia incide a retenção, se realizado em outro município não incide a
retenção.

Seria possível acrescentar este campo no XML gerado do Tax One afim de
informarmos se o serviço foi prestado em Cotia ou em Outro Município? 

  

Suporte: 

Durante o processo de geração da obrigação municipal
para Cotia, foi identificado que não gera a tag <LocalPrestacao>. 

Atualmente, a TAG <LocalPrestacao> está presente
no manual de layout, mas não foi incorporada na regra de negócio para geração
do arquivo. Por gentileza avalie a possibilidade de ajustar a regra de negócio
para garantir que a TAG <LocalPrestacao> seja gerada no arquivo. 

 

Regra: Não gera a tag para o Município Cotia 

 

 

Manual de layout

## Root Cause

A tag LocalPrestacao passou a ser obrigatória na geração do xml, necessitando assim da sua inclusão.
