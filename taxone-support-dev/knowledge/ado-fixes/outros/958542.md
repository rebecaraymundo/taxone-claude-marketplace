# TKT156490 - MRV - ONESOURCE TAX ONE - MUNICIPAL - CAMPINAS/TERESINA'NOTA JOSEENSE' - Correção de duplicidade na geração de notas fiscais e otimização na geração de arquivos TXT

## Descricao

Durante a validação dos arquivos magnéticos gerados para o Município de Campinas, foi identificado que o sistema está gerando registros de notas fiscais duplicados. A análise dos arquivos demonstrou que, para notas fiscais que possuem múltiplos itens com o mesmo código de serviço, o sistema está criando uma linha para cada item, sem consolidar os valores. Consequentemente, o mesmo número de nota aparece diversas vezes no arquivo, um para cada item, ao invés de um único registro com o valor totalizado por código de serviço. 

 

 

 

Outro ponto crítico identificado refere-se aos arquivos TXT gerados para o período. O sistema está segmentando os dados em múltiplos arquivos, com 50 notas em cada arquivo conforme a regra de negocio, porem não cria um sequencial para não sobrepor os arquivos, e ao selecionar todos arquivo e salvar, não aparece na nuvem. 

 

 

 

Anexo simulação das situações informadas.

## Root Cause

Alteração de regra de negócio.

## Solucao

SaaS

## Release Notes

Carta do Patch – DW e TAXONE: 

Módulo Municipal > Campinas, 

Menu  Obrigações> Meio Magnético 

     Alteração da regra de geração dos registros, passando a agrupar os itens também para Campinas.
