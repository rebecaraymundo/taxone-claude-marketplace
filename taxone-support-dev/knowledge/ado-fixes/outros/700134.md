# [ARAPIRACA ]TKT113407 - TAXONE - MUNICIPAL - ARAPIRAPA - Tag <InscricaoMunicipal> do Prestador do Serviços não deve ser considerado no arquivo - Validador NFSE ABACO

## Descricao

CLIENTE: 

Ao validar o meio magnético do modulo
Municipal > Arapiraca, tivemos um erro referente ao campo
<InscricaoMunicipal> do Prestador do Serviço, quando essa informação não
constar no cadastro, essa tag não deve ser gerada. 

Realizamos um teste retirando a tag sem a
informação e o arquivo foi validado corretamente, só deve ser gerado a tag de
inscrição municipal quando houver essa informação no cadastro do prestador. 

 
 
SUPORTE: 

Avaliar a regra referente a tag <InscricaoMunicipal> do
Prestador do Serviço para o Município de Arapiraca - validador NFSE ABACO, a tag não é obrigatória e
quando não existir informação ela não deve ser gerada. 

Manual de layout 

 
Regra de negócio  

 
 Simulação aba analysis

## Release Notes

Informações p/a Carta do
Patch: DW e TAX ONE 

Módulo: Municipal > Arapiraca
- AL.

 

Menu: Obrigações >
Meio Magnético.

 

Objetivo: Alterar a
geração da tag <InscricaoMunicipal> da TAG <IdentificacaoPrestador> passando
a verificar, se não existir valor para essa TAG, não será exibido na
estrutura do arquivo magnético, e não será gerada, conforme orientação da Prefeitura de Arapiraca.
