# TKT170528 - SAVEGNAGO - ONESOURCE TAX ONE - SPED - FISCAL - Registro C500: Notas de entrada com o Campo 16 - VL_SERV_NT, valor dos serviços não tributados pelo ICMS zerado

## Descricao

Solicitamos apoio em relação a geração do campo 16 - VL_SERV_NT, valor dos serviços não tributados pelo ICMS, para as notas de entrada do modelo 66. 

 

Após a escrituração de uma NF3e modelo 66, cuja classificação seja mercadoria, cliente reportou que o campo VL_SERV_NT não apresentou o valor esperado mesmo estando com as bases necessárias para o preenchimento informadas (no caso da situação em questão seria a Base ICMS Outras). 

 

Abaixo regra atual para a geração do campo 16 - VL_SERV_NT. 

 

Cliente não marcou o parâmetro “Não gerar os itens de serviço nos registros C510, C590, C610 e C690”, logo enquadra-se na regra abaixo: 

 

 

 

Segue detalhamento das evidências encaminhadas pelo cliente: 

 

Estamos realizando uma geração com um item sem tributação de ICMS, mas não gera o campo VL_SERV_NT. 

 

 

 

O Parametro não está marcado, ou seja ele poderia pegar a informação da X08. 

 

 

 

 

 

 Tanto que no item do registro C590 traz o CST 090 = Outras tbm.

## Root Cause

O campo "Valor dos Serviços Não Tributados" (VL_SERV_NT) do registro C500 do SPED Fiscal estava sendo gerado de forma incorreta para notas fiscais de energia elétrica eletrônica (modelo 66 / NF3e). Isso ocorre por dois motivos combinados: o sistema não reconhecia o modelo 66 como um tipo de documento válido para esse registro, excluindo essas notas do processamento, e o cálculo do campo não considerava os valores de Base de ICMS classificados como "Outras", resultando em valores incompletos ou zerados. Como consequência, as obrigações fiscais relacionadas a esse tipo de nota de entrada eram reportadas de forma incorreta ao fisco.
