# [DIMED] TKT146450 - TAX ONE - MUNICIPAL - Criciúma BETHA - "Inconsistência no Comprimento de Linha ao Validar Arquivo com a Prefeitura de Criciúma"

## Descricao

Ao realizar a validação do arquivo junto à Prefeitura de Criciúma, foi identificado um erro relacionado ao comprimento das linhas do arquivo. A mensagem de erro apresentada é: 

"Comprimento de linha inválido. Informados 635 caracteres e são esperados 122., 123., 636." 

 

Esse erro indica que o sistema da prefeitura está esperando linhas com comprimentos específicos (122, 123 ou 636 caracteres), mas o arquivo contém uma linha com 635 caracteres, o que não é aceito pelo validador. 

 

Bati campo a campo conforme o manual de layout encaminhado.  

 

  

Vi que o campo que não esta sendo gerado é a Natureza de Operação.  

  

Abaixo segue um passo a passo para preenchimento do campo: 

  

✅ Passo a Passo: Notas Recebidas 

1. Identificar os dados da nota fiscal 

Para cada nota verifique: 

- Serviço cadastrado 
- Município do prestador 
- Local de prestação do serviço 
- Município do estabelecimento 
- Campo COD_MUNICIPIO na tabela SAFX07
- Campo IND_CLASSE_PFJ na tabela X04_PESSOA_FIS_JUR
2. Verificar se há parametrização com COD_PARAM = "471" (Tributação no município)

Preencher com ‘1’ se qualquer das condições abaixo for verdadeira:

- O serviço e município do prestador estão parametrizados na tela Parâmetros – Classificação de Serviços > Município com COD_PARAM = 471
- O campo COD_MUNICIPIO está nulo e o serviço + município do prestador estão parametrizados com COD_PARAM = 471
- O campo COD_MUNICIPIO é igual ao município do estabelecimento e o serviço + município do prestador estão parametrizados com COD_PARAM = 471
- O local de prestação do serviço é igual ao município do estabelecimento ou ao campo município ISS do estabelecimento, e o serviço + município do prestador estão parametrizados com COD_PARAM = 471
3. Verificar se há parametrização com COD_PARAM = "472" (Tributação fora do município)

Preencher com ‘2’ se qualquer das condições abaixo for verdadeira:
