# [Tax Automation] GNRE > Geração de arquivos por UF (segregação) - QA

## Descricao

 Explicação dada pela Vetor está neste link. 
 No Automation estamos gerando o arquivo XML da GNRE unificado por estabelecimento. Para atender as necessidades advindas do Portal Webservice para emissão de guias, necessitamos efetuar a segregação do arquivo por UF. 
 Foi identificado que existe uma tabela (ID) que guarda os processos e notamos que existe um processo pai, como principal, e os demais como processos filhos, como secundários. 
 Hoje, o Automation gera os dados e o arquivo XML no Tax One, mas quando vai buscar as informações no processo, não encontra nada, pois os dados estão segregados. 
 Quando o parâmetro no Tax One é feito para gerar os arquivos por UF, existe um processo principal e um processo distinto para cada UF. 
 Inicialmente, Michel Paiva (Vetor) identificou algumas possibilidades de ajustes no Tax One para que o processo no Automation tenha êxito: 
- Alternativa 1 - segregar por processo úncio, um tipo para cada UF; 
- Alternativa 2 - segregando por processo - primário e secundários (como é hoje).
Necessita ter um PROC_ID principal que vincule aos processos secundários, usando o campo PROC_ID_ORIGEM para fazer o link; 
- Alternativa 3 - colocar dentro do processo principal o ID dos processo filhos.   A necessidade do Automation é ter uma forma de linkar as coisas e a aplicação conseguir capturar isso para levar para o Portal Webservice (transmissão e geração da guia PDF). 
 
 
 Evidência de cenário
Execução realizada em 18.12.2023 no ambiente produtivo do cliente Siemens

## Release Notes

Carta do Patch DW e TAXONE: 

  

Resumo da Alteração: A geração do Arquivo da GNRE Online passa a gravar todas as UF's selecionadas para geração, num
único número de processo por estabelecimento.   

Originalmente a rotina gerava vários processos, um para cada UF.  

Com a redução do número de processos gerados, o controle e conferência dos arquivos se torna mais simples.
