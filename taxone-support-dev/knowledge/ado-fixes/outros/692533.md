# [Internal Bug] T1 - SPED - EFD-Reinf - Quando é excluído uma informação do contribuinte sem Registro de Informação de Software preenchido a tela não dá refresh nas informações excluída.

## Passos de Reproducao

URL do ambiente de testes utilizado (Tax One): ONESOURCE: TAX ONE (onesourcetax.com)
   
 Credenciais de acesso para reprodução da falha: 
- Tennant / Database: T1-QA 
- Usuário: lara.paulo 
- Senha:  
- Empresa: 077 
- Estabelecimento: REPA 
- Período: 09/2024  
 Pré-requisito(s): (optional)
 
 Steps: 
- Cadastrar informação do contribuinte sem informação de software e tentar excluir  

 

Resultado encontrado: Ao tentar excluir o cadastro de informação de contribuinte que não possui informação preenchida no Registro de Informação de Software, o cadastro é excluido porém não é dado refresh na tela e quando pesquisar pela pastinha os dados na tela em segundo plano fica apenas os campos do Registro de Informação de Software.
 

Resultado esperado: que após excluir a tela seja feita um refresh limpando os campos preenchidos sem mudar os campos demonstrados em tela.
 

Evidência da falha:
 

Cadastro sem informação de Registro de Informação de Software: 

 

 

 

Excluindo o cadastro: 

 
 Informações continuam sendo demontradas em tela: 
 Abrindo a pasta: 
   
 
 Conforme vemos em segundo plano demonstra apenas os campos de software mas o registro foi excluído corretamente no banco.
