# [ENGIE] TKT99716 - DW - DCTF - Erro ao replicar Dados iniciais campo "Regime de Apuração"

## Descricao

**TKT 99716 [ENGIE] TAXONE - FEDERAL - OTF - DCTF - Erro ao replicar Dados iniciais campo "Regime de Apuração" assume valor numérico** 

 Prezados, boa tarde!
 
 Cliente relata:
 Estamos com um problema ao replicar os dados iniciais no dentro do programa DCTF via DW. Ao selecionar “Replicar” para utilizarmos como base os parâmetros mais recentes, o DW está retornando o item “Regime de Apuração da Contribuição para o PIS/PASEP e da COFINS” com o valor 5, o que é invalido e acaba gerando erro no PVA. 
 Suporte:
 Efetuamos um teste no ambiente de Suporte (em anexo a demanda) e o erro esta ocorrendo.

## Release Notes

Carta para o
patch: 

Essa alteração tem
como objetivo corrigir o botão de Replicar para preencher corretamente o campo:
Regime de Apuração da Contribuição para o PIS/Pasep e da Cofins. 

  

Módulo Impactado: OTF 

Menu: Outras
Obrigações > DCTF > DCTF Mensal (a partir de 01/01/2006) > Dados
Iniciais > Por Empresa
