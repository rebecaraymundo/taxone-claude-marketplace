# [MUNICIPAL]: T1/Automation - POC - Implementação por Validadores Municipais - Validar - Nomenclatura dos arquivos - Validador ISS WEB

## Descricao

Eu como usuário(a) fiscal das obrigações municipais. 
   1) Gostaria que fosse ajustado a nomenclatura dos arquivos do meio Magnético para cada município gerado via Tax Automation dos Validadores implementados. 
  “Código Empresa + Código Estabelecimento + Município + MMAAAA”, como exemplo, teríamos Barretos:  
“931 0105 – BARRETOS 102024” 

A exceção ocorre para Curitiba que possui um padrão diferente “PMC_10_2024” (mas não atenderíamos) 
   Devido ao volume de município do portifólio do produto T1 e demais que serão implementados, dessa forma facilita a identificação de cada arquivo. 
 Essa nova nomenclatura será liberada conforme a liberação dos validadores via Automation. 
 
  

 
 
 
 
  Validadores 
  Quarter 
  Sprint 
 
 
  ISS WEB 
  Q2 
  2025\Q2\2025_S11_May21-Jun03 
 
 
     
2) Incluir uma validação na geração da obrigação do ISS WEB para que a seja respeitada a parametrização feita na tfix105 (tabela PRT_VALIDADOR_MUNIC).

No TaxOne essa validação é feita pela tela, porém no Automation é feito a criação de um fluxo, onde é relacionado o estabelecimento ao validador independente do relacionamento município x validador.

 
  Funcionalidade Existente:  Agrupamento: Básicos Módulo: Tax Automation Menu: Obrigações Fiscais Municipais
