# 172748 [RAIZEN] – MUNICIPAL - DES-e - SÃO GONÇALO (RJ) ERRO CÓD SERV ZERO ESQUERDA CAMPO: CodigoTributacaoMunicipio

## Descricao

COD_EMPRESA: JV1 Raízen S.A.
   COD_ESTAB: 0001
Ambiente Tax One: UAT – Q8M
URL: 'https://uat.onesourcetax.com/platform/apps/oms/home/oms/safmunic/home/oms/safmunic/safmunic1/w_lib_proc_safmunic 
UF e Município: RJ - São Gonçalo - Código Município Tax One 4904 

Serviço Lei 116 (TFIX60.TXT): 07.02 

Serviço municipal (TFIX106) :  7.02 

Exigência Layout DES-e>Prefeitura: 7.02 

NF Entrada: 3382 

Data Fiscal: 18/09/2025
Período: 01/09/2025 a 30/09/2025. 

Taxo One >  Municipal > São Gonçalo > DES-e 
 
Cliente relata que após configurações (prints/documento anexo) ao gerar o XML (anexo DES-e), o arquivo gera no campo CÓD SERV.: 07.02, mas a Prefeitura REJEITA. 

Após manipular o xml indicando CÓD SERV.: 7.02, o arquivo processa com sucesso. 
Imagem xml

 

RESUMO 
Identificamos erro na geração/validação do DES‑e – São Gonçalo/RJ para a empresa JV1 / 0001 (competência 09/2025, após mudança do validador): o arquivo está saindo com Código do Serviço (LC116) com zero à esquerda (ex.: 07.02), porém o validador da Prefeitura exige sem zero à esquerda (ex.: 7.02). Ao remover manualmente o zero, o arquivo é aceito. Pela regra o campo é preenchido a partir do DWT_SERVICO_LEI_116 / COD_SERV_LEI_116, e a orientação é desprezar zeros à esquerda, mas na prática não está funcionando (a informação está na escrituração da NF e consta também na SAFX2018/rotina de geração). 

Sugestão de correção: ajustar a rotina/regra de geração do layout para normalizar o LC116 removendo zeros à esquerda antes de montar o meio magnético (ex.: 07.02 → 7.02) ou aplicar de/para quando necessário.

Observação importante (para evitar impacto para outros clientes): 

A correção deve ser controlada por município/layout (São Gonçalo/RJ – DES‑e), para não impactar outras empresas/municípios que possam aceitar/exigir formatação diferente do LC116.

Anexo evidências e xml gerado, segue Layout_São_Gonçalo_RJ:

Cliente traz essa sugestão: "aperfeiçoar essa regra com "tratamento" para trazer o código de serviço
