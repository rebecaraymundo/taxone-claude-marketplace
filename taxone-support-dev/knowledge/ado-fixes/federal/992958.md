# TKT162231 - MRV - ONESOURCE TAX ONE - MUNICIPAL - SIGISS - Erro ao submeter o arquivo no validador do Município de Londrina' Valor da Base de calculo nao pode ser menor que o Valor da NF"

## Descricao

Ao validar o meio magnético para o município de Londrina, verificou-se que, para os documentos fiscais sem valor de base de cálculo (campo preenchido com 0,00), ocorreu o seguinte erro durante a validação: "O valor da base de cálculo não pode ser menor que o valor da nota fiscal". 

 

O cliente está realizando a validação dos documentos fiscais para o município de Londrina e, nos casos em que o ISS é retido pelo prestador de serviços, informou que, conforme a legislação vigente, não é informado o valor da Base ISS no item do documento fiscal. Ao gerar o meio magnético desses documentos, o campo "Valor Base ISS" é preenchido com 0,00. Contudo, ao submeter o arquivo ao validador, ocorre um erro e o arquivo não é validado. 

 

 

 

Ao verificar a regra de Geração do campo, atualmente a regra do campo considera o Valor da Base ISS dos itens: 

 

 

 

https://trten.sharepoint.com.mcas.ms/:w:/r/sites/REQ_MTZ/_layouts/15/Doc.aspx?sourcedoc=%7BEFA1E0AC-B07B-443D-843C-FBCAC247799E%7D&file=MTZ-SIGISS-Geracao_do_Meio_Magnetico.doc&action=default&mobileredirect=true 

 

Ao orientar ao cliente, o preenchimento deste campo para que o campo seja gerado devidamente com valor para que possa ser validado na prefeitura, o mesmo informou que por se tratar de ISS retido pelo prestador, este campo não deve ser preenchido, e para validação no validador o arquivo não deveria ter valor preenchido ou deveria ser preenchido com o valor total da Nota fiscal. (Na simulação, anexarei dois testes de validação). 

 

Poderiam verificar se há possibilidade de atendimento para essa situação?
