# [MUNICIPAL]: T1/Automation - POC - Implementação por Validadores Municipais - Fase 08 - Validação - Final - T1 + Automation -  ISS.NET/SIGISS/BETHA/ISS WEB LAR PAULI

## Descricao

Demanda de DEV: https://dev.azure.com/tr-ggo/Mastersaf%20Fiscal%20Solutions/_workitems/edit/819205 
    Eu como usuário(a) fiscal das obrigações municipais. 
 1) Gostaria que fosse realizado um estudo de implementação por validadores municipais no TAXONE e posterior viabilizar a chamada no Tax Automation, dando a possibilidade do cliente realizar o geração de determinado município no que compete o validador.   Dessa forma simplificaria o processo de implementação de municipal, sendo assim um ganho para o produto aumento o portfólio de prefeituras atendidas. 
 2) Gostaria que fosse ajustado a nomenclatura dos arquivos do meio Magnético para cada município gerado via Tax Automation dos Validadores implementados.  
 Devido ao volume de município do portifólio do produto T1 e demais que serão implementados, dessa forma facilita a identificação de cada arquivo. 
 Essa nova nomenclatura será liberada conforme a liberação dos validadores via Automation. 
  “Código Empresa + Código Estabelecimento + Município + MMAAAA”, como exemplo, teríamos Barretos:  
“931 0105 – BARRETOS 102024” 

A exceção ocorre para Curitiba que possui um padrão diferente “PMC_10_2024” (mas não atenderíamos) 
  
 ​Considerações Importantes. 
 Essa demanda tem comtemplar o processo de validação como um todo geração e Validação de pelo menos 1 município de cada validador em T1 e Automation.  Nessa primeira entrega não esta comtemplando a GISSONLINE.
