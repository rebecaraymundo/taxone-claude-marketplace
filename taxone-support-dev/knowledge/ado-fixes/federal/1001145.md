# TKT162566 - SANTISTA - ONESOURCE TAX ONE - MUNICIPAL - ISS ONLINE 4R - Tatuí -  Reavaliação regra campo "Situação" na parametrização Parâmetros – Classificação de Serviços - Município.

## Descricao

Reavaliação regra campo "Situação" na parametrização Parâmetros – Classificação de Serviços - Município. 

Avaliar para no momento da parametrização a tela permita que o COD_PARAM = “495” e o COD_PARAM = “472” permita realizar a parametrização para códigos de serviços iguais. 

Atualmente, para fornecedores que prestam serviços para ambas estabelecimento no município de Tatuí e Americana, como o exemplo de um fornecedor de Campinas, é necessário cadastrar o mesmo serviço em diferentes parâmetros de Classificação de Serviço por Município, conforme abaixo: 

 

- Para atender ao ISS Tatuí: 
- CAMPINAS >> Parâmetros – Classificação de Serviços com o COD_PARAM = “495” >> serviço 1446 
- CAMPINAS >> Parâmetros – Classificação de Serviços com o COD_PARAM = “495” >> serviço 1448  
 

 

- Para atender ao ISS Americana: 
- CAMPINAS >> Parâmetros – Classificação de Serviços com o COD_PARAM = “472” >> serviço 1446 
- CAMPINAS >> Parâmetros – Classificação de Serviços com o COD_PARAM = “472” >> serviço 1448  
 
Problema Identificado: 

A tela de cadastro atualmente não permite relacionar um mesmo serviço para mais de um parâmetro dentro do mesmo município. Na tela consta a seguinte: 

"Não será aceito, para o mesmo município, o relacionamento de um mesmo serviço para mais de um parâmetro." 

 

Solicitação Técnica: 

Solicito análise e ajuste no sistema para permitir que, para o mesmo município, seja possível relacionar um mesmo serviço a mais de um parâmetro, visando atender à necessidade dos estabelecimentos que utilizam diferentes parâmetros para o mesmo fornecedor/serviço. 

 

Justificativa: 

A limitação atual impede o correto atendimento às exigências fiscais das filiais, prejudicando o cadastro e a operação dos fornecedores que prestam serviço para mais de uma filial.
