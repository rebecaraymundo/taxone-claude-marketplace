# [OTF] T1 -  IURD - Geração de Informe de Rendimento - Menu > Rendimentos Outros

## Descricao

Eu como usuária do modulo Obrigações de Tributos federais 

  

Gostaria que fosse feita uma melhoria, com a
inclusão do filtro por UF na Geração dos informes de Rendimento Cliente IURD. 

  

Na geração os arquivos devem serem gerados
cada um com suas respectivas UF. Todos os informes que foram da UF de SP, devem
gerar num único arquivo, com a seguinte nomenclatura. 

  

Exemplos: SP_90000684015_29744778000197_2025 

                   PE_70002042495_29744778000197_2025 

  

Modulo Existente 

Modulo: Federal 

Menu: Obrigações de Tributos Federais > Rendimentos >
Rendimentos Outros  

 

 
 Informações Adicionais:  Experiencia do Cliente IURD Atualmente, geramos aproximadamente 16 mil Informes de Rendimentos. O arquivo é salvo no formato padrão: CPF_CNPJ_matriz_ano (Ex.: 90000684015_29744778000197_2024). Este formato exige um enorme retrabalho para separar os informes por Unidade Federativa (UF), o que é fundamental para a distribuição dos documentos aos estados.
 Para eliminar o retrabalho na separação, nossa proposta é que o sistema implemente uma das seguintes opções na próxima geração:
 
                   1.	Geração por Pasta Individualizada: O sistema deve gerar os Informes em pastas separadas para cada UF.
                   2.	Inclusão da UF no Nome do Arquivo: O nome do arquivo deve incluir o código da UF no início.
                   Formato Sugerido: UF_CPF_CNPJmatriz_ano
 
 Exemplos:
  SP_90000684015_29744778000197_2025
  PE_70002042495_29744778000197_2025
 
 Esta alteração é fundamental para garantir a agilidade e a correta distribuição dos documentos.

## Release Notes

Informações p/a Carta do Patch: 

Produto: TAXONE 

Melhoria 

Objetivo:  
Informamos que realizamos uma atualização importante em nosso produto com o objetivo de aprimorar a experiência de uso e oferecer maior precisão nas emissões de declaração de rendimentos (Grandes Volumes), do módulo Federal > Outras Obrigações de Tributos Federias, menu Rendimentos/ Rendimentos Outros
 

Principais melhorias implementadas 

1. Inclusão do campo UF no filtro das telas

Agora é possível filtrar as informações diretamente pela Unidade Federativa (UF), permitindo buscas mais rápidas, segmentadas e alinhadas às necessidades de cada operação. 

2. Alteração da nomenclatura dos arquivos

Quando houver UF cadastrada, os arquivos gerados passarão a incluir essa informação na nomenclatura, garantindo maior organização e fácil identificação dos documentos. 

3. Criação de pastas específicas por UF

Na geração dos arquivos, o sistema passará automaticamente a estruturar pastas dedicadas a cada UF cadastrada
