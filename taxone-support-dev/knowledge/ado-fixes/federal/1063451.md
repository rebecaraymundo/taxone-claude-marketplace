# TKT166316 - MRV - ONESOURCE TAX ONE - MUNICIPAL - SIGISS - SANTA LUZIA - Reavaliação da regra para geração eventual não considerar as notas do mesmo município - Copy

## Descricao

licitação Suporte 

Para o validador SIGISS, município Santa Luzia. 

 

O cliente está gerando o arquivo como eventual (Arquivo de Entradas de Serviços (Const. Civil/Utilities/Telecom) e nele estão sendo consideradas notas em que o prestador é do mesmo município de geração do arquivo, o que não é correto. 

Neste caso, para gerações eventuais, em que o cadastro do estabelecimento é outro município, a geração não atende a regra para desconsiderar os documentos em que o Prestador está no mesmo município em que o tomador. 

A solicitação é para uma reavaliação da regra de requisitos. 

 

Avaliar para considerar que notas de prestador dentro do município não façam parte do arquivo (cliente utiliza modelo 70 para NFSe)

## Solucao

SaaS

## Release Notes

Informações p/a Carta do Patch:
DW e TAX ONE 

Regra 

Objetivo: Inclusão da regra para desconsiderar nota fiscais
em que o Prestador está no mesmo município em que o tomador da eventual, na
geração do Arquivo de Entradas de Serviços (Const. Civil/Utilities /Telecom),
para o validador SIGISS, contemplando a geração do município de Santa Luzia/MG.
