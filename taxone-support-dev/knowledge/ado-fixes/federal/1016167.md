# TKT167264 - TAX ONE - MUNICIPAL -  Vitória - Arquivo de Entradas de Serviços (Consti. Civil/Utilities/Telecom) " Inconsistências no Registro DST no campo Código do Município"

## Descricao

Ao gerar o arquivo Arquivo de Entradas de Serviços (Consti. Civil/Utilities/Telecom) no módulo de Vitória, o campo Código do Município esta sendo preenchido com o resultado presente no cadastro do estabelecimento do campo Município conforme imagem abaixo:  

Estamos trabalhando com o municipal de Vitória (inscrição eventual) e quando o cliente foi tentar validar o arquivo, foi apresentado erro de código de município de IBGE. Ao invés de considerar o código do município de Vitória, está considerando o que consta no cadastro da empresa estabelecida e não está correto. 

 

 
 

 

O que esta descrito em nossa regra que realmente seria desta forma: 

 

 

 

Porém esse comportamento do sistema esta causando o erro no validador : 

Erro : -Linha 1: O campo [Código do Município (T1-08)] deve ser informado como 5309.  

 

 

Cliente informa que o campo que deve ser considerada a informação é o do cadastro da Inscrição Municipal eventual :
