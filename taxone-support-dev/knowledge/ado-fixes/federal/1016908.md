- Ambiente que acontece o problema: UAT

Contexto e
Problema Observado 

  

Na geração do arquivo
de Serviços Tomados (GIAP-PRESCON) para Araraquara/SP, o TaxOne está utilizando
a Data de Emissão para definir a competência, contrariando a especificação
oficial do validador Prescon (RN 2754), que exige seleção por Data Fiscal (competência).
A tag data Nota Fiscal deve permanecer com a Data de Emissão (RN63 27.a). 

Fundamentação
Normativa 

 

 

  

- • RN54 – Recuperação das notas fiscais de
serviços tomados: Data Fiscal da nota dentro do período de referência.  
- • RN63 – Tag <dataNotaFiscal>: Preencher
com DATA_EMISSAO. 
- • Layout Araraquara: Tags DataCompetencia (Data
Fiscal Emissão) e DataEmissao (Data de Emissão).  
 
 Evidências 
Arquivo gerado para 

Empresa: BRL Brado Logística
S/A 

Estabelecimento: 0022 –
Brado Logística S.A 

Competência: 11/2025 

NFS-e: 71588 

Data de Emissão = 26/05/2025 
Caminho: Básicos > Data Warehouse >
Manutenção > Documento Fiscal > Doc.Fiscal de Mercadoria e Serviço

 
 
Geração do Arquivo 

Caminho: Municipal
> Araraquara > Obrigações > Meio Magnético 

 
Demonstrando que o
sistema está considerando emissão para competência. 

 
Erro -
Simulado 

Competência
divergente: NF 71588 – Data Fiscal: 05/11/2025, Data Emissão: 26/05/2025 –
Incluída em 11/2025 com base na emissão. 

 RN27.a  não atendida: Seleção deveria ocorrer por
Data Fiscal.

a regra de preenchimento do campo Data de Competência, para que seja considerada
a informação da data de emissão do documento fiscal, para o município de Araraquara/SP,
do validador NFSE GIAP-PRESCON. 

  

Layout Padrão: XML
