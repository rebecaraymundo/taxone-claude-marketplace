# TKT141776 - RED BULL - BRAVO (BPO) - DW - MUNICIPAL - NOTA CARIOCA - Ajuste do manual Web Help

## Descricao

Conforme conversado com Rosemeire abro essa demanda para ajuste do manual do WEB Help  :Módulo Nota Carioca 

 

 

 

 

nosso manual do web help esta incoerente porque no nosso manual informa que ao colocar a validade ele buscaria a partir de aquela data, mas quando clica na lupa ele mostra apenas dados daquela data para trás.  

 

 

 

 

 

Expliquei que a data de validade inserida na parametrização é crucial para a listagem de serviços que o sistema apresenta. Quando inserimos uma data de validade, o sistema busca os serviços cuja data de início de validade esteja dentro do período informado. 

  

Por exemplo, ao realizar a Relação Código de Serviços Msaf x Serviços Municipais, se o serviço tem sua data de validade inicial em 10/10/2023, a data de validade a ser inserida deve ser posterior a essa data (exemplo 11/10/2023 mas lembrando que se eu colocar a data exata da validade ele aparece, isso foi apenas um exemplo). Se estamos criando uma nova parametrização, o serviço precisa ter uma data de validade anterior à data da parametrização. Em outras palavras, a data de validade da parametrização deve ser associada a um serviço que já exista antes dela. 

 

Como no exemplo das imagens e prints tirados por mim e pela Rose minha data é de 01/01/2010 e aparece tudo que tenho cadastrado para anterior a essa data e nunca a partir dessa data. 

No print da Rose a validade dela é 01/01/2011 e aparece tudo anterior a essa data ou igual a data e nunca a partir da data de validade como informado no manual.
