# TKT161470 - MRV - ONESOURCE TAX ONE - MUNICIPAL - EGOVERO - SÃO LEOPOLDO -  Avaliar a regra para não considerar notas fiscais quando o prestador é no mesmo município do estabelecimento.

## Descricao

Validador: EGOVERNO 

Município: São Leopoldo 

 

Cliente possui Inscrição Municipal Eventual para o município de São Leopoldo. 

Avaliar a regra para não considerar notas fiscais quando o prestador é no mesmo município do estabelecimento ( São Leopoldo) no menu de geração - Arquivo de Entradas de Serviços (Const.Civil/Utilities/Telecom) 

 

Na regra de Geração do Meio Magnético Situação Especial não consta informação para desconsiderar notas fiscais quando o prestador é no mesmo município do estabelecimento, não tendo necessidade de gerar a nota fiscal no arquivo, visto que já cai de forma automática na prefeitura. 

 

 

Por favor avaliar.

## Solucao

SaaS

## Release Notes

Informações p/a Carta do Patch:
DW e TAX ONE 

Ajuste de regras 

Objetivo: Ajustado o processo de geração para o município de São Leopoldo do validador EGOVERNO, para que sejam
desconsideradas as notas fiscais eletrônicas emitidas por prestadores
estabelecidos dentro do município da inscrição eventual.  

  

Layout Padrão: XML 

Módulo Impactado 

Módulo: Municipal > São Leopoldo/RS 

Menu: Geração 

 

- Geração do Arquivo de Entradas de Serviços (Const.
Civil/Utilities/Telecom)
