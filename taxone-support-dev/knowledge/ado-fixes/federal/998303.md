# TKT163649/163607 - MRV - ONESOURCE TAX ONE - MUNICIPAL - ATENDE-NET - GRAVATAÍ/PALHOÇA-  Reavaliação de regra campo Cod. Situação tributária quando existe Inscrição Eventual.

## Descricao

1) Munícipio GRAVATAÍ/PALHOÇA - Erro geração código da Situação Tributaria" -  Reavaliação de regra campo Cod. Situação tributária quando existe Inscrição Eventual. 

Mesmo com o parâmetro indicado pelo suporte em outro chamado do mesmo problema e validador, parâmetro classificação do serviços cod 473, o parâmetro esta feito e o código da situação tributaria sai com 01 tributado integralmente, quando deveria ser 04. 

O cliente possui estabelecimento domicilio em Porto Alegre, mas possui Inscrição Eventual para GRAVATAÍ - RS. Esta escriturando um documento fiscal que possui redução na base de cálculo, de acordo com o Validador AtendeNet o campo Código situação tributária deveria ser gerado como 04, mas o campo é gerado no meio magnético como 01. 

Ao verificar a regra de negócio a regra sugere a seguinte parametrização: O código '473' deve ser parametrizado na tela "Parâmetros – Classificação de Serviços" utilizando o COD_PARAM (cod_classif) da seguinte forma: 

Se Cod_classif = 473, Cod_municipio_Nf = Cod_municipio_Estab e Ind_tp_ret = 1, então cod_sit_tributaria será igual a 04. 

O Tax One está validando o estabelecimento (Joinville) para a geração do código de substituição tributária e não está considerando o cadastro da inscrição municipal eventual e a parametrização realizada para gerar o código 04, resultando na geração do código situação tributária com o código 01. 

Demanda aberta anteriormente: https://dev.azure.com/tr-ggo/Mastersaf%20Fiscal%20Solutions/_workitems/edit/920398, cliente relata que após a liberação desta demanda, o erro não foi corrigido.
