# TKT160970 - MRV - ONESOURCE TAX ONE - MUNICIPAL - MARINGA - FINTELISS Erro ao importar o arquivo xml e  para não gerar notas de prestador do mesmo município

## Descricao

Ao realizar a importação de arquivo em xml com no máximo 500kb. 
 Cabeçalho correto para Maringa: <EnviarLoteServicoTomado xmlns="http://www.isse.com.br/dmse.xsd" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://www.isse.com.br/dmse.xsd dmse_v1.0.xsd "> <LoteServicoTomado Id="" versao="1.00">    
Houve liberação da demanda https://dev.azure.com/tr-ggo/Mastersaf%20Fiscal%20Solutions/_workitems/edit/923046 com infolegis 700/25 - B - ISSQN - MARINGÁ/PR - Migra Sistema (FINTEL). 

Avaliar a regra para não considerar notas fiscais quando o prestador é no mesmo município do estabelecimento ( Maringá), para Eventual. 

  

Porem o cliente retornou com o erro acima. 
 
Validador: DMSe 

Município: Maringá  

Conforme informado pelo time de Base Tributária as as operações registradas em NFS-e serão lançadas automaticamente na Declaração Mensal de Serviços Eletrônica (DMS-e). 

 

 

Regra do validador DMS-e 

 

 

OBS: Verifiquei que existe uma demanda de atualização do validador de Maringá para o validador FintelSS, porem conforme informado pelo time de Base Tributária podemos considerar que o validador ou plataforma para Maringá é a FINTEL, e dentro dela, a obrigação é a DMS-e.
