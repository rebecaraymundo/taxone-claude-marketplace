# TKT136396 - ERNST YOUNG - TAX ONE - MUNICIPAL - NOTA JOSEENSE - Município de Campinas

## Descricao

TKT136396 - ERNST YOUNG - TAX ONE - MUNICIPAL - NOTA JOSEENSE - Município de Campinas 

 

Após a liberação da atualização de layout do município de Campinas, o cliente validou e nos retornou alguns pontos: 

 

Registro tipo “H” - Identificação (Header): 

Campo 2 Inscrição Municipal: No Manual de Layout descreve que este campo tem um tamanho total de 30, porém o Tax One gera somente com o tamanho da IM de fato informada no cadastro do estabelecimento. 

 

 

Quando o cliente importa o arquivo no portal da prefeitura, acusa erro de Layout: 

 

 

 

Solicitado ao cliente que realizasse um teste, que acrescentasse manualmente espaços no arquivo, para completar o tamanho do campo, e o erro não apresentou mais. 

 

Quando, o arquivo foi importado sem apresentar erro de layout, o mesmo foi processado, retornando outros erros, em outros campos: 

 

No Registro tipo “T” - Notas Tomadas: 

 

Campo 11 - Código do município do prestador de serviços e Campo 28 - "Código do município do Local da Prestação do Serviço: 

Estes campos possuem um tamanho total de 7 caracteres, e no arquivo gerado pelo Tax One gera nesses campos o código da UF + Código do município. Quando o município tem o código com 4 caracteres ou menos, o campo é gerado com espaços no final para completar o tamanho, e isso gera erro na validação do arquivo (apesar de que no manual de layout orienta a preencher com espaços no final) 

 

 

 

O cliente realizou um teste, inserindo 0 entre o Código UF e o Código Município, para completar o tamanho do campo, e estes erros não mais aconteceram. 

 

Campo 16 - Tipo de logradouro do prestador 

Neste campo, segundo a regra busca a informação da parametrização Tipo Logradouro Msaf x Tipo Logradouro, e nesta parametrização traz uma listagem, que é a mesma que tem no Manual de Layout, mas para outro campo: 

 

 

 

 

 

Veja no Manual de Layout, esta lista é referente ao campo 19 e não 11:
