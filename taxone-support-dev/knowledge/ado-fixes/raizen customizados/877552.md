# Criação da Tabela Acessória - Tacess - Desenvolvimento - Validação modulo Parametros QA

## Descricao

Tabela Acessória 

Objetivo: Desenvolver uma tabela acessória com a finalidade de alimentar os dados e manutenção de acordo com a tabela obrigatória DOOTAX (Tax Payments). 

link Cópia de Planilha de Apoio - Campos Obrigatorios JSON (1) (003).xlsx 

Campos : UF_FAVORECIDA; COD_ARRECADACAO, COD_RECEITA, DET_RECEITA 

Detalhes: 

- Fonte de Dados: Tabela Obrigatória do Dootax (Tax Payments). Carregar os dados. 
- Módulo a ser Utilizado: 
Menu Básicos,  Módulo: Ferramentas, Menu Tabelas Internas à
Manter à Tabelas 
 
- Finalidade: A tabela acessória deve facilitar a manipulação e a visualização dos dados, permitindo uma gestão mais eficiente das informações relacionadas a pagamentos de impostos e manutenção pelo usuário  Desenvolvimento técnico: 
 Mapeamento de Campos: Realizar uma análise detalhada para identificar quais campos são necessários para o cruzamento entre a tabela acessória tace119 e a SAFX 2080. É essencial definir precisamente as chaves de integração, como IDs e códigos fiscais, para garantir a correta correspondência e integração dos dados.

 Em anexo estão os documentos MTZ e a planilha de apoio 
 ------- Teste parâmetro realizado,  as informação dos COD_RECEITA, COD_ARRECADACAO e DET_RECEITA gerados nas guias estão vindo sem informação e não esta aparecendo para seleção. 
 Dado esperado: Trazer no T1 no parâmetros de Guias na opção COD_RECEITA, COD_ARRECACAO e DET_RECEITA  a opção de buscar via matchcode os códigos  vigentes para seleção que estão cadastrados na tabela taces119.  Aonde ao abrir o matchcode de seleção aparecer, UF/ Código Receita, UF/ Código ARECADACAO e UF/DET RECEITA. .
