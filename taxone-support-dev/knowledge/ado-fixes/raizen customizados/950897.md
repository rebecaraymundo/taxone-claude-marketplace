# Tax Payments - Ajuste na regra do Campo Data de pagamento - Preenchimento obrigatório. "Correção erro MS"

## Descricao

Objetivo:
Incluir o campo "Data de Pagamento" como preenchimento obrigatório no arquivo JSON para todas as guias emitidas. 

Regra de Negócio:
 

O campo "Data de Pagamento" será adicionado como um elemento obrigatório no arquivo JSON para todas as guias emitidas. Este campo será preenchido automaticamente pelo sistema, seguindo as mesmas regras e o mesmo formato já configurados para o campo "Data de Vencimento", que se encontra na tela de parâmetros. 

As regras disponíveis são selecionadas pelo usuário: 

- Dias Corridos 
- Dias Fixos 
- Dias Úteis 
- Dias Corridos Postergados 
- Dias Úteis com Dias Fixos Antecipados 
- Dias Postergados  
A "Data de Pagamento" no arquivo JSON terá o mesmo valor da "Data de Vencimento", assegurando consistência no processo de geração das guias de acordo com a opção selecionada pelo usuário. 

Alteração na tela: 

Para alinhar com a inclusão do campo "Data de Pagamento" e para facilitar o entendimento da funcionalidade, haverá uma alteração no rótulo do campo na tela de parâmetros: 

- De: Regra de Vencimento 
- Para: Regra de Vencimento/Pagamento  Relatório de Guias: 
  No relatório, a "Data de Pagamento" será automaticamente igual à "Data de Vencimento". Essa padronização facilita o acompanhamento e a gestão das guias, garantindo que ambas as datas sejam consistentes e precisas, além de refletir claramente a visualização para o usuário. 
 Geração do arquivo JSON:  No arquivo, a "Data de Pagamento" será automaticamente igual à "Data de Vencimento". 
 Jornada do usuário: Acessar o Tax One Módulo: Básico Data warehouse>Manutenção>Guias de pagamentos>Parâmetros de Guias de pagamentos
