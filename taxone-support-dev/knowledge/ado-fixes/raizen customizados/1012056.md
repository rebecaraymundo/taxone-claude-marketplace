# [Dootax APIS- Integração] Guia Complementar - Lançamento do calculo, geração do arquivo JSON e do Relatório de Status referentes os tributos PIS, COFINS, ICMS‑Próprio, ICMS Antecipado e ICMS‑ST. - Copy

## Descricao

Objetivo: Tela 

 
 
 
  
Desenvolver uma tela específica para o cálculo e emissão de guias complementares quando houver diferença a pagar decorrente da reapuração dos impostos: ICMS‑Próprio, ICMS‑ST, IPI, PIS‑Cumulativo, COFINS‑Cumulativo, PIS–Não Cumulativo e COFINS–Não Cumulativo. 

A tela deve permitir identificar, comparar e consolidar a diferença entre: 

- Valor reapurado (apuração atual) e 
- Valor pago da guia original, obtido via API de retorno da Dootax (status “Guia Pagar”).  
possibilitando o cálculo automático da diferença a pagar e a geração do arquivo JSON correspondente para cada tributo. 
 
- Regra de Negócios:  
- 
1. Consulta de Apurações: O sistema deve consultar o valor atual da apuração para o período.  
- 
  
- 2. Consulta de SAFX Retorno: Valor da Guia Pagar de guias com Status "Guias Pagas".   
3. Cálculo da Diferença a Pagar: A diferença deve ser calculada individualmente por tributo, aplicando a seguinte fórmula: 
  Diferença a pagar = Valor Reapurado – Valor Pago da Guia (Status “Guia Pagar”  API Dootax) 
 Tooltips informativos para usuários. 
- 
  
- MTZ: MTZ_Tela de Guia Complementar.docx  
- 
  
- Jornada de usuário:  
- 
- Agrupamentos: Básico 
- Módulo: Manutenção >> Guia de Pagamento >> Guia Complementar 
- Menu: Acesso Principal >> Guia de Pagamentos     

 

Objetivo Geração: 

Realizar a geração e emissão de guias complementares quando houver diferença a pagar decorrente da reapuração dos impostos. A funcionalidade deve permitir que o usuário gere o arquivo dadiferença pagar para os tributos ICMS‑Próprio, ICMS‑ST, IPI, PIS‑Cumulativo, COFINS‑Cumulativo, PIS–Não Cumulativo e COFINS–Não Cumulativo. 
 
Regra de negócio: 

- O sistema deve gerar o arquivo JSON com todas as diferenças a pagar por tributo, seguindo o padrão estabelecido. 
- O arquivo deve ser disponibilizado na Tela de Status e Relatório de Status
