# Dootax - APIs de Integração Chamado 171645 ICMS-ST Arquivo JSON gerando as UFs do Estabelecimento/ Chamado 171637 Erro preenchimento do Parâmetro em tela/Erro transmissão do arquivo Json e Correção de Caracteres especiais no Menu de Geração da Guia

## Descricao

--Dootax - APIs de Integração Chamado 171645 Grupo de imposto de ICMS-ST - Arquivo JSON gerando as UFs do Estabelecimento 
 Objetivo: 
 É corrigir o erro do reportado pelo usuário abaixo: 
 O arquivo JSON gerado pelo sistema quando geramos as guias está gerando todas as UFs favorecidas com o estado do estabelecimento ao invés de considerar as IEs cadastradas do favorecido. 
 Regra de Negócio: 
 O campo UF_FAVORECIDA será preenchido conforme os
valores apresentados na tabela Resumo_APUR_ST para correção do erro.
 
Fonte dos Dados 

Os valores apresentados serão extraídos da tabela Resumo_APUR_ST, considerando os seguintes filtros: 

- COD_TIPO_DO_LIVRO = 108 ou 165 
- COD_ESTADO = UF_FAVORECIDA 
- COD_OPER_APURAR = 013 
- Campo considerado: VLR_ICMS  Jornada do usuário: 
 Manutenção> Guias de Pagamentos>Geração 
  Link: MTZ-ICMS-Geracao_GUIA_ICMS-ST (1).doc
 
 Abaixo exemplo de erro do erro: 
 
 
 
 ---------------------------------------------x----------------------------------- 
 --Dootax API Integração Chamado #171637 Erro no preenchimento do Parâmetro em tela 
 Objetivo: 
 Trata-se da correção do erro reportado pelo usuário ao tentar criar um parâmetro de guia de pagamento para ICMS_NORMAL. 
Atualmente, o TaxOne exibe a mensagem informando que “o cadastro já existe”, porém o parâmetro nunca foi criado pelo usuário.
Esse comportamento incorreto impede a continuidade do processo. 

Impacto: O erro impede o usuário de configurar os Parâmetros necessários para a geração das guias de pagamento, bloqueando todo o fluxo de criação e envio das guias. 
 Jornada do usuário: 
 Manutenção> Guias de Pagamentos>Parâmetros

 --Dootax API de Integração - Correção de Erro de Caracteres no Menu de Geração da Guia Objetivo: O objetivo é corrigir o erro na geração das guias relacionado ao tratamento de caracteres especiais.
