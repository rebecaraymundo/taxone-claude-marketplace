# [Dootax APIS- Integração] ID- Rastreabilidade de Guias entre os sistemas T1 X OBI - Desenvolvimento

## Descricao

Objetivo: Desenvolver uma regra única e consistente para a identificação (ID) de cada guia de pagamento no novo módulo "Guia de Pagamento", integrado ao sistema Onesource Tax One/OBI para plataforma  TAX PAYMENTS( DOOTAX), garantindo que cada guia seja identificável, rastreável e livre de duplicidades. 

Definição do ID da Guia de Pagamento para Rastreabilidade entre Sistemas 

O ID da guia de pagamento de cada guia emitida é essencial para garantir a rastreabilidade entre os sistemas. A estrutura do campo num_doc_origem deve incluir os seguintes dados:

Estrutura do ID

O ID será composto pelos seguintes elementos, com tamanhos ajustados:

- Cod_Empresa: 3 caracteres (Ex.: "001").
- Cod_Estabelecimento: 6 caracteres (Ex.: "000001").
- Cod_Receita: 6 caracteres (Ex.: "131701").
- Hora de Geração: 4 caracteres (Ex.: "1536").
Formato Final

[Cod_Empresa][Cod_Estabelecimento][Cod_Receita][Hora]

Exemplo Final:
0010000011317011536

Validações

- Unicidade: Certificar-se de que cada ID gerado seja único dentro do sistema.
- Comprimento: Garantir que o ID tenha exatamente 20 caracteres.
- Consistência: Verificar se os valores de empresa, estabelecimento, receita e hora estão corretos e correspondem à realidade dos eventos descritos.
Observação:
Estabelecer que, sempre que o campo 'num_doc_origem' estiver preenchido com o número da nota fiscal, esse mesmo número seja adotado como ID da guia. Isso assegurará a rastreabilidade consistente entre os sistemas.

Validações Necessárias para o ID :

- 
Unicidade do ID: Certifique-se de que cada ID gerado seja único para evitar duplicidade dentro do sistema.

- 
Validação de Formato e Comprimento:

- Cod_Empresa: Deve sempre ter 3 caracteres, preenchido corretamente (Ex.: "001").
- Cod_Estabelecimento: Deve sempre ter 4 caracteres, preenchido corretamente (Ex.: "00001").
- Cod_Receita: Deve sempre ter 5 caracteres, preenchido corretamente (Ex.: "131701").
- Hora de Geração: Deve sempre ter 4 caracteres, sendo uma combinação eficien
