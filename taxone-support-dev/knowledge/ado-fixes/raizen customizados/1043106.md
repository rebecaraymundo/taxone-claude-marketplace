# API | Contabilização – Inclusão do Campo “Nº Documento Contábil” na Tela de Lançamento Complementar

## Descricao

Objetivo: 

Incluir o campo Número de Documento Contábil,
retornado pela SAFX de Retorno nº 347, na lógica de Lançamento
Complementar, impedindo que o usuário prossiga com o Lançamento
Complementar quando não houver número de documento disponível na tela. 

Regra de
Negócio:  

 
- O
     campo Número de Documento Contábil deve ser preenchido
     automaticamente pelo sistema com o valor retornado da SAFX de Retorno
     nº 347, campo NRO_DOC_CONTABIL, considerando os filtros existentes na
     tela: 
 
  
- Empresa 
  
- Estabelecimento 
  
- Grupo de Imposto 
  
- Imposto 
  
- Mês da Apuração 
  
- Ano da Apuração 
  
- Tipo do Parâmetro 
  
- Código
      de Ajustes ou Outros Lançamentos 
 
 
- O
     sistema deve validar obrigatoriamente se o número foi retornado
     antes de permitir a continuidade do Lançamento Complementar criado pelo usuário. 
 
- Caso não
     exista número retornado, o sistema deve: 
 
  
- Exibir mensagem de erro; 
  
- Impedir
      o avanço geração do arquivo do arquivo JSON. 
  
- Manter
      o campo como obrigatório e não permitir que fique vazio. 
  
 Mensagem de Erro  

“O Lançamento Complementar só poderá ser realizado se
houver Número de Documento Contábil. 

Por favor, verifique se existe lançamento contábil
para o período informado e Ajustes ou Outros Lançamentos informados. 

Link: MTZ_Tela de Complemento de  Lançamentos Contábeis - Manual Diferença de valores.docx
