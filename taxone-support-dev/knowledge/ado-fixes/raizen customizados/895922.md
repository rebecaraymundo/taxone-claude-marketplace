# [Dootax APIS- Integração]Produtização do Costumizado-  Módulo  de telas - Agrupamento Básico/Datawhouse /Manutenção - Guias de pagamentos - Copy

## Descricao

Objetivo:  

Criação do novo módulo de "Guia de Pagamento" é integrar a funcionalidade customizada existente do DOOTAX ao sistema Onesource Tax One e Onesource BR, transformando-a em um módulo produtizável. Isso visa otimizar a implementação, garantir a consistência operacional e facilitar a manutenção futura, assegurando que o módulo atenda às necessidades específicas de processamento de Guias de Pagamento. 

 

Proposta Inicial:  

A proposta inicial é reaproveitar a estrutura já existente do DOOTAX, integrando-a a um novo menu no sistema Tax One, utilizando APIs para automação de processos relacionados a Guias de pagamento de tributos. 

 

Regras de negócio / Comportamentos: 

 

- Transformar a funcionalidade customizada em um módulo genérico, sem referências específicas ao cliente original. 
- Renomear tabelas, procedures, views e outros objetos do banco de dados para evitar conflitos. 
- Ajustar chamadas de procedures, tabelas e views para refletir novos nomes. 
- A integração deve permitir a automação de processos de Guias de pagamento via APIs. 
- Implementar funcionalidades como inserção, exclusão, confirmação de registros, geração de relatórios, e envio de guias para pagamento.  Requisitos: MTZ-GUIAS DE PAGAMENTO_INTEGRAÇÃO_DOOTAX _TELAS E MENUS - ICMS PROPRIO.docx
 Reverse_ICMS_NORMAL.docx
 Layout_integracao_txt - Entrada.xlsx
 
 
Jornada do usuário:  

O usuário poderá acessar o novo módulo através do menu no sistema Tax One, onde poderá consultar, inserir, modificar, excluir e enviar guias de pagamento. O usuário também poderá gerar relatórios e visualizar logs de processos. 

 

Maquetes/Protótipos/Ícones: Não há detalhes específicos sobre mockups ou ícones no documento. 

 

Considerações técnicas: 

- Garantir que o módulo seja compatível com o sistema Tax One. 
- Utilizar APIs para integração com a plataforma DOOTAX. 
- Assegurar que a implementação seja limpa e genérica, sem dados do cliente original.  
Dependências: 

- Integração com a plataform
