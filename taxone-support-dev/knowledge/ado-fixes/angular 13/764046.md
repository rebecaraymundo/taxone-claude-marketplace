 Pré-requisito(s): (optional)
 
 Steps:

Evidência da falha: 
 

Foram encontrados diversos problemas para a tela de documento fiscal com a mudança do Angular 13, mesmo que esta tela não esteja passando pelo conversor, foi aberto o bug para análise. 
  
- O botão “fechar” não é renderizado na
versão do Angular 13. 
- O campo de seleção do Estabelecimento
está “travado” no Angular JS, enquanto no Angular 13 o mesmo fica “destravado”.
Ao trocar o Estabelecimento, o filtro considera o outro estabelecimento para
seleção. 
- As opções para os documentos listados
possuem um comportamento diferente entre as versões, onde alguns botões
aparecem em momentos diferentes. A opção “Limpar Filtros” não apareceu na
versão do Angular 13. 
- Na versão do Angular 13, ao clicar em um
filtro aparece uma aba “valores” que não faz nada.

habilitado, mas aparece a mensagem: “É
necessário salvar a capa da nota fiscal antes de editar as demais abas.” Já no Angular 13, o menu fica
desabilitado. 
- A tela não recupera o Estabelecimento no
Angular 13, ficando preenchido “Selecione um”, porém o campo está bloqueado. 
- A opção Tipo de Documento, não fica
habilitada para versão do Angular 13. 
- A opção Modelo de Documento Fiscal, não
fica habilitada para versão do Angular 13. 
- A opção Inscrição Estadual, não retorna
nenhuma informação no Angular 13.
