# [Drogaria São Paulo] TAX ONE - BÁSICOS - JOB SERVIDOR - Criação de SAFX341 para Geração do Registro 1400 Sped Fiscal (QA)

## Descricao

Branch: MFS794593 REQ 794589
 DEV 794591 QA 794593 
 
 As a tax user of the SPED FISCAL module 
 I would like a SAFX file to be made available so that I can upload the information necessary to generate Record 1400 of the Sped Fiscal.  Today the product offers two ways to fulfill the 1400 registration: - Maintenance screen for manually entering this information.  Location: EFD Module - Digital Tax Accounting - Generation menu >> Maintenance >> Record 1400 - Automatic generation of some 1400 codes requested by the states, such as  “RJVAF20002”,  “RJVAF20003” 
 Importing the new SAFX file saves the records in the EFD_REG_1400 table. 
 Existing Functionality :  Grouping:  Basic Module:  Server Job Menu : Load > Execution 
Import > Execution 

Batch Import > Execution 
             Export Data > Execution 
  Additional Details 
 
Creating  SAFX and making available the functionalities of  Loading,  Normal and Batch Import, and Export. 
 

 
  
 
Final Table: EFD_REG_1400    Ticket related to this improvement: https://atendimentotr.zendesk.com/agent/tickets/126698
 
See new guidelines issued by the Technical Evolution Team on 01/08/25: 

ALL current and new SAFXs must have a virtual field called chave_safx<respective numbering> and an index called IX_CHAVE_SAFX<respective numbering>. 

 Example: SAFX07 must have a field called chave_safx07 and an index called IX_CHAVE_SAFX07(chave_safx07).
