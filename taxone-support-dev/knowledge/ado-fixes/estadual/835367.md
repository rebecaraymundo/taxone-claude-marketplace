# TAXONE - Report Fiscal - Ajustar Report REL_DOCTO_ANLTCO_TRANSP_FPROC (QA)

## Descricao

Descrição do Problema Encontrado: Módulo Básicos >> Report Fiscal, menu: Gerenciais >> Documentos Fiscais >> Analíticos >> Conhecimento de Transporte Objeto: REL_DOCTO_ANLTCO_TRANSP_FPROC  Na avaliação do cenário Shopee foi detectado que o relatório deve ser reavaliado.
 
Avaliação do Pessanha: 

Fala pessoal boa tarde, seguinte la no grupo de incidentes o Alceste pegou o problema do relatorio da shopee q demora horrores e tb aumenta CPU : 
 
Issue do Alceste aberta sobre o mesmo tema relacionada no Related, 835122 contem o plano de execução da query 

 

-  ele possui um union de duas queries onde uma acessa a tabela X07 por data_fiscal e a outra nao. Isto gera um full scan, mesmo q nao retorne nenhum registro nesta querie. Antes no produto DW nao possuiamos as tabelas particionadas, porem agora isso no T1 sera o padrao para estas tabelas mais volumosas. Na minha visao este relatorio deveria ser revisado em conjunto com um BA pois com a tabela particionada, tentar fazer um acesso a ela sem usar a chave de particionamento vai aumentar muito o tempo e tb CPU. 
- A shopee gera varias vezes este relatorio ao longo do dia    
 

  

Solução Proposta: 

O "Período" passa a ser critério obrigatório. Hoje ele só é habilitado quando escolhido o critério "Período/Pessoa Fisica ou Jurídica" 

 
Com isso a tela terá as seguintes modificações: 

1) O critério "Período/Pessoa Fisica ou Juridica" será renomeado para "Pessoa Fisica ou Juridica" 

2) O campo Período será reposicionado e ficará depois do Estabelecimento

## Release Notes

Alteramos o relatório Analítico de Conhecimento de Transporte, disponível no menu Básicos > Report Fiscal > Gerenciais > Documentos Fiscais > Analíticos > Conhecimento de Transporte, tornando o campo Período um critério obrigatório para a geração do relatório. Esta mudança visa garantir maior precisão na consulta dos dados
