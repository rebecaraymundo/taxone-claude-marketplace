# [Internal Bug] T1 - SPED - EFD - Escrituração Fiscal Digital - Não indica ao usuário que é necessário preencher COD_ESTADO e não permite gravar registro acusando data igual a de hoje como "nula ou data inválida"

## Passos de Reproducao

Credenciais de acesso para reprodução da falha:
       
- Tennant / Database: T1 QA (tenant VO4)   
- Usuário | Senha: acesso próprio 
- Empresa: 076 
- Estabelecimento: login Todos, ao acessar tela pode ser qualquer um, porém utilizamos o '00' 
- Período: independe  
 Pré-requisito(s): (optional)
 
 Steps: 
- Acessar "EFD - Escrituração Fiscal Digital" sem logar com estabelecimento 
- Navegar até o menu: Parâmetros >> Registro 0200 >> Tributação Interna ICMS - Produto x UF 
- Preencha o campo GRUPO_PRODUTO  
- 02  
- Preencha o campo IND_PRODUTO 
- Acabado: '01' no banco  
- Preencha o campo de COD_PRODUTO  
- PRODUTO 60 POSIÇÕES 0123456789012345678901234567890123456789 
- Campo DESCRICAO será automaticamente preenchido  
- Preencha o campo de DAT_VIGENCIA_INI  
- Data atual do dia que a reprodução está sendo feita  
- Clique em  
- 
 
- Preencha o campo DAT_VIGENCIA_INI 
- Acabado: '01' no banco 
- Imediatamente após selecionar a opção '01', dados do produto são trazidos    
 
 

Evidência da falha: O campo COD_ESTADO é obrigatório e por esse motivo o registro não pode ser gravado, no entanto, em momento algum, o pop-up indica esse ponto ao usuário. 

DAT_VIGENCIA_INI = '30/10/2024' + preenchimento do campo COD_ESTADO 

Registro salvo com sucesso ✅ 

 

 

 

Query: 

select *
  from X100_PARAM_REG75MM
 where cod_produto =
       'PRODUTO 60 POSIÇÕES 0123456789012345678901234567890123456789'
   and cod_estado = 'AP';

 

 

 

 

DAT_VIGENCIA_INI = '30/10/2024' 

Mensagem de erro: 

"900025 - O parametro data vigencia inicial nao pode ser nulo e deve ser uma data valida. Linha de Referência: 001" 

 

 

 
 

 
 
 

DAT_VIGENCIA_INI = '29/10/2024' 

Mensagem de erro: 

"91081 - Nao foi possivel inserir o registro na Tabela. Linha de Referência: 001"
