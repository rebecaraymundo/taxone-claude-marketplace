# TAX ONE - Avaliação Melhorias LOG Processos Datadog - SP12

## Descricao

Avaliar melhorias para servico log Datadog

 Esta melhoria visa corrigir um problema encontrado na rotina de resiliencia.
 A rotina de resiliencia hoje funciona da seguinte forma:
    As mensagens possuem 3 Status que sao: 0 - Qdo sao criadas, 1 - Qdo estao sendo enviadas/consumidas pelo queueControl e 2 - Qdo foram consumidas e o processo terminou. 
   Caso o Servico do queuecontrolproducer e/ou queuecontrol fiquem fora do ar ou sem consumir mensagens por mais de uma hora e existem mensagens que tenham ficado com status = 1, na volta dos servicos nenhuma mensagem sera enviada para o queuecontrol consumir pois considerara q ainda ha uma remessa pendente e é nesse ponto que a resiliencia atua. Nesta situação a rotina verificava a partir da menor data_log se havia passado uma ou mais de uma hora, se sim acionava a resiliencia q nad mais é que alterar o status das mensagens de 1 para 0 novamente, para que no próximo envio fossem pegas. Porém foi observado que se o serviço fica fora muito tempo, a resiliencia é acionada o tempo todo, causando concorrencia entre o envio(colocar com status 1) e a própria resilência(colocar com status 0).
 
 A correção consistiu em criar uma nova data (data_Status) que sera utilizada em conjunto com o campo Status quando ele for <> 0, ou seja, a rotina de resiliência passará a validar o tempo de ausência pela data_status e não mais pela data_log.
 
 
 Cenario de teste:
 
 1) parar a fila queueExportDbLog(necessitará de auxilio de techops);
 2) executar um processo (com isso fará q a mensagem vá para o staus 1), se ja nao tiver mensagens neste status = 1;
 3) acompanhar no Datadog se para o queueControlProducer estao correndo a mensagem "msgservice=NENHUMA MENSAGEM ENVIADA POIS EXISTEM MENSAGENS SENDO CONSUMIDAS. QTDE MSG EM CONSUMO DO DATADOG:";
 4) Se estiver ocorrendo, o dev devera alterar a data_status na tabela_log_process_tenant para uma hora antes da data_log das mensagens que estao com status = 1;
