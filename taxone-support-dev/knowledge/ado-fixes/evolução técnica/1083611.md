# [SNYK] [taxone-ws-carga-safx] Correções de vulnerabilidades

## Descricao

Resumo executado 

Remediação Snyk Open Source (Maven) no repositório taxonecomp_WS-CargaSAFX, branch MFS1083611 → PR GitHub #116 (base dev). Commits com AB#1083611 para vínculo automático Boards ↔ GitHub. 

Scan Snyk (MCP) — antes 

Na rc com spring-boot-starter-parent 3.5.7: 20 achados (CVEs em Spring/Tomcat/Netty/Logback/Jackson/commons-lang3, transitivas commons-configuration/commons-lang, licenças LGPL). 

O que foi alterado no pom.xml (entregue no PR) 

- spring-boot-starter-parent 3.5.7 → 3.5.12 — eleva BOM (Spring Web/MVC, Security, Actuator, Tomcat embed, Logback, Jackson geridos pelo Boot, etc.). 
- commons-lang3 via propriedade commons-lang3.version = 3.18.0 — mitigação CVE-2025-48924 (Snyk). 
- org.json:json (json.version) 20231013 → 20251224. 
- net.minidev:json-smart 2.5.2 → 2.6.0. 
- io.netty:netty-handler — removida versão explícita (4.2.0.Final); dependência passa a seguir o BOM do Spring Boot. 
- Versão do artefato taxone-ws-carga-safx 7.0.2 → 7.1.0 (semver: minor por json-smart 2.5→2.6, alinhado à skill).  
Build 

mvn clean install executado com sucesso após as alterações. 

Scan Snyk (MCP) — depois do PR 

6 achados restantes (não corrigidos neste PR): 

- jackson-core 2.19.4 — CWE-770 (high); Snyk sugere linha 2.21.1; requer avaliação com BOM Spring Boot 3.5.12 antes de forçar jackson-bom. 
- commons-configuration 1.10 — CVE-2025-46392 (medium), transitiva (ex.: cadeia OWASP ESAPI). 
- commons-lang 2.6 — CVE-2025-48924 (high), transitiva (mesmo contexto). 
- Licenças LGPL-2.1 (low): hibernate-commons-annotations, hibernate-core, xom — política de licenciamento, não upgrade de versão único.  
Próximos passos sugeridos 

- Subir/atualizar ESAPI ou gerir transitivas (exclusão + coordenadas seguras) para commons-configuration / commons-lang 2.x. 
- Avaliar upgrade coordenado Jackson 2.21.x com matriz Spring Boot. 
- Tratar achados de licença via processo interno se necessário.
