# TAX ONE - Retorno alterado para API ConsultaLote - 367

## Descricao

Bom dia, 
 Após o incidente ocorrido na semana retrasado, o retorno dos erros ao consultar via API passou a vir de forma diferente, não deixando claro o problema relacionado ao registro, como segue abaixo:

Retorno anterior, com descrição do campo incorreto: 

6 | COD_DOCTO | Campo obrigatório deve ser preenchido. Favor verificar layout. 

Retorno atual:  

6 | Campo obrigatório deve ser preenchido. Favor verificar o layout. | 007-001-20190221-9---2-04622116001195-6-@--R002-@-@--1- 
Alguns clientes nos procuraram com dúvida sobre o que se tratava a mensagem e como identificar qual o campo incorreto. Podem verificar por favor a possibilidade de identificar o campo não preenchido (em verde) ao invés da linha (em laranja)?
