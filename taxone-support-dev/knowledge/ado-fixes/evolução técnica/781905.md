# TAX ONE - Aplicar ajuste de controle CPU na TENANT_MGMT

## Descricao

Compartilhado pelo @Siqueira Junior, Alceste (TR Technology & Ops) nova versão da query para precisão na captura do percentual da CPU de controle de resiliência dos serviços  
 "Por alguma razão, talvez bug, a view da oracle está computando mais de 60 segundos por minuto, ocorre em situações adversas, fiz uma query para quando ocorrer essa situação, retornar o valor de 50%" WITH
    FUNCTION f_double
        RETURN NUMBER
    IS
        y   NUMBER (10, 2);
    BEGIN
        SELECT MAX (VALUE)
          INTO y
          FROM v$parameter
         WHERE name = 'cpu_count';
 
        RETURN y;
    END;
SELECT VALUE,
       end_time,
       Begin_time,
       CASE
           WHEN (VALUE > (f_double * 60)) THEN 50
           ELSE TRUNC (VALUE / ((f_double) * 60) * 100)
       END
  FROM v$con_sysmetric_history a
WHERE     metric_name = 'CPU Usage Per Sec'
       AND GROUP_ID = 18
       AND Begin_time >= SYSDATE - 3 / 24 / 60;
