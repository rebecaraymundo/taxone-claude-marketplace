# TAXONE CargaSAFX- Melhorias "Data Arquivo" - Fase 3 - SP04/05

## Descricao

- Aplicar a mesma solucao do obi-consumer no cargasafx implementada na issue #952399  

- limitar o range da data inicial e data final da importação não somente pelos parâmetros do lote, mas também pelo conteúdo do arquivo    Fazer isso só para as SAFX07, SAFX08 e SAFX09, para as demais pode continuar como já está hoje.
Ler o período que veio na mensagem (json)
 Guardar a menor e maior data por safx, independente da quantidade de arquivos
 Se a menor data da safx, for maior que o período inicial da mensagem
   Então usar a menor data da safx
   Senão, usar o periodo inicial da mensagem
 Se a maior data da safx, for menor que o período final da mensagem
   Então usar a maior data da safx
   Senão, usar o período final da mensagem
   
 Observação:
 Para a SAFX08 e SAFX09 usar a DATA_FISCAL (campo 3 do arquivo)
 Para a SAFX07, tem que usar a regra abaixo:
 	Se o MOVTO_E_S (campo 3 do arquivo) for '9' ou
 	   DATA_SAIDA_REC (campo 20 do arquivo) for (null, '00000000', '@')
 	   Então usar DATA_EMISSAO (campo 11 do arquivo)
 	Se o MOVTO_E_S (campo 3 do arquivo) for diferente de '9'
 	   Então usar DATA_SAIDA_REC (campo 20 do arquivo) 
  As datas salvas nos campos data_ini e data_fim das tabelas protocolo_carga e det_job_import  (após iniciar a importação). 

- trocar pra chamar diretamente as sequences das respectivas tabelas
