# TAXONE - Correção na recuperação dos TENANT_CODE para gravação dos LOGs dos objetos de BD - 366

## Descricao

TAXONE - Foi identificado no ambiente de PROD03 que o schema do suporte(user_taxone_suporte) esta associado a um tenant_code iniciado por (GCR), desta forma a query não estava considerando este tenant_code e retornando erro para qualquer geração efetuada neste banco.
 
 
 
 Motivo:  Correção de situação Ambiente TR 
  Resultado Esperado:  Apoio e avaliação e fix. 
 O resultado esperado é para o schema que tenha na tabela tenant_connection.connection_config apenas um tenant_code associado, este devera ser usado para gravação do log na tabela log_process_tenant. Caso o schema possua mais de um tenant_code associado, pelo padrao de cadastro que Techops adota deveremos desconsiderar todos os tenants_code que iniciam por Q8M (em ambiente AMERS) ou GCR (em ambiente EMEA), se ainda assim retornarem mais de um tenant o erro acima indicado deverá continuar ocorrer.  
           Para garantir o funcionamento ainda no ambiente de QA, executar uma geracao de Equalizacao de Datamart no schema OCLOUD e no schema da EY, esta geração deverá ocorrer sem erros. a partir do DashBoard da FILA-EXEC(TAXONE [ALL] FILA-EXEC FLOW | Datadog) sera possivel verificar os logs criados, e deverao estar associados respectivamente aos tenants (VO4 e VV3_001).
