# [PenTest2023][Low][TaxOne] 36184:3.13 - CORS misconfiguration - Parte4 - 351

## Descricao

36184:3.13: The application does not validate cross origin resource sharing requests from another sub origin based on a allow list. Access is granted to any arbitrary sub domain based on the origin header. Cross origin resource sharing is a mechanism that allows restricted resources on web page to be requested from another domain outside the domain for which the resources originated.
 
 
- A malicious user logs in to the application and access search user functionality and triggers a request. 
- The user intercepts the request and modifies the Origin header to include a malicious origin. 
- Note that the response header now reflects the presence of the malicious origin in the Access-Control-Allow-Origin field and for some endpoints the value of Access-Control-Allow-Origin is set to wildcard (*) value

(https://uat.onesourcetax.com/uat/useradmin-service/api/Users/SearchUsers?migrationCompatibilityMode=true)
