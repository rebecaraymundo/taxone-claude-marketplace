# [PROD] - Camel SFTP

## Descricao

Foi identificado um fluxo do camel onde ao enviar arquivo com um nome especifico, o camel crio uma pasta como o nome tinha um caracter especial deu erro:
Cenario:

1) Pelo SFTP utilizando o usuário INTG, envio o arquivo com o seguinte nome 130208_20240531__+de1docnum.txt 2) Quando o arquivo tem __ no meio do nome do arquivo, o camel acha que o que vem depois é o nome do usuário, com isso ele crio a pasta +de1docnum e moveu o arquivo 130208_20240531.txt e quando o camel tento retornar o arquivo para o SFTP, como tem um caracter especial começou a dar erro.

Deve ser avaliado como resolver isso:
Bloqueia arquivo e move para rejeitado.
Renomeia o arquivo e segue o fluxo.
