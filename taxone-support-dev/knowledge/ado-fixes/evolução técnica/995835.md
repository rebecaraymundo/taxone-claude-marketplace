# [UAT02] - Serviço REINF-PRODUCER

## Descricao

Foi identificado um loop de mensagem na fila dos eventos do REINF.

Como mostra na imagem um cliente tem uma consulta que leva 20s para concluir, com a consulta do tenant do suporte que acessa a mesma base são mais 20s.

Com isso um cliclo de do serviço esta levando 40s, e a cada 30s ele coloca todos os tenants para essa validação.

Dessa forma a fila nunca fila limpa e sempre aumenta, deve ser analisado uma forma de não entrar em Loop, para mitigar foi colocado 2m para um novo ciclo.

Impactos que podem causar.

Lentidão no envio de eventos dos clientes devido a essa demora para retornar a consulta e pegar um novo tenant para validar:

Evento foi gerado dia 02/12/2025 as 10h e só foi enviado 03/12/2025 as 17h como mostra na imagem.
