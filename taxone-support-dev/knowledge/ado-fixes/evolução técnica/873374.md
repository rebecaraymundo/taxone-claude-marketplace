# TAXONE - [Deploy] - Extrator API - Avaliacao Consumo Mensagens - SP16

## Descricao

Avaliar consumo de mensagens do intergrador API  
- consumo de cada mensagem por consumers diferetente  Foram avaliados vários pontos e em conversa com o time foi decido que a melhor estratégia era criar um novo serviço que lê mensagens da fila do RabbitMQ assim que chegam e enviam para outra fila no ActiveMQ, pois no ActiveMQ conseguimos realizar melhor o controle das mensagens, coisa que no RabbitMQ não funciona muito bem. Portanto, o extrator agora passou a ler mensagens do ActiveMQ e devolve o json de retorno na fila do RabbitMQ como já fazia.
