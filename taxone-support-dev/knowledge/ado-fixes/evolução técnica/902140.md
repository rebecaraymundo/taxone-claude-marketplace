# TAXONE - Teste e Deploy - Ajustar unificação serviço CargaSAFX - SP19

## Descricao

Durante a janela de implantação da aplicação camel-consumer unificado foi identificado um estouro no número de conexões com o banco, fazendo com que o revert da versão fosse realizado. 
 Identificou-se que o problema é ocasionado quando tenta-se realizar carga de arquivos que já não existem mais. 
 Tal comportamento também ocorre na última versão anterior a unificação. Logo, trata-se de um bug pré-existente.    
 Motivo: Impedir o estouro de conexões no banco de dados quando o consumer tentar realizar carga de arquivo inexistente. 
 Resultado Esperado:
    
 
  
 
 Não ocorrer variação no número de conexões com o banco de dados.
