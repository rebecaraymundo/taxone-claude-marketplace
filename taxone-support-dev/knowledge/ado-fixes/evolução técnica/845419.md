# TAXONE -TDM Implementação Novos Campos Extras Cliente Piloto - Fase 1

## Descricao

Objetivo:
Garantir a completa aderência da solução TDM à necessidade do cliente piloto Stellantis, por meio da criação e disponibilização dos 4  como pendentes. durante a análise dos 67 campos solicitados, sendo que 40 campos já foram entregues ao cliente, anteriormente. 

Justificativa:
Estes campos, inicialmente identificados na minha análise como duplicados, foram posteriormente confirmados como necessários após alinhamento com o cliente. Portanto, a inclusão destes campos é fundamental para fechamos o piloto TDM com o cliente. 

Proposta:
Criação e disponibilização dos 4 campos a seguir: 

- COD_FISJUR_LSG ( API - Capa de Documento Fiscal) 
- COD_SITUACAO_PIS_ST (API - Itens de Mercadorias)  
- COD_SITUACAO_COFINS_ST (API - Itens de Mercadorias)  
- COD_NATUREZA_OP (API - Itens de Serviços)   Anexo os Dicionário de dados a seguir: 
-  Capa de Documento Fiscal 
- Itens de  Mercadorias 
- Itens de Serviços
