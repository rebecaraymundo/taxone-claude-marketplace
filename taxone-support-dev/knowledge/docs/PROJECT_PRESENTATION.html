Visao Geral do Projeto

O que e o TAX ONE Support Dev?

Uma plataforma de orquestracao multi-agente que automatiza o ciclo completo de suporte ao sistema fiscal TAX ONE (Thomson Reuters / Mastersaf). Desde a triagem de bugs no Azure DevOps ate a criacao de Pull Requests no GitHub, passando por analise de impacto, implementacao, revisao de codigo, testes automatizados e planejamento de sprints.

O sistema atua sobre um ERP fiscal com 4.028 tabelas Oracle, 66.067 colunas, 5.425 procedures/packages PL/SQL e cobre todos os modulos fiscais brasileiros (ICMS, IPI, PIS/COFINS, ISS, SPED, eSocial, REINF, ECD, etc).

Principios de Design

  • Knowledge-First: Todo agente carrega contexto do KB antes de agir
  • Separation of Concerns: Agentes com escopos definidos (read-only vs read-write)
  • Backwards-Compatible: Fallback gracioso quando artefatos nao existem
  • Human-in-the-Loop: Confirmacao obrigatoria em acoes destrutivas
  • Compound Engineering: Toda solucao gera conhecimento reutilizavel

Regras de Operacao Criticas

Regras de Seguranca

  • PL/SQL: ISO-8859-1 + CRLF (binary edit, nunca Edit tool)
  • DDL: NUNCA alterar existente (criar nova com WI_ID)
  • Compilar PL/SQL ANTES de testes (porta 1521)
  • tests/ NUNCA vai para Git (somente ADO Attachments)
  • Confirmar antes de: branch, commit, push, PR, ADO update
