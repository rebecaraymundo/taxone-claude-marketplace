# NFSE ABACO Manaus - Natureza operacao gera 16 ao inves de 19

## Root Cause

Na funcao `setDadosItem` do package `MUN_NFSEABACO_GERA_TOMAD`, a logica de determinacao
da natureza de operacao para o layout ABACO nao tratava corretamente o cenario de
prestador enquadrado no **Simples Nacional** com **ISS nao retido** em **Manaus**.

O sistema gerava o codigo `16` (generico) quando deveria gerar `19` (Simples Nacional
sem retencao em Manaus), conforme regras do layout ABACO.

## Solucao

Adicionada condicao especifica na funcao `setDadosItem` para verificar:
- Municipio = Manaus
- Prestador Simples Nacional
- ISS nao retido

Quando todas as condicoes sao atendidas, a NATUREZA_OPERACAO e setada como '19'.

## Arquivos Modificados

1. `artifacts/sp/msaf/municip/munic/MUN_NFSEABACO_GERA_TOMAD.fon` (+13/-2)
   - Funcao `setDadosItem`: adicionada condicao para cenario Manaus + Simples + ISS nao retido

## Licoes Aprendidas

1. **Layouts municipais ABACO**: Cada municipio pode ter codigos de natureza de operacao
   especificos. Ao corrigir bugs de natureza, consultar a documentacao do layout ABACO
   do municipio especifico.

2. **Cenarios combinados**: Bugs de natureza de operacao geralmente envolvem combinacoes
   especificas de regime tributario (Simples/Lucro Real/Presumido) + tipo retencao + municipio.
   Testar todas as combinacoes relevantes.
