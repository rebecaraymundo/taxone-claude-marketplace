# WI 1058914 - Reemissao IPI nao encontra livros no TaxOne

## Problema

A tela **Federal > IPI > DATA MART > Reemissao do Livro Fiscal** no TaxOne busca dados na tabela `ict_lvr_reimp`, mas o fluxo de **emissao** do livro IPI (P8) grava apenas na tabela `lvr_reimp` (DW). Resultado: reemissao retorna "nao existe nenhum livro gravado".

### Fluxo quebrado

```
Emissao P8 (PB) → SAF_IPI_LIVRO8 → dados em ICT_LVR
               → gravar_arquivo_grp → gravar_lvr_reimp → UPDATE lvr_reimp (DW) ✓

Reemissao TaxOne (PB m_reimp_livros_ipi_t1) → SELECT ict_lvr_reimp → VAZIO ✗
```

### Causa raiz

- `pkg_grava_relatorio_dwi.gravar_lvr_reimp` apenas fazia UPDATE em `lvr_reimp`
- Nenhum INSERT em `ict_lvr_reimp` era feito durante a emissao IPI
- A reemissao TaxOne (diferente do DW) busca EXCLUSIVAMENTE em `ict_lvr_reimp`

### Modelo de referencia

WI 585367 (PRODEPE) resolveu o mesmo problema para livros estaduais:
- PB faz INSERT direto em `ict_lvr_reimp`/`ict_lvr_reimp_rel` durante emissao
- `pkg_grava_relatorio_dwi` foi alterado (+138 linhas) para suportar o fluxo

## Solucao implementada

### 1. Sequence `SEQ_ICT_LVR_REIMP` (DDL)

**Arquivo**: `artifacts/bd/V2R010/DDL_MFS1058914.sql`

A sequence nao existia no banco. Criada com START WITH = MAX(ident_reimp) + 1 = 19445 para evitar conflito com dados PRODEPE existentes (1940 registros).

### 2. Alteracao `pkg_grava_relatorio_dwi.gravar_lvr_reimp` (PL/SQL)

**Arquivo**: `artifacts/sp/pkg_grava_relatorio_dwi_body_MFS1058914.sql`

Logica adicionada apos o UPDATE existente em `lvr_reimp` (linhas 928-984):

1. **Condicao**: Apenas quando `p_insc_estadual IS NULL` (fluxo federal/IPI generico)
2. **SELECT metadados** de `lvr_reimp` via `proc_id` (empresa, estab, tipo_livro, periodo)
3. **Verificacao duplicata**: Se ja existe registro em `ict_lvr_reimp` para mesmo livro/periodo, DELETE das filhas (`ict_lvr_reimp_rel`) e pai antes de re-inserir
4. **INSERT em `ict_lvr_reimp`**: Metadados do livro com novo `IDENT_REIMP` via sequence
5. **INSERT em `ict_lvr_reimp_rel`**: BLOB do relatorio com `SEQ_REL=1`
6. **Exception handler**: Loga erros sem interromper o fluxo principal (UPDATE em lvr_reimp ja executou)

### Indice unico respeitado

`IX_ICT_LVR_REIMP` (UNIQUE): (COD_EMPRESA, COD_ESTAB, COD_TIPO_LIVRO, DATA_PER_INI, DATA_PER_FIM, SERIE_LIVRO, SUB_SERIE_LIVRO, INSC_ESTADUAL)

A logica de DELETE + INSERT evita violacao de PK/UK em reemissoes subsequentes.

### 3. Nenhuma alteracao PB necessaria para o fluxo basico

O PB ja chama `gravar_lvr_reimp` durante a emissao. A alteracao no PL/SQL e transparente.

**Nota para dev PB**: Se for necessario popular `LST_REL` (lista de DataWindows) e `DATAWINDOW` na `ict_lvr_reimp_rel` para o fluxo completo de reemissao PSR/PSRX, sera preciso ajustar `m_arg_livros_emissao.srm` para passar esses dados. O fluxo basico (encontrar o livro + download do BLOB) funciona sem isso.

## Tabelas afetadas

| Tabela | Acao |
|--------|------|
| `ict_lvr_reimp` | INSERT (novo) |
| `ict_lvr_reimp_rel` | INSERT (novo) |
| `lvr_reimp` | UPDATE (existente, mantido) |

## Regressao

- **ICMS/generico**: `p_insc_estadual IS NULL` → mesma condicao do UPDATE em `lvr_reimp`. Nao afeta `lvr_reimp_am`
- **PRODEPE**: PB faz INSERT direto, nao usa `gravar_lvr_reimp`. Sem impacto
- **AM (Amazonas)**: `p_insc_estadual IS NOT NULL` → bloco novo nao executa
