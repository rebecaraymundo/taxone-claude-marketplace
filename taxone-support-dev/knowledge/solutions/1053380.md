# Balancete XLSX situacao especial - saldo anterior zerado em periodo parcial

## Root Cause

No package `REL_BALANCETE_EXCEL_FPROC` (REL_BALANCETE_EXCEL_FPROC.pck), as subqueries de saldo
inicial e final assumiam periodos mensais completos usando `LAST_DAY()`:

```sql
-- Saldo inicial (ERRADO):
AND x02a.DATA_SALDO = LAst_day(pDataIni)

-- Saldo final (ERRADO):
AND x02a.DATA_SALDO = LAst_day(Pdatafim)
```

Em situacao especial (periodo parcial, ex: 01/03 a 16/03), `LAST_DAY(pDataIni)` retornava 31/03,
uma data sem registro de saldo na X02_SALDOS, resultando em saldo anterior zerado.

Adicionalmente, no cursor `C_Calc_Conta_Consolidada`, a expressao CASE WHEN:
```sql
CASE WHEN x02.data_saldo BETWEEN Pdataini AND last_day(Pdataini)
```
limitava o filtro ao primeiro mes do periodo, causando zeragem em periodos anuais tambem (WI #1062301).

## Solucao

1. **Saldo inicial**: Substituido `LAST_DAY(pDataIni)` por subquery `MIN(DATA_SALDO)` dentro
   do periodo, capturando o saldo mais proximo disponivel
2. **Saldo final**: Substituido `LAST_DAY(Pdatafim)` por `Pdatafim` diretamente
3. **Consolidado**: Substituido `last_day(Pdataini)` por `Pdatafim` no CASE WHEN
4. Correcao aplicada em 4 secoes: X02_SALDOS e X80_SALDOS_CCUSTO (package spec e body)

## Licoes Aprendidas

1. **LAST_DAY() assume meses completos**: Em situacao especial (periodo parcial), `LAST_DAY()`
   aponta para uma data que pode nao ter registro de saldo. Usar a data real ou MIN() dentro
   do periodo.

2. **C_Calc_Conta_Sint vs C_Calc_Conta_Consolidada**: O package tem 2 cursores para cenarios
   diferentes (sintetico vs consolidado). Bugs em um podem nao existir no outro. Verificar ambos.

3. **Mesmo package, multiplos WIs**: `REL_BALANCETE_EXCEL_FPROC` e usado para Balancete XLSX
   em todos os cenarios. Um fix pode resolver multiplos WIs (neste caso, #1053380 e #1062301).
