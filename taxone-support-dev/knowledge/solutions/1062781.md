# Performance SAF_RF_NF_ICMS_DIFAL - Report Fiscal DIFAL

## Root Cause

A procedure SAF_RF_NF_ICMS_DIFAL (s_rf_nf_icms_difal.sql) apresentava degradacao de
performance em ambientes com alto volume de dados (Magazine Luiza). Cinco anti-patterns
de performance foram identificados:

1. **SUBSTR em coluna indexada**: `substr(X2012.COD_CFO,1,1) in ('2','6')` impedia
   uso do indice em X2012_COD_FISCAL
2. **NVL em colunas filtradas**: `NVL(CAPA.SITUACAO,'N') <> 'S'` e
   `nvl(CAPA.Ind_Origem_Info,' ') <> '1'` forcavam full scan por aplicar funcao
3. **DELETE row-by-row via cursor**: Loop `FOR C2_DEL IN (SELECT ROWID...)` com
   DELETE individual e SAF_NUM_OP_COMMIT a cada linha
4. **INSERT row-by-row (FORALL comentado)**: FORALL original foi comentado e
   substituido por WHILE loop com INSERT individual
5. **Indice sem coluna USUARIO**: IX_ITEM_APURAC_DIFAL_NF_01 nao incluia USUARIO,
   forcando full scan no DELETE por sessao

## Solucao

### Fix 1: SUBSTR → LIKE (linhas 65, 121)
```sql
-- ANTES:
substr(X2012.COD_CFO,1,1) in ('2','6')

-- DEPOIS:
(X2012.COD_CFO LIKE '2%' OR X2012.COD_CFO LIKE '6%')
```
LIKE com prefixo permite range scan no indice. SUBSTR aplica funcao em cada linha.

### Fix 2: NVL SITUACAO → IS NULL OR (linhas 57, 115)
```sql
-- ANTES:
NVL(CAPA.SITUACAO,'N') <> 'S'

-- DEPOIS:
(CAPA.SITUACAO IS NULL OR CAPA.SITUACAO <> 'S')
```

### Fix 3: NVL Ind_Origem_Info → IS NULL OR (linhas 58, 116)
```sql
-- ANTES:
nvl(CAPA.Ind_Origem_Info,' ') <> '1'

-- DEPOIS:
(CAPA.Ind_Origem_Info IS NULL OR CAPA.Ind_Origem_Info <> '1')
```

### Fix 4: DELETE cursor → DELETE unico (linhas 169-182)
```sql
-- ANTES:
FOR C2_DEL IN (SELECT ROWID FROM ITEM_APURAC_DIFAL_NF WHERE ...) LOOP
   DELETE FROM ITEM_APURAC_DIFAL_NF WHERE ROWID = C2_DEL.ROWID;
   SAF_NUM_OP_COMMIT(CONT_COMMIT_W);
END LOOP;
COMMIT;

-- DEPOIS:
BEGIN
   DELETE FROM ITEM_APURAC_DIFAL_NF
    WHERE COD_EMPRESA = P_COD_EMPRESA
      AND COD_ESTAB   = P_COD_ESTAB
      AND DATA_INI BETWEEN P_DATA_INI AND P_DATA_FIM
      AND USUARIO     = P_USUARIO;
   COMMIT;
EXCEPTION
   WHEN OTHERS THEN ...
END;
```

### Fix 5: WHILE INSERT → FORALL SAVE EXCEPTIONS (linhas 206-279)
```sql
-- ANTES:
I := rGerRel.FIRST;
F := rGerRel.LAST;
While i <= f loop
  BEGIN
    INSERT INTO ITEM_APURAC_DIFAL_NF (...) VALUES (rGerRel(I)...);
  EXCEPTION WHEN OTHERS THEN ...
  END;
  I := rGerRel.next(i);
end loop;

-- DEPOIS:
BEGIN
  FORALL I IN rGerRel.FIRST .. rGerRel.LAST SAVE EXCEPTIONS
    INSERT INTO ITEM_APURAC_DIFAL_NF (...) VALUES (rGerRel(I)...);
EXCEPTION
  WHEN bulk_errors THEN
    FOR j IN 1..SQL%BULK_EXCEPTIONS.COUNT LOOP
      -- log cada erro individual com chave do registro
    END LOOP;
  WHEN OTHERS THEN ...
END;
COMMIT;
```

### Fix 6: DDL_MFS1062781.sql — Indice com USUARIO
```sql
DROP INDEX IX_ITEM_APURAC_DIFAL_NF_01;
CREATE INDEX IX_ITEM_APURAC_DIFAL_NF_01 ON ITEM_APURAC_DIFAL_NF
  (COD_EMPRESA, COD_ESTAB, NUM_PROCESSO, USUARIO);
```

## Licoes Aprendidas

1. **SUBSTR em WHERE impede indice**: Sempre usar LIKE com prefixo em vez de
   SUBSTR(col,1,N) para filtros por inicio de string. LIKE 'X%' faz range scan.

2. **NVL em WHERE impede indice**: Substituir `NVL(col,'default') <> 'valor'` por
   `(col IS NULL OR col <> 'valor')`. O otimizador consegue usar indice na segunda forma.

3. **DELETE em batch e mais eficiente**: Um unico DELETE com WHERE e ordens de
   magnitude mais rapido que cursor row-by-row com COMMIT intermediario.

4. **FORALL com SAVE EXCEPTIONS**: Permite bulk insert mantendo tratamento de erros
   individual. O handler bulk_errors itera SQL%BULK_EXCEPTIONS para logar cada falha.
   COMMIT deve ficar FORA do BEGIN/EXCEPTION/END para executar em ambos os caminhos.

5. **Indice deve cobrir todas as colunas do WHERE**: O DELETE filtra por USUARIO mas
   o indice original nao incluia essa coluna. Recriar com coluna adicional.

6. **DDL imutavel**: Nunca alterar DDL existente (DDL_MFS759431). Criar nova DDL
   com ID da WI atual (DDL_MFS1062781).

7. **FORALL nao aceita EXCEPTION direto**: O FORALL precisa estar dentro de um bloco
   BEGIN...EXCEPTION...END explicito para capturar bulk_errors.
