# DMS Manaus - Filtrar Prestadores Locais

## Root Cause

A procedure DMS_MANAUS_GERA_PREST.pck processava todos os prestadores retornados pelo cursor
C_SERV_EMIT_PRESTADOS (PKG_MMAG_ISSQN), sem verificar se o prestador é do mesmo município
do estabelecimento. Prestadores locais (COD_MUNICIPIO_PFJ = COD_MUNICIPIO_ESTAB) já são
escriturados automaticamente pela prefeitura de Manaus, gerando duplicidade na declaração.

## Solução

Adicionado filtro após a verificação de recolhimento_emit_prestador (linhas 406-410):

```sql
-- MFS1071002: Desconsiderar notas de prestadores do mesmo município do estabelecimento
-- Prestadores locais já são escriturados automaticamente pela prefeitura
If rGrupo.COD_MUNICIPIO_PFJ = rGrupo.COD_MUNICIPIO_ESTAB Then
   raise IgnorarRegistro;
End If;
```

O cursor C_SERV_EMIT_PRESTADOS já retorna ambos os campos:
- `COD_MUNICIPIO_PFJ` (linha 229 do PKG_MMAG_ISSQN): município do prestador (X04.COD_MUNICIPIO)
- `COD_MUNICIPIO_ESTAB` (linha 246): município do estabelecimento (E.COD_MUNICIPIO)

## Lições Aprendidas

1. **Cursor já fornecia dados**: COD_MUNICIPIO_PFJ e COD_MUNICIPIO_ESTAB já estavam no cursor,
   bastava adicionar a comparação na procedure consumidora.

2. **Filtros em cascade**: A procedure já tinha pattern de IgnorarRegistro para outras condições
   (recolhimento_emit_prestador). O novo filtro segue o mesmo pattern, adicionado logo após.

3. **Regra municipal**: Prestadores locais são escriturados pela prefeitura automaticamente.
   Incluí-los no DMS gera duplicidade e potencial rejeição do arquivo.
