# SAVEGNAGO - Produto Geral Tributacao Interna NullPointerException

## Root Cause

Na tela w_prt_x100_param_reg75mm (Parametrizacao Reg 0200 - Tributacao Interna),
evento `ue_grava_reg`, quando o usuario selecionava "Produto Geral" (ind_produto='*',
cod_produto='*'), a variavel `ldt_data_prod` nunca era inicializada.

Na linha 370, `String(NULL, 'yyyymmdd')` causava NullPointerException no runtime
Appeon Web (PowerBuilder compilado para Java/Web).

Fluxo do bug:
1. Usuario seleciona Produto Geral (ind_produto='*')
2. Bloco ELSE na linha 339 seta ls_cod_estab=' ', mas NAO ldt_data_prod
3. Linha 370: `String(ldt_data_prod, 'yyyymmdd')` → String(NULL, ...) → NPE

## Solucao

Adicionar inicializacao `ldt_data_prod = ldt_data_ini` no bloco Produto Geral
(linha 340), consistente com o fallback server-side em ix100.sql (linhas 179-180:
`IF p_data_produto IS NULL THEN usar p_dt_ini`).

```powerscript
if ls_ind_produto = "*" and ls_cod_produto = "*" then
    ls_cod_estab = ' '
    ldt_data_prod = ldt_data_ini    // FIX: previne NULL datetime
else
```

## Licoes Aprendidas

1. **String(NULL, format) em Appeon**: PowerBuilder desktop trata String(NULL) como
   string vazia. Appeon Web (Java runtime) lanca NullPointerException. Sempre
   inicializar datetime antes de String().

2. **Produto Geral como caso especial**: Quando ind_produto='*', o fluxo pula
   a busca de datas do produto. Variaveis dependentes desse fluxo ficam NULL.

3. **Verificar consistencia PB↔PL/SQL**: O server-side (ix100.sql) ja tinha
   fallback para NULL. O bug era apenas no client-side (PB/Appeon).

4. **Heranca PB**: Janelas filhas herdam eventos do pai. Corrigir no pai
   propaga para todos os filhos — nao duplicar fix.
