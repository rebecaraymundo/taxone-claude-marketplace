# SAFX3009 - Procedures desatualizadas causando erro de Codigo de Servico

## Root Cause

Procedures de importacao SAFX3009 (OL_IMP_X3009, SAF_IB_X3009, SAF_IMP_X3009, PKG_IMP_X3009)
desatualizadas no banco do cliente apos aplicacao do DDL DDL_MFS848597.

O DDL adicionou os parametros P_ALIQ_TRIB_REG_IBS_MUN e P_VLR_TRIB_REG_TRIB_IBS_MUN a assinatura
de OL_IMP_X3009 (posicoes 49-50). Com a versao antiga compilada no banco, a chamada posicional
de SAF_IB_X3009 e SAF_IMP_X3009 passa os argumentos nas posicoes erradas, causando desalinhamento.

O erro exibido ao usuario — "Codigo do Servico nao cadastrado na Tabela de Codigos de Servico
(SAFX2018)" — ocorre porque o campo que deveria receber o codigo do servico recebe outro valor
(ou NULL), e a validacao subsequente falha mesmo que o codigo exista corretamente na SAFX2018.

**Codigo-fonte no repositorio: 100% CORRETO** em todos os fluxos de importacao.

**Mesmo padrao da WI 1066543** (SAFX3007/DDL_MFS848597 — mapeamento UF/MUN invertido).

## Solucao

Recompilacao bottom-up das procedures X3009 no banco do cliente:

```sql
-- Passo 1: Procedure folha (core INSERT/UPDATE)
ALTER PROCEDURE OL_IMP_X3009 COMPILE;

-- Passo 2: Procedures de importacao
ALTER PROCEDURE SAF_IMP_X3009 COMPILE;
ALTER PROCEDURE SAF_IB_X3009 COMPILE;

-- Passo 3: Package import MERGE
ALTER PACKAGE PKG_IMP_X3009 COMPILE;
ALTER PACKAGE PKG_IMP_X3009 COMPILE BODY;

-- Passo 4: Orquestrador DW
ALTER PACKAGE PKG_TMP_DOCTO_FISCAL_FPROC COMPILE;
ALTER PACKAGE PKG_TMP_DOCTO_FISCAL_FPROC COMPILE BODY;

-- Passo 5: Validar — nenhuma linha deve retornar
SELECT object_name, object_type, status
FROM user_objects
WHERE object_name IN (
    'OL_IMP_X3009',
    'SAF_IMP_X3009',
    'SAF_IB_X3009',
    'PKG_IMP_X3009',
    'PKG_TMP_DOCTO_FISCAL_FPROC'
)
AND status = 'INVALID';
```

## Diferenca em relacao ao X3007 (WI 1066543)

| Aspecto | SAFX3007 | SAFX3009 |
|---------|----------|----------|
| SAF_IB usa | Leitura posicional de arquivo texto (SUBSTR via SAF_IB_PROC_LINHA) | SAFX3009%ROWTYPE por nome de campo |
| Vulnerabilidade SAF_IB | **VULNERAVEL** a desalinhamento de dados do arquivo | **SEGURO** contra desalinhamento de dados |
| Ponto de falha | SAF_IB_X3007 (dados do arquivo) | Assinatura de OL_IMP_X3009 (chamada posicional) |
| Sintoma | Valores UF/MUN invertidos | Erro "Codigo de Servico nao cadastrado" |

A protecao via `SAFX3009%ROWTYPE` (acesso por nome de campo) imuniza SAF_IB_X3009 contra
desalinhamento de dados do arquivo, mas nao protege contra desalinhamento na assinatura da
procedure que ela chama (OL_IMP_X3009). O problema e um nivel acima.

## Hipotese Alternativa (se recompilacao nao resolver)

Se apos a recompilacao o erro persistir, investigar a funcao SAF_PEGA_GRUPO com modulo 2014:
- GRUPO_SERVICO incorreto pode causar falha na validacao SAFX2018 independentemente do desalinhamento
- Verificar se o codigo de servico esta associado ao grupo correto na configuracao tributaria do cliente

## Licoes Aprendidas

1. **Familia X3007/X3008/X3009 compartilha DDL_MFS848597**: Se uma safx falhar apos o DDL,
   verificar imediatamente as demais da mesma familia — todas podem estar com procedures desatualizadas.

2. **OL_IMP_X3009 tem 71+ parametros posicionais**: Qualquer mudanca de assinatura (adicao,
   remocao ou reordenacao de parametros) quebra todos os callers que usam chamada posicional.
   Procedures com 50+ parametros sao especialmente frageis — recompilacao obrigatoria apos DDL.

3. **ROWTYPE protege dados, nao assinatura**: `SAFX3009%ROWTYPE` protege SAF_IB_X3009 contra
   desalinhamento de campos do arquivo (diferente de SAF_IB_X3007), mas nao protege a chamada
   posicional a OL_IMP_X3009. Sao duas camadas de vulnerabilidade distintas.

4. **Validacao de Codigo de Servico depende de contexto**: O erro "Codigo de Servico nao
   cadastrado" pode ter duas origens: (a) desalinhamento de assinatura passando valor errado
   ao parametro de codigo, ou (b) GRUPO_SERVICO incorreto via SAF_PEGA_GRUPO modulo 2014.
   Recompilacao resolve (a); investigar configuracao tributaria resolve (b).
