# ECD Livro Razao Auxiliar - UTL_FILE.FOPEN com path local Windows

## Root Cause

A procedure `SAF_SPED_IMP_ARQ_LRAZAO` recebia o caminho do diretorio local da estacao
do usuario (via PowerBuilder `GetFileOpenName()`) e passava diretamente ao `UTL_FILE.FOPEN`
como primeiro argumento.

```sql
-- ANTES (bugado): P_PATH = 'C:\Users\fulano' (path local Windows)
ARQ_W := UTL_FILE.FOPEN(ltrim(rtrim(P_PATH)), P_NOME_ARQUIVO, 'r');
-- Erro: UTL_FILE.INVALID_PATH (UTL_FILE espera Oracle Directory Object, nao path fisico)
```

O `UTL_FILE.FOPEN` espera como primeiro argumento o **nome de um Oracle Directory Object**
(ex: `DIR_JOB_SERVIDOR`), nao um path fisico do sistema operacional.

## Solucao

Adicionada consulta a `PRT_DIRETORIOS_SERVIDOR` para obter o Oracle Directory Name
antes de chamar `UTL_FILE.FOPEN`:

```sql
-- DEPOIS (corrigido): busca directory name do banco
SELECT DIRECTORY_NAME
  INTO V_DIR_NAME_W
  FROM PRT_DIRETORIOS_SERVIDOR
 WHERE COD_MODULO = 'JOB SERVIDOR'
   AND ROWNUM = 1;

ARQ_W := UTL_FILE.FOPEN(V_DIR_NAME_W, P_NOME_ARQUIVO, 'r');
```

Handler `NO_DATA_FOUND` adicionado para retornar mensagem clara caso o diretorio
nao esteja configurado.

Assinatura da procedure mantida (P_PATH continua como parametro IN mas e ignorado)
para backward compatibility com tela PowerBuilder.

## Padrao Utilizado: PRT_DIRETORIOS_SERVIDOR

Tabela padrao do TAX ONE para mapear modulos a Oracle Directory Objects:

```sql
-- Estrutura:
--   COD_MODULO      VARCHAR2  (ex: 'JOB SERVIDOR')
--   DIRECTORY_NAME  VARCHAR2  (ex: 'DIR_JOB_SERVIDOR' — Oracle Directory Object)
--   DIRECTORY_PATH  VARCHAR2  (ex: '/opt/oracle/data/taxone/')

-- Padrao de uso (confirmado em 15+ procedures):
SELECT DIRECTORY_NAME INTO v_dir
  FROM PRT_DIRETORIOS_SERVIDOR
 WHERE COD_MODULO = 'JOB SERVIDOR';
```

Procedures que usam esse padrao:
- `SAF_ATUALIZA_DIRECTORY_TAXBR`
- `JD_FNC_GET_DIR_CPROC`
- `s_expto8.sql` (MFS34287)
- Procedures de integracao (eSocial, ECF)

## Fluxo PB → PL/SQL

```
PB: GetFileOpenName() → 'C:\Users\x\arquivo.txt'
PB: Split → diretorio='C:\Users\x' + arquivo='arquivo.txt'
PB: EXECUTE SAF_SPED_IMP_ARQ_LRAZAO(:emp, :estab, :mes, :ano, :arquivo, :diretorio, :livro)
PL: SELECT DIRECTORY_NAME FROM PRT_DIRETORIOS_SERVIDOR → V_DIR_NAME_W
PL: UTL_FILE.FOPEN(V_DIR_NAME_W, P_NOME_ARQUIVO) → abre arquivo no servidor
```

O usuario deve garantir que o arquivo TXT esteja no diretorio do servidor Oracle
(nao na maquina local).

## Pre-requisito Cliente

Registro em `PRT_DIRETORIOS_SERVIDOR`:
- `COD_MODULO = 'JOB SERVIDOR'`
- `DIRECTORY_NAME` = nome do Oracle Directory Object
- Oracle Directory Object deve existir: `CREATE DIRECTORY <nome> AS '<path>';`
- Permissao: `GRANT READ ON DIRECTORY <nome> TO <schema>;`
