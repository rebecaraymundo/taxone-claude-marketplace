# MULTIPLAN - Saldo anterior incorreto no Balancete

## Root Cause

No package `REL_BALANCETE_EXCEL_FPROC.pck`, os cursores `vlr_saldo_ini` e `ind_saldo_ini`
usavam subqueries com `MAX(data_saldo)` para encontrar o saldo anterior:

```sql
AND s.data_saldo = (SELECT MAX(s2.data_saldo)
                      FROM SALDO_CONTABIL s2
                     WHERE s2.ident_conta = s.ident_conta
                       AND s2.data_saldo < Pdataini)
```

O `MAX(data_saldo)` buscava o saldo da data mais recente ANTES do periodo, considerando TODO
o historico. Se existissem saldos futuros ja calculados, o MAX poderia retornar uma data
posterior ao mes anterior esperado.

O padrao correto no TAX ONE e que saldos contabeis sao gravados no LAST_DAY de cada mes.
O saldo anterior de um periodo deve ser sempre o do ultimo dia do mes anterior ao periodo
solicitado.

## Solucao

Substituir a subquery `MAX(data_saldo)` por `LAST_DAY(Pdataini - INTERVAL '1' MONTH)`:

```sql
-- ANTES (subquery):
AND s.data_saldo = (SELECT MAX(s2.data_saldo) FROM SALDO_CONTABIL s2 ...)

-- DEPOIS (deterministico):
AND s.data_saldo = LAST_DAY(Pdataini - INTERVAL '1' MONTH)
```

Aplicado em ambos os cursores (vlr_saldo_ini e ind_saldo_ini).

Fix: **3 insercoes, 18 delecoes** (subqueries correlacionadas eliminadas).

## Licoes Aprendidas

1. **MAX vs LAST_DAY para saldos**: No TAX ONE, saldos contabeis sao gravados no ultimo dia
   do mes (LAST_DAY). Usar `MAX(data_saldo)` e fragil porque depende de quais dados existem
   na tabela. `LAST_DAY` e deterministico e independente dos dados.

2. **Subquery correlacionada eliminada**: Alem de corrigir o bug, o fix MELHORA performance
   ao eliminar a subquery correlacionada (que executava para cada linha do cursor principal).
   `LAST_DAY` e uma funcao pura que o Oracle calcula uma unica vez.

3. **Padrao de datas no TAX ONE**: Quando se trabalha com saldos contabeis, sempre usar
   `LAST_DAY` para referenciar o saldo de um mes. Nunca usar MAX/MIN que dependem dos dados
   existentes.

4. **Verificacao de premissa**: Antes de aplicar LAST_DAY, verificar que o cliente segue o
   padrao de gravar saldos no LAST_DAY. Script de validacao fornecido (01_validacao_saldo.sql).
