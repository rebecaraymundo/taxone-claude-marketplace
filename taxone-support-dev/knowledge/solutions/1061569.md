# Informe Rendimentos PF nao gera cod 5204 junto com 3208

## Root Cause

No metodo `obterInformes()` da classe `CedulaDDAO.java`, ao processar o arquivo CSV
de informes, a logica de **agrupamento por beneficiario** usava apenas o campo `cpfCgc`
para detectar duplicatas:

```java
if (cpfCgc.equals(cpfCgcAnterior)) {
    // agrupa retencoes no mesmo registro
}
```

Quando o mesmo beneficiario (CPF) tinha retencoes em **grupos de retencao diferentes**
(ex: 3208 = Alugueis PF e 5204 = outro grupo), o sistema tratava como mesmo informe
e tentava agrupar. Como o CSV e ordenado por CPF, o segundo grupo (5204) era adicionado
como retencao do primeiro grupo (3208) e nao gerava informe separado.

O resultado: ao selecionar 3208 e 5204 juntos, o informe do 5204 era "engolido" pelo 3208.
Selecionando individualmente, cada um gerava corretamente.

## Solucao

Adicionado `grupoRetencao` na comparacao de duplicidade:

```java
if (cpfCgc.equals(cpfCgcAnterior) && grupoRetencao.equals(grupoRetencaoAnterior)) {
    // agrupa retencoes no mesmo registro
}
```

Agora, mesmo CPF com grupos de retencao diferentes gera informes separados.

## Arquivos Modificados

1. `artifacts/java/.../dao/CedulaDDAO.java` (+10/-6)
   - Metodo `obterInformes()`: adicionada variavel `grupoRetencaoAnterior` e comparacao

## Licoes Aprendidas

1. **Chave de agrupamento incompleta**: O bug classico de usar apenas parte da chave
   para agrupar. Em informes de rendimento, a chave correta e `(CPF, grupoRetencao)`,
   nao apenas `CPF`. Ao processar CSVs sequenciais, sempre considerar TODOS os campos
   que definem unicidade.

2. **Selecao multipla vs individual**: Quando o bug so aparece na selecao conjunta
   (3208+5204) mas nao individual, indica problema de agrupamento/merge indevido
   entre os resultados.

3. **CedulaDDAO pattern**: O metodo `obterInformes()` le um CSV gerado pelo banco
   (via `_lista_informes.csv`) e agrupa em objetos Java. O CSV e ordenado por CPF,
   entao a logica de "proximo CPF diferente = novo informe" precisa considerar
   todos os campos relevantes.
