# Base Outras ICMS-S nao salva no DW (X08ItensMerc JPA entity)

## Root Cause

Na entidade JPA `X08ItensMerc.java` (tabela permanente `X08_ITENS_MERC`), o campo
`VLR_BASE_ICMSS` (Base Outras ICMS-S) nao estava mapeado com `@Column`.

O campo existia em:
- Tabela do banco: `X08_ITENS_MERC.VLR_BASE_ICMSS NUMBER(17,2)` (DDL_OS3690.sql)
- Model temporario: `TmpX08ItensMerc.vlrBaseIcmss` (linha 262)
- DTO: `TmpX08ItensMercDTO.vlrBaseIcmss` (linha 93)

Mas faltava no model permanente `X08ItensMerc.java`, que so tinha `VLR_BASE_ICMSS_N_ESCRIT`.

Resultado: ao consolidar dados da tabela temporaria para a permanente, ou ao ler da tabela
permanente, o campo era ignorado pelo JPA e retornava null.

## Solucao

Adicionado o campo na entidade JPA (PR #160, commits 40a30bb + 345ce49):

```java
@Column(name = "VLR_BASE_ICMSS", precision = 2)
private BigDecimal vlrBaseIcmss;

// Tambem adicionados (commit 345ce49):
@Column(name = "VLR_BASE_ICMSS_2", precision = 2)
private BigDecimal vlrBaseIcmss2;
@Column(name = "VLR_BASE_ICMSS_3", precision = 2)
private BigDecimal vlrBaseIcmss3;
@Column(name = "VLR_BASE_ICMSS_4", precision = 2)
private BigDecimal vlrBaseIcmss4;
@Column(name = "IND_CRED_ICMSS")
private String indCredIcmss;
```

### Logica de distribuicao (OL_IMP_X08 linhas 3605-3620)

O campo BASE_REDU_ICMSS (CAMPO 100) e o equivalente ICMSS do BASE_ICMS (CAMPO 61):

```sql
-- Analogia ICMS:  TRIB_ICMS  indica tipo, BASE_ICMS  carrega o valor
-- Analogia ICMSS: TRIB_ICMSS indica tipo, BASE_REDU_ICMSS carrega o valor

IF P_TRIB_ICMSS = '2' THEN VLR_BASE_ICMSS_2_W := RX08.BASE_REDU_ICMSS;
ELSIF P_TRIB_ICMSS = '3' THEN VLR_BASE_ICMSS_3_W := RX08.BASE_REDU_ICMSS;
ELSIF P_TRIB_ICMSS = '4' THEN VLR_BASE_ICMSS_4_W := RX08.BASE_REDU_ICMSS;
END IF;
```

## Licoes Aprendidas

1. **Consistencia entre Tmp e permanente**: Os models JPA para tabelas temporarias (TmpX08*)
   e permanentes (X08*) devem ter os mesmos campos. Ao adicionar um campo no Tmp, verificar
   se o permanente tambem precisa.

2. **@Data do Lombok**: Com `@Data`, basta adicionar o campo private — getters/setters sao
   gerados automaticamente.

3. **Modulo correto**: O DW web module e `taxone_modules_t1dw` (repo separado), nao JMsafDW_NEW
   (que e apenas para Informe de Rendimentos).

4. **Nomes diferentes no fluxo online**: Campos VLR_BASE_ICMSS nao aparecem com esse nome
   no SAF_OL_DW_X08/OL_IMP_X08. Mapeamento: VLR_BASE_ICMSS = BASE_SUB_TRIB_ICMS (CAMPO 66),
   e BASE_REDU_ICMSS (CAMPO 100) e o equivalente ICMSS do BASE_ICMS (dinâmico por TRIB_ICMSS).

5. **Analogia ICMS/ICMSS**: TRIB_ICMS:BASE_ICMS :: TRIB_ICMSS:BASE_REDU_ICMSS.
   TRIB_ICMS_AUX:BASE_ICMS_AUX :: TRIB_ICMSS_AUX2:BASE_ICMSS_AUX2.
