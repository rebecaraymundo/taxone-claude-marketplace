# SAFX3007 - Mapeamento invertido UF/MUN na importacao

## Root Cause

Procedures de importacao da SAFX3007 desatualizadas no banco do cliente.
O DDL DDL_MFS848597 adicionou novas colunas (ALIQ_TRIB_REG_IBS_MUN, VLR_TRIB_REG_TRIB_IBS_MUN)
as tabelas SAFX3007 e X3007_DOCTO_FISCAL_RT, porem as procedures nao foram recompiladas apos
a aplicacao do DDL. Isso causa desalinhamento no mapeamento posicional dos campos, fazendo com
que o valor do campo 67 (VLR_TRIB_REG_TRIB_IBS_UF) seja gravado na posicao do campo 69
(VLR_TRIB_REG_TRIB_IBS_MUN).

**Codigo-fonte no repositorio: 100% CORRETO** em todos os fluxos de importacao.

## Solucao

Recompilacao das procedures no banco do cliente, na ordem correta de dependencias:

```sql
-- Passo 1: Procedures folha
ALTER PROCEDURE OL_IMP_X3007 COMPILE;
ALTER PROCEDURE OL_IMP_X3009 COMPILE;

-- Passo 2: Procedures de importacao
ALTER PROCEDURE SAF_IMP_X3007 COMPILE;
ALTER PROCEDURE SAF_IB_X3007 COMPILE;

-- Passo 3: Package orquestrador
ALTER PACKAGE PKG_TMP_DOCTO_FISCAL_FPROC COMPILE;
ALTER PACKAGE PKG_TMP_DOCTO_FISCAL_FPROC COMPILE BODY;

-- Passo 4: Validar
SELECT object_name, object_type, status
FROM user_objects
WHERE object_name IN ('OL_IMP_X3007','OL_IMP_X3009','SAF_IMP_X3007',
                       'SAF_IB_X3007','PKG_TMP_DOCTO_FISCAL_FPROC')
AND status = 'INVALID';
```

## Fluxos de Importacao SAFX3007

O Job Servidor pode acionar 3 caminhos distintos:

| Fluxo | Arquivo | Mecanismo | Seguranca |
|-------|---------|-----------|-----------|
| SAF_IB_X3007 | s_ib3007.sql | Leitura posicional via SAF_IB_PROC_LINHA | **VULNERAVEL** a desalinhamento apos ALTER TABLE |
| SAF_IMP_X3007 | s_ix3007.sql | Cursor SELECT + acesso por nome (l_row(i).campo) | **SEGURO** - acesso por nome |
| PKG_IMP_X3007 | pix3007.pck | Cursor + MERGE INTO (acesso por nome) | **SEGURO** - acesso por nome |

Orquestrador: s_ibatc2.sql (linha ~5083) — seleciona fluxo baseado em `p_ind_utiliza_tab`:
- `'A'` = SAF_IB_X3007 (arquivo direto)
- Outros = SAF_IMP_X3007 ou PKG_IMP_X3007 (staging)

### Core INSERT/UPDATE
- OL_IMP_X3007 (ix3007.sql) — procedure chamada por SAF_IB e SAF_IMP
- PKG_IMP_X3007 usa MERGE direto (nao chama OL_IMP)

### Exportacao
- SAF_EXP_X3007 (s_ex3007.sql) + PKG_FORMATA_EXPORTACAO.pck

### DDL
- DDL_MFS848597.sql — versao atual (com colunas MUN)
- DDL_MFS851815.sql — versao anterior (SEM colunas MUN)

## X3008 vs X3009 - Reforma Tributaria IBS/CBS

| Aspecto | X3008 (Mercadorias) | X3009 (Servicos) |
|---------|-------------------|-----------------|
| IBS por ente | IBS **UF** (estadual) | IBS **MUN** (municipal) |
| Coluna aliquota | ALIQ_IBS_TRIB_REG_UF | ALIQ_TRIB_REG_IBS_MUN |
| Coluna valor | VLR_TRIB_REG_TRIB_IBS_UF | VLR_TRIB_REG_TRIB_IBS_MUN |
| Tabela definitiva | X3008_ITENS_MERC_RT | X3009_ITENS_SERV_RT |
| Tabela temporaria | TMP_X3008_ITENS_MERC_RT | TMP_X3009_ITENS_SERV_RT |

**BY DESIGN**: Na Reforma Tributaria brasileira, mercadorias sao tributadas pelo IBS da UF
e servicos pelo IBS do Municipio. Consistente em todas as 6 camadas do sistema.

## Licoes Aprendidas

1. **Desalinhamento posicional apos DDL**: Quando ALTER TABLE adiciona colunas, procedures com
   leitura posicional (SAF_IB_PROC_LINHA) ficam desalinhadas se nao recompiladas. Procedures com
   acesso por nome (l_row(i).campo) sao imunes.

2. **Ordem de recompilacao**: Sempre recompilar bottom-up (folhas primeiro, orquestradores por ultimo)
   para evitar cascata de INVALID.

3. **Named parameters vs posicionais**: Preferir named parameters em chamadas a OL_IMP para
   eliminar risco de inversao. SAF_IMP_X3007 ja usa acesso por nome; SAF_IB_X3007 depende de posicao.

4. **Verificar DDL anterior vs atual**: Comparar DDL_MFS* para entender quais colunas foram
   adicionadas e quais procedures podem estar afetadas.

5. **X3008 vs X3009**: Ao investigar gaps de colunas entre mercadorias e servicos, verificar se
   a diferenca e BY DESIGN (Reforma Tributaria UF vs MUN) antes de assumir bug.
