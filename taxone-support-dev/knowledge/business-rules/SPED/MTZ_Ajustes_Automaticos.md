|  | Overview:   A proposta é criar uma tela nova no sistema, permitindo a criação de regras, combinando parâmetros disponibilizados na tela, para que seja possível identificar NFs que atendam às regras criadas e adicione ajustes de créditos disponibilizados nas telas de apuração. Esta demanda atende apenas aos ajustes para créditos nos registros M510/M110 sem considerar a geração dos registros filhos.  Proposta inicial:    Criação de tela de Regras de Ajustes automáticos: Criação de uma nova tela, que permita a criação de regras para realizar ajustes, identificando parâmetros que estejam nas NFs. Parâmetros disponíveis na tela: Nome da regra: campo livre para digitação do nome da regra. Alfanumérico com 100 caracteres aceitando caracteres especiais. Ao tentar preencher o campo com 101 caracteres, o campo não aceita mais caracteres. Se o usuário colar um valor de outra fonte (ctrl+c e ctrl+v) com mais de 100 caracteres, o campo em questão aceitará apenas até o 100 caracter ignorando o restante. Detalhes: campo livre para digitação do nome da regra. Alfanumérico com 500 caracteres aceitando caracteres especiais. Ao tentar preencher o 501o caracter, o campo não aceita mais caracteres. Se o usuário colar um valor de outra fonte (ctrl+c e ctrl+v) com mais de 500 caracteres, o campo em questão aceitará apenas até o 500 caracter ignorando o restante. Campo Estabelecimento: Campo para seleção do estabelecimento que a regra vai ser executada. Se o usuário selecionou um estabelecimento na barra principal o campo em questão já vem preenchido read-only. Se o usuário selecionar a opção "Todos", o campo em questão apresentará uma lista dos estabelecimentos disponíveis para seleção. Campo SCP: Campo para seleção do código de SCP para utilização da regra. Só é possível digitando o código no inputText. Caso não haja um SCP para o valor informado, o inputText ao lado apresentará a mensagem "Código da SCP não encontrado", e adicionalmente a regra não pode ser salva. Campo Cód Ajuste: Campo para seleção do código de ajuste que a regra vai gerar os valores quando o processo ler a regra e identificar valores que devem ser ajustados. Os códigos são trazidos de uma tabela de cadastro de Ajustes. Campo do tipo "DropDownList", permitindo apenas a seleção de valores pré-estabelecidos. Campo Cod Tipo de Crédito: Campo para seleção do tipo de crédito que a regra vai gerar os valores quando o processo ler a regra e identificar valores que devem ser ajustados. Os códigos são trazidos de uma tabela de cadastro de Ajustes. Campo do tipo "DropDownList", permitindo apenas a seleção de valores pré-estabelecidos. Campo CST PIS: Campo para seleção dos CSTs de PIS para serem considerados na regra que está sendo criada. O campo permite múltipla seleção de códigos que já existem em tabelas de cadastro de CST. Campo similar a um "DropDownList", permitindo apenas a seleção de valores pré-estabelecidos. Campo CST COFINS: Campo para seleção dos CSTs de COFINS para serem considerados na regra que está sendo criada. O campo permite múltipla seleção de códigos que já existem em tabelas de cadastro de CST. Campo similar a um "DropDownList", permitindo apenas a seleção de valores pré-estabelecidos. O campo Movimento da Nota Fiscal: Vai trazer 2 valores para seleção do usuário (ENTRADA E SAÍDA). Caso o movimento da NF seja = SAÍDA, o sistema vai identificar que o ajuste que deve ser feito é de REDUÇÃO de CRÉDITOS. Caso o movimento da NF seja = ENTRADA, o sistema vai identificar que o ajuste que deve ser feito é de ACRÉSCIMO de CRÉDITOS. Campo Tipo de Nota Fiscal: Trará 2 valores para seleção do usuário (Normal e Devolução). Alíquota PIS: Campo para seleção das alíquotas de PIS que devem ser consideradas para a regra que se deseja criar. Campo no formato Tag box, que permite a digitação livre, e ao clicar “Enter” o valor é aceito e permite uma nova digitação de valor. Os valores aqui adicionados serão utilizados como filtros na rotina de ajustes. Supondo que eu adicione a tag "TESTE", a rotina vai filtrar as NFs que possuem alíquota "TESTE", não havendo dados que atendam a esse critério, a rotina de ajustes não terá dados para tal e nada ocorre. Isso ocorre tanto para valores "fora do padrão" como para valores Alíquota COFINS: Campo para seleção das alíquotas de COFINS que devem ser consideradas para a regra que se deseja criar. Campo no formato Tag Box, que permite a digitação livre, e ao clicar “Enter” o valor é aceito e permite uma nova digitação de valor. Os valores aqui adicionados serão utilizados como filtros na rotina de ajustes. Supondo que eu adicione a tag "TESTE", a rotina vai filtrar as NFs que possuem alíquota "TESTE", não havendo dados que atendam a esse critério, a rotina de ajustes não terá dados para tal e nada ocorre. Isso ocorre tanto para valores "fora do padrão" como para valores Os campos de alíquotas não terão limitador para inserção de "tags" de alíquotas. O formato do campo é limitado a no máximo 7 dígitos para inserção das alíquotas. considerando no máximo 4 campos décimais, e separados por (.) ponto. exemplo do comportamento do campo formato Tab Box: bento.ui.int.thomsonreuters.com/#/components/tags-input Ações disponíveis na tela: A tela vai permitir as ações que são consideradas nas demais telas do TaxOne e que terão as mesmas CONFIRMAR:  A opção permite que as regras sejam salvas. LIMPAR: Esta opção permite que os dados preenchidos na tela, sejam limpos. FECHAR: Fecha a tela de regras.   REGRAS CADASTRADAS Visualização da regra na própria tela: A tela permite que as regras já cadastradas sejam visualizadas na própria tela, trazendo na linha os parâmetros que foram utilizados em cada uma das regras. Na própria linha serão disponibilizadas 2 ações: REMOVER: Ao selecionar esta opção a regra selecionada vai ser deletada do sistema. É apresentada a mensagem "Confirma a exclusão do registro?" com opções de "Sim" e "Não". Se sim, formulário é limpo e grid atualizada. Se não, nada ocorre. ALTERAR: Esta opção permite que a regra possa ser alterada. Ao selecionar esta opção, os parâmetros serão carregados na própria tela, para que seja possível editar os parâmetros desejados. Ao finalizar a edição, o usuário deverá selecionar a opção confirmar para salvar as alterações. É apresentada a mensagem "Confirma a exclusão do registro?" com opções de "Sim" e "Não". Se sim, formulário é limpo e grid atualizada. Se não, nada ocorre.  Observação Importantes: As regras devem ser criadas sempre pelo estabelecimento matriz. Processo de Ajustes Manuais: O usuário vai criar as regras no sistema, conforme a sua necessidade, utilizando a tela que será criada e disponibilizada.   Execução do processo de Ajustes Manuais: O processo vai ser executado a partir do processo que já existe hoje “automação Ajustes de Crédito – Nota Mercantil”. Este processo realiza a simulação e também a execução das regras que foram criadas na tela de “regras”.  Resultado do processo de Ajustes Manuais: Ao executar a simulação (processo que já existe atualmente no sistema TAXONE > SPED > EFD – Escrituração Fiscal Digital das Contribuições > Obrigações > Automação Ajustes de Crédito – Nota Mercantil), o sistema vai executar a geração de um arquivo excel, com o resultado da leitura das regras que foram criadas no sistema (A disponibilização do arquivo será feita no sistema na opção "nuvem", área de download). Ao executar o processo, o sistema vai gerar os ajustes que foram realizados a partir das regras, que foram exibidos no arquivo excel, e vai fazer a geração dos ajustes nos códigos de ajustes e códigos de créditos nos registros M510 para COFINS e M110 para PIS (como é feito atualmente).    Jornada do usuário:     TAXONE > SPED > EFD – Escrituração Fiscal Digital das Contribuições > Parâmetros > Ajustes Automáticos > Regras TAXONE > SPED > EFD – Escrituração Fiscal Digital das Contribuições > Obrigações > Automação Ajustes de Crédito – Nota Mercantil    Alterações visuais:   Tela de Regras de Ajuste: Tela de Cadastro de Regras para Ajustes Automáticos: Criação de regras para considerar Notas Fiscais que atendam os parâmetros combinados e faça a geração dos ajustes de “créditos” nos registros M510 e M110: A regra deve permitir a combinação de mais de um CST para PIS e COFINS no momento do cadastro e deve permitir a digitação de mais de uma alíquota para PIS e COFINS. Movimento da NF: Entrada e Saída Tipo de NF: NF Normal ou Devolução Alíquota PIS: campo para digitação de alíquotas para PIS. Alíquota COFINS: campo para digitação de alíquotas para COFINS. Cod Ajuste: campo para seleção do código do ajuste. Cod Tipo de Crédito: campo para seleção do código do tipo de crédito. Detalhes: Campo livre para detalhamento do objetivo da regra criada.    Botões/Ícones: CONFIRMAR: Opção para salvar a regras que está sendo criada no sistema, caso tente salvar as informações sem preencher todos os campos obrigatórios, o sistema apresentará uma mensagem indicando que há campos obrigatórios não preenchidos. Os campos que são exibidos na tela de regras, serão trazidos de tabelas que já existem no sistema, e serão disponibilizadas na tela de regras para a utilização.  Processo de execução do Ajuste Automático O processo para execução do Ajuste Automático, vai ler as regras que foram criadas na tela de “REGRAS”, e vai identificar quais Notas Fiscais contemplam os parâmetros cadastrados nas regras e vai gerar os valores a serem ajustados nos códigos de ajustes e nos códigos do tipo de créditos selecionados. Estes valores serão disponibilizados nos Registros M110 e M510 (PIS e COFINS). Para emitir as NFs para conferência:  TAXONE > SPED > Básicos > Report Fiscal > Gerenciais > Documentos Fiscais > Analíticos > Conferência de Documentos Fiscais (ICMS/IPI/PIS/COFINS) - Formato XLS O processo de geração do Ajuste Automático, vai ser executado a partir de um processo que já existe atualmente no TaxOne: “automação Ajustes de Créditos – Nota Mercantil”, que permite a simulação da execução do processo (gerando um arquivo excel, com o resultado que deve ser gerado como ajuste) e permite também a execução efetivamente das regras criadas. A execução vai gerar nos Registros M110/M510 o resultado do arquivo excel que foi gerado na simulação (como é feito atualmente) O processo atual que vamos utilizar, faz a geração apenas dos Registros M510 e M110, não gerando dados nos Registros “filhos”. Gravação dos ajustes nos Registros M110/M510 Quando a regra criada for cadastrada com o campo: MOVIMENTO DE NF = SAÍDA, o sistema vai considerar que aquele ajuste deve ser gerado como REDUÇÃO. A gravação do registro no M110/M510 vai ser gerado com "TIPO DE AJUSTE = 0 - AJUSTE DE REDUÇÃO. Quando a regra criada for cadastrada com o campo: MOVIMENTO DE NF = ENTRADA, o sistema vai considerar que aquele ajuste deve ser gerado como ACRÉSCIMO. A gravação do registro no M110/M510 vai ser gerado com "TIPO DE AJUSTE = 1 - AJUSTE DE ACRÉSCIMO.    Pré-condições / Pré-requisitos:  A execução dos ajustes vai buscar Notas Fiscais que tenham sido carregadas do ERP, utilizando o Job do sistema O sistema só vai executar este processo de ajustes para créditos, utilizando a geração dos valores nos Registros M510/M110. O processo atual que será usado para execução dos ajustes “automação Ajustes de Créditos – Nota Mercantil”, executa apenas a geração dos resultados nos Registros M510 e M110, sem a geração dos valores nos respectivos “filhos”. Restrições de escopo:    Vamos alterar o processo atual, para não fazer a leitura das regras atualmente em hardcoded, para realizar a leitura das regras, de acordo com a tela que vamos desenvolver de cadastro de regras. Não vamos realizar nenhuma alteração no processo atual que já existe de “automação Ajustes de Créditos – Nota Mercantil”. Os ajustes não serão gerados para os respectivos “filhos” dos Registros M510/M110. Não vamos realizar nenhuma alteração no arquivo excel que é gerado na simulação do processo existente “automação Ajustes de Créditos – Nota Mercantil”. Não vamos desenvolver nenhum relatório que exiba as regras criadas na tela. Não faz parte do escopo desativar a tela utilizada atualmente para ajustes do M510/M110. TAXONE > SPED > EFD – Escrituração Fiscal Digital das Contribuições > Parâmetros > de Ajustes de Créditos dos Registros M110/M510 Não vamos desenvolver queries para trazer as regras da tela existente, para a tela nova que vamos desenvolver.    Riscos/Mitigações propostas:   Risco: O usuário vai ter total liberdade para criar as regras para a execução do ajuste, o sistema não vai fazer nenhuma validação se a regra é válida, ou se está de acordo com o Guia Prático EFD. Os campos que são exibidos na tela de regras, serão trazidos de tabelas que já existem no sistema, e serão disponibilizadas na tela de regras para a utilização.  Processo de execução do Ajuste Automático O processo para execução do Ajuste Automático, vai ler as regras que foram criadas na tela de “REGRAS”, e vai identificar quais Notas Fiscais contemplam os parâmetros cadastrados nas regras e vai gerar os valores a serem ajustados nos códigos de ajustes e nos códigos do tipo de créditos selecionados. Estes valores serão disponibilizados nos Registros M110 e M510 (PIS e COFINS). Para emitir as NFs para conferência:  TAXONE > SPED > Básicos > Report Fiscal > Gerenciais > Documentos Fiscais > Analíticos > Conferência de Documentos Fiscais (ICMS/IPI/PIS/COFINS) - Formato XLS O processo de geração do Ajuste Automático, vai ser executado a partir de um processo que já existe atualmente no TaxOne: “automação Ajustes de Créditos – Nota Mercantil”, que permite a simulação da execução do processo (gerando um arquivo excel, com o resultado que deve ser gerado como ajuste) e permite também a execução efetivamente das regras criadas. A execução vai gerar nos Registros M110/M510 o resultado do arquivo excel que foi gerado na simulação (como é feito atualmente) O processo atual que vamos utilizar, faz a geração apenas dos Registros M510 e M110, não gerando dados nos Registros “filhos”. Gravação dos ajustes nos Registros M110/M510 Quando a regra criada for cadastrada com o campo: MOVIMENTO DE NF = SAÍDA, o sistema vai considerar que aquele ajuste deve ser gerado como REDUÇÃO. A gravação do registro no M110/M510 vai ser gerado com "TIPO DE AJUSTE = 0 - AJUSTE DE REDUÇÃO. Quando a regra criada for cadastrada com o campo: MOVIMENTO DE NF = ENTRADA, o sistema vai considerar que aquele ajuste deve ser gerado como ACRÉSCIMO. A gravação do registro no M110/M510 vai ser gerado com "TIPO DE AJUSTE = 1 - AJUSTE DE ACRÉSCIMO.    Pré-condições / Pré-requisitos:  A execução dos ajustes vai buscar Notas Fiscais que tenham sido carregadas do ERP, utilizando o Job do sistema O sistema só vai executar este processo de ajustes para créditos, utilizando a geração dos valores nos Registros M510/M110. O processo atual que será usado para execução dos ajustes “automação Ajustes de Créditos – Nota Mercantil”, executa apenas a geração dos resultados nos Registros M510 e M110, sem a geração dos valores nos respectivos “filhos”.
