# <a id="_Toc48585380"></a><a id="_Toc48588059"></a>Registro 0035 \- Identificação da Sociedade em Conta de Participação

__RN0035__

Este registro será gerado somente quando o processamento for realizado através da “Geração dos Registros” do menu “Obrigações SCP” e sua origem será o Cadastro da SCP \(Manutenção >> Registro do Bloco 0 >> Cadastro da SCP\)\.

Quando a geração for realizada através de um estabelecimento indicado como “Sócio Ostensivo”, poderão ser gerados “N” registros 0035, sendo um para cada SCP no qual o Sócio Ostensivo participa, ou seja, um para cada código de SCP vinculado ao estabelecimento que está gerando o arquivo \(considerar centralizados, para geração por estabelecimento centralizador\)\.

Se a geração for realizada através de um estabelecimento indicado com o Código da SCP, será gerado “1” registro 0035, demonstrando o Cadastro da SCP para o qual a informação está sendo gerada\.

A chave de geração deste registro é o campo 02 – COD\_SCP\. Se no Cadastro da SCP constar mais de um estabelecimento com o mesmo código para um grupo de centralização de estabelecimentos, deverá ser considerado o primeiro registro lido do Cadastro para geração deste 0035\.

__\[ALTERADA \- CH18232\_2018 \(MFS\-19805\)\]__

__Tratamento:__

- Se o campo “Data de Validade Final” do Cadastro da SCP estiver preenchido e estiver fora do período de geração do arquivo magnético, o registro deve ser ignorado;
- Se o campo “Data de Validade Final” do Cadastro da SCP estiver preenchido e estiver dentro do período de geração do arquivo magnético, o registro deve ser gerado;
- Se o campo “Data de Validade Final” do Cadastro da SCP não estiver preenchido, o registro deve ser gerado\.

# <a id="_Toc48588073"></a>Registro 0500 – Plano de Contas Contábeis

RN0500

Este registro deve ser gerado a partir das contas contábeis referenciadas nos diversos registros da EFD\-PIS/COFINS\.

Gravar um registro 0500 para cada conta contábil \(campo COD\_CTA constante na movimentação do período\)

O registro da SAFX2002 \(Tabela do Plano de Contas\) a ser considerado, deve ser sempre o de maior data de validade, que seja <= a data final do período\.

# <a id="_Toc48588074"></a>Registro 0600 – Centro de Custos

RN0600

Este registro deve ser gerado a partir dos centros de custo referenciados nos diversos registros da EFD\-PIS/COFINS\.

# <a id="_Toc48588075"></a>Registro 0900 – Composição das Receitas do Período – Receita Bruta e Demais Receitas

RN0900

__Regra Geral__

Este registro deve ser gerado a partir da somatória dos valores de Receita Bruta e Demais Receitas, de alguns registros da EFD\-Contribuições\.

Registro de nível 2, ocorrência de 1 por arquivo e obrigatório somente se a escrituração for transmitida após o prazo regular de entrega\.

__Fluxo da Geração do Registro 0900__:

Passo 1: Geração dos Registros

menu Obrigações >> Geração dos Registros

Nesta etapa os valores referentes à receita total dos blocos A, C, D, F, I e 1 serão calculados e gravados em uma tabela que estará disponível para consulta via Tela de Demais Receitas não Brutas \(0900\)\.

Passo 2: Manutenção das Parcelas das receitas não classificada como receita bruta

Menu Manutenção >> Registro do Bloco 0 >> Demais Receitas não Brutas \(0900\)\.

Nesse etapa o usuário poderá visualizar os valores totais de receita bruta dos blocos A, C, D, F, I e 1, gerados no passo anterior e incluir valores que considerar como receita __não brutas__ para os respectivos blocos\.

Passo 3: Geração do Meio Magnético

menu Obrigações >> Geração do Meio Magnético

O registro 0900 é gerado no arquivo com os valores das receitas brutas e não brutas dos blocos A, C, D, I e 1\.

__Observações:__ 

Este registro deverá ser informado sempre que a escrituração original for transmitida após o prazo regular de entrega \(após o 10º dia útil do 2º mês subsequente ao período de apuração a que se refere a escrituração\)\.
