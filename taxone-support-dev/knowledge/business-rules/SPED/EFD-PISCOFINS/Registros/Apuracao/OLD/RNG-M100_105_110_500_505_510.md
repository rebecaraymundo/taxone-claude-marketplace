__Regras de Recuperação para os registros do bloco M:__

__Registro M100 – Crédito de PIS/PASEP relativo ao período__

__RNG\-M100__

Deverá existir um registro M100 para cada Tipo de Crédito e alíquota informados nos documentos que constam dos registros A100/A170, C100/C170, C190/C191, C395/C396, C500/C501, D100/D101, D500/D501, F100, F120, F130, F150, conforme <a id="RG_M100_CALC"></a>Regra de Cálculo do Crédito abaixo:

- 
	1. Os registros A100 e F100, quando o campo IND\_OPER = 0;
	2. Os Registros C100 cujo COD\_MOD <> 55 quando o campo IND\_OPER = 0;
	3. Os Registros C100 cujo COD\_MOD = 55 quando o campo IND\_OPER = 0 e o campo IND\_ESCRI do registro C010 <> 1 ;
	4. Todos os registros C395, C500, D100, D500, F120, F150, F200 e filhos;
	5. Os registros C190 quando o campo IND\_ESCRI do registro C010 <> 2;

Caso sejam informados registros F800, deverão existir registros M100, considerando os campos COD\_CRED \(conforme COD\_CRED informado no registro F800\) e IND\_CRED\_ORI \(igual a 1\), não sendo informado registro M105\. 

Se COD\_INC\_TRIB do registro 0110 é diferente de 2 e caso tenham sido informados registros F205 e F210, deverá existir um registro M100/M500 com COD\_CRED igual a 109, gerado exclusivamente a partir destes registros, não sendo informado o registro M105\.

O agrupamento desse registro deve ser feito com base das informações dos seguintes campos do registro M100:

02 – Código de Tipo de Crédito apurado \(COD\_CRED\)

03 \- Indicador de Crédito Oriundo \(IND\_CRED\_ORI\)

05 – Alíquota PIS \(percentual\) – \(ALIQ\_PIS\)

07 – Alíquota PIS \(em reis\) – \(ALIQ\_PIS\_QUANT\)

__Tratamento para Geração de SCPs__

Quando a geração for realizada através do menu "Obrigações SCP", além dos critérios definidos na regra padrão de seleção, devem ser considerados apenas os registros referentes à SCP de geração para composição deste registro, considerando:

Se for geração de um Estabelecimento Centralizador identificado como "Sócio Ostensivo", será considerado para composição deste registro e seus filhos apenas os registros pertencentes a este estabelecimento e os seus centralizados \(considerando a regra padrão de seleção\) e que não tenham o campo Código do SCP preenchido\.

Se for geração de um Estabelecimento Centralizador identificado como um Código SCP, será considerado para composição deste registro e seus filhos apenas os registros pertencentes a este estabelecimento e os seus centralizados \(considerando a regra padrão de seleção\) e que tenham o campo Código do SCP preenchido com o indicado na geração\.

Se for geração de um Estabelecimento Descentralizado identificado como "Sócio Ostensivo", será considerado para composição deste registro e seus filhos apenas os registros pertencentes a este estabelecimento \(considerando a regra padrão de seleção\) e que não tenham o campo Código do SCP preenchido\.

Se for geração de um Estabelecimento Descentralizado identificado como um Código SCP, será considerado para composição deste registro e seus filhos apenas os registros pertencentes a este estabelecimento \(considerando a regra padrão de seleção\) e que tenham o campo Código do SCP preenchido com o indicado na geração\.
