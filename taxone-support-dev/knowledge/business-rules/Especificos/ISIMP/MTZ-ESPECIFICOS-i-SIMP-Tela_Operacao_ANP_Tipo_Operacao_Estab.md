Tipo de Operação

Alfanumérico 

S

S

O campo Tipo de Operação deve permitir que o usuário selecione quantos tipos de operações forem necessárias para a parametrização\.

O campo deve exibir os campos “COD\_OPERACAO” \+ “ \- “ \+ “DESCRICAO” da tabela SAFX2001\.

Neste campo devem ser apresentados os tipos de operação válidos em relação à data de Validade Inicial informada na tela\. 

Se para uma data de Validade Inicial \(EX 01/01/2012\) um código de operação já estiver parametrizado para um Código de Operação ANP que tenha o campo Tipo de Operação \(campo da TFIX116\) igual a “E – Entrada”, este código também pode ser parametrizado para um novo Código de Operação ANP que tenha Tipo de Operação seja igual a “S – Saída”\.

Se um código de operação já estiver parametrizado para um Código de Operação ANP que tenha o campo Tipo de Operação \(campo da TFIX116\) igual a “S – Saída”, este código também pode ser parametrizado para um novo Código de Operação ANP que tenha Tipo de Operação seja igual a “E – Entrada”\. 

Se um código de operação já estiver parametrizado para um Código de Operação ANP que tenha o campo Tipo de Operação \(campo da TFIX116\) igual a “O – Outros”, este código de operação não pode mais ser associado a um novo Código de Operação ANP \(independente do Tipo de Operação do Código ANP\), da mesma forma que códigos de operação associadas a Códigos ANP que tenham Tipo de Operação “E – Entrada” ou “S – Saída”, não podem ser associados a um Código ANP de Tipo “O – Outros”\.

O mesmo Código de Operação só pode ser associado a um único Código ANP de Entrada e a um outro de Saída\. Não é possível associar o mesmo código de operação para “N” códigos de entrada e “N” códigos de saída\. Neste contexto, se o usuário associar um Código de Operação a um Código ANP de Entrada e indicar uma nova parametrização para um outro Código ANP de Entrada, o Código de Operação associado ao código anterior não deve ser exibido, por se tratar da mesma operação\. Isso se aplica tanto para entrada quanto para saída\.
